# v1.1.0.0 - Inventory Parity And Doc Truth

**Released:** 2026-07-16

Brings the packet to full inventory parity with its hub siblings and executes the researched sk-design de-duplication directive: `mcp-refero` is now the single canonical home of the Refero tool surface, and the old duplicated catalog inside `sk-design` is a pointer. All new content traces to `references/tool-surface.md` and the phase-001 research synthesis; no new tools, arguments, or limits were invented.

---

## What's New In 1.1.0.0

### Examples (new directory)

- `examples/README.md` — directory page mirroring the `mcp-aside-devtools/examples/` conventions: per-walkthrough purpose and enforcement notes, common patterns (mandatory `tool_info` confirmation, SKIP-valid marking for OAuth-gated steps, synchronous calls with cited evidence), troubleshooting, see-also.
- `examples/funnel-styles-screens-flows.md` — the full styles -> screens -> flows research funnel as a worked Code Mode walkthrough, doubled-prefix callables throughout, metadata-first ordering at every layer.
- `examples/metadata-first-lookup.md` — a single-tool metadata-first screen lookup, including the elements facet (`ui_elements`/`ux_patterns`) in place of the non-existent `search_elements`.
- `examples/screen-image-fetch.md` — the image-last lane: `thumbnail` before `full`, never `response_format` on `refero_get_screen_image`.

### Scripts

- `scripts/install.sh` (new) — non-interactive, verify-only posture check: Node >=18 and npx present, `refero` manual presence via read-only grep (presence = OK; registration stays an operator decision), the Node-25 SIGSEGV warning, and the OAuth boundary marked operator-only. Installs nothing, modifies nothing; `bash -n` clean.

### Manual Testing Playbook (6 -> 9 scenarios)

Three scenarios added, grounded in the research synthesis §D workflows and §C limits record; live parts SKIP-valid:

- `FUNNEL-001` (`read-only/funnel-walk.md`) — the complete three-layer funnel on one brief, with ID typing checked at every layer transition.
- `FORMAT-001` (`read-only/format-text-retrieval.md`) — `response_format` discipline: per-tool `tool_info` checking, JSON vs markdown shapes, unknown-field preservation, and the image-tool exclusion.
- `QUOTA-001` (`safety-gate/quota-recovery.md`) — quota/429 honesty: verbatim provider relay, declared unknowns, no invented retry contract, no config/auth mutation as recovery; the live 429 capture is SKIP-valid by design.

Root playbook updated: coverage table, wave planning, scenario summaries, cross-reference index (9 IDs / 9 files), and release-readiness rule.

### Feature Catalog (per-tool leaves)

Eight per-tool leaf files added so each documented tool has its own home, one per tool under its layer directory: `styles/{search_styles,get_style}.md`, `screens/{search_screens,get_screen,get_similar_screens,get_screen_image}.md`, `flows/{search_flows,get_flow}.md`. Domain overview files link their leaves; the root catalog's capability-count summary now tracks 3 domain files + 8 per-tool leaves.

### sk-design De-Duplication (the research §G directive)

- `sk-design/design-interface/references/mcp_tooling/refero_tools.md` slimmed from a full duplicated tool catalog to a POINTER document: canonical transport home named (`.opencode/skills/mcp-tooling/mcp-refero/`), with only sk-design's own judgment-side guidance retained (styles-first critique-against framing, the anti-averaging mandate).
- design-interface prose that described the old in-skill catalog updated to "mcp-refero transport over mcp-code-mode" phrasing (SKILL.md, README.md, mobbin_tools.md cross-link, design_references_mcp.md catalog links, and the ID-010 playbook anchor role).
