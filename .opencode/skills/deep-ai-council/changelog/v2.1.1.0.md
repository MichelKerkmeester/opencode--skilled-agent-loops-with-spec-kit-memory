# v2.1.1.0: Deep-Mode Docs + Script Tests

Date: 2026-05-24

Follow-on to the v2.1.0.0 release cleanup, run as packet `131-deep-skill-evolution/000-release-cleanup/011-deep-ai-council-deep-mode-docs-and-tests`. It closes the five follow-ons the 004 phase-5 deep-research loop deferred: documents the deep-mode session hierarchy and the findings registry, cross-links the graph-replay script, reconciles the DAC-001 rename narrative, and adds test coverage for five previously-untested scripts. Documentation and tests only. No deep-mode runtime or script logic changed. No backward-incompatible changes.

## What Changed

#### Added

- `references/deep_mode.md`: the deep-mode session -> topic -> round hierarchy, the `session-state.jsonl` and `round-state.jsonl` state files, cost guards, and the `deep-loop-runtime/lib/council/` dependency (F-002).
- `references/findings_registry.md`: the cross-topic findings-registry schema, fingerprint-based dedup, cross-topic priors, and filesystem locking (F-003).
- `scripts/tests/`: five new vitest files covering `lib/persist-artifacts.cjs`, `lib/rollback.cjs`, `lib/audit-trail.cjs`, `advise-council-completion.cjs`, and `replay-graph-from-artifacts.cjs` (F-006; 60 test cases).

#### Changed

- `references/graph_support.md`: cross-links `replay-graph-from-artifacts.cjs` and its derivation in the recovery path and cross-references (F-004).
- `feature_catalog/01--runtime-routing-and-rename/01-...md`: reconciled the DAC-001 narrative — the agent files are `ai-council.*` (`@ai-council` identity, the `deep-` prefix dropped in v1.2.0.0) while the skill keeps the `deep-ai-council` name (DAC-001 flag).
- `SKILL.md`: wired the two new references into §3 RESOURCE_MAP (additive intent entries, no router-pseudocode change) and §6 REFERENCES; version bumped `2.1.0.0` -> `2.1.1.0`.
- `README.md`: added the two new references to §9 RELATED DOCUMENTS.

#### Deferred

- Full `vitest run` of the five new tests is deferred to the project's standard test invocation: the skill's `scripts/` has no local vitest config (the existing four `.vitest.ts` tests share this). The new tests are syntax-verified (`node -c`), match the existing import/`createRequire` style, and target the real scripts.
