This release adds the missing sk-doc section dividers to cli-opencode's reference docs. Several files ran their numbered H2 sections together with no `---` separator between them, two reference docs lacked the leading divider before section 1, and `agent_delegation.md` had two H2s whose only nearby `---` sat mid-content rather than immediately before the heading. The separators are correct now, so the docs scan cleanly and match the rest of the skill set. No dispatch behavior changed.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/130-prompt-knowledge-layering/014-doc-divider-sweep-and-sk-prompt-cleanup/` (Level 1)

---

## What Changed

#### Documentation

- **Section dividers added** -- `references/agent_delegation.md`, `references/integration_patterns.md`, `references/opencode_tools.md`, `references/permissions-matrix.md` and `references/context-budget.md` were missing the standalone `---` thematic break that the sk-doc templates require before each numbered H2 section. We inserted the missing blank-padded dividers before every non-first H2, and added the section-1 leading divider to `permissions-matrix.md` and `context-budget.md`. In `agent_delegation.md`, sections 4 and 9 had a `---` floating in the previous section's body but none immediately before the heading, so they now get a proper immediate divider. The edits add only blank and `---` lines, so a content-skeleton diff against the prior version is byte-identical on every non-blank, non-divider line, verified.
- **One doc deferred** -- `references/cli_reference.md` carried an unrelated uncommitted content edit (a new model-id row) at sweep time, so its divider fix was held back to avoid bundling that work.
