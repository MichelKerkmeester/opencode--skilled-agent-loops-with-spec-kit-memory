This release folds the sk-code slice of the packet 146 ponytail-based refinement into the code-work skill — reusable restraint teachings transplanted from an external reference and added on top of the existing routing without changing surface precedence or the Iron Law. The theme is doing the least that is still correct: before writing new code the implementer now walks a short restraint ladder, builds the simplest correct version instead of stalling on a scope question, names a deliberate shortcut so it reads as intent rather than ignorance, and a new validator keeps the surface routing map honest against what is actually on disk.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/146-sk-code-ponytail-based-refinement` (Level 2)

---

## New Features

Three additions land in the always-loaded universal references plus one new verification script. None of them change surface detection, surface precedence (OPENCODE over WEBFLOW over UNKNOWN), or the Iron Law; they sit after routing as pre-write reflexes and one structural check.

#### Design Restraint Ladder (pre-write)

An agent with a green light to implement tended to reach for a custom solution first. A new roughly thirteen-line Design Restraint Ladder in the always-loaded `references/universal/code_quality_standards.md` makes the agent climb from least to most code before writing: does this need to exist at all (YAGNI), then a standard-library primitive, a native platform or runtime feature, an already-installed dependency, a one-line expression, and only then a minimal custom implementation. It is a post-read reflex that runs AFTER surface and intent routing, so it consumes the detected surface but changes neither surface precedence nor the Iron Law (Phase 3 verification is still required). The ladder is gated to the Phase 0 to Phase 1 transition in `references/phase_detection.md` — the 0 to 1 gate now requires that, for implementation intent, the laziest viable rung has been selected — and it is named in the SKILL.md Phase Overview so the implementer meets it before the first line of new code.

&nbsp;

#### Implementer anti-stall rule

A restraint ladder can backfire into paralysis: an agent that spots a possibly unnecessary requirement might stop and ask rather than ship. A new §4 ALWAYS bullet in `SKILL.md` closes that gap. The implementer builds the simplest correct implementation of the stated requirement and does not stall. When part of the requirement looks unnecessary, it implements the requirement as specified AND raises a scope-amendment recommendation in the same response — it never silently cuts scope (SCOPE-LOCK still holds) and never blocks solely to ask when a safe minimal version already satisfies the requirement.

&nbsp;

#### Intentional-simplification convention

When a simplification is deliberate — a known shortcut chosen on purpose, not an oversight — it should read as intent. A new convention in `references/universal/code_style_guide.md` §4 introduces a neutral `ceiling:` comment that names the shortcut, its known ceiling, and the upgrade path or trigger. It is a durable WHY, not a brand or tool prefix (a brand prefix reads as a perishable, cargo-cult label), and it is explicitly NOT added to the comment-hygiene checker's allowed-pattern list — it already passes because it carries no forbidden id, and allow-listing it would let a forbidden id on the same line slip through. This is the producer-side companion to the reviewer-side downgrade rule shipped in sk-code-review v1.4.0.0.

&nbsp;

#### STACK_FOLDERS validator

The smart router declares each surface in a `STACK_FOLDERS` map and then loads that surface's patterns from `references/<surface>/` and `assets/<surface>/`. If a declared surface had no folders, or a surface-shaped folder existed with no declaration, the router would silently load nothing or drift from its documented contract with no warning. A new `assets/scripts/verify_stack_folders.py` parses the `STACK_FOLDERS` literal out of SKILL.md and asserts that every declared surface resolves to both an on-disk references folder and an assets folder, and that no orphan surface folder exists undeclared in either the `references/` or the `assets/` tree (the shared `universal/` material and the `scripts/` tooling directory are exempt by design). It exits non-zero with a per-problem report when the declaration and the folders disagree.

## Upgrade

No migration required. The ladder, the anti-stall rule, and the `ceiling:` convention are additive guidance that change how the implementer approaches new code, not the surface routing or the verification contract. The new validator is a standalone check; run `python3 .opencode/skills/sk-code/assets/scripts/verify_stack_folders.py` to confirm every declared surface resolves to real folders.
