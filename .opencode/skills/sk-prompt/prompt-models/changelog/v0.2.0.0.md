## [**0.2.0.0**] — 2026-05-18

Expands the dispatch matrix from a single-executor-per-model model to a multi-path matrix where each small model declares all available executor + provider combinations. Adds GLM-5.1 to the active rotation. Adds new manual testing scenarios that exercise `sk-small-model` alongside `sk-prompt` and the relevant CLI executor as a triple-skill flow.

> No spec packet — refactor lands directly on `main` per user direction. Triggered by user directive 2026-05-18 clarifying the executor + provider matrix.

---

## What Changed

#### Multi-executor dispatch matrix in `model-profiles.json`

`sk-prompt-small-model/assets/model-profiles.json` v1.1 introduces an `executors` array per model entry. Each executor item declares its `executor` (cli-devin / cli-opencode), `provider` (cognition / deepseek-api / opencode-go / anthropic / google), `quota_pool`, and `status`. A `primary_quota_pool` field preserves the preferred path for fallback-router scoring. The schema lets the fallback engine pick a different pool when one is exhausted — DeepSeek-v4-pro now has three independent paths (Cognition Pro via cli-devin, DeepSeek API direct via cli-opencode, opencode-go pool via cli-opencode) instead of one.

---

#### GLM-5.1 moved from out-of-scope to active rotation

Active small-model rotation is now: SWE-1.6, DeepSeek-v4-pro, Kimi-k2.6, Qwen3.6, GLM-5.1. Frontier models (Opus, Sonnet, gpt-5.5) remain out of scope. GLM-5.1 has 2 active dispatch paths: cli-devin via Cognition Pro and cli-opencode via opencode-go.

#### Stale skill-graph compile rebuilt

`.opencode/skills/system-skill-advisor/mcp-server/scripts/skill-graph.json` had been generated 2026-05-15 (before sk-small-model existed) and was missing the sk-small-model node + reverse `enhances` edges. Post-rebuild: skill_count 20 → 21, sk-util family registers sk-small-model, reverse `enhances` edges visible from cli-devin and cli-opencode.

---

#### Testing playbook scenarios for triple-skill flow

`cli-devin` playbook gains DV-028 (SWE-1.6 dispatch via sk-small-model + sk-prompt) and DV-029 (DeepSeek-v4-pro via cli-devin Cognition Pro through sk-small-model + sk-prompt). `cli-opencode` playbook gains CO-035 (DeepSeek-v4-pro via opencode-go provider through sk-small-model + sk-prompt) and CO-036 (Kimi-k2.6 via opencode-go through sk-small-model + sk-prompt). Each scenario validates the full advisor → sk-small-model surfacing → sk-prompt composition → CLI dispatch → output verification chain.

#### AGENTS.md / CLAUDE.md + 114 spec status drift

`AGENTS.md` dispatch rule body lists the active model set inline and points at `sk-small-model` as the canonical home for the full executor matrix. The 114 phase parent `spec.md` Status drift identified in deep-review iter-6 is resolved: all 6 phase children now show Status: Complete.
