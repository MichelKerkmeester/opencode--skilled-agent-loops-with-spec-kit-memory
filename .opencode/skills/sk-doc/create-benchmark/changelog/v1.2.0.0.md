# v1.2.0.0 — skill-benchmark and model-benchmark added; Smart Routing header normalized

`create-benchmark` now authors four benchmark families, up from two. The MCP-promotion (`mcp_server/benchmarks/benchmark-<YYYY-MM-DD>/`) and behavior_benchmark families are unchanged in contract. This release adds two more authored families and names two code-owned ones for router disambiguation, and normalizes the SKILL.md section structure to the canonical sk-doc shape.

The two new authored families:

- **Skill-benchmark (Lane C)** — this packet owns the *storage convention* for a hub's `benchmark/` tree and the *template for its `benchmark/README.md` run-label index*. The per-run `skill-benchmark-report.md` is a `build-report.cjs` anti-drift render and is never hand-authored or templated; the D1-D5 scoring contract stays lane-local in deep-improvement and is cross-linked.
- **Model-benchmark (Lane B)** — this packet owns the *input fixtures* the model under test answers (code-task oracle, pattern/capability evidence contract, reviewer-prompt) and the *run profiles* that drive a run. The evaluator, scorers, and reviewer-verdict contract stay lane-local in deep-improvement and are cross-linked.

Lane A (agent-improvement) is named in Smart Routing only for disambiguation — its artifacts are code-owned and run entirely in-lane, so this packet ships no template, guide, or index for it (a documented non-goal).

## What Added

- **`references/skill_benchmark/skill_benchmark_storage_guide.md`** — storage-and-naming standard for a hub `benchmark/` tree: sibling run-label folders, the frozen `baseline/` anchor, what lands in each run-label dir, and the hard boundary that the report `.md` is renderer-owned. The D1-D5 measurement contract is cross-linked, never restated.
- **`assets/skill_benchmark/skill_benchmark_readme_template.md`** — fillable scaffold for a hub `benchmark/README.md` run-label index. It templates the index of runs only, never the rendered per-run report.
- **`references/model_benchmark/model_benchmark_fixture_guide.md`** — authoring guide for Lane B inputs: the three fixture families and their shape markers, the profile shape and sweep keys, the reviewer-fixture shape at a glance, and the lane boundary for scoring.
- **`assets/model_benchmark/model_benchmark_code_task_fixture_template.md`** — scaffold for one tiered code-task oracle fixture (`fn_name` + `tests[]` + `hidden_tests[]`); oracle values come from a verified reference implementation.
- **`assets/model_benchmark/model_benchmark_pattern_fixture_template.md`** — scaffold for the two non-code-task shapes: the pattern evidence-contract fixture and the reviewer-prompt capability fixture.
- **`assets/model_benchmark/model_benchmark_profile_template.md`** — scaffold for a Lane B run profile (fixtures, models, frameworks, scoring, sampling, gate) authored against the lane's profile validator.
- **`SKILL.md` §10 SKILL-BENCHMARK STORAGE AND INDEX and §11 MODEL-BENCHMARK FIXTURES AND PROFILES** — the in-contract summaries of the two new families, each mirroring §9's shape: what it is, package/artifact shape, template map, authoring workflow, ALWAYS/NEVER (including the renderer-owned and lane-local boundaries), and success criteria.
- **`SKILL.md` §12 REFERENCES** — a new REFERENCES H2 routing to the packet's family guides and the lane-owned normative contracts (behavior framework, Lane C scoring contract and renderer, Lane B evaluator and reviewer schema).

## What Changed

- **Smart Routing header normalized.** The merged `## 1. WHEN TO USE + SMART_ROUTING` header was split into `## 1. WHEN TO USE` and `## 2. SMART ROUTING`, and the Benchmark Families table plus the routing/decision content moved under SMART ROUTING. The families table gained an OWNS-vs-ROUTES column covering all six families (four authored, two code-owned non-goals). All H2s were renumbered contiguously (MCP-promotion sections 2-7 → 3-8, behavior 8 → 9), and intra-file `section N` cross-refs were updated to match. The packet now passes `package_skill.py --check` (previously failed on the missing `SMART ROUTING` and `REFERENCES` sections).
- **Frontmatter.** `version` bumped 1.1.0.0 → 1.2.0.0; the description and Keywords comment now cover all four authored families.

## What Fixed

- Updated the two cross-file `SKILL.md` section-number references that the renumber made stale: `references/behavior_benchmark/behavior_benchmark_guide.md` (MCP promotion "sections 2 through 7" → "3 through 8") and `assets/shared/benchmark_report_template.md` (report contract "Section 4" → "Section 5").

## Known Pre-Existing Issues (not changed here)

- `package_skill.py --check` still emits advisory warnings (non-blocking): the packet has no `INTEGRATION POINTS` / `RELATED RESOURCES` recommended sections, `SKILL.md` exceeds the 3000-word recommended soft cap (it is under the 5000-word hard cap), and the SMART ROUTING section does not carry the `discover_markdown_resources` / `_guard_in_skill` / `UNKNOWN_FALLBACK` pseudocode markers. The last is expected: this is a prose-routed workflow packet with no machine router, so the canonical router pseudocode does not apply.
- The broken `assets/benchmark/...` template path noted in `v1.1.0.0.md` still appears in `changelog/v1.0.0.0.md` (a historical record) and `references/shared/worked_example.md` (outside this release's scope). Flagged for a follow-up link sweep.

## Notes

This packet has no packet-local `graph-metadata.json` and no bound `/create:` slash command; the single advisor identity lives at the `sk-doc` hub root. The skill-benchmark and model-benchmark artifacts are authored into their owning lane (`system-deep-loop`), not into this packet — this packet ships only the templates and the guides.
