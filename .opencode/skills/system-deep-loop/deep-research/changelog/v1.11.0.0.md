This release lands the deep-research portion of the packets 108 plus 110 auto-review uplift program. Two reusable teachings from the upstream `dzianisv/opencode-plugins` auto-review package now apply to every deep-research iteration: a role-banner marker at the rendered-prompt boundary, and an anti-repetition rule across iterations. The synthesis content-hash dedup that shipped in deep-review v1.8.0.0 is intentionally NOT applied here because deep-research is single-dimension by design and does not produce the cross-dimension restatements that motivated H-7. Both mechanisms cleared a 5-of-5 unanimous deep-ai-council review by `cli-codex gpt-5.5 xhigh fast` before any code shipped.

## New Features

#### DEEP-RESEARCH marker at the rendered-prompt boundary

Long research iterations occasionally drifted out of role when the banner scrolled past the visible context window. The fix places a `DEEP-RESEARCH` marker on line 1 of `prompt_pack_iteration.md.tmpl`, where it stays at the top of the rendered prompt regardless of template content. A matching `step_marker_scan` runs in the auto dispatcher YAML and fails fast when the marker is missing after rendering. Reference templates that ship YAML frontmatter were deliberately left untouched, because a stray marker inside frontmatter would corrupt the loader.

&nbsp;

#### Anti-repetition rule across iterations

The CONSTRAINTS section of the deep-research SKILL.md now prohibits restating a finding already reported in a prior iteration of the same research arc. Researchers must either omit the duplicate or extend the prior entry with new evidence. This is a prompt-level rule so the researcher carries the intent through iteration n while still seeing iterations 1 through n minus 1 in context.

---

## Why H-7 dedup is NOT applied to deep-research

The synthesis content-hash dedup that landed in deep-review v1.8.0.0 keys partly on a `dimension` field. Deep-research iterations are single-dimension — there is no second-dimension restatement to collapse — so applying the same dedup mechanism would add metadata fields without behavioral payoff. This decision is recorded for cross-reference with the deep-review v1.8.0.0 changelog where the dedup mechanism is documented in full.

## Council-Gated Workflow

Packet 108 ran through a 5-seat deep-ai-council deliberation before any source change. The council returned REQUEST-CHANGES on the first pass with HIGH confidence. The revisions that landed include: H-2 markers placed on `prompt_pack_iteration.md.tmpl` line 1 (not inside frontmatter-bearing reference templates); H-4 anti-repetition added to CONSTRAINTS rather than to a softer "guidance" section so the rule is binding. Implementation followed via `cli-opencode + deepseek-v4-pro --pure`.
