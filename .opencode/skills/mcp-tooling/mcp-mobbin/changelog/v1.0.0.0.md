# v1.0.0.0 - First Release

**Released:** 2026-07-16

First release of the `mcp-mobbin` skill, the read-only Mobbin design-research TRANSPORT packet of the `mcp-tooling` hub (`packetKind: transport`, `mutatesWorkspace: false`). It teaches an agent to search **Mobbin's library of real app UI screenshots** through a `mobbin` Code Mode manual — **drafted but NOT yet registered** (registration is a later phase) — and to hand every design-affecting conclusion to `sk-design`, the mandatory cross-hub judgment partner. The packet is grounded exclusively in the completed three-lineage deep-research fan-out synthesis of the Mobbin MCP developer surface, and it is the closest structural sibling of `mcp-refero`: remote-MCP-via-`mcp-remote`, read-only, OAuth-gated, no CLI machinery.

### Key design decisions (from the research synthesis)

- **One documented tool, four intents**: `search_screens` (`query`, `platform` `ios`|`web`, `limit` default 5 / ~15 ceiling) is the complete public baseline; app, screen, flow, and element research are query intents over it, never invented tool families. Flow sequence is reconstruction, always labeled inference.
- **No API key, answered in the negative**: Mobbin MCP auth is browser OAuth only (DCR RFC 7591, PKCE S256, `openid`, live-probed 401 challenge). The manual `env` is intentionally empty; no `MOBBIN_API_KEY` or auth env var exists to name, and the separate REST workspace key never enters MCP wiring.
- **Pre-registration honesty**: the `mobbin` manual is a paste-ready DRAFT (byte-identical across two research lineages), NOT registered; `doctor.sh` reports absence as expected INFO, and the packet never registers or edits `.utcp_config.json`.
- **Inferred callable**: `mobbin.mobbin_search_screens` is convention-derived, never observed live; post-registration `tool_info` confirmation is mandatory, fail-closed on drift, mutation-capable tools refused outright.
- **Documented limits only**: plan gate Pro/Team/Enterprise (Free excluded; denial semantics unverified); 60 requests / 60 seconds / user with `Retry-After` + backoff on 429; per-plan caps and Finance+ coverage left undocumented.
- **Ten open questions carried as UNKNOWN**, including the authenticated tool inventory, the `deep` input-vs-server-behavior conflict (preserved, not resolved), inline-image fidelity through Code Mode, and the end-to-end OAuth round trip.

### Boundaries

- Minimal runtime `allowed-tools` (`Read, Bash, Grep, Glob, mcp__code_mode__call_tool_chain`); Write, Edit, and Task are forbidden for this transport.
- The official `mobbin/skills` repository (single `mobbin-search` skill) is referenced, not vendored; its optional evidence-board writer is excluded by the read-only boundary.
- No hub files are modified by this packet's release; hub registration (mode-registry, hub-router, hub docs) and the `.utcp_config.json` manual application ship in later phases.
