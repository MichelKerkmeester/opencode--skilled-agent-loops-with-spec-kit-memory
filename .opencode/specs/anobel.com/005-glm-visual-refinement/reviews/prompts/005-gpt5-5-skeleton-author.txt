## 2. PROBLEM & PURPOSE

### Problem Statement
The deterministic geometry skeleton (phase 004) recovers many 2D-positioned tiles, but the hardest ones — dense matrix / node-diagram / branch / connector-map / popover tiles — still fail the phase-001 spatial gate (RC-1 overflow, RC-2 node collisions, RC-3 title-zone break) even after the best-of-3 deterministic recompute. For those residual high-value 2D tiles there is no path between "deterministic skeleton failed" and "downgrade to a linear primitive", so genuinely diagram-shaped content is lost rather than recovered. Linear-flow tiles already score 86-94 and need no help.

### Purpose
Add a cost-capped escalation that hands the hardest residual 2D-risk tiles to GPT-5.5 as a geometry-skeleton author: it computes a coordinate/safe-zone skeleton (the same JSON schema phase 004 emits) that phase 004 then compiles to a locked scaffold for GLM-5.2 to render — recovering diagram tiles that the deterministic skeleton cannot, without spending a paid model on linear tiles that are already strong.

> Grounding (cite RC-ids): A6 final recommendation `../../004-bento-visuals/research/iterations/iter-r2-A6.md`; pipeline step 8 escalation `../../004-bento-visuals/research/iterations/iter-r4-pipeline.md`; cost ceiling `../../004-bento-visuals/research/iterations/iter-r4-risk.md`; dispatch contract `.opencode/skills/cli-opencode/SKILL.md`.

## 3. ARCHITECTURE

### Pattern
Escalation provider behind a deterministic default (strategy + failure-triggered fallback). The deterministic skeleton author stays the default; GPT-5.5 is the escalation author, fired only after deterministic failure on a qualifying tile (AdaCoder adaptive-planning principle, iter-r2-A6).

### Key Components
- **EscalationClassifier**: applies `trigger_gpt55_if` / `skip_gpt55_if` to the tile's primitive, item count, connector needs, and the phase-004 failure state; emits a per-tile decision + reason.
- **Gpt55SkeletonAuthor**: builds the skeleton-author prompt, runs the cli-opencode dispatch, and validates the returned JSON against the phase-004 schema.
- **CostCapAccountant**: enforces `<=1` call and `<=4200` tokens per tile and the batch budget derived from the pre-spend `N_2D`; refuses + logs over-cap calls.
- **SkeletonAuthorRouter**: in `gen-tile.mjs`, routes the 2D skeleton step deterministic-author -> (on failure + escalate) GPT-5.5-author -> (on failure) downgrade-to-linear (phase 004).

### Data Flow
1. Phase 002 labels the tile 2D-positioned; phase 004 computes the deterministic skeleton; GLM renders; phase 001 gate runs.
2. On gate failure, phase 004 best-of-3 recompute runs; if it still fails, control reaches the EscalationClassifier.
3. If `trigger_gpt55_if` holds and `skip_gpt55_if` does not, and the batch budget allows, dispatch GPT-5.5 to author a skeleton.
4. Validate the JSON skeleton -> hand to the unchanged phase-004 compile + GLM-render path -> re-run the phase-001 gate.
5. Pass -> SHIP; fail -> downgrade-to-linear (phase 004). Either way the CostCapAccountant records the spend.

### Exact dispatch (cli-opencode)
Dispatched from the bento generation harness (a Node process), which is the legitimate external-dispatch use case - not an OpenCode self-invocation. If the harness ever runs inside OpenCode, route the openai dispatch through a sibling cli-* / fresh shell per the cli-opencode self-invocation guard.

```bash
# Provider auth pre-flight first (once per run): require openai; never substitute silently.
PROVIDERS=$(opencode providers list 2>&1)
echo "$PROVIDERS" | grep -q "openai" && OPENAI_OK=1 || OPENAI_OK=0
# OPENAI_OK=0 -> disable escalation, fall back to deterministic skeleton + downgrade-to-linear, ASK operator.
# Confirm the live slug before first dispatch: opencode models openai

opencode run \
  --model openai/gpt-5.5-fast \
  --variant xhigh \
  --format json \
  --dir "$REPO_ROOT" \
  "$SKELETON_AUTHOR_PROMPT" </dev/null
# Notes:
#   - openai/gpt-5.5-fast = low-latency Fast variant (OpenAI priority service tier); paid premium provider.
#   - --variant xhigh = max reasoning effort (gpt-5.5 effort range tops out at xhigh).
#   - omit --agent (top-level --agent is rejected); the author role is stated in the prompt body.
#   - </dev/null gives stdin EOF so a non-interactive run does not hang.
```

### Skeleton-author prompt (geometry only; returns JSON, never tile code)
Faithful to iter-r2-A6; the canvas contract + coordinate system are the locked phase-004 zones.

```text
You are the geometry skeleton author for a 480px-high maritime B2B dashboard bento tile.
You do not write final UI code. You produce a prevalidated JSON geometry skeleton that will be
compiled into locked HTML/CSS for GLM-5.2 to render.

Hard canvas contract:
- Tile height is 480px.
- diagramFrame = { x: 24, y: 24, w: "calc(100% - 48px)", h: 312, maxY: 336 }.
- titleFrame   = { x: 24, y: 368, w: "calc(100% - 48px)", h: 88 }.
- All diagram content fits inside diagramFrame; nothing may enter y=336..368; title renders bottom-left in titleFrame.

Coordinate system:
- BBoxes are relative to diagramFrame. x and w use 0..1000 horizontal units scaled to diagramFrame width; y and h are pixels 0..312.
- A bbox is valid only if x >= 0, y >= 0, x + w <= 1000, y + h <= 312.

Diagram constraints:
- Min node gap 16px; min text-to-node gap 12px; at most 4 visible rows (collapse remainder to "+N more"); row height 36-42px; row gap 6-8px.
- Edge labels and status pills get their own bbox and may not overlap nodes; connectors use explicit anchors (top/right/bottom/left/center); prefer orthogonal.
- Eyebrow is Title Case; avoid CSS text-transform uppercase; use the intended anchor glyph only.

Return JSON only. No prose.
```

The returned object is the phase-004 skeleton schema: `layoutPrimitive`, `riskFlags`, `zones {diagramFrame, titleFrame}`, `coordinateSystem`, `nodes[]`, `rows[]`, `edges[]`, `overflowPlan`, `textRules`, `validation`.

### Escalation trigger + cost cap
Source: iter-r2-A6 `trigger_gpt55_if` / `skip_gpt55_if` / `cost_caps`; batch ceiling from iter-r4-risk.

```yaml
escalation:
  # Precondition: tile is 2D-positioned AND the deterministic skeleton path (incl. best-of-3) failed the phase-001 gate.
  trigger_gpt55_if:
    - layoutPrimitive in [matrix, flow, hub, branch, popover, node-diagram, network, connector-map]
    - itemCount > 4 and layoutPrimitive not in [linear, stack, simple-kpi, text-card, timeline]
    - brief requires cross-links, connector lines, branching, overlays, pills, edge labels, or popovers
  skip_gpt55_if:
    - layoutPrimitive in [linear, stack, simple-kpi, text-card, timeline]
    - no connector geometry
    - no more than 4 visible rows
    - tile already passes the phase-001 gate

cost_caps:
  max_gpt55_calls_per_2d_tile: 1
  max_gpt55_tokens_per_2d_tile: 4200
  no_gpt55_for_linear_tiles: true
  # Batch: report N_2D (qualifying failed tiles) BEFORE spend; abort batch if projected paid calls exceed N_2D.
  # GLM render cost = 0 (Z.AI Coding Plan subscription); GPT-5.5 (openai) is paid.
  # Worst case if all ~18 baseline failures are high-risk 2D: <=18 GPT-5.5 calls, <=75.6K tokens (18 x 4.2K).
  # Exact dollar ceiling is UNKNOWN without live token prices: track paid_calls x avg_tokens x provider_price.
```
