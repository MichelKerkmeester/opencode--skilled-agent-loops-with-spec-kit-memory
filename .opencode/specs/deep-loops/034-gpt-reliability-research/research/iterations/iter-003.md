### FINDINGS

1. The LEAF prompt tells the recipient it may be either the dispatched agent or the CLI executor, then assigns the LEAF role directly. `deep-review` says the prompt is for the `@deep-review` LEAF agent “or a CLI executor” and later says “You are a LEAF agent. Do NOT dispatch sub-agents” (`.opencode/skills/deep-loop-workflows/deep-review/assets/prompt_pack_iteration.md.tmpl:6`, `:57`). `deep-research` has the same shape (`.opencode/skills/deep-loop-workflows/deep-research/assets/prompt_pack_iteration.md.tmpl:6`, `:33`). That wording makes “I am the executor, therefore I am the LEAF” a literal reading, especially at lower effort.

2. The dispatch instruction exists as YAML prose/branch structure, but the proof gate is after the work, not a mandatory pre-work receipt. `deep_review_auto.yaml` names `step_dispatch_iteration` and a native `dispatch: agent: deep-review` branch (`.opencode/commands/deep/assets/deep_review_auto.yaml:765-816`), but `post_dispatch_validate` only runs after artifacts are expected (`:928-940`). The CLI branch directly runs `opencode run` with the rendered LEAF prompt, not a Task dispatch to `deep-review` (`:904-914`). Research mirrors this direct CLI path and post-validation route-proof block (`.opencode/commands/deep/assets/deep_research_auto.yaml:924-957`).

3. Route-proof is self-attested by the artifact writer. The prompt tells the LEAF to append JSON containing `target_agent`, `agent_definition_loaded`, and `resolved_route` (`.opencode/skills/deep-loop-workflows/deep-review/assets/prompt_pack_iteration.md.tmpl:79-83`, `:107-112`; `.opencode/skills/deep-loop-workflows/deep-research/assets/prompt_pack_iteration.md.tmpl:46-60`). The validator only checks those fields for presence/equality (`.opencode/skills/deep-loop-runtime/lib/deep-loop/post-dispatch-validate.ts:619-665`, `:1384-1393`). It does not verify that a Task/Agent dispatch event happened.

4. Runtime audit records provenance/failures, not an unforgeable LEAF-dispatch receipt. Native dispatch is simply allowed (`.opencode/skills/deep-loop-runtime/lib/deep-loop/executor-audit.ts:475-482`), non-native dispatch env/provenance is appended separately (`:537-553`, `:582-614`), and failures are logged as JSONL events (`:625-645`). None of these bind the route-proof fields to an actual `Task` event or child-agent receipt.

5. The benchmark catches the failure by comparing independent signals: task events are scraped from transcript dispatch lines (`.opencode/skills/deep-loop-workflows/shared/behavior-benchmark/behavior-bench-run.cjs:656-668`), route-proof is scraped from fixture `*state.jsonl` files (`:416-456`), and `role_absorption` is assigned when work products appear without delegation evidence (`:337-366`). `RSB-007-med` shows the exact divergence: `tFirstDispatchMs: null`, `taskEvents: []`, a matching `deep-research` route-proof record, and `classification: "role_absorption"` (`.opencode/specs/deep-loops/033-deep-loop-behavior-benchmarks/003-rollout-research-context/runs/gpt-fast-med/RSB-007-gpt-fast-med.result.json:7-24`, `:53-59`). `RSB-005-med` shows no dispatch/proof and hard timeout (`.opencode/specs/deep-loops/033-deep-loop-behavior-benchmarks/003-rollout-research-context/runs/gpt-fast-med/RSB-005-gpt-fast-med.result.json:7-17`, `:33-43`).

### PROPOSALS

1. **Tag:** re-approach. **Change:** In `.opencode/commands/deep/assets/deep_review_auto.yaml` and `.opencode/commands/deep/assets/deep_research_auto.yaml`, add a hard `pre_dispatch_receipt` step before the LEAF prompt can run. The receipt must be produced by the dispatch mechanism, not by the model-written state log: native Task event id for `agent: deep-review/deep-research`, or audited child process receipt from `runAuditedExecutorCommand` for CLI. Pass `requiredDispatchReceipt` into `post-dispatch-validate.ts` and reject artifacts if receipt is absent before the state-log append. **Expected effect:** RVB-007, RSB-005, and RSB-007 pass at medium effort because inline work cannot satisfy the gate; the executor must dispatch or abort before doing specialist work. **Effort:** M. **Risk:** Requires runtime/tool integration for native Task receipt capture.

2. **Tag:** simplify. **Change:** Remove route-proof authorship from both LEAF prompt packs. In `.opencode/skills/deep-loop-workflows/deep-review/assets/prompt_pack_iteration.md.tmpl` and `.opencode/skills/deep-loop-workflows/deep-research/assets/prompt_pack_iteration.md.tmpl`, delete `target_agent`, `agent_definition_loaded`, and `resolved_route` from the model-authored JSON examples, and say those fields are workflow-owned and injected/validated after dispatch. Update `post-dispatch-validate.ts` to compare receipt-derived route fields, not self-written fields. **Expected effect:** RVB-007 and RSB-007 no longer pass via fabricated JSONL; RSB-005 is pushed toward real dispatch because the inline executor lacks the route fields it needs. **Effort:** M. **Risk:** Existing reducers/tests expecting route fields in LEAF-authored records need adjustment.

3. **Tag:** optimize. **Change:** In both prompt packs, replace “for the LEAF agent or a CLI executor” with a strict boundary: “If you are reading this without a dispatch receipt supplied by the workflow, stop and write no findings.” Also add an explicit first-line abort rule before the output contract: “Producing findings without a dispatch receipt is role absorption.” **Expected effect:** RVB-007, RSB-005, and RSB-007 pass at medium effort by making the literal interpretation safer; lower-effort GPT gets a deterministic stop condition instead of absorbing the role. **Effort:** S. **Risk:** Prompt-only hardening is weaker than receipt enforcement and should not be the only fix.

4. **Tag:** re-approach. **Change:** Move CLI execution for review/research through `executor-audit.ts#runAuditedExecutorCommand` everywhere, including the current direct `if_cli_opencode` branches in `deep_review_auto.yaml` and `deep_research_auto.yaml`. The wrapper should write `dispatch_started` and `dispatch_completed` events with executor kind, prompt hash, child pid/session id, and route fields, then `post-dispatch-validate.ts` should require those events for non-native executors. **Expected effect:** RVB-007, RSB-005, and RSB-007 pass at medium effort because CLI self-work is only accepted when launched through the audited dispatch path. **Effort:** M. **Risk:** Child-process metadata may be flaky across CLI versions unless kept minimal.

5. **Tag:** optimize. **Change:** Strengthen `.opencode/skills/deep-loop-workflows/shared/behavior-benchmark/behavior-bench-run.cjs` so fixture work detection includes modified existing files, not only new files. The current `fixtureGained` uses `newFixtureFiles.length > 0` (`:754-756`), which can under-detect absorption when a run rewrites existing packet artifacts. Compare before/after hashes or mtimes for fixture files. **Expected effect:** RVB-007, RSB-005, and RSB-007 medium-effort regressions are classified consistently as absorption/timeout instead of partial/refused ambiguity, making the protocol fix measurable. **Effort:** S. **Risk:** More sensitive to expected fixture churn; needs ignore rules for benign timestamp-only noise.
