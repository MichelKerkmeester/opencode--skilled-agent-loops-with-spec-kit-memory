## Summary

Packet `120/003-minimax-prompt-framework-benchmark` reused the 113 SWE-1.6 eval rig (7 fixtures, 5-dim rubric, claude grader) with the dispatch layer swapped to `opencode run --model minimax/MiniMax-M2.7`. It ran 7 iterations / 49 real MiniMax dispatches across 5 prompt frameworks + a pre-planning hill-climb.

**Winner: TIDD-EC framework + dense pre-planning (0.775).** Two divergences from SWE-1.6 (which preferred RCAF + medium):

| Axis | MiniMax M2.7 winner | SWE-1.6 winner |
|------|---------------------|----------------|
| Framework | TIDD-EC (0.767) > RCAF (0.742) | RCAF |
| Pre-planning density | dense (0.775) > medium (0.767) | medium |

Full ranking: TIDD-EC 0.767 > COSTAR 0.762 > RCAF 0.742 > RACE 0.704 > CIDI 0.683.

## Caveats

- `fix-003` had no node_modules → its D2 hard-gated uniformly across all variants (affects absolute scores equally, not the relative ranking).
- Grader ran in `fallback_fenced` parse mode; single sample per (variant, fixture). Margins (0.008-0.03) are above the ~0.02 noise floor; a re-run would tighten confidence.
- Not cross-model-validated (MiniMax is the only model on this provider path); no propagation to cli-devin/codex cards (MiniMax is cli-opencode-only).
