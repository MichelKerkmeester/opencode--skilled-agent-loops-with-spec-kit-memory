## How It Was Delivered

Planned approach (not yet executed). The four OpenCode artifacts will be authored by copying the shape of the sibling `flowchart` command and retargeting every step at the create-diff engine, so the command stays a thin orchestration layer over `create_diff.py` and `validate_report.py`. The engine is consumed as a CLI, never re-implemented: the workflow resolves the operator's inputs, shells out to `compare`/`compare-pair`, validates the produced report, and presents the result — the source document is read-only throughout. After the router and assets exist, the Codex stub is generated (not hand-written) via `sync-prompts.cjs`, and the `mode-registry.json` `command` field is flipped from `null` to `"/create:diff"`. Verification then confirms routing, asset existence, the tool-surface subset, the engine-never-bypassed invariant, the registry wiring, stub idempotency, and recursive strict validation.

<!-- ANCHOR:decisions -->
## Key Decisions

| Decision | Why |
|----------|-----|
| Follow the full sibling pattern (router + presentation.txt + auto.yaml + confirm.yaml) even though create-diff is engine-backed | Consistency with all ten existing `/create:*` commands: one predictable shape for operators and canon gates. The operator explicitly chose the full pattern over a minimal engine-only wrapper. |
| Keep the command a thin layer over the engine; never re-implement diffing | The comparison, extraction, snapshotting, and rendering are already shipped and hardened (phases 002–008); duplicating any of it in the command layer would create a second, drift-prone source of truth. REQ-003 freezes this as a P0 invariant. |
| Generate the Codex stub rather than hand-author it | `sync-prompts.cjs` already emits Codex prompt stubs from OpenCode commands; a generated stub stays idempotent and free of hand-edit drift, matching how the other commands' Codex prompts are produced. |
| `:confirm` is the default mode when no mode token is given | Matches the sibling grammar (`flowchart` and the other `/create:*` commands default to the interactive, checkpointed workflow); `:auto` is opt-in for gate-free runs. |
| `allowed-tools` is a strict subset of the mode surface with no `Task` | The router must not widen the create-diff mode's capability surface; excluding `Task` preserves the LEAF, no-sub-agent boundary the mode declares. |
