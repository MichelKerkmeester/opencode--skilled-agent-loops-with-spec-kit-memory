## Concrete Spec-Kit-native adoption sketch (027/004 + 027/005)
027/004 causal-edge tombstone lifecycle for SQLite:

Add minimal state to causal edges, or a sidecar table if altering hot rows is risky. Preferred columns on `causal_edges`: `edge_state TEXT NOT NULL DEFAULT 'active' CHECK(edge_state IN ('active','tombstoned'))`, `tombstoned_at TEXT NULL`, `tombstone_reason TEXT NULL`, `tombstoned_by TEXT NULL`, `tombstone_run_id TEXT NULL`, `purge_after TEXT NULL`, `source_kind TEXT NOT NULL DEFAULT 'manual'`, `source_ref TEXT NULL`, `source_hash TEXT NULL`. Keep a `causal_edge_tombstones` append-only audit table keyed by `edge_id` or by the natural edge key if hard-delete removes the row.

Algorithm: every active causal-edge delete path must run one transaction. First `UPDATE causal_edges SET edge_state='tombstoned', tombstoned_at=COALESCE(tombstoned_at, :now), tombstone_reason=COALESCE(tombstone_reason, :reason), tombstoned_by=COALESCE(tombstoned_by, :actor), tombstone_run_id=COALESCE(tombstone_run_id, :run_id), purge_after=COALESCE(purge_after, :purge_after) WHERE edge_id IN (...) AND edge_state='active'`. Then `INSERT OR IGNORE` into `causal_edge_tombstones` with the same first-tombstone snapshot. Only after that may a purge job `DELETE FROM causal_edges WHERE edge_state='tombstoned' AND purge_after <= :now`.

Idempotent re-run: repeated tombstone calls do not rewrite the first tombstone timestamp or extend grace. Repeated purge deletes zero rows after the first successful hard-delete. This intentionally improves on MemClaw’s `memory_soft_delete`, which rewrites `deleted_at` on direct repeat.

027/005 deterministic metadata-edge promotion guard:

“Validated” means all of these pass before inserting a causal edge: relation type is in an allowlist such as `parent`, `child`, `parent_chain`, `phase_child`; source frontmatter parsed successfully; source and target IDs resolve to existing Spec Kit memory/spec records; no self-edge; parent-chain edges do not create a cycle; target status is not deleted/tombstoned; relation source is deterministic frontmatter or graph metadata, not inferred co-occurrence; the source document hash matches the parsed relation snapshot used for promotion.

Provenance should be explicit: `source_kind='metadata_promoter'`, `generated_by='metadata-edge-promoter'`, `source_ref='<spec-or-memory-id>#frontmatter.<field>'`, `source_hash=<frontmatter hash>`, `promoter_version=<version>`. Manual causal links keep `source_kind='manual'`.

Duplicate prevention: enforce a natural active-edge key like `(from_id, to_id, edge_type)` for active causal edges. On promotion, if an active manual edge with the same natural key exists, skip promotion and record `skipped_reason='manual_edge_exists'`; never add a generated duplicate. If an active generated edge from the promoter exists, update only `source_hash`, `last_validated_at`, and confidence/status metadata. If a tombstoned generated edge exists and the same validated metadata relation reappears, create a new active edge only if policy allows resurrection, and link it to the prior tombstone audit row.

## Negative knowledge (confirmed)
Do not port MemClaw’s multi-tenant/fleet lifecycle fanout. It exists because core-operations posts action fanouts and core-api publishes per-org messages: `core-operations/src/core_operations/tasks.py:24-85`, `core-api/src/core_api/services/lifecycle_audit.py:227-236`. Spec Kit Memory is local SQLite and should use a single local transaction plus local audit rows.

Do not port Pub/Sub redelivery machinery. MemClaw’s handlers are designed around event redelivery, DLQ, and per-delivery audit transitions: `common/events/lifecycle_handlers.py:127-139`, `common/events/lifecycle_handlers.py:217-246`. Spec Kit needs idempotent local reruns, not distributed consumer semantics.

Do not port fleet scoping or org hard-purge tables. MemClaw has tenant/fleet hard-purge table lists and explicitly preserves lifecycle audit for org deletion observability: `core-storage-api/src/core_storage_api/services/postgres_service.py:162-193`, `core-storage-api/src/core_storage_api/services/postgres_service.py:195-223`. Spec Kit should scope by local memory/spec IDs.

Do not port co-occurrence-inferred `related_to` as causal truth. It is created from shared memory links above a threshold and lacks `evidence_memory_id` on insert: `core-api/src/core_api/pipeline/steps/entity_linking/infer_relations.py:35-74`, `core-api/src/core_api/pipeline/steps/entity_linking/infer_relations.py:147-164`.
