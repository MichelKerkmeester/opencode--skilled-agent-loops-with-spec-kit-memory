# v4.1.0.0 — two-axis hub: surface packets added

sk-code becomes a **two-axis** parent hub. The four workflow modes plus review are unchanged in what they do; alongside them the hub now carries three read-only **surface evidence packets** and bundles zero-or-more of them next to the primary workflow mode. The Y-segment bump reflects packets added within the v4 hub architecture — no breaking change to the routing contract (Lane C's projection ignores unknown keys).

## The two axes

- **Workflow axis** (modes that act): `implement`, `quality`, `debug`, `verify`, `review`.
- **Surface axis** (read-only domain evidence): `webflow`, `opencode`, `animation` — `packetKind: surface`, `backendKind: evidence-base`, read-only `toolSurface`, `routingClass: metadata` (advisor-invisible). Declared via `extensions.surface-axis`.

The hub resolves one workflow mode as primary and bundles surfaces as evidence through the new `routerPolicy.outcomes.surfaceBundle` outcome (workflow ordered first, surfaces after). "review my webflow animation for jank" → `[review, webflow, animation]`.

## Restructure

- Surface evidence consolidated out of the workflow modes into `webflow/`, `opencode/`, `animation/` packets (each with `SKILL.md`, `README.md`, `references/`, `assets/`, `changelog/`). The workflow modes keep only their process contracts, universal checklists, and verify scripts.
- `code-review` folded to `review` (packet, `packetSkillName`, and SKILL name renamed; the `code-review` keyword is retained for search). The rule-canary CI path, both deep-review agents, the prompt-pack template, and `check-rule-copies.js` (+ its test) were repointed in the same change.

## Wiring

- `mode-registry.json` (→ 1.1.0) — `packetKind` on every mode, three surface entries, `extensions.surface-axis`, review repoint; the surface aliases (`webflow implementation`, `opencode system code`, `motion.dev animation`) moved from `implement` to their own surfaces.
- `hub-router.json` (→ 1.1.0) — surface `routerSignals` + owned vocabulary classes; `surfaceBundle` outcome and a full `tieBreak`. The vocabulary was reconciled so `parent-hub-vocab-sync` passes clean (0 orphan aliases, 0 collisions, 0 ownership drift — previously 13/4/1).
- `shared/references/smart_routing.md` — every RESOURCE_MAP and prose path normalized to hub-root-relative, packet-qualified form; `router-replay.cjs` and the router-sync guard updated to match.
- Hub `SKILL.md` (→ 4.1.0.0), `README.md` (→ 4.1.0.0), `description.json` (→ 4.1.0.0), and `graph-metadata.json` repointed to the new layout.
