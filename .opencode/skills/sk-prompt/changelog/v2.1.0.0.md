This release makes `sk-prompt` cleanly forkable by removing the one asset that was not generic. The CLI-dispatch-specific prompt quality card moved out to the model-craft hub, leaving `sk-prompt` as a pure, fork-portable prompt-framework engine with no small-model or CLI-dispatch ties.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/130-prompt-knowledge-layering/013-card-relocation-and-guard/` (Level 1)

---

## What Changed

#### Architecture

- **CLI quality card moved out** -- `cli_prompt_quality_card.md` was a CLI-dispatch-specific asset sitting inside the otherwise-generic `sk-prompt` framework engine, which blocked a clean standalone fork. We moved it via git rename to `sk-prompt-small-model/assets/`, so `sk-prompt` now owns only the generic framework knowledge: the 7-framework definitions plus CLEAR scoring (`references/patterns_evaluation.md`), DEPTH thinking (`references/depth_framework.md`) and the format guides plus framework registry in `assets/`. The asset folder was verified to carry only generic engine assets.
- **Graph metadata cleaned** -- with the card gone, `graph-metadata.json` still listed it in `enhance_when`, `key_files`, `entities` and `source_docs`, which would have left dangling references. The card was removed from all four, the file still validates as JSON and no dangling references into the moved card remain, verified.
