# Implementation Plan: 027 Spec-Tree Six-Track Consolidation

<!-- SPECKIT_LEVEL: 2 -->
<!-- SPECKIT_TEMPLATE_SOURCE: plan-core | v2.2 -->

---

<!-- ANCHOR:summary -->
## 1. SUMMARY
A move-only structural reorganization: collapse 31 flat top-level children into six themed parents (`000-release-cleanup` kept; five new tracks), renumber each parent's children contiguously, re-derive identity metadata, restructure the changelog, and realign the root tracking docs. Modeled on the `026-graph-and-context-optimization` Wave-4 consolidation.
<!-- /ANCHOR:summary -->

---

<!-- ANCHOR:quality-gates -->
## 2. QUALITY GATES
- `validate.sh --recursive` on the 027 root: 0 errors.
- Grep sweep: zero stale old top-level paths in canonical navigable docs.
- `git show --stat HEAD` confined to the 027 subtree.
<!-- /ANCHOR:quality-gates -->

---

<!-- ANCHOR:architecture -->
## 3. ARCHITECTURE
Three deterministic scripts: (1) `git mv` the 30 phases + relocate untracked leftovers; (2) bare-prefix textual path rewrite plus deterministic identity-field re-derivation across the moved tree; (3) lean-trio generation for the five parents. Root docs and the changelog are realigned by hand; `timeline.md` is regenerated by `scratch/gen-timeline.py`.
<!-- /ANCHOR:architecture -->

---

<!-- ANCHOR:affected-surfaces -->
## FIX ADDENDUM: AFFECTED SURFACES
- The six themed parents and all moved phase subtrees.
- Root: `spec.md`, `graph-metadata.json`, `description.json`, `before-vs-after.md`, `context-index.md`, `timeline.md`, `changelog/`.
- Parent wiring: `000-release-cleanup/graph-metadata.json` + `description.json` gain this child.
<!-- /ANCHOR:affected-surfaces -->

---

<!-- ANCHOR:phases -->
## 4. IMPLEMENTATION PHASES
1. Scoped baseline commit of daemon metadata churn.
2. Create the consolidation spec folder; wire into the parent.
3. Create five themed parents (lean trio).
4. `git mv` + path rewrite for all 30 phases.
5. Restructure `changelog/`; realign root docs; regenerate `timeline.md`.
6. Validate (`--recursive`, `--strict`, grep sweep); scoped commit.
<!-- /ANCHOR:phases -->

---

<!-- ANCHOR:testing -->
## 5. TESTING STRATEGY
`validate.sh --recursive` (tree-wide), `validate.sh --strict` on this folder, a grep sweep for stale paths over canonical docs, and a spot-check that representative grouped-parents (old 018 → 003/002, old 029 → 005/003) resolve their `parent_id`/`children_ids`.
<!-- /ANCHOR:testing -->

- `validate.sh` and the phase-parent detection helpers.

<!-- ANCHOR:rollback -->
## 7. ROLLBACK PLAN
The reorg lands as scoped commits on top of a clean baseline; `git reset --soft` / `git revert` of those commits restores the prior flat structure. Moves are history-preserving and reversible.
<!-- /ANCHOR:rollback -->

## L2: PHASE DEPENDENCIES
- Baseline commit MUST precede any `git mv` (clean rename-only diff).
- Parents MUST exist before children are moved into them.
- Path rewrite MUST follow the moves; `timeline.md` regeneration MUST follow all moves.
<!-- /ANCHOR:phase-deps -->

<!-- ANCHOR:enhanced-rollback -->
## L2: ENHANCED ROLLBACK
Because the daemon re-derives metadata continuously, a clean baseline commit isolates churn from the structural diff. If a rewrite error is found post-commit, re-run the deterministic identity re-derivation (idempotent) rather than reverting the whole move.
