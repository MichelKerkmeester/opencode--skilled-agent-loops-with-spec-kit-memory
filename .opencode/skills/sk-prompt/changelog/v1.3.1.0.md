# sk-prompt v1.3.1.0 — cross-CLI propagation of bundle-gate + anti-hallucination guidance

**Released**: 2026-05-17
**Status**: Documentation addendum to `assets/cli_prompt_quality_card.md`. Three new composition-guidance notes added to §3 (Pre-planning density softening, Bundle-gate strictness, Anti-hallucination wording) based on cross-model validation in packet 118.

---

## Summary

The 003-eval-loop run on cli-devin SWE 1.6 surfaced three composition-level findings beyond framework selection:

1. **Medium-density pre-planning beats dense pre-planning.**
2. **Standard bundle-gate language beats strict (verbose constraints push models toward defensive output).**
3. **Framework choice dominates anti-hallucination wording (~2.4× more impactful).**

Packet 117 propagated only the medium pre-plan finding cross-CLI, holding the other two pending cross-model validation. Packet 118 measured all five council-seeded variants against **deepseek-v4-pro** (via cli-opencode + DeepSeek direct API) and **kimi-k2.6** (via cli-devin's preset). Both held cross-model:

| Gate | deepseek-v4-pro Δ | kimi-k2.6 Δ | Verdict |
|------|-------------------|-------------|---------|
| Bundle-gate-aversion | +0.0525 | +0.0305 | HOLDS on both |
| Framework > anti-hallucination | +0.0732 | +0.0756 | HOLDS on both |

This release records both findings cross-CLI and softens the v-002 BUILD-dense claim from "without yield" to "without clear yield — medium matches or beats dense on every measured model" (deepseek-v4-pro v-002 was 0.7129 vs v-004's 0.7600 — close, but medium still wins).
