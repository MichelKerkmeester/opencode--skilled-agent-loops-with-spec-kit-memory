This first release lets other AI assistants delegate focused work to Cursor's terminal command-line tool. It packages safe execution rules, delegation patterns, and reference docs grounded in a live-verified Cursor CLI contract, so teams can use Cursor more deliberately. The goal is to make the workflow easier to follow and easier to trust.

---

#### Structured delegation into Cursor

Other assistants needed a clear way to decide when Cursor was the right helper for a task. The skill introduced routing patterns for generation, review, Composer-model dispatch, read-only plan/ask exploration, and agent-based delegation. Requests can be handed off more intentionally instead of treating Cursor as a generic fallback.

---

#### Safety and self-invocation guarding

Delegated work needed guardrails around approval flags, the always-zero exit code, and Cursor's shared editor-config surface. The package documents the approval/sandbox mapping and a self-invocation guard built from live-confirmed `CURSOR_AGENT`/`CURSOR_CONVERSATION_ID` env signals plus process ancestry, so a session already running inside Cursor CLI never dispatches itself.

---

#### Reusable support material

People needed guides and templates they could reuse instead of rebuilding delegation prompts from scratch. The release shipped a main guide, references covering Cursor's unique worktree/cloud-worker/plugin surfaces and shared editor config, and prompt templates. Cursor becomes easier to adopt and easier to use consistently.
