## Review: canonical-fingerprint.ts (seat opus-memory-daemon)

### What the module actually does
`canonicalizeInput` builds a `CanonicalJson` tree (sort object keys, normalize CRLF/CR→LF in strings, normalize `-0`→`0`, reject non-finite numbers) then `JSON.stringify`s it; `canonicalFingerprint`/`codeHash` sha256 that string and return **bare** 64-hex (`.digest('hex')`, line 61). Canonicalization is sound and deterministic for the plain-JSON input domain: keys sorted via `Object.keys().sort()` (UTF-16 code-unit order, platform-stable), insertion order = sorted order so `JSON.stringify` output is stable, line-endings normalized for cross-checkout stability. The test at `canonical-fingerprint.vitest.ts:6-18` confirms key-order + line-ending invariance.

### Trace of real consumers (de-rates the angle's headline risk)
The only production importer is `incremental-index.ts:17` → `planMemoizedIndexing` (line 744), which is itself **only called from tests** (`incremental-index-foundation.vitest.ts`); the module header marks it "Deferred lexical-only indexing." It feeds a memo cache keyed `(component_path, input_fingerprint, code_hash)` (`memo.ts:49`).

Crucially for my assigned angle: **continuity-freshness does NOT use this module.** The continuity contract requires a `sha256:`-prefixed digest — `thin-continuity-record.ts:105` `FINGERPRINT_RE = /^sha256:[a-f0-9]{64}$/`, enforced as `MEMORY_011` (line 702-703). `canonicalFingerprint` emits a bare hex with no prefix and is never bridged into the continuity record (no import path exists). So there is **no active fingerprint mismatch** in continuity-freshness — the two sha256 surfaces are independent and use incompatible string formats by construction. No P0/P1 there.

### Real but low-severity findings
1. **Container-dependent handling of non-JSON values** is inconsistent: as an object value, `function`/`symbol`/`undefined` are silently dropped (`line 41`), but as an array element the same `function`/`symbol` **throws** via the line 53 fallback while `undefined` becomes `null` (`line 33` map → `line 49-51`). For a contract named "canonical," whether an unsupported value crashes or is silently elided depends on where it sits. Diverges from `JSON.stringify` (which never throws on array functions, emits `null`). Low impact in the intended plain-JSON domain.

2. **Non-finite numbers throw, uncaught at the call site.** `normalizeNumber` throws on `NaN`/`Infinity` (`line 13-15`); `planMemoizedIndexing` calls `canonicalFingerprint(input.canonicalInput)` with no try/catch (`incremental-index.ts:744`), so one non-finite numeric field would abort the whole memo-planning batch rather than degrade. The throw itself is defensible hardening; the unguarded batch call site is the smell. Not live today (test-only path).

Both are cited, real, and modest — no data-loss, crash-in-production, or security exposure. The module is correct for its documented input domain; the prior deep-review's remediations are intact.
