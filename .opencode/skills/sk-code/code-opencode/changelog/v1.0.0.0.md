# v1.0.0.0 — opencode surface established

First release of the `opencode` surface packet. The system-code evidence that previously lived under the workflow modes (`code-implement/references/opencode/`, `code-quality/assets/opencode-checklists/`, `shared/references/opencode-shared/`) is now consolidated under one read-only surface packet on the sk-code hub's surface axis.

## New

- `SKILL.md` — the surface contract: when the hub bundles this evidence, the language-sliced reference map, the non-negotiable system-code standards (plugins never write to the TUI, descriptors are load-bearing, alignment drift is a verification gate, one language per task), and the on-demand authoring checklists.
- `README.md` — packet at-a-glance and layout.
- `references/` — TypeScript/Python/shell/config/JavaScript standards plus the language-agnostic `shared/` tier (universal patterns, code organization, hooks, alignment verification).
- `assets/checklists/` — component authoring (skill/agent/command/mcp_server) and per-language quality gates.

## Notes

This packet is `packetKind: surface`, `backendKind: evidence-base`, read-only (`toolSurface` ⊆ {Read, Bash, Grep, Glob}), and advisor-invisible (`routingClass: metadata`). Spec-folder authoring is out of scope — it lives in system-spec-kit. The hub bundles this evidence alongside a workflow mode and slices it by the detected language; it never routes as a primary and mutates nothing.
