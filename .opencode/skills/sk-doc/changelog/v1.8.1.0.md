sk-doc's skill-authoring contract is now enforced end-to-end: the packaging validator checks reference/asset frontmatter, versioning, naming, and the smart-router pattern, and the create commands + markdown agent seed those from the templates by construction.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/154-sk-design-parent` (phase parent — the sk-design family build that surfaced the gap)

---

## Why

A fast multi-model build of the `sk-design` skill family produced skills that PASSED the old `package_skill.py` despite kebab-case reference names, references/assets missing the `version` field, and SKILL.md SMART ROUTING sections with no smart-router pseudocode. The documented contract (the four `assets/skill/*_template.md` files) was stronger than the enforced one, so drift was the path of least resistance. This release closes that gap on both sides: detection (validator) and construction (create commands + agent).

## What Changed

#### Validator (`scripts/package_skill.py` + tests)

- Now validates references (`references/**/*.md`) and assets (`assets/**/*.md`) for the full 5-field frontmatter (`title`, `description`, `trigger_phrases`, `importance_tier`, `contextType`) plus `version`.
- snake_case filename enforcement is now **recursive** into domain subfolders (the old check only scanned the top level, so `references/color/oklch-workflow.md` was never flagged).
- Requires `version` in SKILL.md AND in every reference/asset/feature-catalog/manual-testing-playbook/changelog/README `.md` that carries frontmatter (frontmatter-less docs are skipped, not failed).
- Checks SKILL.md Section 2 SMART ROUTING for the smart-router pseudocode markers (`discover_markdown_resources`, `_guard_in_skill`, `UNKNOWN_FALLBACK`).
- New checks emit as **warnings, not hard errors**: a blast-radius sweep showed hard-erroring would newly fail 16 of 25 existing skills, so the contract is advisory to avoid breaking the corpus. 12 new regression tests; full suite green; 0 new hard failures repo-wide.

#### Create machinery (`commands/create/assets/create_sk_skill_{auto,confirm}.yaml`)

- Full-create now seeds SKILL.md Section 2 with the smart-router pseudocode (from `skill_smart_router.md`) and stamps the SKILL.md version.
- Reference/asset creation now requires snake_case filenames and the full 5-field + `version` frontmatter, seeded from `skill_reference_template.md` / `skill_asset_template.md`.
- `package_skill.py --check` is pinned as a blocking completion gate.

#### Markdown agent (`.opencode/agents/markdown.md`, `.claude/agents/markdown.md`)

- Added a Skill Authoring Contract: follow all four skill templates, version every authored `.md`, seed the router pseudocode, and run `--check` before claiming done.

#### Skill-creation references

- `references/skill_creation/creation-workflow.md` — fixed the smoking gun: `version` was marked optional with a 3-part example, directly contradicting the validator's mandatory 4-part requirement. Now required + 4-part; refs/assets seed from templates; SMART ROUTING step seeds the pseudocode.
- `references/skill_creation/validation-and-packaging.md` — added a Completion Gate section and corrected the required-fields list.
