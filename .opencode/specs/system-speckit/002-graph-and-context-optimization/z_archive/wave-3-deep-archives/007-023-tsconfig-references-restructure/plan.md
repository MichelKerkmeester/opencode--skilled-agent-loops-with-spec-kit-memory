## 1. SUMMARY

### Technical Context

| Aspect | Value |
|--------|-------|
| **Language/Stack** | TypeScript, Node.js |
| **Framework** | MCP server under `.opencode/skills/system-code-graph/mcp_server` |
| **Build Config** | `tsconfig.json` extending system-spec-kit base settings with `rootDir: ".."` and `outDir: "./dist"` |

### Overview

The fix removes the explicit project reference to `../system-spec-kit/shared` from the extracted system-code-graph tsconfig. The existing `include` already covers shared TypeScript sources; without project-reference ownership, TypeScript emits those files into `dist/system-spec-kit/shared/` under the system-code-graph output tree.

<!-- ANCHOR:architecture -->
## 3. ARCHITECTURE

### Pattern

Single-project emit ownership for included cross-skill source files.

### Key Components

- **system-code-graph tsconfig**: Owns the extracted build's root and output path.
- **system-spec-kit shared sources**: Included in the extracted build because runtime imports compiled shared files from the extracted dist tree.
- **system-code-graph launcher**: Runtime smoke test for whether compiled relative imports resolve.

### Data Flow

`tsc` reads sources from `.opencode/skills/system-code-graph`, `.opencode/skills/system-spec-kit`, and `.opencode/skills/system-skill-advisor` through the include list. With `rootDir: ".."` and `outDir: "./dist"`, included shared files emit under `dist/system-spec-kit/shared/`.
