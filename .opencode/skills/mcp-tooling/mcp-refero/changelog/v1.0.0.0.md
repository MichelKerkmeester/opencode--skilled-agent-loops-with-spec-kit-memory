# v1.0.0.0 - First Release

**Released:** 2026-07-16

The initial release delivered `mcp-refero` as the read-only Refero design-reference transport in the `mcp-tooling` hub (`packetKind: transport`, `mutatesWorkspace: false`). It searched real shipped UI styles, screens, and flows through the registered Refero MCP manual and Code Mode, supplied evidence to `sk-design` as the mandatory judgment partner, and packaged the live-discovered tool contract, examples, catalogs, tests, diagnostics, and template-conformant documentation without granting the transport design authority.

---

## What's In 1.0.0.0

### Core Transport And Backend

- `SKILL.md` defined Refero-specific activation signals, a guarded five-intent router, discovery-first execution, the styles to screens to flows research funnel, access limits, and read-only safety rules.
- The transport used the existing `refero` Code Mode manual, which launched `npx -y mcp-remote https://api.refero.design/mcp`, and preserved the registered object as a verify-only asset rather than editing `.utcp_config.json` or creating a second manual.
- Runtime authority stayed minimal: Read, Bash, Grep, Glob, and Code Mode calls were allowed, while Write, Edit, and Task were forbidden.
- `mcp-refero` became the canonical transport home for Refero tool-surface facts. `sk-design` retained only its judgment-side framing and consumed the transport as evidence instead of duplicating its catalog.

### Live-Discovered Code Mode Surface

- The packet documented eight read-only tools across three layers: style search and detail, screen search, detail, similarity and image retrieval, and flow search and detail.
- The 2026-07-16 discovery fixture confirmed all eight registry names before authentication in dotted doubled form, such as `refero.refero.refero_search_styles`, and confirmed TypeScript callables such as `refero.refero_refero_search_styles(args)`.
- Each session still ran `list_tools` and `tool_info` before the first call and failed closed on drift. Discovery required no OAuth, while authenticated calls remained operator-gated.
- The live schema confirmed `response_format?: "json" | "md"` with default `"md"` on style and screen search and required `platform: "ios" | "web"` on screen search. Other text-tool formatting stayed a per-tool `tool_info` check, and `refero_get_screen_image` never accepted `response_format`.
- UUID screen identifiers and numeric flow identifiers stayed distinct, unknown response fields were preserved, sparse-flow reconstruction was labeled inference, and screenshots were fetched only after metadata could not answer the question.

### Access, Auth, And Limits

- The packet stated that the Free plan had no MCP access and that Pro was the first supported tier with a published quota of 8,000 tool calls per month.
- The local `mcp-remote` bridge preferred HTTP, used browser OAuth on port 3334, and stored operator-owned auth state under `~/.mcp-auth`. Node.js 18 or newer was required, with this project documented on Node 24 and the observed Node 25 crash called out for diagnosis.
- OAuth completion and authenticated calls remained honestly Inferred until an operator completed authorization. Unpublished burst, concurrency, page-size, `Retry-After`, Bearer-acquisition, and end-to-end OAuth details stayed UNKNOWN rather than being invented.
- The packet preserved the registered manual byte-for-byte and documented the env-backed Bearer header only as an alternative provider-supported shape, not as the adopted configuration.

### Feature Catalog And Examples

- `feature-catalog/` supplied a root inventory, three domain overviews, and one leaf for each of the eight tools, including call forms, argument bounds, identifier types, result-shape cautions, mutation class, and cross-links to the canonical references.
- `examples/` supplied the complete styles to screens to flows funnel, a metadata-first single-tool lookup, and an image-last screen fetch. Every walkthrough began with `tool_info`, used the doubled callable prefix, and treated OAuth-gated execution as SKIP-valid when access was unavailable.
- Element research stayed a screen-metadata facet through `ui_elements` and `ux_patterns`; the packet did not invent a `search_elements` tool.

### Testing And Operator Tooling

- The nine-scenario manual testing playbook covered registered-manual verification, pre-auth discovery, the full funnel, flow detail, text-format discipline, quota honesty, config-mutation refusal, `sk-design` pairing, routing recall, holdouts, and a negative control.
- Live paid or OAuth-gated steps allowed a documented SKIP, while all locally testable routing, schema, refusal, and wiring checks required real execution.
- `scripts/install.sh` verified Node, npx, registered-manual presence, and the operator OAuth boundary without installing or mutating anything. `scripts/doctor.sh` provided read-only diagnostics and an opt-in unauthenticated endpoint probe whose healthy result was HTTP 401.
- `mcp-servers/refero-mcp/README.md` pointed to the hosted provider and manual registration contract without vendoring a server.

### References, Assets, And Documentation

- `references/tool-surface.md` carried the eight-tool contract, funnel, identifier rules, plan gates, deprecated-surface negative knowledge, and open questions.
- `references/mcp-wiring.md` carried the registered manual, bridge and OAuth behavior, doubled-prefix naming, operator-owned auth state, and runtime constraints.
- `references/troubleshooting.md` covered authentication, entitlement, discovery, Node runtime, batching, quota, and drift failures with safe recovery steps.
- `assets/utcp-refero-manual.md` preserved the registered manual snapshot and verify-only posture.
- `README.md` and `INSTALL-GUIDE.md` described current-state use and operator verification. All packet paths used kebab-case, including `feature-catalog/`, `manual-testing-playbook/`, their root indexes, and `INSTALL-GUIDE.md`.

### Template Conformance

- Reference and asset documents followed the create-skill OVERVIEW-first structure with Purpose and Usage while preserving their technical content, frontmatter, filenames, and Related Resources placement.
- `scripts/README.md` followed the create-readme code-folder structure for Overview, Architecture, Directory Tree, Key Files, Entrypoints, Validation, and Related resources, grounded in the actual read-only and exit-code behavior of `install.sh` and `doctor.sh`.
- The package retained one read-only transport contract, one registered backend, and no packet-local design judgment or workspace mutation.

### Boundaries

- Any design-affecting use loaded `sk-design` first. Refero supplied cited references and never issued taste, accessibility, or readiness verdicts.
- The official `referodesign/refero_skill` methodology repository was referenced rather than vendored.
- No hub files or live configuration belonged to this packet's release, and no undocumented tools, arguments, limits, or authentication results were claimed.
