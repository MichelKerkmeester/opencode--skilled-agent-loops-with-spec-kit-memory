## [**1.3.0.0**] - 2026-07-17

Kebab-completion: the five tool-keyed names carved out in 1.2.0.0 are now renamed to hyphens, closing the last snake_case gap in the packet. The carve-out existed only because `package_skill.py`, `parent-skill-check.cjs`, and the skill-benchmark loaders (`load-playbook-scenarios.cjs`, `playbook-generator.cjs`, `validate-playbook-topology.cjs`) hardcoded the snake_case forms; those tools have since been patched to accept both spellings, so the rename is now safe.

#### Kebab-completion renames

- Need: 1.2.0.0 kept `feature_catalog/`, `manual_testing_playbook/`, their two root index files, and `INSTALL_GUIDE.md` snake_case as a tool-mandated carve-out. The packet was otherwise fully kebab, leaving these five as the only exceptions.
- Change: The tooling gained dual-spelling tolerance, so all five were renamed via `git mv`: `feature_catalog/` → `feature-catalog/`, `feature-catalog/feature_catalog.md` → `feature-catalog/feature-catalog.md`, `manual_testing_playbook/` → `manual-testing-playbook/`, `manual-testing-playbook/manual_testing_playbook.md` → `manual-testing-playbook/manual-testing-playbook.md`, and `INSTALL_GUIDE.md` → `install-guide.md`. Every in-packet reference (README table, SKILL.md install link, playbook root index catalog links + scenario tables, feature-catalog leaf cross-links, `scripts/README.md`, `scripts/doctor.sh`, `scripts/install.sh`) was updated to the new paths.
- Why it matters: One naming dialect across the entire packet, with no remaining carve-out. The `skill_asset_feature_catalog` sk-doc template classifier token stays snake_case by design — it is a template identifier, not a filesystem path.
