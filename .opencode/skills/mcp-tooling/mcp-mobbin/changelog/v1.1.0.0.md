# v1.1.0.0 - Registered-State Truth And Inventory Parity

**Released:** 2026-07-16

The `mobbin` Code Mode manual **is now registered** in `.utcp_config.json` (operator-applied 2026-07-16, byte-identical to the researched draft: stdio `npx -y mcp-remote https://api.mobbin.com/mcp`, empty `env`). This release flips every packet doc from pre-registration doctrine to registered-state truth — presence is the healthy wiring result, absence is a failure symptom — and brings the packet to inventory parity with its transport siblings: an `examples/` directory, a verify-only `scripts/install.sh`, an enriched manual-testing playbook (6 -> 9 scenarios), and per-intent feature-catalog leaves carrying the cross-cutting constraints. Discovery and OAuth remain pending: manuals load at Code Mode startup, no fresh session has run discovery, and the callable stays **INFERRED** until `tool_info` confirms it.

## What's New In 1.1.0.0

### Registered-state doc truth (every stale marker flipped)

- `SKILL.md`: the registration trap became a **discovery trap** — manual registered, discovery pending a fresh Code Mode session, OAuth pending the operator; RULES/ESCALATE reframed (absence = escalate a broken registration, never repair); quick reference updated.
- `README.md`, `INSTALL_GUIDE.md`: quick starts, FAQs, troubleshooting, verification tables, and the architecture diagram now describe the registered wiring state; Section 4 of the install guide is "Reconnect and Authenticate" (operator-only) instead of "Register and Authenticate (later phase)".
- `references/mcp-wiring.md`, `references/tool-surface.md`, `references/troubleshooting.md`: the draft-manual framing became the registered-manual reference shape; "no tools resolve" now means a stale session, pending OAuth, or a broken registration — never an expected baseline.
- `assets/utcp-mobbin-manual.md`: re-titled as the registered manual's **reference shape** (live config wins on any disagreement); the 9-item post-registration checklist executed doc-side with evidence (config parses, entry matches field-for-field, `env` still empty, no `.env` line) while the live items (reconnect, OAuth, discovery, baseline check, schema check, smoke search) stay open as valid SKIPs with exact commands.
- `mcp-servers/mobbin-mcp/README.md`, `feature_catalog/feature_catalog.md`: pointer and inventory language flipped to registered-state truth.

### Doctor behavior change

- `scripts/doctor.sh`: manual **absence is now an ERROR** (was expected INFO) — a broken or reverted registration to escalate with the reference shape, never to re-add; presence reports OK plus the bridge shape; the callable-confirmation pointer notes that discovery has not yet run. Still strictly read-only and non-interactive.

### New: examples/

- `examples/README.md`: shared preflight (doctor, SKIP-valid operator OAuth, mandatory `tool_info` confirmation) and shared rules, traced to `references/tool-surface.md` only.
- `examples/smoke-search-limit-1.md`: the minimum verification — confirmed callable, one `limit: 1` search, documented response-shape check.
- `examples/platform-flow-research.md`: platform-filtered flow-pattern research with reconstruction labeled as inference.
- `examples/element-intent-query.md`: element-intent query analyzed within screens and compared across `app_name` values.

### New: scripts/install.sh

- Non-interactive, verify-only posture check: Node 18+/npx, registered `mobbin` manual present in `.utcp_config.json` (read-only grep; presence = OK), and the operator-only first-use OAuth pointer (browser round-trip; no API key exists). Exit 0 = posture verified; `bash -n` clean.

### Playbook enrichment (6 -> 9 scenarios)

- `MANUAL-001` renamed/rewritten: `discovery-setup/manual_absent_expected.md` -> `discovery-setup/manual-registered-expected.md` (presence expected; absence escalated as ERR).
- New `read-only/platform-filter.md` (`PLATFORM-001`): the `ios`|`web` enum, infer-or-ask, and per-platform comparison calls.
- New `limits-access/rate-limit-backoff.md` (`RATELIMIT-001`): the 429 `Retry-After` + backoff-with-jitter observation procedure — SKIP-valid, never a forced load test.
- New `limits-access/paid-gate-taxonomy.md` (`PAIDGATE-001`): the 401 vs entitlement vs 429 error-taxonomy walk with the UNVERIFIED Free-denial caveat relayed verbatim.
- Root playbook updated: coverage table (9 across 5 categories), wave planning (5 waves), REFUSE-001 reframed for the registered state, and the cross-reference index at 9/9.

### Feature-catalog enrichment

- All four intent leaves (`apps/`, `screens/`, `flows/`, `elements/`) gained query-intent recipe tables and a cross-cutting constraints section (rate limit, plan gating, schema honesty), each traced to `references/tool-surface.md`.

### Unchanged

- The single-tool contract (`search_screens`), the no-API-key auth model, the sk-design pairing, the read-only transport boundary (Write/Edit/Task forbidden), the 60/60s rate limit, and the open questions (authenticated tool list, `deep` conflict, inline-image fidelity, end-to-end OAuth) — all carried forward verbatim; only their registration-state framing moved.
