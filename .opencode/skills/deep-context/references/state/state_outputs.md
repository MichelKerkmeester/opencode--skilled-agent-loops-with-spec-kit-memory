# Deep Context Output Files Reference

This reference covers the human-readable and report outputs of the deep-context packet. JSONL records live in `state_jsonl.md`; reducer ownership lives in `state_reducer_registry.md`.

---

## 1. OVERVIEW

### Purpose

Define the deep-context packet outputs that are not raw state log lines: the strategy file, per-iteration markdown narratives, per-iteration delta streams, the auto-generated dashboard, and the Context Report deliverable (markdown + JSON).

### When to Use

Load this reference when writing or validating these outputs rather than the `deep-context-state.jsonl` records.

### Core Principle

Human-readable outputs must stay synchronized with raw state and respect ownership: the host owns strategy, iterations, deltas, and the report; the reducer owns the dashboard. Seats write none of them.

### Key Sources

- `.opencode/commands/deep/assets/deep_start-context-loop_auto.yaml` — `step_create_strategy`, `step_write_iteration`, `step_compile_report`, `step_emit_report_json`.
- `.opencode/skills/deep-context/assets/context_report_template.md` — the Context Report section order and field schemas.
- `.opencode/skills/deep-context/scripts/reduce-state.cjs` — `renderDashboard` (the dashboard generator).

---

## 2. STRATEGY FILE

`deep-context-strategy.md` is the host-owned scope and focus plan, created at init with the seeded frontier and the executor-pool roster.

Seeded contents:

- Scope (gather-subject).
- Frontier: the ranked SLICE list as the initial focus candidates.
- Executor pool roster: one row per seat (label, kind, model, promptFramework).
- Known Context: prior memory context summary (appended only after the file exists), else `None`.
- Thresholds: max iterations, convergence threshold, relevance gate, agreement min.

Update protocol:

- read strategy before each iteration to pick the current focus;
- the host advances `Next Focus` to prioritize uncovered / low-agreement slices for the next shared sweep;
- on a blocked or stuck stop, the host overrides `Next Focus` with the recovery focus (uncovered slices first).

---

## 3. ITERATION FILES

Iteration files are host-written, write-once markdown narratives, one per parallel sweep.

```text
context/iterations/iteration-001.md
context/iterations/iteration-002.md
context/iterations/iteration-003.md
```

Each narrative covers, for that sweep:

- the SHARED focus that was swept;
- per-seat contribution (which seats found what);
- merged findings with their agreement count (`k/N`);
- contradictions surfaced (never auto-resolved);
- the slice-coverage delta for the iteration.

Every cited `file:symbol` must be verifiable against the code graph; stale references are labeled, not shipped silently.

---

## 4. DELTA FILES

`deltas/iter-NNN.jsonl` is the host-written, write-once per-iteration delta stream — a complement to the canonical `deep-context-state.jsonl` append.

```text
context/deltas/iter-001.jsonl
context/deltas/iter-002.jsonl
```

Rule: each delta file MUST contain at least one record whose `type === "iteration"`, so a partial run can be rehydrated from the per-iteration deltas even if the canonical state log append was interrupted.

---

## 5. DASHBOARD

`deep-context-dashboard.md` is auto-generated by the reducer (`renderDashboard`) from the JSONL state log, the per-seat findings, and the merged registry. It is never manually edited.

Sections (in render order):

| Section | Source |
|---------|--------|
| 1. STATUS | config scope/start, derived status, iteration count, lineage |
| 2. PROGRESS | one row per iteration record (focus, findings, NewAgr, sliceCov, agrRate, status) |
| 3. MERGED METRICS | registry `metrics` (findings, gated, low-confidence, agreement-eligible, agreementRate, relevanceFloor, bucket counts) |
| 4. REUSE CATALOG | top agreement-weighted `reuseCandidates` (capped) |
| 5. CONTRADICTIONS | registry `contradictions` (surfaced, never auto-resolved) |
| 6. GRAPH CONVERGENCE | latest `graph_convergence` event rollup (decision, signals, blockers) |
| 7. ACTIVE RISKS | present only when the latest iteration reported `error` status |

Generation rules:

- regenerate after each iteration evaluation (`step_update_registry` runs the reducer);
- compute every derived value from raw state;
- write the full file atomically (temp + fsync + rename), overwriting rather than appending;
- agent or manual edits are discarded on the next refresh.

See `state_reducer_registry.md` for the reducer contract behind this output.

## 6. CONTEXT REPORT

`context/context-report.md` (with its `context/context-report.json` companion) is the host-synthesized deliverable, compiled at convergence from `findings-registry.json`. It ships verified **pointers**, not source bodies.

Section order (reuse-catalog first because it is the highest-value section):

1. REUSE Catalog — id, `symbol (file:line)`, signature, reuse verb, confidence, agreement `k/N`, freshness, notes.
2. Integration Points — `[HARD]` / `[soft]` entry/exit symbols with `file:line`.
3. Touch List — files to change in implementation order.
4. Conventions — `file:line` exemplars (naming, error handling, logging, test layout).
5. Pruned Dependency Subgraph — only edges within the touch radius.
6. Prior Art / Decisions — related specs/ADRs/memory, date-stamped.
7. Gaps & Unknowns — not-found / unverified findings plus marginal-relevance near-misses.
8. Methodology & Confidence (footer) — per-executor attribution, agreement counts, relevance gate, contradictions surfaced, stop decision.

The full section and field schema lives in the report template — `../../assets/context_report_template.md`. Do not restate the field guide here.

`context-report.json` carries the same merged findings in machine-readable form (`reuseCandidates[]`, `integrationPoints[]`, `touchList[]`, `conventions[]`, `dependencySubgraph{}`, `priorArt[]`, `gaps[]`) plus per-finding attribution (`producedBy[]`, `agreement`, `relevance`, `freshness`) and the convergence signals, for `/speckit:plan` and `/speckit:implement`.

Every `file:symbol` MUST be code-graph-verified before the report is finalized; anything unverifiable is labeled `unverified`, not omitted silently. A Convergence Report block (stop reason, total iterations, executor pool, final signals) is appended to the markdown report at synthesis.
