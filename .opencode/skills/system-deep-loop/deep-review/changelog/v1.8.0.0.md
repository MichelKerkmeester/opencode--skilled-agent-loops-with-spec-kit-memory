This release lands the deep-review portion of the packets 108 plus 110 auto-review uplift program. Four reusable teachings from the upstream `dzianisv/opencode-plugins` auto-review package now apply to every deep-review iteration: a parseable verdict on the final line, a role-banner marker at the rendered-prompt boundary, an anti-repetition rule across iterations, and a content-hash dedup that extends the existing synthesis path. A fifth teaching, H-9 bounded evidence, was deferred to packet 111 plus because current deep-review prompts pass state and metadata pointers rather than embedded file contents, so the council found no real interpolation path to harden. Every mechanism cleared a 5-of-5 unanimous deep-ai-council review by `cli-codex gpt-5.5 xhigh fast` before any code shipped.

## New Features

#### Per-iteration verdict on the final line

Deep-review iterations used to end with prose. Synthesis and dispatch logic had to guess at the iteration outcome, which made gating decisions brittle. Every iteration must now end with a plain-text final line of the form `Review verdict: PASS`, `Review verdict: CONDITIONAL`, or `Review verdict: FAIL`. The verdict mapping is fixed: PASS means no P0 and no P1 findings, CONDITIONAL means at least one P1 but no P0, FAIL means any P0. A new `step_derive_verdict` step in the deep-review dispatcher YAMLs reads the iteration record and re-derives the verdict from the actual P0 plus P1 counts, so a misreported verdict line is corrected before it reaches synthesis. The final line is plain text rather than Markdown bold per the council's clarity ruling.

&nbsp;

#### DEEP-REVIEW marker at the rendered-prompt boundary

Long iterations occasionally drifted off-role when the banner scrolled out of context. The fix places a `DEEP-REVIEW` marker on line 1 of `prompt_pack_iteration.md.tmpl`, where it stays at the top of the rendered prompt regardless of how much template content sits below. A matching `step_marker_scan` runs in both the auto and confirm dispatcher YAMLs and fails fast when the marker is missing after rendering. Reference templates that ship YAML frontmatter were deliberately left untouched here, because a stray marker inside frontmatter would corrupt the loader.

&nbsp;

#### Anti-repetition rule across iterations

The CONSTRAINTS section of the deep-review SKILL.md now prohibits restating a finding already reported in a prior iteration of the same review. Reviewers must either omit the duplicate or extend the original entry with new evidence. This is enforced at the prompt level so the reviewer carries the intent through iteration n while still seeing iterations 1 through n minus 1 in context.

&nbsp;

#### Content-hash dedup extends existing synthesis dedup

The synthesis step already deduplicated findings at `deep_review_auto.yaml:1115-1124` using a file-plus-line plus normalized-title key. That key missed cross-dimension restatements (the same security finding described once under "security" and once under "correctness") and missed minor wording drift across iterations. This release extends the existing key with a content hash: `content_hash = sha256(file_path + line_range + finding_type + normalized_description_80chars)`. Synthesis now matches in two tiers, primary on content_hash, fallback to the legacy file plus line plus normalized title for records that predate this release. Cross-dimension matches collapse to one entry with a `dimensions: [<list>]` field that names every dimension that surfaced the finding. There is no parallel dedup path. The council mandate was to extend, not duplicate. H-7 dedup is intentionally NOT applied to deep-research because deep-research is single-dimension. That decision is documented in the deep-research changelog for the same release window.

## Deferred: H-9 bounded evidence

H-9 was the original packet 106 teaching to bound embedded file content in iteration prompts so a single oversized blob could not crowd out the rest of the context. The council's section 10.5 review concluded that current deep-review prompts do not interpolate file contents at all. Iterations pass state files plus metadata pointers and let the reviewer fetch evidence on demand. There is no real interpolation path to harden in the present codebase. H-9 is deferred to packet 111 plus, to be revisited if and when an interpolating path is introduced.
