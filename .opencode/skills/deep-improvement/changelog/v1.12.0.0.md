This release adds the **capability-discrimination layer** to the model-benchmark framework: three new partial-credit fixture packs (hard computational, harder computational, strict-validation), the capability-comparison profiles, a dispatch cwd-isolation fix that keeps real runs from polluting the repo, and the first **reproducible MiniMax-M3 vs MiMo-V2.5-Pro reliability verdict**. Everything is additive over the existing Lane B grader and the config-driven sweep — the full Vitest suite stays green at **158**.

## What Changed

### Discriminating fixture packs

- **Hard pack (T4)** — `hard-merge-intervals`, `hard-parse-csv-line`, `hard-roman-to-int`, `hard-eval-expr`: partial-credit oracle fixtures (16–18 cases each), every oracle validated through the real `code-task-scorer.cjs` (reference impl 1.0, wrong impl 0.33–0.625).
- **Harder pack (T4)** — `harder-semver-compare`, `harder-normalize-path`, `harder-int-to-words` (24 cases each). Outcome: these **saturated** frontier models (a useful negative result — raw difficulty alone does not separate M3 and MiMo).
- **Validation pack (T4)** — `validate-ipv4`, `validate-date`, `validate-semver`: invalid-dominant strict validators (26–28 cases, 64–74% adversarial-invalid; reference 1.0 vs lax 0.75–0.889). These are the **discriminators** — strict input validation, not computation, is where the two models diverge.
- Tiered-taxonomy seeds added: `t1-smoke-echo` (T1) and `t4-adversarial-tokenizer` (T4).

### Capability profiles

- `capability-m3-vs-mimo.json` (hard pack, n=3), `capability-m3-vs-mimo-v2.json` (harder pack, n=5), `capability-m3-vs-mimo-v3.json` (validation pack, n=5) — all `mode: model-vs-model`, `correctnessGate.threshold: 1.0`, `reporting.groupBy: model`.

### Dispatch cwd-isolation (repo safety)

- `sweep-benchmark.cjs` `dispatchCell` now runs each dispatch with `cwd` (and `--dir`) set to a per-cell `os.tmpdir()` temp dir, removed in a `try/finally`. An agentic model's stray file-writes land in the throwaway dir, never the repo. Proven across real runs via a before/after untracked-file diff (**0 new files**).

### Tests + docs

- `sweep-isolation.vitest.ts` adds dispatch-isolation coverage (cwd under `os.tmpdir()`, not the repo root; cleaned up after single + multi-cell sweeps; a simulated model write does not leak) plus fixture-shape + profile-load coverage for the new packs. Suite: **158 passing**.
- New folder READMEs for `scripts/model-benchmark/tests/` and `assets/model_benchmark/benchmark-fixtures/` — the sk-code OPENCODE alignment verifier (`verify_alignment_drift.py`) PASSES on both scopes (0 findings).

### The verdict (the payoff)

- First **reproducible** MiniMax-M3 vs MiMo-V2.5-Pro capability verdict: **M3 is the more reliable model** — perfect 1.0 / correctness-gate-eligible across **32 of 32 cells** over two independent runs with disjoint fixtures (004 computational+roman: 1.0 vs 0.898; 006 validation+roman: 1.0 vs 0.891). MiMo carries a **~1-in-5 catastrophic-failure rate** on hard validation plus a mild roman-numeral gap, keeping it below the correctness gate every run.

### Methodology lessons baked in

- **De-risk at n=2 for saturation, rank at n≥5.** The n=2 de-risk briefly flipped the read (it surfaced a rare M3 truncation); n=5 corrected it. Per-model ranking needs ≥5 samples because the catastrophic failures are ~1-in-5 events.
- **Reproducibility across disjoint fixtures certifies a verdict** when the built-in paired-bootstrap CI defaults to a format-TIE under correctness saturation.

### Known follow-up (P2)

- The trustworthiness reporter prints "TIE-on-format" whenever correctness saturates — misleading when the correctness gate cleanly separates the pair (one model eligible, the other not). Recommended: report the sole gate-eligible model as the correctness winner.
