## 1. SUMMARY

| Aspect | Value |
|--------|-------|
| **Surface** | sk-git references and their path consumers |
| **Change class** | Filesystem rename plus path-pointer rewrite |
| **Execution** | Dependency-closed, path-scoped batch on the pinned migration worktree |

### Overview
The executor will apply the nine-entry semantic source-to-target map from the phase specification, using a collision check before any rename. It will then update active links, router/resource tables, commands, and path-valued pointers in the sk-git reference consumers, while leaving identifiers, keys, fields, and non-path prose unchanged.

<!-- ANCHOR:quality-gates -->
## 2. QUALITY GATES

### Definition of Ready
- [ ] The pinned BASE SHA and phase rename-map hash are recorded.
- [ ] Every source and target in the nine-entry map is present, absent, or explicitly dispositioned.
- [ ] The candidate consumer inventory covers SKILL.md, README.md, assets/worktree-checklist.md, and references/*.md.

### Definition of Done
- [ ] Every applicable source path is renamed once or recorded as already compliant.
- [ ] Active pointers resolve to the kebab-case targets with zero broken links.
- [ ] The candidate diff contains no out-of-scope rename or non-path identifier/key change.
<!-- /ANCHOR:quality-gates -->

<!-- ANCHOR:architecture -->
## 3. ARCHITECTURE

- **Semantic rename map**: maps each historical underscore filename to one explicit hyphenated target; it is not generated by a blind character substitution.
- **Pointer inventory**: scans tracked sk-git Markdown and relevant metadata for path-valued references before and after the rename.
- **Dependency-closed batch**: performs file renames and pointer rewrites together so no committed state points at a missing reference.
- **Exemption filter**: excludes Python names, Python package directories, tool-mandated names, identifiers, keys, fields, and frozen content.
<!-- /ANCHOR:architecture -->

<!-- ANCHOR:phases -->
## 4. IMPLEMENTATION PHASES

### Phase 1: Setup
- Confirm the parent and 017 frozen map are available; pin BASE and record the map hash.
- Inventory the nine source paths, nine targets, symlink or mode metadata, and every tracked pointer to them.
- Abort on exact, casefold, or normalization collisions, duplicate targets, or an unknown candidate.

### Phase 2: Implementation
- Rename applicable files from references/ to their semantic kebab-case targets with history and metadata preserved.
- Rewrite SKILL.md, README.md, assets/worktree-checklist.md, and reference-document path pointers.
- Keep source and target paths from coexisting after the batch; record no-op dispositions for targets already present.

### Phase 3: Verification
- Resolve every active reference path and rerun the pointer inventory.
- Review git rename status, mode and symlink parity, and the path-scoped diff.
- Run the phase checklist's source-map, exemption, and broken-link checks; record commands and exit codes in the SOL report.
