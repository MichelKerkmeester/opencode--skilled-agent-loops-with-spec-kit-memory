This release makes the sk-prompt manual-testing-playbook fully forkable by removing its dependence on sibling skills. After the phase-013 card relocation and the phase-015 path repoint, the playbook still leaned on `sk-prompt-small-model` (the relocated `cli_prompt_quality_card.md`) in nine places, including two `rg` test steps and a disk-resolution assertion, plus stale `§8` references and a few `cli-*` mentions. The playbook now tests only sk-prompt's own surface, so a standalone fork runs clean.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/130-prompt-knowledge-layering/016-sk-prompt-playbook-forkability-decouple/` (Level 1)

---

## What Changed

#### Documentation

- **Two escalation scenarios reframed to sk-prompt's own surface** -- `SP-023` and `SP-024` tested the CLI prompt-quality card's fast-path and escalation thresholds, a CLI-dispatch concern that left sk-prompt when the card moved to the hub. They now test sk-prompt's own behavior: `SP-023` verifies a low-complexity prompt passes the CLEAR check inline (anchored on `references/patterns_evaluation.md`) without dispatching `@prompt-improver`, and `SP-024` verifies a high-risk prompt escalates to `@prompt-improver` (anchored on `SKILL.md` §7 contract and §4 ESCALATE IF). Both `rg` steps now target sk-prompt's own files; both resolve (33 CLEAR matches, all 4 escalation terms).
- **All hub-card source refs repointed or removed** -- `SP-019`, `SP-021`, `SP-025`, `SP-026` cited the hub card in their source-ref tables; those rows were removed (redundant with the adjacent `patterns_evaluation.md` row) or repointed to `SKILL.md` §7. The root playbook's coverage note, Realistic Test Model, Global Preconditions disk-resolution list, and §12/§15 descriptions were updated to drop the card.
- **Stale and cross-skill references scrubbed** -- the root playbook referenced a `§8 fast-path asset section` that phase 014 removed from `SKILL.md`; it now points at `§7`. The `cli-claude-code`/`cli-codex`/`cli-opencode`/`cli-copilot` names in the wave-planning operational rules were genericized to "available executor skills". A token grep confirms zero references to `sk-prompt-small-model`, `cli_prompt_quality_card`, or any `cli-*` skill remain in the playbook, and all 37 `rg` targets point only at `sk-prompt/`.
