## Candidates
1. **LT-bg-sleeptime-agent → NET-NEW (H/M) — headline.** A dedicated background agent fires AFTER the foreground step, reorganizing message transcripts via a tool-chain (`store_memories → rethink_user_memory → finish_rethinking_memory`), spawned per-main-agent, async (`safe_create_task`). `voice_sleeptime_agent.py:32-46,87-91`; `sleeptime_multi_agent_v4.py:139-165`. Maps to — internal has **NO background reorganization agent**; `reconsolidation-bridge.ts` is synchronous on-save. A cadence-driven async consolidation pass is a net-new architectural direction.
2. **LT-turn-cadence-trigger → NET-NEW (M/S — cheap).** Persistent `turns_counter` gates background work: `turns_counter % sleeptime_agent_frequency == 0` (default 5) amortizes expensive reorganization across N turns. `sleeptime_multi_agent_v4.py:139-145`; `server.py:784`. Maps to — internal `pressure-monitor.ts` gates on token-ratio, retention sweep is cron/trigger — neither is turn-count cadence.
3. **LT-tool-rule-memory-chain → NET-NEW (M/M) — the loop-governor for CG-agentic-tool-loop.** Strict tool-call DAG (`InitToolRule → ChildToolRule → ContinueToolRule → TerminalToolRule`) forces a full reorganization cycle, no early bail. `voice_sleeptime_agent.py:87-91,132-148`. **This is exactly the bounding mechanism iter-22 found `CG-agentic-tool-loop` lacks** ("no agent-loop step/cost governor; all greenfield") — Letta's tool-rule DAG is a ready template for it.
4. **LT-llm-transcript-chunking → NET-NEW (M/M).** `store_memory(start_index, end_index, context)` lets the LLM slice raw transcript by index-range and store the chunk as an archival passage — LLM decides which segments are worth preserving. `voice_sleeptime_agent.py:137-175`. Maps to — internal `chunking-orchestrator.ts` chunks by markdown structure, not LLM-selected ranges.
5. **LT-block-undo-redo → NET-NEW (L/M — lower priority).** Sequence-based undo/redo stack (`BlockHistory`, `sequence_number`) with linear-history truncation. `block_manager.py:860-1009`. Maps to — internal `checkpoints.ts` is snapshot-based (full DB copy), not a navigable sequence stack. Lower leverage (snapshots already work).
6. **LT-tag-junction-dual-write → EXTENDS (M/S).** Passage tags stored in BOTH a JSON column AND a `PassageTag` junction table → DISTINCT/EXISTS filtering + tag-count aggregation. `passage_manager.py:56-75,115-121,395-415`. Maps to — internal `memory_index.trigger_phrases` is a text column; no junction for structured tag aggregation. EXTENDS.

## Net effect
The Letta sleep-time vein was genuinely productive. Three candidates (bg-sleeptime-agent + turn-cadence + llm-transcript-chunking) form a coherent **"async sleep-time consolidation"** initiative — background, cadence-gated memory reorganization that LLM-selects transcript ranges to archive (distinct from our synchronous reconsolidation-bridge). And **LT-tool-rule-memory-chain directly de-risks CG-agentic-tool-loop** by supplying the loop-governor it was missing. block-undo-redo is lower-leverage (our snapshot checkpoints suffice); tag-junction is a clean EXTENDS.
