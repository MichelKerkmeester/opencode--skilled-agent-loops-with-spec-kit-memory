# v2.2.0.0: Reference And Asset Alignment

Date: 2026-05-24

## 1. OVERVIEW

This release aligns `deep-ai-council` with the shared deep-skill resource model while preserving its planning-only domain. It adds a council-specific quick reference, loop protocol, and runtime asset family so operators see the same resource shape as `deep-research` and `deep-review` without copy-pasting their loop vocabulary.

## What Changed

#### Template Alignment

- Added a sk-doc-shaped quick reference and loop protocol for first-touch operator guidance and full council workflow navigation.
- Updated `SKILL.md` resource domains so markdown references and markdown assets are discovered through the same guarded router pattern used by sibling deep skills.

#### Reference Changes

- Added `references/quick_reference.md` for commands, artifact map, stop conditions, validation, and sibling-skill routing boundaries.
- Added `references/loop_protocol.md` for packet resolution, seat setup, critique, convergence, persistence, handoff, and recovery.

#### Asset Changes

- Added `assets/deep_ai_council_config.json` as the run-config template.
- Added `assets/deep_ai_council_strategy.md` as the round strategy template.
- Added `assets/deep_ai_council_dashboard.md` as the status dashboard template.
- Added `assets/prompt_pack_round.md.tmpl` as the council-seat prompt-pack template.
- Added `assets/runtime_capabilities.json` as the runtime parity and validation matrix.
- Normalized the config template convergence signal to the canonical `two-of-three-agree` value used by runtime persistence and reference docs.

#### Router And README Updates

- Bumped `SKILL.md` to `2.2.0.0` and wired the new reference/asset family into resource loading guidance.
- Updated `README.md` version, structure, related documents, asset inventory, and validation navigation.

No runtime script or workflow behavior changed.
