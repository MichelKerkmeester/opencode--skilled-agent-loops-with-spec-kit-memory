# Checklist: sk-prompt subtree rollup gate

## Verification Protocol

This checklist is the blocking SOL verifier contract for phase 007. The verifier pins candidate and BASE SHAs, reads
all sibling checklist verdicts and maps, enumerates the complete sk-prompt skill surface, records commands and exit
codes, and fails on any incomplete sibling, unknown non-kebab path, stale active reference, or unresolved release finding.
<!-- /ANCHOR:protocol -->

<!-- ANCHOR:pre-impl -->
## Pre-Implementation

- [ ] CHK-001 [P0] Phases 001–006 have completed evidence and their blocking checklist contracts are available
- [ ] CHK-002 [P1] Candidate SHA, BASE SHA, sibling map hashes, and final census command are recorded before the gate runs
<!-- /ANCHOR:pre-impl -->

<!-- ANCHOR:code-quality -->
## Code Quality

- [ ] CHK-003 [P0] No sibling phase has an unresolved ownership conflict, stale handoff, or out-of-scope mutation
- [ ] CHK-004 [P0] The final scope-aware census classifies every retained non-kebab filesystem name; no in-scope authored name remains without evidence
<!-- /ANCHOR:code-quality -->

<!-- ANCHOR:testing -->
## Testing

- [ ] CHK-005 [P0] All sibling P0 checklist items pass, and phase 006 has a matching changelog/version verdict
- [ ] CHK-006 [P0] The complete hub, playbook, benchmark, prompt-improve, and prompt-models surfaces contain no stale active source path after the child maps are applied
- [ ] CHK-007 [P0] Every retained Python/package, tool-mandated, generated, frozen, or other approved non-kebab name has a recorded disposition; unknown classifications fail the gate
- [ ] CHK-008 [P1] Final census, map hashes, sibling verdict matrix, commands, exit codes, and unresolved findings are reproducible from the evidence
<!-- /ANCHOR:testing -->

<!-- ANCHOR:fix-completeness -->
## Fix Completeness

- [ ] CHK-010 [P1] The gate performed no new rename, reference rewrite, metadata repair, changelog edit, or content migration

## Documentation

- [ ] CHK-012 [P1] The final evidence identifies every sibling packet, path classification, exemption, release verdict, and handoff target
<!-- /ANCHOR:docs -->

## File Organization

- [ ] CHK-013 [P1] Only rollup evidence changed during gate execution; no scratch directory or implementation-summary scaffold artifact remains

<!-- ANCHOR:summary -->
## Verification Summary

The subtree passes only when all sibling contracts are green and the final scope-aware census is clean. A discovered
unknown or stale path routes back to its owner; the rollup gate does not absorb or implement that work.
<!-- /ANCHOR:summary -->

<!-- ANCHOR:sign-off -->
## Sign-off

Signed off when the SOL verifier confirms the sibling matrix, final naming census, active-reference closure, and release
evidence, central validation is green, and the pass/block handoff is recorded. No skill-surface migration occurs here.
<!-- /ANCHOR:sign-off -->
