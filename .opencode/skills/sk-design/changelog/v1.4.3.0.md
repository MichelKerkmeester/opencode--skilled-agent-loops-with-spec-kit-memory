# sk-design v1.3.0.0 - v1.4.3.0 - Command Refactor, Open Design Transport, and Manual Playbook Hardening

> Covers the net cumulative change since v1.2.0.0's mode-local procedure operating model: the five `/design:*` commands moved to a router+assets shape, a sixth transport mode joined the registry, and the first-ever full manual execution of the playbook found and fixed a real routing/authoring-boundary defect for every mode.

---

## Added

- `design-mcp-open-design`, a sixth mode on the `transport` axis (`packetKind:"transport"`, `backendKind:"od-cli-transport"`) that bridges to the external Open Design app's `od` CLI and stdio MCP server. It carries `command: null` and is reached only by mandatory pairing with one of the five design-judgment modes, never on its own.
- Router+assets command shape for all five `/design:*` commands, mirroring the `speckit`/`create`/`deep` command family pattern: a thin router file plus `assets/design_<mode>_{auto,confirm,presentation}` files.
- `manual_testing_playbook` grew from 25 to 37 scenarios across three rounds of coverage audits, closing gaps per mode and the parent hub (`AI-004`, `MG-004`, `PB-007`, `HM-004`, plus the multi-probe batteries already covered).
- The first-ever full, genuine manual execution of the entire playbook (56 real dispatches through `cli-opencode`, no analysis-only wrapper) — the automated Lane C benchmark structurally cannot reach roughly a third of the corpus, so this was the first time every scenario ran for real.
- `transformVerbRouting.excludedAliases.audit` (`harden`, `polish`) now has the same documented precedence exception in `SKILL.md`'s Mode Vocabulary Guardrails that `excludedAliases.foundations` (`typeset`, `colorize`) already had.
- `mode-registry.json`'s `transformVerbRouting.note` clarifies that `excludedAliases` (free-text transform-verb framing) and `command-metadata.json`'s `taskProjections` (explicit slash-command ownership) are two intentionally separate resolution layers that name the same verbs, not a collision.

## Changed

- `sk-design/SKILL.md`'s `audit` guardrail bullet gained a "single-axis static review" exception: a review scoped to one `foundations`-owned static axis with its own proof-gate procedure card stays pure `foundations` instead of bundling `audit`.
- Advisor routing for design-scoped review/critique prompts was fixed across BOTH of sk-design's independent scoring backends discovered this cycle: `skill_advisor.py`'s standalone-CLI local-fallback path (`PHRASE_INTENT_BOOSTERS`) and the native TS daemon's live-reloaded path (`graph-metadata.json` `intent_signals`/`derived.trigger_phrases`) — a fix to only one backend does not reach dispatches routed through the other.
- `design-md-generator`'s brief-only authoring-boundary contract (`references/authoring_boundary.md`, `SKILL.md` ALWAYS/NEVER rules) was rewritten from a labeling requirement to a zero-artifact boundary stop after empirical testing showed the labeling-only version still let the mode author a full `DESIGN.md` from a brief with no live site.
- 3 mode packets (`design-foundations`, `design-motion`, `design-audit`) had their front-matter `family` field corrected from a stale `sk-code` to `sk-design`.
- `command-metadata.json`'s `preferSiblingWhen` entries no longer present `/design:design-mcp-open-design` as a runnable sibling command; the transport has none.

## Fixed

- 8 real bugs found by the full manual playbook run (advisor misroutes, a skipped ALWAYS-marked resource load, an unenforced excluded-alias rule, a router-precedence loss on the md-generator authoring boundary, an intake-before-routing violation, a weak-signal transform-verb producing zero routing, an advisor-tier loss to `sk-doc`) — all 12 constituent dispatches re-verified PASS.
- `PB-002`'s advisor-probe and audit-bundling defects, found by a follow-up fresh audit and fixed across the two scoring-backend layers described above.
- `FR-001`'s playbook index prompt, which had drifted from its per-feature file's authoritative text, violating the playbook's own verbatim-prompt rule.

## Preserved

- The public mode taxonomy remains `interface`, `foundations`, `motion`, `audit`, and `md-generator`; the sixth mode is a transport, not a seventh design-judgment mode.
- The advisor identity remains the single `sk-design` hub with exactly one graph metadata file at the hub root.
- `design-md-generator` remains the only mutating design-judgment mode and keeps the Playwright extract-write-validate backend boundary.
