This release lands cli-devin's two deferred reference-doc edits and removes the extra leading divider from two asset docs. The permission-mode reference carried a typo that listed `dangerous` twice, and the quota-fallback doc pointed at a model-profiles.json path that no longer exists. Both are fixed and the docs are internally consistent again. No dispatch behavior changed.

## What Changed

#### Documentation

- **Permission-mode typo fixed** -- `references/cli_reference.md` listed `--permission-mode` as "Choose `auto` (default), `dangerous`, or `dangerous`" and carried a duplicate `dangerous` table row plus a "3-tier risk taxonomy" version-drift note. Devin has exactly 2 permission modes (`auto`/`dangerous`), confirmed against the skill README which states this in five places. We deduped to the 2 real modes, removed the phantom "mostly auto" row, and changed the version-drift wording to "not among the 2 permission modes this skill documents."
- **model-profiles.json path repointed** -- `references/quota-fallback.md` referenced `sk-prompt/assets/model-profiles.json` in four places, a path that does not exist (the registry lives at `sk-prompt-small-model/assets/`). All four now point at the real location. This doc also gains its missing section dividers (deferred from phase 014).
- **Asset leading dividers removed** -- `assets/prompt_templates.md` and `assets/deep-loop-iter-template.md` carried a `---` before section 1 that the sk-doc asset template omits. Removed for consistency with the sibling asset cards.
