## Verdict
**AGREE-WITH-CHANGES** — confidence **0.72**. The external-auditor feedback loop is the right mechanism, but the phase as written overreaches on 2D layout repairs, misorders A4 against the research pipeline, and ships a brittle issue classifier that will misroute fixes.

## What GPT-5.5 got right
- The 18 FIX findings are real, specific, and never fed back — verified from `004-bento-visuals/research/inputs/audit-*.json` (27 SHIP / 18 FIX).
- Keeping generator (GLM-5.2) and auditor (MiniMax-M3) separate is correct; the benchmark notes GLM confabulates audits.
- The `adopt-if` gate with deterministic overflow/contrast/geometry checks is the right guardrail against false-fixes.
- The +1–3 net SHIP prediction aligns with `iter-r4-pipeline.md` §3 adopt/adapt/reject table.

## Gaps / risks / errors
1. **Sequencing conflict with the research pipeline.** The research `iter-r4-pipeline.md` places MiniMax issue→typed JSON at **step 10**, *after* step 7–9 skeleton-first/best-of-3/downgrade. The proposed phase makes A4 **phase 003**, before skeleton-first (004/005). This means A4 will burn round-2 calls on hard 2D tiles that skeleton-first is meant to solve, then still route holdouts to 004 afterward. A4 should run after 004/005, not before.
2. **Adapter regex misclassifies issues.** The `/overlap|collid|crowd|popover|node|branch|hub|matrix|diagram|funnel|connector|truncat/i` pattern fires on almost every issue because words like "matrix," "panel," or "overflow" appear in overflow issues too. Example: `accountbeheer-4` ("matrix rows … overlap bottom title") would emit both `vertical_overflow` and `node_collision`, but a matrix is not a node collision — it is a table/row primitive. `accountbeheer-2` ("red square") would be classified as `typography_or_icon_contract` instead of `off_brand_color`. This will over-generate fixes and push wrong repair contracts.
3. **"Force linear/stacked/bounded-lane" degrades product semantics for 2D tiles.** The spec tells GLM to replace a rights/permissions matrix or a routing diagram with a stacked list to fix overflow (RC-1/RC-2). That may raise the MiniMax score by removing overlap, but it changes the visual encoding — a matrix shows cross-product relationships a list cannot. The "preserve product meaning" invariant is in tension with the layout policy.
4. **Scope ambiguity: FIX-only vs borderline.** `iter-r4-pipeline.md` step 10 says run A4 "after deterministic gates pass **or for borderline failures**." The proposed phase says strictly FIX-only. The two borderline FIX tiles (`een-factuur-1` 88, `kwartaalcijfers-4` 86) are easy conversions; excluding them would understate A4's value and the comparator results.
5. **T0 comparator is not a true no-findings baseline.** Both T0 and T1 inject the failed render image, so T0 is really "look at the broken render and fix it." That overlaps with the A3 render-feedback loop, not with the question "does typed MiniMax feedback beat raw re-rendering?" A cleaner T0 omits the render image and relies only on generic invariants.
6. **Dependency naming drift.** The plan repeatedly references `geometry_check.py`, but phase 001 ships `a1-gate.mjs` / `contrast.mjs` / `a1-repair.mjs`. The contract surface needs to match the actual filenames/schema from 001.
7. **Missing measurement of semantic/primitive regression.** The false-fix gate checks title position, contrast, palette, casing — but not whether the tile still uses the intended primitive (matrix remains matrix, routing remains routing). Add a primitive-consistency check or A4 will silently flatten diagrams into lists.

## Strongest improvement
Narrow A4 to a **post-skeleton, post-downgrade audit translator**: let phase 001 (deterministic gate) and phases 004/005 (skeleton-first/downgrade) own geometry, then run A4 only on the remaining failures for the defects those phases cannot express well — RC-5 contrast, RC-6 off-brand color, RC-4 casing/glyph drift, and minor clipping. For hard 2D layout, A4 should *describe* the defect but route to 004/005, not attempt a prose-driven geometric repair.

## One thing to test before building
Run `classifyFixes` against all 18 issue strings and manually audit the emitted fix types. If the regex maps `matrix` to `node_collision` or `red square` to `typography_or_icon_contract`, the adapter needs a rule-based, not keyword-based, classifier before any GLM calls are wasted.
