**Highest-Risk Item**

The single highest-risk item is advisor mode collapse. It combines high likelihood with critical blast radius because the migration deletes five rankable graph identities, creates one folder-bound ID, and then relies on a new mode discriminator that does not yet exist in the observed scorer aliases/boosts. A false pass here is especially dangerous: the advisor can correctly return `deep-loop-workflows` while selecting the wrong internal mode, loading the wrong resource packet, or applying the wrong permissions.

**Most Likely Plan Mistake**

The thing I most expect the plan to get wrong is half-applying the two-tier mode contract. The existing repo already has three different discriminators: runtime `--loop-type` for graph-backed modes, improvement `loop-host.cjs --mode` for lanes, and Lane D markdown router setup. A mechanical migration will be tempted to normalize them. That would violate the pass-1 architecture and likely break byte-identical artifacts before tests point clearly at the cause.
