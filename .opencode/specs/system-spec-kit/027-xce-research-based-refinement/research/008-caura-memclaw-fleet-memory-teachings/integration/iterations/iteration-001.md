## Integration sequence

1. Amend `008` first because it prevents building the wrong thing. The current `008` parent and children still plan active causal and retention consumers; MemClaw evidence says the safe v1 is event capture plus diagnostics only.

2. Amend `002` provenance next. `source_kind`, manual/constitutional overwrite guards, and automated-mutation audit are prerequisites for every later automated writer.

3. Add 010/P1 idempotency receipts before broader reconciliation. `memory_save` and `memory_update` are retryable write surfaces today, and 006 statediff/replay will make idempotency more important.

4. Align 006 with the receipt primitive and async enrichment. This keeps retry/replay semantics explicit before statediff subscribers move mutation hooks out of handlers.

5. Amend 004 tombstones and indexes. Causal cleanup should be idempotent and audit-preserving before generated edge volume grows.

6. Amend 005 edge promotion. Natural-key upsert plus skip-manual is safe only after 002 manual protection and 004 cleanup semantics are settled.

7. Add 010/P3 and 007 stale/status audit. This is independent enough to run early, but it is most useful once status/tombstone semantics are explicit.

8. Add 010/P2 tool-ownership map. It is low-risk and can land anytime, but dependency-wise it should read the current dispatcher modules and stay generated/linted.

9. Confirm 003/T14 during 003 implementation. It mostly validates the existing 003 design, with `last_dedup_checked_at` as an incremental marker.

10. Add constitutional rules last only after wording is stable. The durable rules should be narrow: no automated overwrite of manual/constitutional state, entity evidence is not causal truth, and feedback cannot demote protected memory.
