# v1.4.0.0 - Own the Open Design transport for the real-UI loop

**Released:** 2026-06-17

The shared design loop was split so `sk-design-interface` can stand alone. The generic, transport-agnostic loop now lives in `sk-design-interface` as `real_ui_loop.md`; this skill owns the Open Design transport mechanics for it in a new local reference. No change to the `od` CLI contract, the tool surface, the gating taxonomy, or the mandatory `sk-design-interface` coupling shipped in v1.3.0.0.

---

## What Changed

### New reference: design_parity_transport.md

- Added `references/design_parity_transport.md`, carrying the Open-Design-specific transport mechanics that previously lived in `sk-design-interface`'s parity doc: reading a system via `od`, reuse via `tokens.css`/`components.html`, `od ui respond` revision routing, the `previewUrl`/`get_run`/`get_artifact` fidelity check, and run self-healing. It references `../sk-design-interface/references/design-process/real_ui_loop.md` for the judgment.

### Repointed references

- `SKILL.md` (§2 resource row), `README.md`, `INSTALL_GUIDE.md`, and `feature_catalog/03--grounding/design-system-grounding.md` repointed from `../sk-design-interface/.../claude_design_parity.md` to the local `references/design_parity_transport.md` plus `real_ui_loop.md`.

---

## Notes

- The mandatory `sk-design-interface` pairing (the MANDATORY PAIRING banner, the routing hard gate, RULES ALWAYS/NEVER, and the success-criteria gate) is unchanged.
- `sk-design-interface` no longer names this skill; the integration is one-way (this skill names that one, not the reverse).
