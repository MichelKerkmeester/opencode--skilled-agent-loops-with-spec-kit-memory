## Zero-manual principles

- Put every invariant at the writer/dispatcher boundary, not in operator instructions.
- Prefer derived metadata over authored metadata: `source_kind`, ownership, stale predicates, and tool counts should be computed from runtime facts.
- Make read-only audits default-on and fail-open; make mutating guards default-on and fail-closed only when data integrity is at risk.
- Use append-only ledgers for automated actions, with deterministic event keys to avoid noisy repeats.
- Run catch-up repair from existing maintenance loops: startup, index scan, retention sweep, `/doctor`, and pre-commit.
- Keep 008 feedback reducers shadow-first: capture events automatically, but do not mutate ranking/retention until measured quality gates pass.

## Top risks + mitigations

| risk | mitigation |
|---|---|
| `mutation-hooks.ts` becomes a dumping ground for policy that must run inside transactions. | Split hook phases: pre-mutation guard, transactional writer, post-mutation invalidation/audit. Keep cache clearing in `mutation-hooks.ts`; keep integrity decisions in write handlers/storage modules. |
| Default-on automation silently changes recall or ranking. | Keep near-duplicate advisory-only and feedback reducers shadow-only; default-on writes can record metadata/audit, not demote memories. |
| Audit ledgers become noisy and unactionable. | Use deterministic event keys, bounded retry counters, actor/source fields, and summary diagnostics in `/doctor`/health instead of raw row dumps. |
| Auto edge promotion overwrites manual causal meaning. | Natural-key dedup must preserve manual `created_by` and evidence; auto can skip or add low-strength parallel evidence only if schema explicitly supports it. |
| Stale/deprecated audit produces false alarms because deprecated rows are intentionally excluded today. | Audit should distinguish intended exclusion from “silent hard exclusion of relevant rows”; report as diagnostic until a policy decision changes recall behavior. |
| Tool-ownership lint drifts if it reads docs as source of truth. | Derive from `TOOL_DEFINITIONS` and command allowed-tool frontmatter; docs are generated/validated outputs, not the authority. |
