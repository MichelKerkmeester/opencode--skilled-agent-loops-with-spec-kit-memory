# /create:agent Presentation Contract

This file is the single source of truth for user-facing presentation in `/create:agent`: startup questions, setup/status dashboards, and result display. The command router owns asset routing only. The workflow YAML owns execution behavior.

## 1. Phase 0 Verification Display

Run this automatic self-check before setup. This is not a user question.

```text
SELF-CHECK: Are you operating as the @markdown agent?

Indicators:
- Invoked with @markdown prefix
- Template-first workflow capabilities are available
- Templates load before content creation
- Template alignment validates after creation

If all indicators are present:
- create_agent_verified = true
- Continue to setup

If any indicator is missing or uncertain:
- Stop before loading workflow YAML
- Display the hard-block message below
```

Hard-block message:

```text
MARKDOWN AGENT REQUIRED

This command requires the @markdown agent for:
- Template-first workflow
- Frontmatter validation
- sk-doc skill integration

Restart with:
@markdown /create:agent [agent-name]

STATUS=FAIL ERROR="Markdown agent required"
```

## 2. Auto Setup Resolution

For `:auto`, resolve setup using the system auto-mode contract, then load `create-agent-auto.yaml` only when every required field is available.

Resolution tiers:

| Tier | Use When | Output |
| --- | --- | --- |
| Tier 1 | `$ARGUMENTS`, flags, pre-bound answers, and defaults resolve all required fields | Persist config and load auto workflow |
| Tier 2 | One or two required fields remain genuinely ambiguous | Ask one narrow question per missing field |
| Tier 3 | Required fields remain unresolved | Emit missing-inputs error and do not load YAML |

Pre-bound setup schema:

```yaml
PRE-BOUND SETUP ANSWERS:
  agent_name: quality-gate
  agent_path: .opencode/agents/
  spec_choice: new
  spec_path: <spec-folder>
  execution_mode: AUTONOMOUS
  memory_choice: skip
```

Default resolution table:

| Field | Required | Resolves Via | Default | Tier-2 Candidate |
| --- | --- | --- | --- | --- |
| `agent_name` | Yes | Positional agent name or marker `agent_name` | none | No |
| `agent_path` | Yes | `--path`, marker `agent_path`, or runtime profile default | runtime agent path | No |
| `spec_choice` | Yes | marker `spec_choice` or targeted choice | none | Yes, when folder choice is ambiguous |
| `spec_path` | Conditional | marker `spec_path`, derived from `spec_choice`, or null when skipped | none | No |
| `execution_mode` | Yes | `:auto`, `:confirm`, or marker `execution_mode` | `AUTONOMOUS` under `:auto` | No |
| `memory_choice` | No | marker `memory_choice`, prior-session detection, or default | `skip` when no prior continuity exists | No |

## 3. Consolidated Startup Prompt

Ask one prompt containing only applicable questions. Do not split these questions across turns.

```text
Before proceeding, please answer:

Q0. Agent Name (if not provided)
What agent should be created?
Format: kebab-case, for example quality-gate or security-audit.

Q1. Spec Folder
A) Use existing: [suggest if related found]
B) Create new spec folder
C) Update related spec: [if partial match found]
D) Skip documentation

Q2. Execution Mode (if no :auto/:confirm suffix)
A) Interactive - confirm at each step
B) Autonomous - execute without prompts

Q3. Prior Session Context (if existing spec has continuity)
A) Load handover plus implementation-summary continuity
B) Load canonical spec docs up to a recent cap
C) Skip

Reply with answers, for example: quality-gate, B, A, C
```

Hard stops:

- Do not proceed until all applicable questions are answered.
- Do not auto-create spec folders without explicit confirmation.
- Do not auto-select execution mode without suffix or explicit choice.
- Do not infer agent names from context, screenshots, or conversation history.

## 4. Setup Dashboard

After parsing setup, display or maintain this dashboard internally before loading YAML.

| Field | Required | Value | Source |
| --- | --- | --- | --- |
| `create_agent_verified` | Yes | `[value]` | Phase 0 self-check |
| `agent_name` | Yes | `[value]` | Q0 or `$ARGUMENTS` |
| `agent_path` | Yes | `[value]` | `--path` or runtime default |
| `spec_choice` | Yes | `[value]` | Q1 |
| `spec_path` | Conditional | `[value]` | Derived from Q1 |
| `execution_mode` | Yes | `[value]` | Suffix or Q2 |
| `memory_loaded` | Conditional | `[value]` | Q3 when applicable |

Proceed only when every required field has a value.
