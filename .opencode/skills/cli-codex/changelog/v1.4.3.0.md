# cli-codex v1.4.3.0 — pre-planning density guidance propagated

**Released**: 2026-05-17
**Status**: Documentation-only addendum. RCAF default in `assets/prompt_quality_card.md` §3 was already present (mirrors `sk-prompt`'s general default). This patch adds the "Pre-planning density" composition-guidance note that complements RCAF.

---

## Summary

The 003-eval-loop empirical run on cli-devin SWE 1.6 measured that **medium-density pre-planning** (3-4 ordered steps with per-step acceptance + verification) outperforms dense pre-plans (4+ steps with full I/O contracts per step) by ~10-15% on aggregate score. The finding likely generalizes across CLI orchestrators because just-enough-structure-not-too-much is a model-agnostic prompt-engineering principle. This card now records it as a recommendation alongside the framework selection table.

The other two SWE-1.6 findings — "standard-over-strict bundle-gate" and "framework-dominates-anti-hallucination" — were NOT propagated. They came from a small coding-specialized model and may not generalize to frontier models (claude-opus-4.7, gpt-5.5, deepseek-v4-pro, gemini-2.5-pro). Packet 118-cross-model-validation will measure whether they hold on deepseek-v4-pro + kimi-k2.6 before any cross-CLI propagation.
