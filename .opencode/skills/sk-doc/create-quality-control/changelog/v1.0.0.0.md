# v1.0.0.0 - Initial Release

**Released:** 2026-07-06

Initial release of `create-quality-control`, the existing-document audit and optimization workflow packet in the `sk-doc` parent hub's ten-packet workflow-only architecture. It evaluates a target markdown document, extracts its structure and metrics, computes a Document Quality Index (DQI) score and quality band, applies Human Voice Rules (HVR) for natural writing, and, only when edits are explicitly requested, makes targeted improvements to that same document's structure, clarity, and AI-friendliness.

## What's Included

- `SKILL.md` -- the inline workflow contract: confirm target and mode (report-only audit, structure validation, content optimization, or batch snapshot), run structure extraction, interpret quality gates (blocking failures, warnings, recommendations), apply HVR voice review, and report findings; a separate content-optimization path for question-answering transformation patterns when edits are in scope.
- `references/workflows.md` -- extended mode examples, phase interactions, batch patterns, and troubleshooting for the core workflow.
- `references/optimization.md` -- full transformation-pattern examples and AI-friendly documentation heuristics for the content-optimization path.
- No packet-local `assets/` or `scripts/` directories. The packet runs the shared `../shared/scripts/extract_structure.py`, `validate_document.py`, and `quick_validate.py`, and reads the shared `../shared/references/global/` standards (core standards, validation bands, HVR rules) rather than duplicating them locally.
