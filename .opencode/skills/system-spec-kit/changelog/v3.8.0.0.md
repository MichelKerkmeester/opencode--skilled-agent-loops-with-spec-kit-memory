# v3.8.0.0, The Dark Flags Earn Their Verdict

v3.7.0.0 closed with a row of experiments parked as still dark, awaiting a benchmark. This release is where they face the bench. Each candidate was measured against the real graph and corpus on its production path, the winners earned a production-readiness pass, and the one that mattered most, the bitemporal edge history, was caught corrupting live reads by a deep review before it could ship, then fixed and confirmed by a second review.

The discipline did not change. A flag earns its place on real measurement or it stays off, and nothing here flips a default on the path you use every day. Every graduate ships behind its existing flag, byte-identical when off, and the flip itself remains a separate decision you hold. What changed is that the dark flags now carry verdicts instead of question marks, and the one that hid a corruption was caught reading the code rather than trusting a green test.

The shape of the release is a reckoning followed by a repair. A benchmark suite measured eight dark-flag families and applied the verdicts, four graduated and four were cut or held. A production-readiness pass then wired the graduates for real and proved each one byte-identical when off. A ten-iteration deep review on opus read every line of that pass and failed it on three blockers, all in the bitemporal wiring. The fixes closed the blockers, a second review confirmed the repair and a final pass closed every remaining recommendation and follow-up.

## Code Graph

The structural index is where this release earns its title. A dark experiment to give every edge a readable history was wired up, caught broken and rebuilt, and the staleness repair that bounds re-parsing got a real number behind its default.

#### The Staleness Cap, Chosen by a Cost Sweep

The reverse-dependency degree cap decides how many importers get re-bound when a high-fan-in dependency is renamed. Before this release it defaulted to uncapped, so renaming a thirty-importer module re-parsed all thirty. The cap now defaults to fifteen, and that number is not a guess. A degree-sweep benchmark drove the real scan handler across importer counts from five to twenty-five and found the re-bind cost is linear with no knee, about a third of a millisecond to under a millisecond per importer over a seventeen-millisecond scan floor. Fifteen fully repairs the common-to-moderate refactor at negligible cost while bounding the unmeasured hundred-plus-importer tail. The honest cost is recorded in the docs: above the cap, a high-fan-in rename leaves the overflow importer edges stale until each one is next edited. The cap is read only inside the force-parse branch, which is off by default, so the value is inert until you opt in.

&nbsp;

#### The Bitemporal Writer, Wired and Then Caught

The headline is a save from a silent corruption. A pre-graduation review had found the bitemporal close-and-insert writer was built but never wired into the reindex path, so it was held at refine. The production-readiness pass wired it in, and the test suite went green. A ten-iteration deep review on opus then read the code and failed it on three blockers, because the tests were green only for cases that never touched the real failure modes.

Before the fix, turning the flag on would have corrupted every live read. The live edge readers had no filter for closed edges, so after a reindex they returned both the superseded edge and its replacement. The production prune that runs at the end of a full scan deleted dangling edges regardless of the flag, and node replacement deleted a symbol's edges before the close path could run, so there was nothing left to close and an as-of query came back empty. After the fix, node replacement and the dangling prune close instead of delete when the flag is on, the live readers return only the current edge and a generation-stamp correction gives every superseded edge a real readable window instead of a zero-width one. A second deep review confirmed the repair with no blockers remaining.

&nbsp;

#### History You Can Query

A preserved history is worthless if nothing can read it. The first wiring left the as-of reader with no caller, so the review held it as honestly out of scope rather than letting it claim a feature that did not exist. This release wires it through: code graph queries take an optional as-of generation parameter that routes edge reads through the temporal readers and a new inbound mirror, deliberately skipping the dangling-edge exclusion because a historical edge legitimately points at a node that has since been deleted. Ask for a past generation and you get the old target, ask without the parameter and you get the current graph exactly as before. The whole surface stays byte-identical when the flag is off.

## Search Intelligence

The track-B search appends that v3.7.0.0 listed as waiting on a rewire reached the deep-reader path, and a budget-trim edge case that could have dropped the best result was closed.

#### The Multi-Hop Appends Reach the Deep Reader

The deterministic multi-hop and lane-champion backfill appends were measured and graduated for deep-K readers. On the labelled multi-target queries, complete recall at twenty rose from just over half to almost all while the shallow ranks stayed flat, so a reader that asks for a wide result set now gets the related matches that the pipeline used to leave on the floor, and a reader that asks for the top few sees no change. The win is deep-K only and is recorded as such rather than dressed up as a blanket recall gain.

&nbsp;

#### The Budget Trim Keeps the Best Result

Tail-appended rows are now exempt from the response-serialization token budget, so a backfill row added past the limit is not the first thing cut. The review found the sharp edge in that exemption: under the hardest squeeze the trim could drop every primary row before touching an exempt backfill, returning a response made entirely of backfill with the top-scored result evicted. The trim now reserves at least one primary row and keeps a pinned constitutional rule ahead of any additive backfill, so the squeeze can thin the result set without ever sacrificing the answer you asked for.

&nbsp;

#### A Density Probe for the Citation Ledger

The true-citation emitter stays at refine, sound but gated on real session-carrying traffic rather than more code. This release adds a density probe that surfaces through the health check and reports when the ledger has accumulated enough session-scoped pairs, with a per-class floor and a minority-ratio floor so a lopsided ledger cannot graduate the reranker on a count alone. The probe observes, it does not flip anything on.

## Skill Advisor

The advisor's ranking spine graduated, and the quiet penalty that keeps it honest about recommending itself was documented, locked and widened.

#### RRF Fusion Graduates With Its Safety Net

The reciprocal-rank-fusion spine was measured on a widened forty-two-prompt set and graduated, lifting top-1 by one with zero regressions and deterministic, near-total agreement. It ships paired with the conflict-rerank seam because the seam is RRF's own regression net, repairing a case that plain fusion would otherwise break. The separate self-recommendation guard was cut in the same pass, measured as behaviorally redundant because the generic explainer floor and an un-flagged penalty already did its work.

&nbsp;

#### The Self-Recommendation Penalty, Documented and Widened

Cutting that guard left an implicit penalty as the only thing stopping the advisor from recommending itself on an audit or explainer prompt, and that penalty was undocumented, so a future refactor could have removed it without anyone noticing. It now carries a durable explanation and a contract test, validated two ways so the lock is real. The review then found the penalty matched only the advisor's exact id and missed its alias, which still recommended itself to the top in the default configuration. The predicate now uses the canonical id set, so both the exact name and the alias are demoted, and a pre-existing test that had encoded the alias bug was corrected to the right behavior. The penalty value itself is unchanged.

## Deep Loop

The finding dedup graduated and then learned to read titles, and the lag gauge stopped crying wolf.

#### Dedup That Respects the Title

The near-duplicate finding collapse was graduated on a clean benchmark, then scale-validated past its original handful of synthetic records on a larger free-text set. That scale test surfaced the exact risk the earlier validation had flagged: the dedup key ignored the finding title, so two genuinely-distinct findings that shared a body but differed only in their title collapsed into one, a measured one-in-two false collapse on that shape. The key now applies a title-overlap gate, so disjoint titles stay separate while a true restatement still merges. The title-only false collapse dropped from one-in-two to zero, while exact duplicates still collapse and distinct recall holds.

&nbsp;

#### A Lag Gauge That Means It

The lag-ceiling gauge graduated alongside the dedup, but its metric was measuring the wrong thing. It read time since a worker was queued, which is queue back-pressure, so any healthy fan-out wider than its concurrency tripped the alarm. The metric is now time since the last completion while work is still pending, a real stalled-worker signal. A healthy busy pool reads quiet, a genuinely hung one fires once and the recommended ceiling dropped from five minutes to two now that it no longer drifts on healthy load. The committed default stays off because turning the gauge on is the separate graduation step.

## The Review That Caught a Masked Corruption

This release is the clearest case yet for the project's shipping rule, that a result-affecting change is not done until something adversarial has read it. The production-readiness pass shipped with a green test suite. A cli executor ran the tests and they passed. By the old standard the bitemporal wiring was ready to graduate.

A ten-iteration deep review on opus, reading the code under rotating lenses rather than running the existing tests, returned a FAIL with three must-fix blockers, eight should-fix findings and four advisories. The blockers were not subtle once seen: the live readers returned closed edges, the production prune deleted history the feature existed to keep, and node replacement deleted the edges before the close path could run. The tests were green because every flag-on assertion went through the one reader that happened to filter correctly, while the readers production actually calls were only ever checked with the flag off. The gap between green and correct was exactly the gap the review was there to close. Each blocker was fixed, a second review on opus confirmed the repair at zero blockers, and a final pass closed every remaining recommendation and follow-up at zero as well.

## The Graduation Timeline

The arc is worth reading as a sequence, because each step depended on the honesty of the one before it.

1. **Dark.** v3.7.0.0 shipped the multi-hop appends, the bitemporal reads, the edge governance vocabulary, the RRF spine, the self-recommendation guard and the near-duplicate dedup as default-off experiments, explicitly parked until a benchmark could speak.
2. **Benchmarked.** A graduation suite measured eight families against the real graph and corpus on their production paths. Four graduated, the multi-hop appends for deep-K, the staleness degree cap, the bitemporal writer and the RRF spine, and four were cut, the retrieval-class weights, the save reconsolidation, the seeded PageRank and the edge governance vocabulary. The self-recommendation guard was cut as redundant and the true-citation ledger was held at refine.
3. **Validated.** A pre-graduation review and research pass confirmed the winners were correct and byte-identical when off, but qualified the posture: the bitemporal writer was unwired, the cap and gauges defaulted to off, the dedup was synthetic-only and the multi-hop win was deep-K only.
4. **Followed up.** A production-readiness pass cleared those qualifications, wiring the bitemporal writer, setting the cap and gauge values, exempting the search appends from the budget, adding the citation density probe and documenting the advisor penalty.
5. **Caught.** A ten-iteration deep review on opus failed that pass on three bitemporal blockers the tests had masked.
6. **Fixed and confirmed.** The blockers were fixed, a second review confirmed PASS, and a final pass closed the remaining recommendations and follow-ups, wiring the as-of query surface, setting the degree cap from a sweep, turning the lag gauge into a true stall detector and making the dedup title-aware. A third review confirmed PASS.

## The Flag Reckoning

The dark-flag suite measured eight families and applied the verdicts in one program rather than on a hunch.

#### Graduated

The multi-hop tail appends graduated for deep-K readers, the code-graph staleness degree cap graduated with a benchmarked default, the bitemporal edge writer graduated after the review caught and the repair closed its corruption, and the advisor RRF fusion spine graduated paired with its conflict-rerank seam. The deep-loop near-duplicate dedup graduated with its lag and heartbeat gauges. All ship behind their existing default-off flags, byte-identical when off.

&nbsp;

#### Held at Refine

The true-citation ledger is sound but stays at refine, gated on real session-carrying traffic accumulating before it can clear the reranker bar. The new density probe exists to tell you when that day arrives.

&nbsp;

#### Cut

Five experiments were built, measured and declined because they did not earn their place: the per-class retrieval weights, which hurt single-hop precision to guard a cost that did not exist, the save reconsolidation cosine band, which merged almost nothing it should have, the seeded PageRank, which tied the flat walk on every metric, the closed-vocabulary edge governance, and the advisor self-recommendation guard, redundant with logic already running.

## Upgrade Notes

No migration is required and nothing changed on the common path. Every graduate in this release ships behind its existing default-off flag and is byte-identical when off, so the production flip remains a separate, evidence-gated decision. If you opt into the bitemporal edge reads, the recommended degree cap is fifteen, the as-of query parameter is available on code graph queries, and the live readers return the current edge. If you opt into the deep-loop gauges, the recommended lag ceiling is two minutes against the new stall metric and the recommended heartbeat is thirty seconds. The advisor self-recommendation penalty change is the one deliberate behavior shift that is not flag-gated, because demoting the advisor's own alias on an audit prompt is a correctness fix that the un-aliased path already applied.

## Verification

This release follows the same shipping rule as the last. A structural protection that only ever makes the system safer or cleaner ships on for everyone, and anything that changes a result ships off by default behind a named flag until a benchmark earns it on. Here that rule was enforced by a benchmark suite and then by three independent deep reviews on opus.

#### Earn or Cut, Measured

The eight dark-flag families were decided by real measurement, not by argument. Four graduated on a measured win, five flags were cut on a measured loss or a measured tie, and the true-citation ledger was held on a clear-eyed read that the signal is sound but the data is not there yet. The multi-hop recall gain, the RRF top-1 lift and the degree-cap cost curve were each read off the real path.

&nbsp;

#### The Review That Earned Its Keep

The bitemporal corruption is the proof that a green test is not a finished feature. The cli test pass was honest about what it ran and wrong about what that meant, because it never exercised the readers production calls. A deep review reading the code found the gap, the fix closed it, a second review confirmed the fix and a third confirmed that every remaining recommendation and follow-up was closed. The two residual advisories that survived are honest about their scope: they touch only the as-of read surface under the flag and never a live read. They are tracked for the day the as-of surface graduates.

&nbsp;

#### Byte-Identical When Off, Proven

Every flag-gated change in this release was proven byte-identical with its flag off, by dedicated off-path tests rather than by assertion. The bitemporal readers run their original queries, node replacement and the prune delete as before, the search appends never mark a row, the lag gauge emits no field and the dedup keeps both distinct findings. The system you run today behaves exactly as it did, until you decide to turn one of these verdicts on.
