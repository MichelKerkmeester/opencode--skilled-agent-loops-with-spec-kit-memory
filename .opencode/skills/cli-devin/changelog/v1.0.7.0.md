# cli-devin v1.0.7.0 — prompt_quality_card.md semantic fork reconciled

**Released**: 2026-06-02
**Status**: Documentation + SKILL.md alignment. No invocation-shape changes.

---

## Summary

`references/prompt_quality_card.md` had grown a parallel prompt-framework taxonomy that conflicted with the project-wide canonical one owned by `sk-prompt`. CLEAR + the 7-framework table (STAR, BUILD, ATLAS, CONTEXT, TIDD-EC, RACE, COSTAR) are the authoritative prompt-craft framework — delegated to `sk-prompt`. Inside cli-devin those same acronyms (STAR, BUILD, ATLAS, CONTEXT, CLOUD-HANDOFF) serve a different role: they are Devin **model-selection and dispatch mechanics**, not competing prompt frameworks. The card had been presenting both layers with equal standing, creating confusion about which surface governs prompt construction vs. which governs model/mode selection.

This change reframes the card so that distinction is explicit, moves the permission-mode / Risk-Tier check out of the CLEAR pre-flight into a non-CLEAR pre-dispatch addendum, and delegates the SWE-1.6 prompt-quality contract to the hub profile at `references/models/swe-1.6.md`. A shared 3-tier precedence rule (`sk-prompt` framework → model profile → card addenda) and a cross-link to the hub profile were added to both the card and SKILL.md so the precedence order is visible from the first entry point a caller reads.

## Precedence Rule (summary)

1. **`sk-prompt` 7-framework table** — governs prompt construction (CLEAR scoring, framework selection).
2. **Model hub profile** (`references/models/<model>.md`) — governs per-model prompt contracts (output shape, budget, gotchas).
3. **Card addenda** (`prompt_quality_card.md` pre-dispatch section) — governs Devin-specific dispatch mechanics (model-selection mnemonics, permission-mode / Risk-Tier gate, cloud-handoff trigger).

---

## Upgrade Notes

No runtime or invocation changes. Callers who were using the card as a prompt-framework reference should switch to `sk-prompt` for that concern. The dispatch-mechanics mnemonics (STAR/BUILD/ATLAS/CONTEXT/CLOUD-HANDOFF) remain valid for model selection; they are documented in the card's "Dispatch Mechanics" section as before.
