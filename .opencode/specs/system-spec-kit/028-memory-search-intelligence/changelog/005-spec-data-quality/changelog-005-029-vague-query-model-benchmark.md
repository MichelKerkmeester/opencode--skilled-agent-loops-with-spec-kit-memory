## 2026-06-22

> Spec folder: `.opencode/specs/system-spec-kit/028-memory-search-intelligence/005-spec-data-quality/029-vague-query-model-benchmark` (Level 2)
> Parent packet: `.opencode/specs/system-spec-kit/028-memory-search-intelligence/005-spec-data-quality`

### Summary

A characterization benchmark that measured how four LLMs drive the `/memory:search` command on deliberately vague queries. It dispatched a 144-cell matrix of bare read-only queries, four models by twelve vagueness-graded queries by three samples, and parsed each event stream into per-model behavioral profiles. The headline is that the citation surface is fully model-robust at a cite-correct rate of 1.0 while the only cross-model variance is in envelope fidelity. This phase ships measured data and a model recommendation, not code.

### Added

- A reproducible benchmark harness in `scripts/`, a config naming the matrix, a concurrent dispatch driver, and an event-stream parser that aggregates to mean plus variance.
- `benchmark-results.md`, the full per-cell and aggregate data tables with cross-sample variance.

### Changed

- No production code changed. The run is read-only against the `/memory:search` command, its envelope, and the underlying retrieval, all left untouched.

### Fixed

- No fixes recorded. There is no correct answer to a vague query, so the deliverable is a characterization not a gate.

### Verification

- All 144 of 144 cells parsed - `results/metrics.json` reports zero cells failed or truncated below the 200-byte floor.
- Every analysis number is sourced - each value in `benchmark-results.md` and the summary traces to `metrics.json`, generated by the parser from the raw streams.
- The run wrote no memory record - only bare-query retrieval was dispatched, no analysis subcommand, so the run is read-only.
- The matrix reproduces from the committed config - `node scripts/run-benchmark.mjs` regenerates it, skipping any cell that already has a raw file.
- `validate.sh --strict` on this phase exits clean.

### Files Changed

- `scripts/benchmark-config.json`: the model, query and sampling matrix.
- `scripts/run-benchmark.mjs`: the concurrent dispatch driver.
- `scripts/extract-metrics.mjs`: the event-stream parser and aggregator.
- `results/`: the raw outputs, timing sidecars and `metrics.json`.
- `benchmark-results.md`: the full per-cell and aggregate data tables.

### Follow-Ups

- Consider a standing red-team query for the `kubernetes` off-corpus false-relevance, where every model awarded a good verdict at 0.78 on an unrelated doc.
- A follow-on running every model at both medium and high would separate the model from the reasoning tier, since gpt-5.5 ran at medium against a high trio.
