# cli-opencode v1.3.5.0 — Xiaomi Token Plan (Europe) + MiMo-V2.5-Pro integration

**Released**: 2026-06-01
**Status**: Documentation + registry addendum. Adds the Xiaomi Token Plan (Europe) provider and the MiMo-V2.5-Pro model as an explicitly-selectable dispatch target. No default change — `opencode-go/deepseek-v4-pro` stays the skill default.

---

## Summary

A live probe on opencode 1.15.13 (2026-06-01) confirmed the Xiaomi Token Plan (Europe) provider `xiaomi-token-plan-ams` is configured and responsive: a one-shot dispatch to `xiaomi-token-plan-ams/mimo-v2.5-pro` returned cleanly with **no** `--agent` flag. This release wires MiMo-V2.5-Pro into the skill's registry and prompt assets so the model becomes an explicitly-selectable target, mirroring how MiniMax (`minimax-coding-plan`) is already wired.

MiMo is an **explicitly-selectable** model — it does not change any default. The best prompt framework is **not yet determined**: it will be benchmarked in 126/004, so no framework winner is claimed (no TIDD-EC / RCAF / any call). `context_length` and `--variant` behavior remain unknown pending 126/003.

## MiMo Additions

- **Provider**: `xiaomi-token-plan-ams` (credential display: "Xiaomi Token Plan (Europe)"). Auth via `opencode auth login` → pick "Xiaomi Token Plan (Europe)". The endpoint is provider-managed (base URL not published — do not fabricate one).
- **Primary model**: `xiaomi-token-plan-ams/mimo-v2.5-pro` (live, lowercase slug — never CamelCase). Sibling live ids in the text/coding family: `xiaomi-token-plan-ams/mimo-v2.5`, `xiaomi-token-plan-ams/mimo-v2-pro`, `xiaomi-token-plan-ams/mimo-v2-omni`.
- **Quota pool**: `xiaomi-token-plan`.
- **Free gateway path**: `opencode/mimo-v2.5-free` (via the opencode-go gateway) — cheap-iteration / probe path; shares the opencode-go credit pool and is the v2.5 (not -pro) tier.
- **Registry entry**: MiMo registered as an explicitly-selectable target; the skill default is unchanged (`opencode-go/deepseek-v4-pro`).
- **Prompt template**: Template 15 documents the dispatch shape only (framework neutral, pending benchmark).
- **Pre-flight detection**: confirm live ids with `opencode models xiaomi-token-plan-ams` before dispatch.

## Dispatch Contract

- **`--agent`**: OMIT. On opencode 1.15.13, `--agent general` warns ("agent general is a subagent, not a primary agent. Falling back to default agent") and falls back — same contract as MiniMax. State any agent role in the prompt body instead.
- **`--variant`**: behavior UNVERIFIED → omit by default (pending 126/003).
- **`context_length`**: UNKNOWN → null placeholder (pending 126/003). No number is claimed.
- **Best prompt framework**: UNKNOWN → pending 126/004. No framework winner is claimed yet.
