# system-deep-loop v2.0.0.0 — Reunification with deep-loop-runtime

> Folds `deep-loop-runtime` (the frozen, MCP-free backend) into `deep-loop-workflows` as a nested `runtime/` infrastructure layer, and renames the unified skill `system-deep-loop`. Major version reset signals the second era of this family: era 1 was the FULL_ISOLATE_NO_MCP split into two skills (packet 116, 2026-05-22); era 2 is this reunification.

---

## Added

- `runtime/` — the former `deep-loop-runtime` skill, nested as infrastructure (not a mode): `lib/{deep-loop,coverage-graph,council}/`, `scripts/`, `tests/`, `database/`, `feature_catalog/`, `references/`, `changelog/` (frozen at v1.5.0.1), `manual_testing_playbook/`. No `graph-metadata.json` of its own — not independently advisor-routable.
- `runtime/README.md` §9A "Operating Rules" — folded in from the former `runtime/SKILL.md`'s ALWAYS/NEVER/ESCALATE rules and the per-runtime agent-mirror convention, since that file's routable-skill frontmatter no longer applies but its operational content remains genuinely load-bearing.

## Changed

- Skill identity: `deep-loop-workflows` → `system-deep-loop` across `SKILL.md`, `description.json`, `mode-registry.json`, `hub-router.json`, `README.md`, `graph-metadata.json` (`skill_id`).
- `graph-metadata.json` — fresh-authored, not merged from the two prior source files. The `deep-loop-runtime ↔ deep-loop-workflows` inter-skill edges (`depends_on`/`siblings`) are dropped entirely — that relationship is now intra-skill structure, not a graph edge. `depends_on: system-spec-kit` weight raised 0.6→0.7 to also cover `runtime/`'s TypeScript tooling-borrow.
- Bidirectional relative-path coupling repaired at the new nesting depth across ~45 code files: forward (`runtime/` → workflows-content) kept the same hop-count and dropped the `deep-loop-workflows` path segment; reverse (workflows-content → `runtime/`) dropped one hop and renamed the segment to `runtime/`. One file (`deep-ai-council/scripts/replay-graph-from-artifacts.cjs`) needed a segment *inserted* rather than hop-math, since it does its own repo-root detection walk.
- `runtime/package.json` (`typecheck` script) and `runtime/tsconfig.json` (`typeRoots`) — the pre-existing `system-spec-kit` TypeScript-tooling borrow, repointed for the new depth (one more `..`). `system-spec-kit/mcp_server/package.json` (`test:council`) and `vitest.config.ts` (test-include glob) — the reverse half of that same borrow, repointed to the new `system-deep-loop/runtime/` path.
- `system-spec-kit/mcp_server/tests/council-playbook-anchor-integrity.vitest.ts` — a `system-spec-kit`-owned test that is itself part of `test:council`'s own run list and hard-crashed on the old hardcoded playbook path; fixed in this release since `test:council`'s own exit gate depends on it.
- `.opencode/commands/deep/{research,review,ai-council}.md` — the 3 router one-liners invoking `render-command-contract.cjs` by hardcoded path, pulled forward from the planned reference-migration phase to close a sequencing gap (this phase's own live-verification step needs a working command surface).
- `deep-improvement/assets/non_dev_ai_system/profiles/deep-loop-runtime.json` — the Lane-D self-target benchmark profile's `packaging_root` and ~20 path fields repointed; its `frozen_surface.live_relpath` repointed from the now-folded `SKILL.md` to `README.md` with a confirmed-present section anchor.
- ~29 additional self-referencing paths inside `deep-improvement/`'s own test suite (`model-benchmark/`, `shared/`, `agent-improvement/` test files) that named `.opencode/skills/deep-loop-workflows/` — simple workspace-root-anchored renames, unaffected by the depth change since `deep-improvement` itself didn't move deeper.
