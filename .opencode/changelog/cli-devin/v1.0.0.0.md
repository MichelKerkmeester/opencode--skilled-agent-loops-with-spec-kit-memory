This first release lets other AI assistants delegate focused work to Cognition AI's "Devin for Terminal" — the official Rust `devin` binary. It packages safe execution rules, permission-mode taxonomy, four-model preset routing, and a documented local-to-cloud handoff so teams can hand work to Devin without losing control of orchestration.

---

#### Structured delegation into Devin

Other assistants needed a clear way to decide when Devin was the right helper. The skill introduces routing patterns matched to Devin's four-model preset: SWE-1.6 for context gathering / tool use / simple-to-medium well-defined code tasks (the default); DeepSeek v4 Pro as the primary pick for complex tasks (ambiguous, multi-step, reasoning-bound); GLM 5.1 as the complex-task fallback for agentic / tool-use shape; Kimi k2.6 as the complex-task fallback for large-context shape. Requests can be handed off intentionally instead of treating Devin as a generic fallback.

---

#### Cloud-handoff capability documented as a first-class capability

Devin's local-to-cloud handoff is the only async-execution capability in the cli-* family. The release ships a dedicated `cloud_handoff.md` reference with a 5-check operator-confirmation gate so the headline differentiator is reachable end-to-end without inferred consent. Cloud sessions transmit local repo state and consume Devin units, so the gate is enforced before any handoff invocation.

---

#### Safety, permission modes, and self-invocation guard

Delegated work needed guardrails. The package documents three permission modes (`normal` / `dangerous` / `bypass`) mapped to the family's risk taxonomy, plus a layered self-invocation guard (env-var probe, process ancestry, speculative lockfile) that fails closed when the calling AI is itself a local `devin` session. The cloud handoff is the documented exception — explicit phrasing + operator confirmation in the same turn.

---

#### Reusable support material

People needed templates and validation scenarios they could reuse. The release ships a SKILL.md mirroring the cli-* family's 8-section contract, a README with the family's 9-section TOC, five references (cli_reference, integration_patterns, agent_delegation, devin_tools, cloud_handoff), two assets (prompt_quality_card, prompt_templates with 6 copy-paste templates), and a 25-scenario manual testing playbook across 9 categories.
