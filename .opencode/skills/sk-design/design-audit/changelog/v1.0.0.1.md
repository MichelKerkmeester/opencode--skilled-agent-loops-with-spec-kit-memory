# v1.0.0.1 - Deep-Review Remediation

**Released:** 2026-06-25

Patch release applying the actionable findings from the family deep review of `audit`.

## Fixed

- **Least-privilege drift:** Removed `Bash` from `allowed-tools`. This is a read-only review skill with no shipped shell use and an explicit rule against claiming scans ran; `allowed-tools` is now `[Read, Grep, Glob, Task]`, matching the read-only `sk-design` siblings.
- **Family field dual meaning:** Set `metadata.family` in `SKILL.md` to `sk-code` so it matches `graph-metadata.json` and the compiler-validated family allow-set (`sk-design` is not an allowed family value).
- **Advisor freshness coverage:** Expanded `graph-metadata.json` `derived.key_files` to include `references/corpus_map.md`, the three feature detail files, and the three manual playbook scenario files, so edits to those routable resources trigger advisor reindexing.
- **Skill-graph weight band:** Brought the `sk-design` sibling weight to `0.6` (was `0.7`, outside the recommended `[0.4, 0.6]` sibling band) to clear the `skill_graph_validate` warning and reciprocate the parent edge.
- **Router pseudocode contradiction:** Removed the unreachable all-zero `AUDIT_CONTRACT` default in `classify_intents` so the documented fallback is consistent with the `UNKNOWN_FALLBACK` low-confidence branch, matching the shared smart-router template.

## Improved

- **Feature catalog coverage:** Added an anti-patterns and production capability row and an explicit map from all five scored dimensions (accessibility, performance, responsive design, theming, anti-patterns) to discoverable detail files.
- **Manual playbook repeatability:** Added an explicit `<TARGET>` slot to each scenario, a global SKIP rule (SKIP only when no target artifact can be supplied), and a corpus-integrity assertion that the a11y/performance gate uses INP rather than the deprecated FID.
