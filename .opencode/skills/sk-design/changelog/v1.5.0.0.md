# sk-design v1.5.0.0 - Styles-Library Utilization

> Covers the shipped styles-library capability across implementation phases 004 through 010: deterministic retrieval, one shared zero-hydration seam, mode-owned consumers, generator schema and STUDY controls and metadata-only Open Design reconciliation.

---

## Added

- A committed retrieval engine over all 1,290 style bundles with byte-stable manifest checks, eligibility-before-ranking, compact candidate cards, disposable in-memory full-text ordering and bounded source-scan fallback.
- Generation-guarded, realpath-contained hydration plus the manifest-bound `CORPUS_USE_PROOF v1` ready gate. Stale generation, stale record, restricted rights and path escape return closed refusals.
- `CORPUS_CONTEXT_PLAN v1` under `shared/corpus-context/`, with zero hydrated styles, one seven-field proof handoff and accepted negative outcomes for no-fit, unavailable, generation mismatch and unknown rights.
- Mode-owned consumers for interface, audit, foundations and motion, each with a closed evidence schema and source-bound tests.
- A metadata-only Open Design grounding receipt, immutable pre-await proposal snapshot and semantic return reconciliation that leaves acceptance with the selected design mode.
- One `V3_SCHEMA` authority for generator emission, prompt instructions and validation, plus bounded STUDY exemplars with injection neutralization, de-literalization, source-leak detection and a real retry without STUDY.

## Changed

- Styles-library evidence now has an enforced authority order: user brief and owned system, selected-mode judgment, target evidence and deterministic checks, corpus reference evidence, then transport output.
- Corpus evidence is advisory-only. It may explain relationships, expose counterexamples, sharpen critique and preserve provenance, but it cannot select a mode, assign severity, prove accessibility or performance, establish copying, authorize exact reuse or accept transport output.
- Interface emits a decision-only relational handoff, audit emits zero to two non-authoritative comparison rows, foundations emits typed relationships and motion runs target-owned restraint before retrieval.
- Generator corpus divergence remains advisory while target, schema and provenance failures remain hard and immutable.

## Preserved

- The public mode taxonomy and routing vocabulary are unchanged.
- `hub-router.json` and `mode-registry.json` remain the routing sources and do not route the shared seam as a mode.
- The corpus does not own taste, target values, audit verdicts, readiness or mutation approval.
- Disabling STUDY restores the ordinary generator write path.
