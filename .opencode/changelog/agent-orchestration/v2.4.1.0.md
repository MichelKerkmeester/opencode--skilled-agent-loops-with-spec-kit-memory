Closes the gap where the Multi-AI Council, when invoked directly as a main agent in opencode, often produced a chat-form report but left the `ai-council/` artifact tree empty. Predecessor packet 098 shipped the writers and scoped-write permissions; this release adds the missing compulsion so the council either persists the canonical artifact set or HALTs and asks where to put it. There is no third path where deliberation happens but artifacts do not.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/100-multi-ai-council-main-agent-write-enforcement` (Level 2)

---

## What Changed

#### Bug Fixes

- **Packet path resolution gate** -- The agent's workflow had no step that asked "which packet's `ai-council/` folder will I write to?" before dispatching seats. Added a new §1 Step 0 RESOLVE that fails closed: it tries the prompt, then the active continuity ladder, then the closest spec-folder ancestor of the working directory, and if none of those yield a path it HALTs and asks the user. No seat dispatch happens until a packet is rooted. Why it matters: this kills the "council ran but nothing on disk" failure mode at its source.
- **Hard write-or-fail rule** -- §7 ALWAYS only enforced upper-bound scope (do not write outside `ai-council/**`); it never enforced the floor (you must write inside it). Added ALWAYS bullets for resolving the packet, persisting before completion, and using the named writer exports, plus matching NEVER bullets banning chat-only delivery, unrooted deliberation, and out-of-scope writes. Why it matters: the agent can no longer interpret persistence as optional.
- **Persistence verification checklist** -- §9 had verification blocks for deliberation quality and plan feasibility but none for "did I actually write the artifacts?" Added a new MANDATORY 9-item PERSISTENCE VERIFICATION block that runs parallel to the existing checks, extended SELF-CHECK from 10 to 11 questions (Q11 is "did I persist the canonical artifact set including `council-report.md`?"), and updated the `DO NOT CLAIM COMPLETION` failure handler to cover Q11. Why it matters: the agent's own pre-delivery audit now catches missing artifacts before it tries to claim done.
- **Output protocol no longer conditional** -- §12 opened with "When invoked with a `spec_folder`, persist council artifacts under `ai-council/`," which implied persistence was optional when no spec folder appeared in the prompt. Flipped the opening line to unconditional and pointed it at §1 Step 0 RESOLVE as the always-on packet-path source. Why it matters: removes the ambiguity that previously let the agent rationalize chat-only output.
- **Invocation contract is now a numbered checklist** -- §13 described the first-call sequence as a prose paragraph that read like guidance, not a contract. Restructured into a 10-step numbered checklist with explicit `writeConfig` then `writeStrategyMd` then `writeStateJsonl(round_start)` then `writeSeat`xN then `writeDeliberation` then `writeReport` then `writeStateJsonl(council_complete)` ordering, and required an `artifact_written` event before each next step. Why it matters: the workflow loop now has a step-by-step contract it can fail on, not a paragraph it can skim.

#### Architecture

- **Codex mirror drift cleanup** -- The `.codex` runtime mirror still carried pre-098 planning-only language in three places: a §7 NEVER bullet about modifying any files, a §9 PLAN VERIFICATION line stating no file modifications attempted, and an anti-pattern row labeled "Direct File Modification." All three contradicted the scoped-write model that 098 introduced and that this packet now compels. Aligned the codex body with the canonical `.opencode` mirror in the same dispatch and updated the Iron Law text from "PLANNING-ONLY EVIDENCE" to "SCOPED-WRITE PERSISTENCE EVIDENCE" across all four mirrors. Why it matters: the four runtimes now carry identical enforcement language so behavior does not silently change based on which runtime invoked the agent.

#### Documentation

- **Reference docs cite the new gates** -- The `multi-ai-council` reference docs described the artifact layout and state format but did not say persistence was mandatory or that runs without `council_complete` were incomplete. Added a top-level "Persistence is mandatory" note to `folder-layout.md` citing the new §1 Step 0 RESOLVE, and a top-level "Every council run MUST close with a `council_complete` event" note to `state-format.md` citing the new §9 verification block and §13 sequence. Why it matters: anyone reading the reference docs first now sees the persistence contract before the layout details.

## Upgrade

No migration required. The new gates apply forward-only to council dispatches after this commit. In-flight `ai-council/` folders from earlier packets are unchanged.

Behavioral changes to be aware of:
- Invoking the council without a resolvable packet path now produces a single user-facing question listing the candidate paths the agent tried, not a chat report. This is intentional. Provide a spec folder when invoking the council from opencode's main-agent slot.
- Council reports that previously completed without writing artifacts will now block at §9 SELF-CHECK Q11 and fail to claim completion. The agent will name the missing artifacts in its abort message.
