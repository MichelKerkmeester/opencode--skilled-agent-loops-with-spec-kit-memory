5. **The 23 typed-gold rows improve exact recall but do not automatically produce perfect efficiency.** With one expected scenario pair and output containing that leaf plus two defaults, typed recall is `1/1`, making D1-intra resource recall and D2 equal 1.0. Typed D3 is precision over all observed pairs, so it is `1/3`. Assuming no intent mismatch, recipe cap, advisor probe, or live-only dimensions, the deterministic row score is `round((13*1 + 20*1 + 15*(1/3)) / 48 * 100) = 79`; if an exact `expected_intent` misses while leaf recall succeeds, D1-intra is 0.6 and the row is 68. D3 reaches 1.0 only if both defaults join each row's typed gold or the contract distinguishes defaults from selected leaves. D1-inter and D4 remain unscored in router mode, D5 remains a structural hard gate, malformed/unresolvable gold is excluded as topology error, and absent/unresolved router contracts fail closed. Thus typed coverage becomes 23 valid rows, recall becomes meaningful, selector accuracy can still fail, and efficiency stays systematically degenerate until default semantics are resolved.
