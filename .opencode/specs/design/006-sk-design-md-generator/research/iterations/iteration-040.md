# Iteration 040 — Track C (deepseek)

## Focus

## Findings
1. **[P0] Cardinal Rule 6 vs Rule 3 — section-existence conflict** — Rule 6 (prompt_template.md:52) mandates 'Keep all 17 sections; do not collapse or rename them.' Rule 3 (§2 template:43) mandates 'If a value is not in the tokens, it does not appear in the document.' When a mandatory section's token data is empty (e.g., a site with zero observed animations, or zero ARIA patterns), the model must either violate Rule 6 (omit the section) or Rule 3 (invent content). No ABSENT-stamp escape hatch exists.
   - Recommendation: Insert rule 9: 'STAMP ABSENT FOR EMPTY SECTIONS: When a required section's token data is empty, write the section with a single ABSENT stamp line (e.g., "**ABSENT**: no motion data captured in extraction"). A stamped-ABSENT section counts as present and valid.'
   - Evidence: design_md_prompt_template.md:40-52 (Rules 3, 6); conflict repeated in cardinal_rules_card.md:33-44 (bullets 1, 9); format.md §6.5:589-592 promotes minimal-absence section for motion alone, no generalized mechanism
2. **[P0] §1 Visual Theme & Atmosphere — unconstrained interpretive prose** — Format spec §1:115-127 mandates an opening sentence that 'must pass the could-this-describe-3-other-sites test,' named design principles with evocative labels, and a comparative-framing sentence. None of these requirements carry a token-citation obligation. The cardinal rules (prompt_template.md:41-52) constrain hex/pixel/font-weight/shadow/radius only — qualitative claims about atmosphere, intent, or design philosophy are ungoverned.
   - Recommendation: Insert rule 7: 'NO INTERPRETIVE INVENTION: Every qualitative claim (adjectives, atmosphere, design principles, strategy names, recommendations) MUST cite a specific token value from tokens.json that supports it. Example: "Authoritative (weight 300 at 56px display size, tokens.fontTokens[3])". A claim with zero cited tokens is a fabrication and must be removed.'
   - Evidence: prompt_template.md:41-52 (cardinal rules scope gap); format.md §1:115-117 (opening sentence), §1:122-128 (named principles), §1:130 (comparative framing); cardinal_rules_card.md:34 — fidelity check only lists 'hex, pixel, font-weight, shadow, and radius'
3. **[P0] §0 Brand Context personality adjectives — unbacked inference** — Format spec §0:89 requires 3-5 brand-personality adjectives 'each with a parenthetical rationale tied to an observable design decision.' The format spec itself demands a bridge between personality claim and concrete token, but the prompt template's cardinal rules never enforce this — Rule 1 only covers hex/pixel/font-weight/shadow/radius, not text claims. An adjective like 'Premium' with a rationale invoking 'blue-tinted shadows' may be completely hallucinated if no shadow tokens with blue tint exist.
   - Recommendation: Insert rule 8: 'CITE A TOKEN FOR EVERY QUALITATIVE CLAIM: Prose assertions about brand personality, atmosphere, or design intent MUST include at least one concrete value in backticks traceable to tokens.json. Undecorated prose (adjectives or philosophy without a cited value) is a fabrication.'
   - Evidence: prompt_template.md:41-43 (Rule 1 scope); format.md §0:89-94 (adjective-rationale requirement); cardinal_rules_card.md:34 (checklist item 1 omits prose qualifiers)
4. **[P0] §7 Vibe Paragraph — uncitable pure prose** — Format spec §7:717-721 mandates a 'Vibe Paragraph: 2–3 sentences capturing the brand personality as expressed through copy.' This is the only subsection in the 17-section spec that has zero token-citation requirement — no hex, no font, no observed copy text required inline. The model is explicitly instructed to produce interpretive prose with no data anchor at all.
   - Recommendation: Amend the Vibe Paragraph spec (§7 in format.md) to require each sentence cite at least one observed copy string or token value; alternatively, add to Rule 8: 'The Vibe Paragraph must cite specific observed copy examples (from §7 Voice Examples) for each tonal claim.'
   - Evidence: format.md §7:717-721 (Vibe Paragraph spec); no corresponding validation item in cardinal_rules_card.md:34-43
5. **[P0] §8 Do's and Don'ts — entirely invented guidance, no token source** — Format spec §8:741-761 requires 8-12 Dos and 8-12 Don'ts, each requiring 'specific css-value or token reference' for Dos and a 'specific threshold or value' for Don'ts. However, the cardinal rules (prompt_template.md:41-52) never instruct the model that every Do/Don't MUST be derivable from observed site behavior in tokens.json. The model is free to invent plausible-sounding guidance using tokens in isolation — a 'Don't exceed 8px border-radius' could reference a real 8px token but fabricate the prohibition.
   - Recommendation: Add to Rule 7: 'Every Do/Don't must trace to an observed pattern in tokens.json (e.g., "Use 4px radius on buttons" must be supported by tokens showing 4px as the dominant button radius). Prohibitions not derivable from token data are fabrication.'
   - Evidence: prompt_template.md:41-43 (Rule 1 scope excludes prose rules); format.md §8:741-761 (Do's and Don'ts spec); cardinal_rules_card.md:34-43 (no Do/Don't fidelity check)
6. **[P1] Cardinal Rule 1 value-type scope gap** — Rule 1 (prompt_template.md:41-43) mandates verbatim copy for: hex, pixel, font-weight, shadow, radius. Missing from this enumeration: line-height, letter-spacing, spacing units, easing-functions, duration values, rem equivalents, stroke-widths, contrast ratios, and OpenType feature tags. A literalist model could interpret Rule 1 as the exhaustive list and estimate/normalize everything else.
   - Recommendation: Expand Rule 1 enumeration: 'Every hex, pixel, rem, font-weight, line-height, letter-spacing, shadow, radius, spacing unit, easing function, duration, stroke-width, and OpenType feature tag MUST be copied verbatim from the tokens.'
   - Evidence: prompt_template.md:41 (enumerated types); format.md §3:407 (letter-spacing in table), §6.5:608 (duration scale), §10:615 (easing functions), §12:938 (stroke weight) — all require verbatim values not covered by Rule 1
7. **[P1] Validation gate only checks hex + section count — prose fabrication unchecked** — The post-write validation gate (prompt_template.md:54-56) runs `validate.ts` and instructs: 'Resolve every hex mismatch and missing section before reporting completion.' Prose fabrication — invented personality adjectives, hallucinated design principles, fake copy examples — produces zero hex mismatches and zero missing sections. The validation gate is blind to the highest-risk fabrication category.
   - Recommendation: Add to the validation step: 'After validation, run a prose-fabrication check: for every qualitative claim (personality adjective, design principle, Do/Don't, philosophy statement), verify (a) at least one cited token supports it, and (b) the token exists in tokens.json. Flag any uncited qualitative claim as a finding.'
   - Evidence: prompt_template.md:54-56 (validation step); cardinal_rules_card.md:43 (last checklist item: 'validate.ts passes with zero hex mismatches and zero missing sections')
8. **[P1] §2.5 Dark Mode Strategy Name — evocative-label fabrication** — Format spec §2.5:348-359 requires naming the dark mode approach with 'an evocative label' (Inverted Gray Ladder, Warm Dark, True Black OLED, Dimmed Luminance) plus 'a comparative observation: how does the dark mode FEEL different from light? Not just "colors invert" but a structural observation.' This demands the model produce original strategy names and structural interpretation — no tokens.json field maps to a 'strategy name.' The model invents both the label and the structural insight.
   - Recommendation: Add to Rule 10: 'LABEL INFERENCE: Any statement that draws a conclusion from data rather than transcribing it MUST be prefixed with [INFERRED]. The Dark Mode Strategy Name and structural observation are [INFERRED] and must cite the specific variableDiff entries that support the claimed structure.'
   - Evidence: format.md §2.5:348-359; prompt_template.md:40-52 (no inference-labeling rule); cross-ref §6:544-550 (similar named-principle fabrication path in Depth & Elevation)
9. **[P1] §13 Agent Prompt Guide — self-contained prompts are recomposed prose** — Format spec §13:1020-1030 requires 5-6 'completely self-contained' prompt strings (100-200 words each) and an Iteration Guide of 6-8 steps. These are prose compositions that recompose tokens into natural language. The cardinal rules constrain individual token values but not recomposition fidelity — the model could fabricate a plausible-sounding hero-section prompt that uses real hex values but invents the layout, spacing, or behavior because those details aren't individually tokenized.
   - Recommendation: Add to cardinal rules: 'All component prompts (§13) must only compose values that appear in the corresponding sections (§0-§12). A prompt that specifies layout, spacing, or behavior not documented in the relevant section is fabrication.'
   - Evidence: prompt_template.md:54-56 (validation can't check prompt fidelity); format.md §13:1015 self-containment test — 'Can an AI agent build this component without looking anything else up?' — validates self-containment, not fidelity to tokens
10. **[P2] Word-count guidance as filler incentive** — Format.md §22:1171-1191 provides per-section word-count targets (e.g., §1: '200-300 words for prose,' §7: '150-250 words'). When token data is sparse (few observed page samples), these minima incentivize the model to pad with generic prose rather than stamping ABSENT on underpopulated sections. The cardinal rules never subordinate word counts to data availability.
   - Recommendation: Add to cardinal rules: 'Word counts are ceilings, not floors. If token data is insufficient to meet a word target without invention, write fewer words and stamp the section foot with [SPARSE: only N tokens available].'
   - Evidence: format.md §22:1171-1191 (word count table); §22:1195-1198 ('Under 350 indicates missing detail'); prompt_template.md:40-52 (no word-count subordination rule)
11. **[P2] Absence-noting rule is scoped to §9 only** — Rule 5 (prompt_template.md:49-51) states 'If no a11y data was captured, note the absence rather than inventing values' but only for the accessibility section. No equivalent instruction exists for §6.5 Motion (where the format spec itself provides an absence template), §8 Do's/Don'ts, §12 Iconography (which also has an inline absence note in the format spec), §11 State Matrix (cells with `-` for unobserved), or §14-17 optional sections.
   - Recommendation: Generalize Rule 5: 'NOTE ABSENCE, NEVER INVENT: For any section or subsection where token data is missing, write an explicit absence note ("ABSENT: no [X] data captured") rather than inventing content. This applies to ALL sections, not only accessibility.'
   - Evidence: prompt_template.md:49-51 (Rule 5 scoped to accessibility); format.md §6.5:587-592 (motion absence template exists but is not enforced by prompt); §12:968-974 (iconography absence template exists but is not enforced by prompt)
12. **[P2] No inference-labeling rule anywhere** — The format spec is dense with sections that require inference: §6 depth-principle naming, §6.5 motion philosophy, §9 WCAG target inference, §3 named typography strategies, §5 whitespace philosophy, §7 vibe paragraph, §12 substitution recommendation. The cardinal rules (prompt_template.md:40-52) have zero instruction to label any of these as inference, so the model presents all judgments as fact.
   - Recommendation: Insert Rule 10: 'LABEL INFERENCE: Any statement that draws a conclusion from data rather than transcribing it MUST be prefixed with [INFERRED]. Unlabeled inference is a fabrication.'
   - Evidence: prompt_template.md:40-52 (no inference-labeling rule); format.md §6:544-545 (named-principle classification), §9:780-781 (WCAG inference), §6.5:597-602 (motion philosophy), §3:415-423 (named strategies) — all inferential, none labeled
