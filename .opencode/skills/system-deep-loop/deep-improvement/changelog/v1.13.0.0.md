# deep-improvement v1.13.0.0 — model-benchmark outputs routed to sk-prompt-models hub

**Released**: 2026-06-03
**Status**: Behavior change for model-benchmark mode only — output path switched to hub. Agent-improvement and all other modes are unchanged.

---

## Summary

The `model-benchmark` command now writes all outputs (benchmark run data, results, synthesis) to the `sk-prompt-models/benchmarks/{run_label}` hub directory instead of the spec-local default. This is **hub-only**: there is no spec-local fallback and no override mechanism. The `.cjs` scripts were already path-agnostic and required no changes.

---

## What Changed

| File | Change | Why |
|------|--------|-----|
| `SKILL.md` | `model-benchmark` output-path doc updated to `sk-prompt-models/benchmarks/{run_label}` (hub only, no override) | Keeps benchmark evidence collocated with the model registry it measures |
| `assets/model-benchmark/auto.yaml` | `output_path` set to hub path | Matches new routing |
| `assets/model-benchmark/confirm.yaml` | `output_path` set to hub path | Matches new routing |

### What did NOT change

- `.cjs` scripts (`sweep-benchmark.cjs`, `run-bench.cjs`, etc.) — already path-agnostic; no edits.
- Agent-improvement mode, skill-benchmark mode, and all other deep-improvement modes — spec-local output paths unchanged.
- Vitest suite — no test changes; all existing tests pass.

---

## Operator Notes

- **No migration needed for past runs.** Previously written spec-local benchmark outputs remain where they are. The routing change applies only to new `model-benchmark` invocations going forward.
- **Hub path resolves:** `sk-prompt-models/benchmarks/{run_label}` under the skills root. The `{run_label}` value comes from the benchmark profile or command argument, unchanged.
