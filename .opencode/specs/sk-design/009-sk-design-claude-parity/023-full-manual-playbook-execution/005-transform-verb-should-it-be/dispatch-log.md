## Notable cross-cutting finding

The standalone advisor-probe script (`skill_advisor.py`) and the live orchestrator dispatch disagreed on TV-002-V2 and TV-002-V3: the probe returned an empty result (`[]`, no candidate above threshold 0.8) for the *exact same clean prompt text* that the live dispatch still correctly routed to `sk-design` -> `audit`. This means the live orchestrator's routing path is not a 1:1 proxy for the standalone `skill_advisor.py` CLI probe on these transform-verb "should it be" phrasings — the live dispatch has additional context/session-level routing signal the isolated CLI probe does not receive. TV-002-V4 is the one variant where BOTH paths failed together (probe empty AND live dispatch skipped skill routing entirely), suggesting "delight ... gratuitous" carries meaningfully weaker sk-design vocabulary signal than "quieter ... conversion clarity" or "distill the interface ... elements" even within the live dispatch's richer routing context.
