This release makes cli-codex authenticate through ChatGPT OAuth only. The OpenAI API key path (`OPENAI_API_KEY`) is removed from the dispatch flow, the auth pre-flight, the reference tables, and the troubleshooting guidance. cli-codex now surfaces `codex login` and waits when the CLI is not authenticated, and never substitutes an API key.

---

## What Changed

#### Auth Model

- **ChatGPT OAuth only** -- `codex login` (browser flow, requires a ChatGPT Plus/Pro/Business/Edu/Enterprise account) is the single authentication path. The `OPENAI_API_KEY` env-var method is no longer offered.
- **Auth pre-flight simplified** -- the two-variable (`OPENAI_KEY_OK` / `CODEX_OAUTH_OK`) decision tree collapses to a single OAuth check. When not logged in, the skill asks the user to run `codex login` and does not dispatch.
- **Reference and troubleshooting reframed** -- `cli_reference.md` §3 (Authentication), §13 (Troubleshooting), and §14 (Environment Variables) drop the API-key method, the `OPENAI_API_KEY` env var, and the `platform.openai.com` key-management link.

#### Unchanged

- Model roster, routing, sandbox modes, and every dispatch rule are untouched.
- The generic NEVER rule against sending secrets (API keys, passwords) in prompts stays -- that is secret hygiene, not an auth method.

## Upgrade

If a machine relied on `OPENAI_API_KEY` for Codex, run `codex login` once to establish the ChatGPT OAuth session. Dispatch flags are otherwise unchanged.
