This release splits review-mode functionality out of deep-research into the new standalone deep-review skill. deep-research is now research-only - all review keywords, commands, and references have been removed or redirected. The goal is to make the workflow easier to follow and easier to trust.

---

#### Review Mode Split

- All review-mode content removed from SKILL.md: review triggers, REVIEW_* intent signals, review-mode architecture diagrams, review-specific rules, review success criteria, review references.
- Review assets moved to deep-review: `deep_review_strategy.md`, `deep_review_dashboard.md`, `review_mode_expected behavior.yaml`.
- Review testing playbook (`07--review-mode/`, 11 scenarios) moved to deep-review and restructured into 28 scenarios across 6 categories.
- Cross-references added: "For code review, see deep-review" in Sections 1 and 8.

---

#### Command Simplified

- `/deep:research` command stripped of all `:review` suffix handling and review-specific setup questions.
- Mode suffixes now only `:auto` and `:confirm` (`:review`, `:review:auto`, `:review:confirm` removed).
- Review examples replaced with cross-reference to `/deep:review`.
- Command chain updated: review path now references `/deep:review`.

---

#### Reference Documentation

- `references/quick_reference.md`: Review mode section (80 lines) replaced with redirect to `deep-review/references/quick_reference.md`. H2 sections numbered per skill_reference_template.md.
- `references/loop_protocol.md`, `convergence.md`, `state_format.md`: Review-specific sections retained as shared protocol documentation (cross-referenced by deep-review).

---

#### Skill Advisor Routing

- Review keywords (`deep review`, `review mode`, `iterative review`, `code audit`, `:review`, etc.) rerouted from `deep-research` to `deep-review` in both Public and Barter repos.
- Research keywords (`autoresearch`, `deep research`, `research loop`, etc.) unchanged.

---

#### Manual Testing Playbook

- Playbook reduced from 41 to 30 scenarios across 6 categories (review scenarios moved to deep-review).
- Root playbook doc updated: removed Section 13 (Review Mode), renumbered remaining sections.
