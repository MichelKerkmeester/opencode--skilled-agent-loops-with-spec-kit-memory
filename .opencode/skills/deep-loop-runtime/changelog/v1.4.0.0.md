## What Changed

#### Documentation

- **Added a SUCCESS CRITERIA section** -- the SKILL.md had no explicit health contract. It now lists the JSON stdout contract for every script entry point, consumer `lib/` import resolution, runtime SQLite and schema presence, and Vitest discovery. Consumer skills get a concrete "is the runtime working" checklist instead of inferring it.
- **Added a REFERENCES AND RELATED RESOURCES section** -- the references block pointed only at ADRs. It now also names the consumer skills (deep-review, deep-research), the cross-package Vitest config that discovers this skill's tests, and the changelog directory, so a reader can trace the runtime out to everything that depends on it.

#### Documentation Structure

- References stay flat here by design. The runtime carries four focused reference files and few consumers, so it was left flat while the four reference-heavy deep-* skills moved to thematic subfolders.
