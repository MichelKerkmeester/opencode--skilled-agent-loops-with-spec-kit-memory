## [**1.3.0.0**] - 2026-07-17

Online-research refresh plus a Chrome-DevTools parity expansion of the feature catalog and manual-testing playbook. A dated capture of the public Aside developer docs corroborated the CLI/MCP/REPL surface, added the verbatim `mcp.json` wiring and the confirmed permission model, and confirmed installer-only distribution. A new DevTools-parity feature family (6 catalog leaves + 6 playbook scenarios) mirrors the `mcp-chrome-devtools` capability surface through the Playwright `page` API the discovery fixture proves is available in `aside repl`, with honest gaps flagged (no native HAR export, no Lighthouse audit, no parallel isolation, no raw-CDP passthrough).

#### DevTools-parity feature family

- Need: Aside's catalog covered its five intent domains but did not mirror the Chrome DevTools capability surface (`mcp-chrome-devtools`).
- Change: Added `feature-catalog/devtools-parity/` (6 kebab-case leaves: `dom-inspection.md`, `screenshots-and-pdf.md`, `console-and-network-capture.md`, `cookies-and-storage.md`, `performance-timing.md`, `navigation-and-tabs.md`) and `manual-testing-playbook/devtools-parity/` (6 scenarios ASD-016..021). Each shows the actual repl code pattern grounded in the fixture's Playwright `page` API. Root indexes, count summary (12 → 18 capabilities; 15 → 21 scenarios), wave planning, and cross-reference indexes updated; a Chrome-DevTools parity table was added to both the catalog root index and SKILL.md §6.
- Why it matters: Console/network capture — previously flagged UNKNOWN and fail-closed (ASD-012/013) — is now a documented, API-grounded pattern (`page.on(...)`), superseded by ASD-018 with the honesty line that live capture needs a bound authorized session (SKIP-valid). Honest gaps are documented, never claimed: no native HAR export (assemble a HAR-equivalent from events), no Lighthouse audit (Performance-API timing only), no parallel isolated instances, no raw-CDP passthrough.

#### Rule reconciliation

- Need: NEVER rule 7 ("Promise console or network capture — no dedicated verified contract exists") contradicted the now-documented Playwright event pattern.
- Change: Rewrote NEVER rule 7 to ban only the *native dedicated tool / native HAR / Lighthouse / parallel isolation / raw CDP* claims, while acknowledging the caller-assembled `page.on(...)` capture path and its bound-session + independent-verification requirement. README console/network FAQ updated to match.
