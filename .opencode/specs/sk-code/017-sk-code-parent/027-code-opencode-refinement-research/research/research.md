# code-opencode Refinement — Research Synthesis

## 1. Executive verdict

`code-opencode` (the playbook for writing code in this repo's own `.opencode/` tree) is **structurally sound but doctrinally drifted**. The surface skeleton is honest — SKILL.md §2 REFERENCE MAP and §4 ASSETS resolve correctly (all 5 language trios, the 4-file `shared/` tier, the 3 workflow-doctrine **symlinks** → `../../shared/references/`, and all 10 checklists live on disk; code-opencode correctly carries no `graph-metadata.json`, respecting the one-identity invariant). The defects are content-level, not structural, and cluster around **three dominant root causes**: (1) **mid-migration staleness** — the docs are version `3.5.0.x`–`3.6.0.0`, written *before* the two-axis parent-hub restructure + the `system-spec-kit` relocation, leaving a systematic rot of evidence pointers (`lib/common.sh`→`shell-common.sh`, `config/config.jsonc`, `references/opencode/`→`code-opencode/references/`, `skill_creation.md`, phantom `.claude/commands/`+`.opencode/prompts/` mirrors) across the shell/config/shared tiers; (2) **teaching-wrong-thing** — the docs prescribe a flat standalone/hand-authored model while every canonical skill now ships an Option-E parent-hub (`mode-registry.json` + `hub-router.json` + `graph-metadata.json`), the TS trio teaches import/emit patterns that **fail to compile** under the very NodeNext/`verbatimModuleSyntax` tsconfig it documents (extensionless relative imports, bare `require()`, missing `node:` prefix), the command/MCP checklists describe neither the real contract-renderer nor the 3-tier launcher+daemon+socket architecture, and `hooks.md` §4/§5 **fabricate** `mcp_server/hooks/opencode|copilot/` suites that never existed; (3) **missing-coverage** — the verify doctrine is operationally empty (no stale-dist trap, no native-ABI/SIGBUS, no det-env test collision, no `validate.sh` exit codes), and three empirically-hit process hazards the surface *should* own (daemon single-writer, comment-hygiene as a HARD BLOCK gate, native-module ABI) are absent. The fix profile is roughly **40% mechanical re-pathing** (batchable, low-risk), **40% teaching-wrong-thing** (substantive rewrites of TS §7/§9, the four authoring checklists, hooks.md §4/§5), and **20% missing-coverage** (additive, no conflicts). Two underlying `.opencode` code bugs bleed into the docs: `verify_stack_folders.py` is permanently broken (parses a removed `STACK_FOLDERS` map) and `.sh`-extension Python checkers self-suppress the comment-hygiene gate.
