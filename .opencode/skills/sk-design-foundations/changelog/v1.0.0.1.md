# v1.0.0.1 - Color-Role Contract And Routing Precedence

**Released:** 2026-06-25

Remediation patch from the family deep review. All changes are documentation-only inside the skill package.

## Fixed

- Color-role contract: established one canonical color-role set (`primary/accent`, `neutral`, `semantic`, `surface`, `border`, `text`) referenced identically by `SKILL.md`, `references/color/palette_theming.md`, `references/color/oklch_workflow.md`, and the color playbook. `focus` is now documented as an accent state, not a separate role, so the color scenario no longer fails a faithful execution.
- Layout routing precedence: `SKILL.md` and the layout playbook now state that layout, spacing, and grid system decisions resolve to `sk-design-foundations` first, ahead of `sk-code`, which receives the system only for implementation.
- Metadata: aligned `metadata.family` in `SKILL.md` frontmatter to `sk-code` to match `graph-metadata.json`.
- Docs hygiene: normalized the typography catalog detail file's source-path base to the skill-root-relative convention used by the sibling catalog files.
- Router pseudocode: removed the unused `LOAD_LEVELS` dictionary and corrected the low-confidence branch comment to describe a no-signal fallback.
- Graph hygiene: removed the redundant `sk-design` sibling edge in `graph-metadata.json`; the parent relationship is already captured under `depends_on`.
