Structural conformance fix so this packet passes the canonical section contract enforced across the `sk-doc` family. The `WHEN TO USE` and `SMART ROUTING` responsibilities had been merged into a single heading whose text used an underscore, so the automated section check could not detect a `SMART ROUTING` section and the packet failed validation. This release splits them into two distinct sections and gives the packet a top-level `REFERENCES` section so routing and reference discovery are legible to both readers and the checker. No workflow behavior changes.

---

## What Changed

#### Documentation

- **Split merged WHEN TO USE and SMART ROUTING headings** -- The two concerns shared one heading with an underscore-joined title, which hid the routing section from the canonical section matcher. They are now separate H2 sections, with the keyword triggers and family-boundary routing rules moved under `SMART ROUTING` where routing signals belong.
- **Promoted the overflow references to a top-level REFERENCES section** -- The reference route-map lived as a subsection under `RULES`, so no standalone `REFERENCES` section existed for the contract to match. It is now its own H2 section while keeping the same links and guidance.
- **Renumbered sections contiguously** -- Splitting the header and promoting references shifted the section count, so all H2 numbers were renumbered from one with no content changes.
