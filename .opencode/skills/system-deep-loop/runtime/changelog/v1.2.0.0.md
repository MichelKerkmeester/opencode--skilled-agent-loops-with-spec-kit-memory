# deep-loop-runtime v1.2.0.0

> Closes the 36-finding backlog from the Phase 5 deep-research audit. Documentation alignment + council surface expansion + targeted test coverage. Ships from packet `131/000-release-cleanup/001-deep-loop-runtime/001-doc-remediation`.

---

## Why This Release Matters

The v1.1.0.0 release-cleanup pass rewrote the README and fixed 4 surgical SKILL.md issues, but a 10-iteration cli-devin SWE-1.6 deep-research loop run on the v1.1.0.0 baseline surfaced 36 further findings the audit-and-validate phases missed. Three of the highest-impact findings were introduced or propagated by the v1.1.0.0 README rewrite itself: a fabricated `code-surface` node kind, a wrong cross-arc citation echoed into the changelog, and a 22-vs-27 vitest count drift. This taught a clear lesson: structural validation (template conformance) is not fact-checking (content accuracy). The deep-research loop is the fact-checker.

v1.2.0.0 closes all 36 findings across 4 remediation batches and brings the council surface - mandated by packet 131/001/008 ADR-001 Runtime Boundary Decision but never implemented in the catalog/playbook - into full documentation parity.

---

## What Changed

### Batch A - Consolidated cleanup (22 findings)

Cross-arc citations corrected from `129/001` to `131/001/008 ADR-001` at 6 sites. Fabricated `code-surface` node kind replaced with the 10-kind source-of-truth allow-list. Vitest count corrected 22 to 27. graph-metadata.json gained the council domain, key_topics, key_files, entities, related_to, causal_summary mention, and both changelogs in source_docs. references/integration-points.md gained a §7.1 documenting 7 previously-hidden consumers plus the cross-package test-discovery note.

### Batch B - Description-drift full-17 sweep

All 17 feature_catalog descriptions verified against their source-module exported surface. Three drifted and were corrected (with matching playbook OVERVIEW lines): fallback-router (quota-pool framing), bayesian-scorer (computeScore + shouldDemote split), post-dispatch-validate (full validation surface). The other 14 were confirmed accurate. True population drift rate is 3/17 (18 percent), below the Phase-5 N=3 sample projection of 43 percent.

### Batch C - Council surface expansion (12 new files + 7 updates)

New `feature-catalog/08--council/` (5 entries) + `manual-testing-playbook/08--council/` (5 scenarios). Catalog + playbook indexes updated (counts 17 to 22, 7 to 8 categories, new sections + cross-reference rows). README STRUCTURE tree updated.

### Batch D - Test coverage (4 vitest files)

Two new unit test files for the previously-uncovered `coverage-graph-query` (8 tests) and `coverage-graph-signals` (9 tests). Two weak council tests deepened: multi-seat-dispatch (2 to 6 tests) and round-state-jsonl (2 to 6 tests). All 29 tests pass via the system-spec-kit vitest harness.
