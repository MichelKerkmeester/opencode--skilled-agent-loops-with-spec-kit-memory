# v1.0.0.0 - create-agent packet established

Initial release of `create-agent`, one of the ten workflow packets in the `sk-doc` parent hub. It scaffolds or updates a single runtime agent markdown file: decide whether the request needs an agent versus a skill or command, resolve the active runtime agent directory, author frontmatter with the unified `permission:` object and an authority boundary, write the required workflow and verification sections, then validate before delivery.

---

## What's Included

- `SKILL.md` - the primary workflow contract: when to use and smart routing (the agent vs skill vs command decision rule), the output package and runtime directory table (`.opencode/agents/` vs `.claude/agents/`), the canonical frontmatter shape (`permission:` object, `task` gating for orchestration authority), the required body shape (hard boundary, core workflow, capability scan, output verification, anti-patterns, related resources), the ordered 14-step creation workflow, and the validation gate.
- `references/agent_creation.md` - long-form creation standards, current-reality notes, and common-mistake examples, used only for deeper detail beyond the SKILL.md contract.
- `assets/agent_template.md` - the canonical scaffold new agents are copied from before placeholders are replaced with the agent's actual role, permissions, workflow, and boundaries.

## Notes

This packet holds no packet-local advisor graph identity (no `graph-metadata.json`). The single advisor identity and workflow registry live at the `sk-doc` hub root. `scripts/` is not present in this packet; validation reuses the shared sk-doc validators (`validate_document.py`, `extract_structure.py`) under `../shared/scripts/`.
