## [**1.3.0.0**] - 2026-07-14

This release adds sk-git's create-skill-canon documentation surface: a **feature catalog** enumerating every capability with its entry point, and **manual-testing-playbook coverage** for the owner-first worktree tooling. No runtime behavior changes — the SKILL router, rules, and workflows are unchanged; this release makes the skill's capabilities discoverable and its worktree tooling exercised.

---

## What Changed

#### Feature Catalog (new)

- **New `feature_catalog/` package** — `feature_catalog/feature_catalog.md` plus 11 per-feature files across four category directories (`worktree_naming/`, `session_lifecycle/`, `workflow_playbooks/`, `remote_platform_integration/`), authored to the `sk-doc/create-feature-catalog` canon. Each capability carries its purpose, entry point, and when-to-use; script-backed features anchor to their script (the owner-first allocator/validators, the launch-wrapper session isolator, the reaper, and the pre-push naming hook) rather than to volatile line numbers.

#### Manual-Testing Playbook Coverage (expanded)

- **New `owner_first_worktree_tooling/` category** — 19 scenario files (`GIT-023`..`GIT-041`) covering the number allocator (locked unique allocation, owner/slug/branch/pair validation, create + create-detached, wrapper-lane exemption), the launch-wrapper session isolator (top-level isolation, child exec-in-place, runtime-identity validation, session marker, shared-artifact containment), the reaper (auto-reap qualifying pairs, keep non-qualifying, dry-run, orphan-daemon reporting), and the pre-push hook (new-branch-only gating, migration tolerance, fail-open, release-branch exemption, bypass env, wrapper-ref rejection). Playbook total: **41 scenarios across 7 categories** (was 22 across 6).
- **Root index updated** — package-artifacts list, scenario count, coverage note, and the feature-catalog cross-reference now reflect the new category and package.

#### Documentation Fixes

- **Corrected the playbook path pattern** — SKILL.md and README described scenario files as `manual_testing_playbook/<NN>--<topic>/<NNN>-<scenario>.md`; the real on-disk layout is `manual_testing_playbook/<topic>/<scenario>.md` (underscore categories, no numeric prefixes). The root playbook's package-artifacts list and the Section 15 cross-reference index likewise carried stale hyphenated paths for the pre-existing `GIT-001`..`GIT-022` scenarios; both now match the on-disk `underscore_case` names.

## Migration Notes

None. Additive documentation only; no branch grammar, tooling behavior, or command surface changed.
