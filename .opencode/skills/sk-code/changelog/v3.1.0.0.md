## [**3.1.0.0**] - 2026-05-05

This release makes Motion.dev a discoverable cross-stack peer category inside `sk-code` so animation guidance can live in one place instead of being buried under Webflow-only docs. The smart router now sees `motion_dev/` as a sibling to `webflow/` and `opencode/`, the manual testing playbook covers Motion regressions and cross-browser performance, and Webflow CDN-specific guidance stays linked rather than duplicated.

> Spec folder: `specs/skilled-agent-orchestration/069-sk-code-motion-dev-and-playbook` (Level 1, phase parent with 3 children)

---

## What Changed

#### New Features

- **Cross-stack motion_dev references and assets** -- Motion.dev guidance was spread across Webflow examples and hard to surface for non-Webflow callers. We populated `references/motion_dev/` with quick_start, animate_and_timelines, scroll_and_gestures, performance_and_pitfalls, decision_matrix, and integration_patterns docs, plus `assets/motion_dev/` install card, playbook hooks, and reusable JavaScript snippets. Motion.dev is now first-class and works the same across stacks.

- **Manual testing playbook coverage for animation and performance** -- The playbook had no scenarios for Motion regressions, cross-browser drift, or animation-specific performance gates. We extended `manual_testing_playbook/` with Motion integration, animation regression, cross-browser, and performance-gate scenarios. Authors writing animation code now have ready-made test scenarios.

#### Architecture

- **motion_dev as peer resource category** -- The router only recognised `webflow/` and `opencode/` as resource categories, so Motion docs had to live underneath Webflow. We added additive `motion_dev/` See-also pointers to existing Webflow docs that mention Motion.dev, and updated SKILL.md, README.md, router resource docs, and `description.json` so `motion_dev/` shows up alongside the other surfaces. Discovery and routing both improve without duplicating Webflow content.

- **Skill graph derived metadata refresh** -- Without a graph refresh, the new `motion_dev/` category was invisible to memory search. We refreshed the skill graph derived metadata through the existing derived-sync path and re-indexed. The new resources are now reachable through the same query paths that surface every other sk-code resource.
