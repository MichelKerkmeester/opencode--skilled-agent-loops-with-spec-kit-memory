# sk-prompt v2.0.0.0 — Model registry moved out; skill is now a clean, forkable framework engine

**Released**: 2026-06-03
**Status**: Breaking boundary change — model registry and all small-model references removed. Framework engine (DEPTH / CLEAR / framework-registry.json) is unchanged.

---

## Summary

`sk-prompt` is now a **standalone, forkable prompt-framework engine**. Everything model-specific has been relocated to `sk-prompt-small-model`, which becomes the single source of truth for model registry and benchmark data. The skill no longer carries any model names, profile paths, or small-model guidance; `framework-registry.json` and the DEPTH/CLEAR patterns remain.

Three coordinated removals land together:

1. **`assets/model-profiles.json` deleted from this skill.** The file now lives at `sk-prompt-small-model/assets/model-profiles.json`. No model-profile data is retained here.
2. **`references/model-profiles.md` deleted.** The schema and adoption content it carried are now covered by `sk-prompt-small-model`'s pattern-index §4 and the JSON itself.
3. **`assets/cli_prompt_quality_card.md` stripped of all small-model references.** The per-model-override note and the Budget-Awareness (small-model) section are removed. Tier-2 in the 3-tier Prompt-Composition Precedence section is reworded generically — no `sk-prompt-small-model` path, no model names. The file now contains only framework selection, DEPTH density, and CLEAR scoring guidance.

Post-change verification: `grep` across all active surfaces in this skill returns zero hits for model names, `model-profiles`, or `sk-prompt-small-model` paths (historical changelog entries are excluded from this check).

Consumers who were reading `sk-prompt/assets/model-profiles.json` or `sk-prompt/references/model-profiles.md` must repoint to `sk-prompt-small-model/assets/model-profiles.json`.
