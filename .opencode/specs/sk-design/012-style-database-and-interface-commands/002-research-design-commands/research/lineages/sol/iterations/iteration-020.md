# Iteration 20: Final Convergence

## Focus
Consolidate the smallest implementation-ready architecture after exactly twenty evidence-bearing passes.

## Final Recommendation
Replace the five thin user-facing routers with five canonical creation templates while retaining registry-stable internal modes:

| Canonical command | Internal mode | User job |
|---|---|---|
| `/interface:design` | `interface` | create/reshape an end-to-end interface direction |
| `/interface:foundations` | `foundations` | create a static visual system and implementation scaffold |
| `/interface:motion` | `motion` | create temporal behavior and reduced-motion parity |
| `/interface:audit` | `audit` | diagnose, specify remediation, and re-verify |
| `/interface:design-reference` | `md-generator` | extract a provenance-rich validated `DESIGN.md` |

## Converged Architecture
1. Each canonical command owns the shared nine-stage choreography: `Route -> Context Manifest -> Progressive Brief -> Grounding -> Mode Plan -> Creative/Diagnostic Work -> Critique/Revision -> Proof -> Handoff`.
2. Each command specializes only brief fields, exemplar role, stage details, output names, and proof matrix. It does not copy mode taste, references, or quality doctrine.
3. `sk-design` modes own judgment; transports own retrieval/render/extraction; `sk-code` owns application-code mutation and stack verification.
4. The persistent envelope includes resolved brief, assumptions, context, grounding, constraints, accepted decisions, proof plan/status, mutation boundary, and output target.
5. Intake uses `resolved`, `auto-resolvable`, and `confirmation-required`; all high-impact questions are bundled once. Exemplar acquisition stops when evidence changes a named decision and accepts auditable no-fit.
6. Outputs carry typed proof levels (`authored`, `observed`, `measured`, `validated`, `verified`, `blocked`, `not-applicable`) rather than universal completion claims.
7. Public commands never invoke public commands. Composition uses ordered modes and versioned accepted-decision handoffs.
8. Roll out additively: canonical `/interface:*` commands first, existing `/design:*` paths as thin tested aliases, then evidence-based deprecation.

## Implementation Order
1. Author a shared creation-contract reference containing the lifecycle, context/evidence schemas, intake policy, and proof labels.
2. Add five canonical command templates with the command-specific contracts from iterations 8-12.
3. Convert existing routers to minimal compatibility aliases without duplicating prompt bodies.
4. Add static contract tests for route/mode, required blocks, no taste duplication, mutation boundary, and handoff shape.
5. Add scenario tests from iteration 16 plus alias parity and representative rendered/measured tests where transports exist.
6. Update command discovery/docs and measure alias usage before any removal proposal.

## Acceptance Gates
- All five names resolve to the expected stable mode.
- Brief and assumption behavior passes resolved/auto/confirm scenarios.
- Grounding records include provenance/role/decision-change or explicit no-fit.
- No command embeds palette/font/layout defaults or mode reference tables.
- No command mutates application code directly or invokes another public command.
- Output blocks and evidence levels pass schema assertions.
- Legacy aliases preserve arguments, route, permissions, outputs, and handoff.
- Audit and extraction fail closed at their declared evidence/access boundaries.

## Residual Risks
- Runtime command syntax may not support a literal include mechanism; choose the smallest supported single-source pattern during implementation.
- The new `/interface` namespace needs collision/discovery validation in the actual command loader.
- Alias deprecation timing remains evidence-dependent.
- Render/browser transports may be unavailable; the degradation ladder must be tested rather than assumed.

## Eliminated Alternatives
- One mega-command, a standalone brief command, style menus, mandatory mood boards, exhaustive interviews, internal-mode renames, immediate legacy removal, command-to-command recursion, and universal `verified=true`.
