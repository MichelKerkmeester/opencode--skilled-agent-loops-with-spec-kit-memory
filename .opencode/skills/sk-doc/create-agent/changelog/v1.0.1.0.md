# v1.0.1.0 - Smart-routing section conformance

Normalized `SKILL.md` to the canonical section contract so the packet satisfies every required H2. No workflow guidance changed; this is a structural pass only.

---

## What Changed

- Split the merged `WHEN TO USE + SMART_ROUTING` header into two discrete H2 sections: `## 1. WHEN TO USE` and `## 2. SMART ROUTING`. The underscore in the old combined heading meant automated section matching never saw a `SMART ROUTING` heading, so the required section registered as missing even though its content was present.
- Moved the routing-away guidance ("use a different `sk-doc` workflow when") and the agent-vs-alternative decision-rule block under the new `SMART ROUTING` section, keeping the applicability list under `WHEN TO USE`.
- Added a `Keyword triggers:` line to `WHEN TO USE` so the packet surfaces the same trigger vocabulary as the keyword comment.
- Promoted the former `OVERFLOW REFERENCES` subsection out of `RULES` into a top-level `## 6. REFERENCES` H2, preserving every reference link.
- Added a `## 5. SUCCESS CRITERIA` H2 summarizing the delivery bar (runtime placement, runtime-correct frontmatter schema, required body sections, linked-not-pasted depth, clean validation).
- Renumbered all H2 headings contiguously (1 through 6). No numeric intra-file cross-references existed, so nothing else shifted.

## Notes

Content is unchanged apart from the restructure — all prose, tables, decision rules, and reference links were preserved verbatim. Packet still carries no local advisor graph identity; validation continues to reuse the shared sk-doc validators under `../shared/scripts/`.
