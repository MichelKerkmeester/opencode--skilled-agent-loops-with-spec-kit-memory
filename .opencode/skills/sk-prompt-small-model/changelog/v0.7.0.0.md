## Summary

The hub's doc surface was aligned to the sk-doc templates and its identity corrected from the stale "thin sentinel" framing to the per-model prompt-craft hub it became after the registry + profiles moved in:

- **Model-keyed smart router (SKILL.md §2).** The no-op router ("this skill has no runtime router of its own") is replaced with a real router faithful to `skill_smart_router.md`: runtime discovery, an alias map resolving a named model to its canonical id, guarded loads, and three-tier fallback. Routing key = the model id; resource = `references/models/<id>.md`. LOC cap relaxed 200 → 300 to fit the inline router.
- **README rewrite.** Hub identity, every ephemeral spec reference stripped (arc name, phase numbers, research-iter links, spec-folder links), structure + Human Voice Rules aligned to `skill_readme_template`.
- **9 profiles conformed.** Each model profile + `_index.md` now opens with `## 1. OVERVIEW` (Purpose / When to Use / Core Principle) and uses ALL-CAPS numbered H2 sections per `skill_reference_template`.
