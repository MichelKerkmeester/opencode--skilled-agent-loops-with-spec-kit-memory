# v1.0.0.0 - First Release

**Released:** 2026-07-16

The initial release delivered `mcp-mobbin` as the read-only Mobbin app-design research transport in the `mcp-tooling` hub (`packetKind: transport`, `mutatesWorkspace: false`). It searched real app screens, flows, and website sections through the registered Mobbin MCP manual and Code Mode, routed design-affecting conclusions to `sk-design`, and packaged the live-discovered three-tool contract, research intents, examples, catalogs, tests, diagnostics, and template-conformant documentation while keeping OAuth and authenticated-call claims explicitly Inferred and operator-gated.

---

## What's In 1.0.0.0

### Core Transport And Backend

- `SKILL.md` defined Mobbin-specific activation signals, a guarded intent router, discovery-first execution, app, screen, flow, element, and website-section research patterns, and read-only safety rules.
- The transport used the registered `mobbin` Code Mode manual, which launched `npx -y mcp-remote https://api.mobbin.com/mcp` over stdio to bridge the hosted Streamable HTTP provider. The manual kept an intentionally empty `env` because Mobbin MCP used browser OAuth and had no `MOBBIN_API_KEY` or other auth environment variable.
- The registered manual was preserved as a byte-identical reference shape. Manual presence was the healthy state, absence was a diagnostic error, and the packet never edited or re-added `.utcp_config.json`.
- Runtime authority stayed minimal: Read, Bash, Grep, Glob, and Code Mode calls were allowed, while Write, Edit, and Task were forbidden.

### Live-Discovered Code Mode Surface

- The 2026-07-16 pre-auth discovery fixture recorded three read-only registry tools: `mobbin.mobbin.search_screens`, `mobbin.mobbin.search_flows`, and `mobbin.mobbin.search_sections`.
- The fixture confirmed the TypeScript call forms `mobbin.mobbin_search_screens(...)`, `mobbin.mobbin_search_flows(...)`, and `mobbin.mobbin_search_sections(...)`. Each session still ran `list_tools` and `tool_info` before its first call and failed closed on drift.
- `search_screens` exposed deliberate `mode: "deep" | "standard" | "fast"` selection, with `fast` treated as a deprecated alias for `standard`, plus `exclude_screen_ids` and `image_format`. Its declared output used `query` and screen records with `id`, `image_url`, `mobbin_url`, `app_name`, and `platform`; older undeclared fields remained probe-required rather than asserted.
- `search_flows` supplied ordered per-screen previews for journey research, and `search_sections` supplied website-section research. Returned ordering was treated as fact, while interpolation, omitted states, and cross-app generalization were labeled inference.
- App and element research remained query intents over the discovered read-only tools rather than invented tool families.

### Access, Auth, And Limits

- The packet documented browser OAuth through DCR, PKCE S256, and `openid`, with operator-owned state under `~/.mcp-auth` or `MCP_REMOTE_CONFIG_DIR`; no API key entered config or `.env`.
- Discovery and full schemas worked before authentication, while OAuth completion, authenticated response shapes, inline-image fidelity, and authenticated calls remained Inferred or UNKNOWN until an operator completed the browser round trip on a paid account.
- Access was documented for Pro, Team, or Enterprise plans. Free-plan denial was relayed from provider evidence while its exact status, payload, and user experience remained unverified.
- The published rate contract stayed 60 requests per 60 seconds per user. On HTTP 429 the workflow honored `Retry-After` and applied exponential backoff with jitter, without forcing load tests or inventing finer burst, concurrency, per-plan cap, or Finance+ coverage contracts.

### Feature Catalog And Examples

- `feature-catalog/` supplied a root inventory and intent pages for apps, screens, flows, and elements, with query recipes, platform rules, schema honesty, plan gates, rate limits, mutation class, and links to the canonical tool reference.
- `examples/` supplied a minimum `limit: 1` smoke search, a platform-filtered flow-research walkthrough, and an element-intent comparison across apps. Every walkthrough used confirmed call forms, cited `mobbin_url` evidence, preserved unknown fields, and labeled reconstruction as inference.
- The catalog and examples treated the three-tool discovery fixture as the baseline while retaining per-session confirmation and authenticated-call gates.

### Testing And Operator Tooling

- The nine-scenario manual testing playbook covered registered-manual reporting, pre-auth discovery, screen search, flow intent, platform filtering, rate-limit observation, paid-gate error taxonomy, config and credential mutation refusal, `sk-design` pairing, routing recall, holdouts, and a negative control.
- Live paid, OAuth, 429, and image-fidelity checks allowed a documented SKIP, while locally testable wiring, schema, routing, and refusal behavior required real execution.
- `scripts/install.sh` verified Node 18 or newer, npx, registered-manual presence, and the operator OAuth boundary without installing or mutating anything. `scripts/doctor.sh` provided read-only diagnostics and an opt-in unauthenticated endpoint probe whose healthy result was HTTP 401.
- `mcp-servers/mobbin-mcp/README.md` pointed to the hosted provider and registered bridge without vendoring server source.

### References, Assets, And Documentation

- `references/tool-surface.md` carried the three live schemas, research-intent workflows, resolved deep-mode contract, response-shape cautions, plan and rate gates, provider disambiguation, and open questions.
- `references/mcp-wiring.md` carried the registered manual, `mcp-remote` bridge, hosted transport, DCR and PKCE behavior, observed naming, operator-owned auth state, and discovery-first contract.
- `references/troubleshooting.md` covered expected pre-auth 401 responses, OAuth and callback failures, stale discovery, entitlement denial, 429 recovery, image fidelity, context pressure, and fixture drift.
- `assets/utcp-mobbin-manual.md` preserved the registered reference shape and separated completed repository checks from operator-owned OAuth and authenticated smoke checks.
- `README.md` and `INSTALL-GUIDE.md` described current-state setup and verification. All packet paths used kebab-case, including `feature-catalog/`, `manual-testing-playbook/`, their root indexes, examples, and `INSTALL-GUIDE.md`.

### Template Conformance

- Reference and asset documents followed the create-skill OVERVIEW-first structure with Purpose and Usage while preserving technical content, frontmatter, filenames, Related Resources placement, and honestly Inferred auth claims.
- `scripts/README.md` followed the create-readme code-folder structure for Overview, Architecture, Directory Tree, Key Files, Entrypoints, Validation, and Related resources, grounded in the actual read-only and exit-code behavior of `install.sh` and `doctor.sh`.
- The package retained one read-only transport contract, one registered backend, and no packet-local design judgment or workspace mutation.

### Boundaries

- Any design-affecting use loaded `sk-design` first. Mobbin supplied cited evidence and never issued taste, accessibility, or readiness verdicts.
- The official `mobbin/skills` repository was referenced rather than vendored, and its optional evidence-board writer stayed outside this read-only packet.
- No hub files or live configuration belonged to this packet's release, and no undocumented tools, parameters, credentials, limits, or authenticated results were claimed.
