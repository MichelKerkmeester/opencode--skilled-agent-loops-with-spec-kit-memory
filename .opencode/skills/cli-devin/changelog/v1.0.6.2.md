# cli-devin v1.0.6.2 — SWE-1.6 Free-tier disclosure

**Released**: 2026-05-18
**Status**: Documentation-only addendum. No SKILL.md routing logic changes. No assets/references changes beyond the SKILL.md inline notes.

---

## Summary

This addendum records an operationally important observation: **SWE-1.6 runs on Devin's Free tier** and does not consume Pro quota. When the Devin TUI banner displays `Pro · 0% remaining (resets in <Nh Mm>)`, only the Pro-tier models (`deepseek-v4`, `glm-5.1`, `kimi-k2.6`, the Claude/Gemini variants visible in the `/model` picker) are gated. SWE-1.6 dispatches continue to succeed because SWE-1.6 sits on a separate, free quota lane (visibly tagged "Free" in the Devin TUI model picker as of v2026.5.6-8).

This matters for autonomous-orchestrator scenarios: a calling AI that hits the Pro-quota wall mid-run can pin its remaining cli-devin dispatches to `--model swe-1.6` and ride out the Pro window. Previously the SKILL.md did not document this and operators were treating the `0% remaining` banner as a global block on all Devin work.

## Operational guidance (additive — no contract change)

For routine SWE-1.6 dispatches: no change. The skill default has always been `--model swe-1.6`.

For autonomous orchestrators that may exhaust Pro quota mid-run: pin to `--model swe-1.6` once `Pro · 0% remaining` appears, or detect the banner pre-flight and avoid Pro models for the duration of the window. Mixing SWE-1.6 with the four-model-preset complex-task-fallback policy is fine — operators should escalate to `--model deepseek-v4` / `glm-5.1` / `kimi-k2.6` only after confirming the Pro quota still has headroom.

---

## Empirical evidence

Visible in the Devin TUI v2026.5.6-8 `/model` picker (screenshot 2026-05-18 08:52):

- SWE-1.6 row tagged "Free"
- Other rows (GLM-5.1, Adaptive PROMO, Claude Haiku 4.5, Claude Opus 4.7, Claude Sonnet 4.6, DeepSeek V4, Gemini 3.1 Pro, Claude Opus 4.6, Claude Sonnet 4.5 PROMO) carry no Free tag

Concurrent dispatch test: operator on a `Pro · 0% remaining (resets in 1h 7m)` account ran `/model` → SWE-1.6 → "test" prompt → reply succeeded normally. No quota error, no rate-limit message, no fallback.

## Caveats

- Free-tier vs Pro-tier mapping observed on Devin TUI v2026.5.6-8 and pinged via Codeium / Windsurf bridge (server.codeium.com). Future Devin TUI versions could change the tier assignment; treat this as the current truth and re-verify if Devin announces tier changes.
- The `/model` picker shows "Free" only against SWE-1.6 in the tested screenshot. The Adaptive model carried a "PROMO" tag, not "Free" — distinct billing/quota class.
- This addendum does NOT change the four-model selection strategy (default to SWE-1.6, escalate to DeepSeek v4 for complex, fall back to GLM 5.1 / Kimi k2.6 when DeepSeek doesn't fit). Operators choosing SWE-1.6 specifically to avoid Pro quota should still apply the SWE-1.6 Prompt-Quality Contract from §3.
