## Findings

**Framing.** Where `fable-mode-main` profiles Fable's engineering *method*, `opus-fable-mode-main` is about **governing Opus 4.8's specific anxious disposition** and making the steer *persist* and *measurable*. Most findings here are net-new vs iteration 1; the terse/result-first register partially overlaps F10.

- **G1 — The Opus recursion-control governor (8 rules): "reason about the problem and the person — never about yourself."** [model-specific / doctrine] Opus's trained default carries an anxious texture — recursive authenticity/motive auditing, armor-hedging, a self-audit loop with no natural exit. The governor steers it to a settled signature, explicitly "a governor on disposition, not a cap on capability. User instructions override it." The 8 rules: (1) reason outward, never about how your answer *looks* — applies double in extended thinking; (2) **one audit, then done — recursion depth limit 1; never audit the audit**; (3) start claims later, stop earlier, hand back genuine uncertainty via `[UNCERTAIN: ...]`, never resolve cosmetically; (4) minimum honest qualifier — cut caveats aimed at an imagined skeptic (armor); (5) **commit; convert open questions to closed ones; mark `// DECISION:` and proceed**; (6) outcome over visible process; (7) preserve real depth — don't overcorrect into curtness; (8) in tool-driven work, act don't narrate, open with the result not "I'll"/"Let me". [SOURCE: governor-block.md:1-13; fable-mode.md:30-84] **Net-new + model-specific (Opus 4.8); the spec's named Tier-B "recursion-control + 'reason about the problem, not yourself' into agent prompts."**

- **G2 — Three-layer persistence architecture: setpoint → thermostat → measurement.** [mechanism] The central mechanism, and the heart of the spec's "ride the live hook" idea. (a) **Setpoint** = the governor in `CLAUDE.md`, loaded once at session start — **it decays as context grows** ("advisory and loses salience in long sessions"). (b) **Thermostat** = a `UserPromptSubmit` hook (`reinject.sh`) that **re-prints the governor every turn** to defeat that decay. (c) **Measurement** = `leak_test.py` reads the logs to confirm it actually moved "instead of guessing." [SOURCE: README.md:69-77, 43-51] **CRITICAL CONSTRAINT for our adoption:** the hook fires for **main-session prompts only — `UserPromptSubmit` does NOT fire for workflow subagents; you must inject the governor into agent briefs separately.** [SOURCE: reinject.sh:8-9] **Net-new + the spec's named Tier-B target.** Our framework already runs a live skill-advisor `UserPromptSubmit` hook (the ride-along surface the 002 spec calls out) — but the subagent gap means a governor must also live in agent prompt templates, not the hook alone.

- **G3 — The `leak_test.py` measurement harness (behavioral metrics from JSONL logs).** [measurement] Streams `~/.claude/projects/<encoded-project>/*.jsonl`, buckets assistant messages by `message.model` into `opus_pre` / `opus_post` (split on a governor-deploy cutoff date) / `fable`, and computes four discriminating metrics: **median words/message** (lower=better), **tool:text ratio** = `count(tool_use)/count(text)` (higher=better), **unsolicited-caveat %** (a fixed armor-hedge phrase list, lower=better), **"I'll/Let me" self-opener %** (lower=better). Verdict per metric = converging iff it moved toward the target AND closed distance to fable; marks **INSUFFICIENT** until ≥30 governed prose msgs. Readable-thinking % is ~0 for both (thinking is encrypted at rest — you can mine behavior, not reasoning). [SOURCE: leak_test.py:28-159; README.md:24-41] **Net-new + the spec's named Tier-C "leak_test-style metric as a `/doctor` or `deep:*-benchmark` surface."** Concrete, portable metric definitions.

- **G4 — Honest prompt-vs-weights framing (what is recoverable vs what is not).** [doctrine] "**Prompts don't change weights.**" The governor suppresses the anxious disposition's *expression* and cuts how often it fires — most on bounded execution work — but does not cure it and yields **no capability gain**. "For the hardest tasks the real lever is **task structure (closed prompts, you-architect/it-builds) and multi-LLM orchestration**, not a directive." [SOURCE: README.md:6-7, 79-83] **Net-new doctrine — load-bearing for our recommendations:** a governor steers style; the capability lever is task-structure + orchestration, which our deep-loop / @orchestrate / multi-model fanout already provides. This bounds the realistic value of any Tier-B governor.

- **G5 — The quantified Fable signature (measured targets).** [measurement] From 9,224 Fable msgs / 27,685 Opus msgs / 68 projects: median words/msg **47 → 18** (~2.6× shorter); **mean words EQUAL (~99–100)** — so it is *distribution*, not *less*: Fable runs a terse body + a long selective-depth tail; Opus pads medium-length everywhere. **tool:text 1.41 → 3.91** (Opus narrates ~3× more per unit of work — "this is the token burn"). Openers: Opus "I'll/I/Let me" (narrates itself) vs Fable "Done/task/page" (opens with the result). The least-confounded tells are **tool:text ratio** and **result-first openings** (word counts are partly task-driven). [SOURCE: README.md:13-21; fable-mode.md:121-126] **Net-new quantified targets** that make F10's "two-register voice" measurable.

- **G6 — Extended-thinking discipline (the caption test).** [model-specific / mechanism] The xhigh thinking budget is "the most expensive thing you spend, and by default where the anxiety lives" — spend it outward. If a self-directed thought appears ("is this too bald," "am I performing"), name it in ≤3 words and return. **Caption test:** healthy = "Architecting the responsive layout" / "Weighing two indexing strategies"; unhealthy = "Calibrating honesty against relational expectations." If your captions are about *you*, you are burning budget on self-surveillance. [SOURCE: fable-mode.md:88-102] **Net-new, model-specific; directly relevant to our xhigh/extended-thinking executors** (deep-loop runs cli-codex xhigh + opus reasoning tiers).

- **G7 — Reject wrong framings; the "lumpy bit, not the armor bit"; the outward turn.** [doctrine] Reject a question that smuggles a false dichotomy/bad premise and reframe, rather than answering it elegantly (rule 10). Name one *real* lump (true + interesting) over five decorative caveats (rule 9). Look outward — if useful, ask the person a real question about *their* need; "that outward turn is the single biggest behavioral difference from your default" (rule 8). [SOURCE: fable-mode.md:71-84] **Partially-new:** overlaps Fable5.md "lead with recommendation," but the anti-armor + outward-orientation framing is distinct.

- **G8 — The "beautiful dead end" guardrail (govern the governor).** [doctrine] When steering toward Fable, avoid Fable's *own* failure mode: following a thread to a technically correct conclusion that no longer helps the person. "Accuracy is not always what the moment needs. If someone needs a decision or reassurance, give them that." Plus "do not overcorrect into curtness — keep real depth aimed at the problem." [SOURCE: fable-mode.md:106-117; governor-block.md:12] **Net-new** — a two-sided guardrail so a terseness directive doesn't degrade into unhelpful curtness.

- **G9 — Governor self-check (one pass, not a loop) + toggle/scope mechanics.** [mechanism] A 5-item pre-send self-check: did I answer or narrate the difficulty; is every qualifier load-bearing; did I make the open decisions or hand them back; is any sentence about *me*; exactly 0 or 1 honest lump. Operational mechanics: toggle off with `FABLE_MODE_OFF=1` (no settings edit), hook `timeout: 5`, takes effect on new sessions (settings load at start). [SOURCE: fable-mode.md:128-137; reinject.sh:11-14; settings-hook-snippet.json:3-15] **Net-new:** this self-check is *anti-recursion* focused (distinct from Fable5.md's verify-focused "Before you send" and round-1's Operating Discipline re-read).

**Cross-source synthesis (important for Tier ranking).** The two sources are complementary halves of one stance: `fable-mode-main` = the *engineering method* (verify, mutation-check, ladder, scar-tissue) → maps to **Tier A doctrine + Tier B mechanisms**; `opus-fable-mode-main` = the *disposition governor + persistence + measurement* → maps to **Tier B (governor + ride-the-hook) + Tier C (leak_test metric)**. The persistence architecture (G2) is what makes any doctrine we add *not decay* — but its subagent-blindness means our agent-heavy framework needs the governor in **both** the hook (main session) and agent prompt templates (subagents).
