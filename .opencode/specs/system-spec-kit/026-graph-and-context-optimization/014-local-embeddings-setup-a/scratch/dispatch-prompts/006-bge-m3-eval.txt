     • "where is provider selection logic" → `shared/embeddings/factory.ts`
     • "Gate 3 classifier" → `shared/gate-3-classifier.ts`

   - `metrics.md` — define top-1 hit, top-5 hit, MRR@10
   - `run-eval.py` — orchestrator script

3) Index code corpus three ways at `006-bge-m3-hybrid-evaluation/eval/indexes/`:
   - `qwen3-4b/` — current Setup A baseline. Use existing `.cocoindex_code/target_sqlite.db` if Qwen3-4B is the active model; otherwise build a parallel index here.
   - `bge-m3-dense/` — bge-m3 dense-only via sentence-transformers (`SentenceTransformer('BAAI/bge-m3').encode(...)`)
   - `bge-m3-hybrid/` — bge-m3 dense + sparse + colbert via the `FlagEmbedding` library or huggingface raw model output. Use `BGEM3FlagModel(...).encode(texts, return_dense=True, return_sparse=True, return_colbert_vecs=True)`. Store sparse + colbert in JSON sidecar.

4) Run all 50 queries against all 3 indexes. Score with MRR@10. Record results in `eval/results.json`.

5) For hybrid scoring, use Reciprocal Rank Fusion (RRF): `score_hybrid = sum(1/(60+rank_i))` across dense/sparse/colbert ranks. Standard RRF k=60.

6) Decision matrix (write to `decision-record.md`):
   - If Qwen3-4B wins OR ties within 2pp MRR@10 → keep Setup A as-is; document bge-m3 as alternative
   - If bge-m3-dense wins by >2pp → swap default to bge-m3 (still pure-dense; no schema work)
   - If bge-m3-hybrid wins by >5pp over best dense → SHIP schema extension (sparse + colbert columns in cocoindex's vec0 table + RRF fusion in query path)

7) **Conditional schema work** (ONLY if hybrid wins by >5pp):
   - Edit `.opencode/skills/system-spec-kit/mcp_server/cocoindex_code/indexer.py` — add `sparse_vector` and `colbert_vectors` auxiliary columns to the Vec0TableDef
   - Edit `.opencode/skills/system-spec-kit/mcp_server/cocoindex_code/query.py` — add hybrid query path using RRF
   - Rebuild + reindex
   - Document in decision-record.md as "hybrid path shipped"
   - Otherwise document as "hybrid path deferred; lift insufficient to justify schema change"
