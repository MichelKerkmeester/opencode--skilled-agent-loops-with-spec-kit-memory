# deep-loop-runtime v1.1.0.0

> First release-cleanup pass on the deep-loop-runtime skill. Documentation-only changes. No code, scripts, tests or storage touched.

---

## Why This Release Matters

The deep-loop-runtime v1.0.0 baseline shipped the arc-118 FULL_ISOLATE_NO_MCP consolidation but the documentation surface had never been audited end to end against the current sk-doc templates or HVR voice standards. The README was undersized at 174 LOC with only 6 of the 9 sections that the system-spec-kit README structural anchor prescribes. The SKILL.md frontmatter description was 240 chars (over the 130-char skill limit). Two artifacts (SKILL.md and README.md) carried a broken changelog path reference (`v1.0.0.md` instead of `v1.0.0.0.md`). A top-level `## Council Primitives` H2 inside section 3 broke the numbered-H2-ALL-CAPS rule.

v1.1.0.0 is the first cleanup pass that addresses all four P1 doc-class deviations with surgical edits while preserving the Smart Router section, the runtime lib code, the script entry points, the tests and the SQLite storage exactly as they shipped in v1.0.0. The release ships from the `001-deep-loop-runtime` packet under `000-release-cleanup` (first of five sibling cleanup packets that follow the same template).

---

## What Changed

### README rewrite (full pass)

The README was rewritten from 174 LOC to 470 LOC. The new structure follows the system-spec-kit 9-section pattern (OVERVIEW, QUICK START, FEATURES with 7 sub-domains, STRUCTURE, CONFIGURATION, USAGE EXAMPLES, TROUBLESHOOTING, FAQ, RELATED DOCUMENTS). The voice is anchored at ~70% of `Public/README.md` intensity. HVR self-scoring caught zero em dashes, zero prose semicolons, zero banned metaphors, zero setup-language patterns and zero copula-avoidance constructions. `validate_document.py --type readme` exits 0.

Coverage by domain:

- 3.1 Executor and prompt (4 modules: executor-config, executor-audit, prompt-pack, post-dispatch-validate)
- 3.2 State safety (4 modules: atomic-state, jsonl-repair, loop-lock, permissions-gate)
- 3.3 Scoring and routing (2 modules: bayesian-scorer, fallback-router)
- 3.4 Coverage graph (3 modules: coverage-graph-db, coverage-graph-query, coverage-graph-signals)
- 3.5 Council primitives (5 modules per packet 131/001/008 ADR-001)
- 3.6 Script entry points (4 .cjs entry points plus cli-guards lib)
- 3.7 Storage (runtime-owned SQLite at storage/deep-loop-graph.sqlite)

Every domain row identifies the consuming skill (`deep-review`, `deep-research`, `deep-ai-council`) so cross-system connections are explicit.

### SKILL.md surgical fixes

Three surgical edits inside SKILL.md, no other changes:

- Line 4: description trimmed from 240 chars to 124 chars
- Line 142: `## Council Primitives` demoted to `### Council Primitives` (H3 sub-section of section 3)
- Line 266: `changelog/v1.0.0.md` corrected to `changelog/v1.0.0.0.md`

Smart Router (SKILL.md section 2) UNTOUCHED. ADR-007 in the spec folder was not triggered.

### Version bump

SKILL.md frontmatter `version: 1.0.0` bumped to `version: 1.1.0`.
