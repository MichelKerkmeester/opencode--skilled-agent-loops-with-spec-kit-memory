# v1.4.0.0 - Decouple from Open Design (standalone skill)

**Released:** 2026-06-17

`sk-interface-design` no longer names the internal `mcp-open-design` skill or the Open Design app anywhere in its live content, so it reads as a standalone skill an external user can adopt on its own. The integration is now one-way: `mcp-open-design` still names `sk-interface-design` as its mandatory design-judgment partner, but `sk-interface-design` knows nothing about it. Mobbin and Refero (public, subscription-gated reference MCPs) stay.

---

## What Changed

### The shared parity doc was split

- The seam doc `references/design-process/claude_design_parity.md` was renamed to vendor-neutral `references/design-process/real_ui_loop.md` and rewritten to drop all Open Design naming. It keeps the transport-agnostic loop (ground -> reuse -> render -> check -> revise -> hand off), the element-target revision grammar, the fidelity-check principle, the handoff manifest, the pre-build direction gate, and the guardrails. Section numbers (§6 handoff, §7 pre-build gate, §8 guardrails) are preserved.
- The Open-Design-specific transport mechanics (the `od` reads, `od ui respond` routing, the `previewUrl`/`get_run`/`get_artifact` fidelity check, run self-healing) moved into `mcp-open-design` as `references/design_parity_transport.md`, which points back to `real_ui_loop.md` for the judgment.

### Open Design naming removed from this skill

- `SKILL.md` (§2 resource rows, §5 references, §7 integration, reference-loading notes), `README.md`, `references/design-grounding/design_inventory.md`, `references/design-grounding/design_references_mcp.md`, `references/design-process/variation_diversity.md`, the feature catalog, and the manual testing playbook were reworded to generic "a real design system you own", and the `claude_design_parity.md` references repointed to `real_ui_loop.md`.
- `graph-metadata.json` dropped `mcp-open-design` from `related_to` and from `prerequisite_for` (the reverse edge in `mcp-open-design` stays).

### Cross-skill repoints

- `mcp-open-design` (SKILL.md, README, INSTALL_GUIDE, feature catalog) repointed from `claude_design_parity.md` to its own `design_parity_transport.md` plus `real_ui_loop.md`.
- `sk-prompt/references/design_generation_patterns.md` repointed its `claude_design_parity.md` citations (§6/§7/§8) to `real_ui_loop.md`.

---

## Notes

- The asymmetry is intentional: interface design does not require Open Design, but Open Design design-work requires interface design (the 145/011 mandatory coupling, unchanged).
- Historical changelogs (v1.1.0.0, v1.3.0.0) are left as point-in-time records and still mention the prior Open Design coupling.
