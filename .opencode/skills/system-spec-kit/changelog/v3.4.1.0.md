# v3.4.1.0, Now You Can Watch It Run

v3.4.0.0 shipped the substrate. v3.4.1.0 is the release where you can see it working. The code graph found the bug that was costing it most of its symbols. The skill advisor finally ships as a real daemon backing live hooks across every runtime that has them. The CocoIndex daemon stops leaking processes and log spam under load. And the doctor commands, trust badges, freshness indicators, and telemetry surfaces this release adds are all built so a single command can answer "is this thing healthy right now?"

Alongside the hardening work, this release is also a polish pass on names, descriptions, and tests. Three skills and agents were renamed to match what they actually do. Skill and command descriptions got trimmed by 42% to fit back under Claude Code's discovery budget, and a new `/doctor:skill-budget` command keeps them honest. Manual testing playbooks for `sk-code-review`, `sk-git`, and `sk-prompt` ship 66 scenarios in natural-human voice. The Multi-AI Council agent now has a full output-protocol contract. And `cli-copilot` is retired, `cli-codex` is the cost-optimized default external executor going forward.

## The Code Graph You Can Actually Use

The code graph was missing 60 to 80 percent of the symbols in your codebase. Not because the parser could not see them, because the parser was never running.

#### The tree-sitter parser path fix

The tree-sitter grammar path was relative to the source tree, not the compiled `dist/` location, so production silently fell back to a regex shim. After the fix, broad-scope scans produce 56,843 nodes versus the regex-fallback baseline of 5,000, roughly 11x more symbols recovered on every scan.


#### A diagnostic command

`/doctor:code-graph` is a new operator command with three modes:

- `auto` and `confirm`: diagnostic-only. Report parser status, scan freshness, scope coverage, and known stale areas.
- `apply`: verification-gated auto-fix. Runs a 28-query gold battery before and after each operation; if the post-flight battery fails, the DB triplet reverts to the last known-good snapshot. Operations map to a three-procedure recovery playbook (SQLite corruption / partial-scan failure / bad-apply rollback) with confidence-tiered exclude rules (high / medium / low). 25 new vitest cases.


#### Freshness, scope, and a self-contained package

Every code-graph response now carries a freshness status and stale-graph highlighting. Default scope excludes are tightened: `z_future/`, `z_archive/`, and CocoIndex's own internal paths are skipped unless explicitly requested. `.gitignore` entries are honored. Cross-file dedup applies on every multi-file query. The whole code-graph package is now self-contained at `mcp_server/code-graph/`. 34 files migrated with zero regressions on the 52 + 85 + 171 test surfaces.


#### Backend resilience

Five hardening streams shipped behind a 10-iteration deep-review and a follow-up sk-code-opencode audit: scan idempotence, query-timeout protection, error recovery, diagnostic accuracy, and test-isolation robustness. 5 P0 + 12 P1 findings closed in the first remediation; 1 P0 + 10 P1 closed in the second. Test suite at 99.8% pass.


#### Hook integration parity

CALLS resolution now disambiguates fully-qualified names instead of failing on symbol-name collisions. Blocked reads return a typed `BlockedResult` with explicit reason instead of an opaque error. CocoIndex ranking fidelity is preserved end-to-end. Graph quality summaries (`graphQualitySummary`) are exposed in startup payloads, and startup-payload parity is verified across all four runtimes.


#### Real-world usefulness trial

A sandbox usefulness trial ran the code graph across five surfaces in production-shaped traffic, queries, hooks, advisor, Gate 3, and runtime smoke. Outcomes mapped back to deferred cells in the implementation track, closing the loop between "we shipped it" and "it works in the way it gets used."

## Skill Advisor: Now a Real Daemon

The advisor finally ships as a daemon. v3.4.0.0 cleaned up the routing dictionaries and shipped the graph compiler. v3.4.1.0 is the runtime that uses both.

#### Native daemon, single-writer lease

A daemon now backs the advisor with watcher-driven freshness and a single-writer lease that prevents concurrent processes from corrupting the compiled graph. The previous Python script and OpenCode plugin paths still work, they route through compatibility shims to the native behavior. A new MCP surface, `advisor_recommend`, `advisor_status`, `advisor_validate`: exposes the advisor as a first-class tool instead of a hook-only side effect.


#### Live hooks across every runtime

Each `UserPromptSubmit` event in Claude, Gemini, Copilot, and Codex now runs `buildSkillAdvisorBrief(prompt, {runtime})` and emits a sanitized skill recommendation with freshness, confidence, and uncertainty fields. The fail-open contract returns `{}` on any error, so a misbehaving advisor never blocks a prompt. Cross-runtime corpus parity: 200 of 200 fixtures match. Cache hit-rate 66.7%, cache hit p95 latency 0.016 ms.

A 650-line reference doc at `references/hooks/skill-advisor-hook.md` (DQI 97/100) documents the contract, and 11 test files plus 10 canonical fixtures lock the behavior in place.


#### Native TypeScript scorer

The five-lane fusion scorer is rewritten in native TypeScript. Family affinity and graph-derived transitive boosts both flow through the new path. The 52-fixture regression suite still passes at 100% top-1 accuracy through the migration.


#### Smart router and plugin hardening

The smart router shipped six remediation areas from the upstream research packet: stale-route resource cleanup, static router validation in CI, conservatively-tuned ON_DEMAND keywords (hit rate climbed from 5.5% to 48.0%), explicit UNKNOWN disambiguation in place of silent CLI fallback, observe-only telemetry, and a bridge-backed OpenCode plugin (`spec-kit-skill-advisor`) that imports the same producer and renderer the hooks use.

The plugin itself was hardened across three deferred audit items: per-instance closure state (no shared module state across plugin instances), an in-flight bridge-promise dedup so concurrent identical cache misses share one bridge call, and bounded payloads with cache LRU eviction. Tests grew from 23 to 30.


#### Setup command, standards alignment, and ambiguity fix

`/doctor:skill-advisor` is a new operator command that analyzes the current skill set, optimizes advisor scoring tables, and re-indexes the skill graph. A standards-alignment pass across the 21 skill metadata files closed routing inconsistencies that accumulated as the advisor evolved.

The ambiguity detector also got a calibration fix: a candidate is now flagged when EITHER score gap ≤ 0.05 OR confidence gap ≤ 0.05, not score-only. 37 scorer tests plus 40 handler tests pass.

## Template Levels, Redesigned

The Level 1 / Level 2 / Level 3 / Level 3+ taxonomy is being retired. It conflated three orthogonal concerns, line count, capability set, and template shape, into one flat dimension that did not scale to phase parents, audit packets, or research-only packets.

A 9-iteration deep-research loop (`cli-codex gpt-5.5 high`) ran on the consolidation question. The PARTIAL backward-compat framing was rejected. The converged design is a **C+F hybrid manifest-driven framework**: capabilities and presets live in a single `spec-kit-docs.json` manifest, scaffolders and validators both read from it, and human-authored scaffold docs are distinguished from command/agent-owned lazy addons. Five ADRs formalize the decisions.

A four-phase implementation plan: ADD manifest and resolver (shipped), MODIFY scaffolder, MODIFY validators, DELETE legacy taxonomy code. The validator stays inclusive of legacy phase parents during the transition. 86 source files collapse into roughly 15 manifest files in the new design.

Four sweep packets cleaned up the surface in parallel: stale references in `SKILL.md` and skill-bundled assets (with brief mentions added for current features like `SPECKIT_POST_VALIDATE`, exit codes, path hardening, save locking, batch inline rendering, and phase-mode syntax), command MD/YAML alignment, fleet-wide `SPECKIT_LEVEL` and `SPECKIT_TEMPLATE_SOURCE` marker validation, and a separate sweep over the `z_archive/` corpus.

## CocoIndex Daemon Resilience

The `mcp-coco-index` daemon was leaking zombie processes (244 open file descriptors held by stale daemons), burning ~90% CPU formatting tracebacks on every client disconnect, growing logs without bound (23 MB observed before the fix), and spawning duplicate daemons under concurrent-start storms.

Nine patches across five hardening phases.

#### Broken-pipe survival

A `_safe_send_bytes` wrapper now catches the broken-pipe condition and recovers instead of double-crashing the writer. The 564 spurious `BrokenPipeError` lines per run are gone, and the disconnect-loop CPU spike with them.


#### Log cap

Logs cap at 10 MB per file with 5 backups (60 MB total) instead of growing without bound. A logging double-write bug, every log line was emitted twice through both a stream handler and a file handler, is fixed via a conditional `StreamHandler` that engages only on TTY.


#### Concurrent-start storms

Concurrent daemon starts now route through an atomic PID-file lock (`fcntl.flock` on POSIX, `msvcrt.locking` on Win32). A sibling-detection block fires inside the lock-held window before the PID write, so two simultaneous starts converge on a single survivor instead of both winning. Listener backlog is also raised from the stdlib default of 1 to 128, so 16-way client storms queue at the OS level instead of failing with connection-refused.


#### Bounded shutdown

Shutdown is bounded by a 10-second `asyncio.wait_for` cap on task-join. The daemon no longer hangs indefinitely on stuck tasks.


#### Operator recovery

A single recovery procedure now works whether the daemon is half-stuck, fully stuck, or duplicated. 20 of 20 pytest pass, 9 unit tests and 7 E2E/integration tests, plus 4 covering the storm scenarios.

## Multi-AI Council, Productionized

The Multi-AI Council agent now ships a complete output protocol contract instead of an implicit one. Agent body §12 through §17 document the output protocol, invocation contract, state schema, convergence signal, caller-persistence protocol, and summary. Four shared reference files under `system-spec-kit/references/ai-council/` cover folder layout, seat diversity patterns, convergence signals, and JSONL state format.

A v1.1 state metadata schema with `schema_version`, `protocol`, and `producer` fields is forward-compatible (v1 readers still parse it). A new `--memory-save-payload-out` flag writes a memory-save payload only on opt-in. An advisory `advise-council-completion.cjs` script reports missing artifacts, missing reports, and seat-count mismatches, and always exits 0, so it works in pre-commit hooks without gating strict validation.

The validator regression test confirms the new `ai-council/` artifact folder pattern is tolerated. Four-runtime mirror parity tests prevent the agent body from drifting between Claude, Codex, Gemini, and OpenCode definitions.

#### Scoped-write authority

The council writes its own artifact tree. Permission across the four runtime mirrors: `write: allow / edit: allow`; `bash: deny / patch: deny`. Path-scope to `ai-council/**` is enforced by the writer library, `scripts/ai-council/lib/persist-artifacts.js` throws `OUT_OF_SCOPE_WRITE` for any out-of-bounds target. The audit-trail schema is additive v1.1 → v1.2 with new `artifact_written` and `rollback` event types; v1 readers continue to parse v1.2 files unchanged. Round-NNN folder is the rollback unit; on seat error or convergence failure, the round artifacts move to `failed/round-NNN-<timestamp>/` and the audit trail records `superseded_by: "rollback"` markers. Three ADRs (write-permission model, audit-trail schema, rollback semantics) document the decisions. 7 new vitest cases cover permission-scope, runtime-parity, audit-trail, and rollback.
