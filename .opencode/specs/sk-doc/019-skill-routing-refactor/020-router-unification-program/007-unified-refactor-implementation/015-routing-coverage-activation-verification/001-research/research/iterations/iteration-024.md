# Iteration 24 — gpt-5.6-sol high — focus: sk-doc-templates

## Headline finding

Default-on currently has two incompatible meanings. Hub `SKILL.md` directives treat a compiled decision as authoritative, while the advisor preserves the legacy recommendation and attaches compiled output only as metadata (`.opencode/skills/sk-code/SKILL.md:52`, `.opencode/skills/sk-code/SKILL.md:56`, `.opencode/skills/system-skill-advisor/mcp-server/handlers/advisor-recommend.ts:362`, `.opencode/skills/system-skill-advisor/mcp-server/handlers/advisor-recommend.ts:371`). Parity conceals this split today; future divergence would produce entry-point-dependent routing.

The intended sk-code alignment gate is also not implemented. `code-opencode/SKILL.md` says a drift guard enforces child/parent `RESOURCE_MAP` equality (`.opencode/skills/sk-code/code-opencode/SKILL.md:45-51`), but `verify_alignment_drift.py` excludes Markdown from supported inputs (`.opencode/skills/sk-code/code-opencode/assets/scripts/verify_alignment_drift.py:39-51`) and dispatches only language/JSON checks (`.opencode/skills/sk-code/code-opencode/assets/scripts/verify_alignment_drift.py:478-491`). Its main loop merely applies those checks to discovered code files (`.opencode/skills/sk-code/code-opencode/assets/scripts/verify_alignment_drift.py:507-519`). It cannot validate the fenced Markdown `RESOURCE_MAP`.

The P4 lockstep scope is incomplete. The cutover plan names two predicates plus seven live hub directives (`.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/012-default-on-decision/plan.md:107`, `.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/012-default-on-decision/plan.md:176`), while create-skill freezes default-off wording into two parent templates (`.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/013-create-skill-alignment/spec.md:90-100`, `.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/013-create-skill-alignment/spec.md:107-108`). Its P4 mitigation mentions only the active scaffold and seven directives, omitting the copy-from template (`.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/013-create-skill-alignment/spec.md:167`).

`ENV-REFERENCE.md` has a provenance problem, not merely a missing row. Its feature table is generated from the memory server’s `search-flags.ts` (`.opencode/skills/system-spec-kit/mcp-server/ENV-REFERENCE.md:27-29`), whereas `SPECKIT_COMPILED_ROUTING` is read by the skill-advisor handler (`.opencode/skills/system-skill-advisor/mcp-server/handlers/advisor-recommend.ts:362`). Its documented flag patterns cover default-on `!== 'false'` and opt-in `=== 'true'`, but not the current literal-`1` predicate or proposed unset-on/`0`-off semantics (`.opencode/skills/system-spec-kit/mcp-server/ENV-REFERENCE.md:18-24`). Yet P0 designates this file as the flag’s source of truth (`.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/012-default-on-decision/plan.md:130-132`, `.opencode/specs/sk-doc/019-sk-doc-router-alignment/020-router-unification-program/007-unified-refactor-implementation/012-default-on-decision/plan.md:154-156`).

Ranked actions:

1. Define one serving contract before P4: either compiled decisions replace legacy recommendations when eligible, or compiled remains shadow metadata everywhere. Default-on cannot mean both.
2. Add a real `RESOURCE_MAP` parser/comparator to `verify_alignment_drift.py`, or move the authority into a machine-readable artifact consumed by both router and verifier.
3. Expand the P4 atomic update set to seven hub `SKILL.md` files, both create-skill parent templates, runtime predicates, `ENV-REFERENCE.md`, and compiled-router catalog entries.
4. Give `ENV-REFERENCE.md` a cross-server flag section or broaden its generator; document unset, `1`, `0`, `false`, and invalid-value behavior explicitly.

Ruled out: the seventh directive is not missing from `sk-git`; the eligibility set identifies `cli-external-orchestration` as the seventh hub (`.opencode/skills/system-skill-advisor/mcp-server/handlers/advisor-recommend.ts:41-49`).
