## [**1.1.1.0**] - 2026-07-16

Live Code Mode discovery ran on 2026-07-16 (direct stdio MCP probe of CodeMode-MCP: `initialize`, then `tools/call` on `list_tools`, `search_tools`, `tool_info`, with `UTCP_CONFIG_FILE=.utcp_config.json`). The dated fixture `references/discovery-fixture-2026-07-16.json` is the packet's new ground truth, and this release flips the naming conflict and the discovery-pending claims to observed facts.

#### Naming conflict resolved with live registry evidence

- Need: The doubled-prefix callable (`refero.refero_refero_<tool>`) was "live-verified locally" but carried an unresolved cross-lineage conflict with a circulating single-prefix derivation, so every doc hedged.
- Change: `list_tools` returned all **eight** tools pre-auth under the dotted doubled registry names `refero.refero.refero_{search_styles,search_screens,get_style,get_similar_screens,get_screen_image,get_screen,search_flows,get_flow}`, and the fixture's `Access as:` line confirms the TS callable `refero.refero_refero_search_styles(args)`. SKILL.md, README, INSTALL_GUIDE, `references/mcp-wiring.md`, `references/tool-surface.md`, the server package README, and the DISCOVER-001 playbook scenario now state the resolution with the fixture cited; the single-prefix derivation is recorded as refuted.
- Why it matters: The conflict was the packet's central hedge; consumers now get one authoritative dual presentation (dotted registry name for discovery queries, doubled underscore form for TS calls) instead of a preserved dispute.

&nbsp;

#### Discovery is pre-auth

- Need: Several docs gated callable confirmation on completed operator OAuth ("after you authenticate", "requires completed operator auth").
- Change: The probe listed tools and returned full schemas WITHOUT OAuth — discovery is pre-auth; authenticated CALLS remain operator-gated. Preconditions in README, INSTALL_GUIDE, and the server README are corrected; OAuth-gated live-call items stay SKIP-valid with their exact commands.
- Why it matters: Agents no longer defer a step that needs no auth, and the real operator boundary (calls, not discovery) is stated precisely.

&nbsp;

#### Schema details recorded from the fixture

- Change: `references/tool-surface.md` open question 2 is partially resolved: the fixture confirms `response_format?: "json" | "md"` (default `"md"`) on `refero_search_styles` and `refero_search_screens`, and required `platform: "ios" | "web"` on `refero_search_screens`. The other six tools' `response_format` exposure remains a per-tool `tool_info` check. Rate limits, page size, OAuth end-to-end, and Bearer acquisition stay UNKNOWN/Inferred.
