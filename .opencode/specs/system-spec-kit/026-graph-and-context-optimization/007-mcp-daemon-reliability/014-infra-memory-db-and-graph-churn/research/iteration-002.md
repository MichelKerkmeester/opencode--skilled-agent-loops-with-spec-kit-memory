## Negative knowledge (directions ruled OUT this iteration, with why)

- Ruled out "marker presence alone means the DB is corrupted." The marker is deliberately written on every DB open and removed only on clean close, so a live process or skipped cleanup can leave it present without proving FTS shadow corruption. Evidence: `.opencode/skills/system-spec-kit/mcp_server/lib/search/vector-index-store.ts:739`, `.opencode/skills/system-spec-kit/mcp_server/lib/search/vector-index-store.ts:1270`, `.opencode/skills/system-spec-kit/mcp_server/lib/search/vector-index-store.ts:1368`.
- Ruled out `memory_index = memory_fts` row-count parity as a complete SQ2 repair/health proof. The known failure is in the FTS5 shadow and the repair decision says the malformed shadow branch was chosen after source-row duplicate IDs were ruled out. Evidence: `.opencode/specs/system-spec-kit/032-infra-memory-db-and-graph-churn/implementation-summary.md:50`, `.opencode/specs/system-spec-kit/032-infra-memory-db-and-graph-churn/implementation-summary.md:79`.
- Ruled out the boot integrity check as the mechanism that heals recurrence. The boot check logs `DETECT-ONLY` and explicitly attempts no DB recovery. Evidence: `.opencode/skills/system-spec-kit/mcp_server/context-server.ts:371`, `.opencode/skills/system-spec-kit/mcp_server/context-server.ts:373`, `.opencode/skills/system-spec-kit/mcp_server/context-server.ts:374`.
- Ruled out a necessary one-to-one link between the SQ1 diagnostic row and `.unclean-shutdown`. With `SPECKIT_LAUNCHER_BRIDGE_DISABLED=1`, a live existing lease can make the launcher report and return before `launchServer()`, which can close the MCP client without opening the DB. Evidence: `.opencode/skills/system-spec-kit/mcp_server/stress_test/substrate/run-substrate-stress-harness.mjs:619`, `.opencode/bin/lib/launcher-ipc-bridge.cjs:322`, `.opencode/bin/lib/launcher-ipc-bridge.cjs:327`, `.opencode/bin/mk-spec-memory-launcher.cjs:883`.
- Ruled out treating packet 031/009 as end-to-end proof that live recurrence cannot happen. It verifies deterministic suites but still calls out the absence of a live two-launcher integration run. Evidence: `.opencode/specs/system-spec-kit/031-embedding-stack-hardening/009-single-writer-durability-cluster/implementation-summary.md:55`, `.opencode/specs/system-spec-kit/031-embedding-stack-hardening/009-single-writer-durability-cluster/implementation-summary.md:111`.
