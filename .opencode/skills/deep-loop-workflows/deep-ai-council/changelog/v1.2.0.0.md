# v1.2.0.0 — rename from deep-ai-council (skill + agent) (2026-05-21)

## Summary

Renamed the skill `.opencode/skills/deep-loop-workflows/deep-ai-council/` → `.opencode/skills/deep-loop-workflows/deep-ai-council/` and the agent `.opencode/agents/deep-ai-council.{md,toml}` → `.opencode/agents/ai-council.{md,toml}` across all three runtime mirrors. Family moved from `deep-loop` to `sk-util` per the parent spec.md §2 rationale: the Council is a planning sentinel skill (closer to sk-prompt's role) rather than an autonomous-loop executor like deep-research/deep-review which iterate to convergence. Behavior, trigger phrases, MCP wiring, council graph projection, and deliberation logic are all unchanged — this is a pure identity refactor.

The agent counterpart drops the `deep-` prefix entirely (becoming bare `@ai-council`) because the Council's identity is "AI Council", not "Deep Anything"; agents do not use the `sk-` family prefix per existing convention (`@code`, `@debug`, `@review`, `@context`, `@markdown`, `@orchestrate`).
