# v1.0.0.0 — webflow surface established

First release of the `webflow` surface packet. The Webflow frontend evidence that previously lived scattered across the workflow modes (`code-implement/references/webflow/` + `assets/webflow/`, `code-debug/references/webflow-debugging/`, `code-verify/references/webflow-verification/`, `shared/references/webflow-shared/`) is now consolidated under one read-only surface packet on the sk-code hub's surface axis.

## New

- `SKILL.md` — the surface contract: when the hub bundles this evidence, the full reference map, the non-negotiable frontend standards (CDN runtime reality, interaction-gated loading, Core Web Vitals as a gate, focus/forms accessibility), and the on-demand asset tier.
- `README.md` — packet at-a-glance and layout.
- `references/` — CSS/HTML/JavaScript standards, implementation and performance patterns, CDN deployment, browser `debugging/` and `verification/`, and the cross-language `shared/` tier.
- `assets/` — integrations, patterns, templates, scripts, and the surface checklists.

## Notes

This packet is `packetKind: surface`, `backendKind: evidence-base`, read-only (`toolSurface` ⊆ {Read, Bash, Grep, Glob}), and advisor-invisible (`routingClass: metadata`). It is reached only by the hub bundling it alongside a workflow mode via `routerPolicy.outcomes.surfaceBundle`; it never routes as a primary and mutates nothing.
