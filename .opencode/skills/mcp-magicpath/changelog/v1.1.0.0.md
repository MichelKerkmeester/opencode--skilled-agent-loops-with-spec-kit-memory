# v1.1.0.0 - Claude Design parity canvas loop

**Released:** 2026-06-14

This release gives `mcp-magicpath` the canvas-side half of the Claude Design parity loop it shares with `sk-interface-design`, plus a fidelity helper. The v1.0.0.0 baseline was the vendored CLI wrapper: find, inspect, install, and author MagicPath components. Everything below is additive and backward-compatible; the CLI directions and the editable-file boundary are unchanged.

Driven by spec `148-sk-interface-design` phases 007 (parity build) and 008 (doc realignment).

---

## What's New

### Canvas-side parity hooks (SKILL.md)

- A new ALWAYS rule wires the skill to the shared protocol `sk-interface-design/references/claude_design_parity.md`:
  - Reuse-before-generate: `search`/`inspect` the active theme's registered components before authoring net-new.
  - Fidelity check: after `code submit --wait`, verify against the backend-rendered `previewImageUrl` (not a `view`/`share` browser screenshot, which is session-gated), gated on the quality floor plus the anti-default critique.
  - Build-error self-healing within the editable boundary (cap two retries).
  - Element-target revision routing: broad feedback re-plans, targeted feedback scopes `code start --component --revision`.
  - The generated/presentational boundary: import installed components, never copy generated markup.

### Fidelity helper

- `scripts/design_fidelity.py` - a standard-library-only, query-only helper that fetches and downloads a component's `previewImageUrl` (via `list-components -o json`) so the fidelity check needs no browser session. Prints the preview path plus a reminder that the verdict is judgment against the quality floor and anti-default critique, not pixel diffing.

### Docs

- `README.md` - a "Design Parity On Canvas" subsection and related-document links to the protocol and helper.
- `scripts/README.md` - documents `design_fidelity.py` alongside `install.sh`.

## Notes
