## [**1.0.0.0**] - 2026-07-06

Initial release of `create-command`, one of the ten workflow packets in the `sk-doc` parent hub's workflow-only architecture. The packet scaffolds and refactors OpenCode slash commands under `.opencode/commands/`, covering command-type selection, frontmatter (`description`, `argument-hint`, `allowed-tools`), mandatory input gates for required arguments, argument dispatch, `:auto`/`:confirm` mode routing, and the thin-router/presentation-asset split for larger mode-based commands.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/125-sk-doc-parent` (Level phase parent)

---

## What Included

#### SKILL.md workflow (primary contract)

- A 13-step creation workflow: decide whether a command is warranted, resolve the invocation path, read existing files first, classify the command type, choose the output package shape, author frontmatter, add mandatory input gates, write the command body, implement argument dispatch, implement mode routing, enforce router/presentation separation, add destructive-action safety, and validate before delivery.
- Rules covering least-privilege `allowed-tools`, mandatory gates for required arguments, and keeping presentation text out of thin routers, plus escalation conditions for unclear invocation contracts.
- Lean and self-contained by design: no packet-local `graph-metadata.json`. The single advisor identity lives at the `sk-doc` hub root.

#### Creation references

- `references/README.md` -- overflow route map indexing the single-concern reference files below.
- `references/worked-example.md` -- the canonical file contract and a fully worked split command (router `.md` and presentation `.txt`).
- `references/router-presentation-split.md` -- router/presentation ownership, the before/after transformation, and the behavior-preserving rule.
- `references/argument-hints-and-modes.md` -- argument-hint design patterns, `:auto`/`:confirm` mode design, and frontmatter/description budget tips.
- `references/common-pitfalls.md` -- command-vs-skill-vs-agent selection and the common command-authoring mistakes table.

#### Assets

- `assets/command/command-template.md` -- command type templates, examples, vocabulary, and a validation checklist.
- `assets/command/command-presentation-template.md` -- the full `_presentation.txt` skeleton for split mode-based command families.
