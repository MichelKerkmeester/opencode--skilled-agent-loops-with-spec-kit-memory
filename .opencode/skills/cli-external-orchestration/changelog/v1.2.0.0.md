# cli-external-orchestration v1.2.0.0

## Added

- Restored `cli-codex` as a third nested workflow mode beside `cli-opencode` and `cli-claude-code`, reversing its earlier deprecation.
- Registered the mode in `mode-registry.json` and `hub-router.json` with field parity to the existing modes, so the advisor and executor-delegation scorer route it through the single hub identity.

## Behavior

- `cli-codex` is availability-gated and fails closed: every routing surface checks `command -v codex` before advertising or dispatching, and refuses the route when the binary is absent.
- Execution delegates to the audited deep-loop runtime (`codex exec`, Codex CLI 0.144.1); the mode advertises the route and owns the availability probe rather than implementing a second adapter.

## Verification

- Hub `mode-registry.json` and `hub-router.json` parse as valid JSON with the `cli-codex` entry at field parity to `cli-opencode`.
- Fail-closed gate proven against the real runtime with the `codex` binary shadowed out of `PATH`.
