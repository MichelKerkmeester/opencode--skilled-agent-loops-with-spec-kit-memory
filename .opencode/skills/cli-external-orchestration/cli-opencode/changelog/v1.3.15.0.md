# cli-opencode v1.3.15.0 — OpenCode Go removal

**Released**: 2026-06-26
**Status**: Logic update. Removes the `opencode-go` gateway provider from all skill logic and switches the default model to `deepseek/deepseek-v4-pro` via the direct DeepSeek API.

---

## Summary

The `opencode-go` gateway is removed from every operational surface of the skill. The default model and provider change from `opencode-go/deepseek-v4-pro` to the direct `deepseek/deepseek-v4-pro` (same underlying model, separately-funded credential). The provider auth pre-flight, decision tree, model-selection tables, prompt templates, examples and keywords no longer reference the gateway.

## Why

The OpenCode Go gateway routes DeepSeek and other open models through one shared credit pool. That pool returned `401 Insufficient balance` (CreditsError) on a live dispatch, and because the skill defaulted to it, the failure surfaced only mid-run rather than at pre-flight. Routing the default through the direct DeepSeek API removes that single point of billing failure.

## New Default

- **Before**: `opencode-go/deepseek-v4-pro --variant high` (gateway)
- **After**: `deepseek/deepseek-v4-pro --variant high` (direct DeepSeek API)
- Same underlying model; the change is the routing provider and its billing pool. Fallbacks remain `openai`, `xiaomi`, `kimi-for-coding`, `minimax` (Token Plan and Direct API).

## Preserved (history, not removed)

- The RM-8 destructive-scope-violation incident in `SKILL.md` (ALWAYS rule 13) and `references/destructive_scope_violations.md` still names the `opencode-go/deepseek-v4-pro` dispatch that deleted 44 files on 2026-05-04. That is a factual incident record; removing it would erase the provenance of the safety lesson.
- Prior changelog entries are unchanged.

## Open

- The `manual_testing_playbook/` test suite still references opencode-go, including two scenario files dedicated to the gateway path (`07--prompt-templates/deepseek-v4-via-opencode-go-with-sk-prompt-models.md` and `kimi-k2-6-via-opencode-go-with-sk-prompt-models.md`). These exercise a now-removed provider and need a separate delete-or-rewrite decision.

## Caveats

- `deepseek/deepseek-v4-pro` requires the `deepseek` provider credential (`opencode providers login deepseek`). If it is not configured, the pre-flight now asks the user rather than silently falling back.
- Gateway-only models that had no direct-provider equivalent (`glm-5.1`, `glm-5.2`, `qwen3.6-plus`) are no longer offered. `kimi-k2.6` is superseded by the direct `kimi-for-coding/k2p7`.
