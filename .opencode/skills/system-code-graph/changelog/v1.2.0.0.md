This release lands the system-code-graph portion of the 028 MCP-to-CLI dual-stack program. The `mk_code_index` daemon gains a second front door: `node .opencode/bin/code-index.cjs` exposes the same eight tools as CLI commands over the same daemon the MCP registration uses, with the registration itself unchanged. The release also repairs the `mk-code-graph.js` OpenCode plugin bridge the right way — the earlier in-process import fix was reverted as a direct-DB dual-writer hazard, and the bridge now routes over CLI/IPC with zero in-process database imports. Warm-only hook fallbacks for Claude Code and Codex complete the pairing: a CLI nobody's runtime calls does not close the transport-down incident class.

> Spec folder: `.opencode/specs/system-spec-kit/028-mcp-to-cli-tool-transition/002-code-index-cli/` (with program-wide phases under the same parent)

## New Features

#### The daemon-backed `code-index` CLI

The CLI generates its command map at runtime from the server's tool definitions, so all eight tools — `code_graph_scan/query/context/status/verify/apply` plus `detect_changes` and the rest of the public surface — are CLI commands with no manifest to drift. `list-tools` enumerates them offline; every other command validates arguments against the existing schemas and speaks JSON-RPC `tools/call` over the IPC socket, auto-spawning the daemon via `mk-code-index-launcher.cjs` when the probe fails and spawning is allowed. Exit taxonomy is shared verbatim with the sibling `spec-memory` and `skill-advisor` CLIs: `0` success, `1` runtime error, `64` usage/schema error, `69` protocol/dist mismatch, `75` retryable daemon error.

Correction (2026-06-12): the CLI does have an asserted manifest in `mcp_server/code-index-cli-manifest.ts`; command generation still uses the server tool definitions, but the manifest exists to lock the expected eight-tool surface.

&nbsp;

#### The false-safe contract carries through

A `status: "blocked"` readiness refusal exits `0` deliberately — blocked is an actionable answer (run the surfaced `requiredAction`, usually `code_graph_scan`), not a CLI failure. A malformed diff to `detect_changes` (`status: "parse_error"`) exits `64`. The deep-review pass also made argument coercion schema-driven: a string `minConfidence` no longer silently zeroes the filter, and a `NaN` exits `64` instead of passing through. Output is drain-awaited on exit, fixing a 64KB pipe-truncation hazard; large payloads were proven to arrive complete.

&nbsp;

#### Warm-only probing and the dist-freshness guard

`--warm-only` (default settable via `SPECKIT_CODE_INDEX_CLI_WARM_ONLY`, with `SPECKIT_CODE_INDEX_CLI_PROMPT_TIME` for the prompt path) probes the socket first and exits `75` instead of cold-spawning. The stable shim refuses a missing or stale dist entrypoint with exit `69` rather than running old code; `SPECKIT_CODE_INDEX_CLI_DEV_ALLOW_STALE=1` is the development override.

&nbsp;

#### The plugin bridge, repaired the right way

The `mk-code-graph.js` bridge had been broken since extraction, and the obvious fix — importing the moved modules in-process — was reverted because a second process touching `code-graph.sqlite` directly is a dual-writer hazard. The repaired bridge (`plugin_bridges/mk-code-graph-bridge.mjs`) routes over the CLI/IPC path instead, synthesizes its transport contract from the status payload, and blocks maintenance tools at prompt time.

&nbsp;

#### Hooks reach the graph without MCP

A shared warm-only helper (`code-index-cli-fallback.ts`) wraps the CLI in a socket probe plus `--warm-only --timeout-ms` invocation for the Claude `session-prime` / `compact-inject` / `session-stop` and Codex `session-start` adapters. No socket means a fail-open return in about a millisecond, and the hook result omits the CLI-backed extras rather than blocking the prompt.

## Backward Compatibility

Fully backward-compatible. The `mk_code_index` MCP registration, tool names, namespaces and response contracts are unchanged; the CLI is an additive second client over the same daemon and database, and the bridge repair removes a broken path rather than changing a working one. Operators who never touch the CLI see no behavior change.

---

## Upgrade

No migration required. `node .opencode/bin/code-index.cjs list-tools --format text` answers offline. If the shim refuses with exit `69`, rebuild the dist (`tsc -p .opencode/skills/system-code-graph/tsconfig.json`). Pass `--warm-only` in prompt-time contexts so a cold daemon yields exit `75` instead of a cold spawn. The new env flags are documented in system-spec-kit's `mcp_server/ENV_REFERENCE.md`.
