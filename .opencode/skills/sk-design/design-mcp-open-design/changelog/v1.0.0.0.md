# v1.0.0.0 - Initial Release

**Released:** 2026-06-14

Initial release of the `mcp-open-design` skill. It teaches an agent to drive the installed Open Design desktop app (nexu-io/open-design, the official open-source local-first Claude Design alternative) from the terminal, through its `od` CLI and stdio MCP server. The agent can wire the MCP server into opencode or Claude Code, read local projects and design-systems read-only, ground a design in one resolved system, and commission headless generation runs, all without typing into the in-app chat. The spine of the skill is the gated tool surface and the `sk-design-interface` integration that keeps the design judgment where it belongs.

## Architecture Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Interface | `od` CLI plus stdio MCP server | Both are faces of the same daemon, so the skill uses whichever fits the step |
| CLI location | `app/prebundled/daemon/daemon-cli.mjs` by bundle path | There is no global `od` shim, and bare `od` is the unrelated octal-dump tool |
| Daemon transport | Socket-discovered on an ephemeral port | The desktop daemon is not on a fixed `127.0.0.1:7456`, which is only the standalone default |
| Tool exposure | Surface read-only, gate mutating, omit destructive from the default path | The running server registers roughly 18 tools including destructive ones, so gating is the spine |
| Design judgment | Delegated to `sk-design-interface` | This skill is the transport, while the taste and anti-default critique live in the design skill |
| Content reuse | Live-read only, never cached into a repo | Vendoring Open Design files would attach their per-source Apache-2.0 or MIT licenses |

## Known Limitations

- The desktop app must be open for any tool call, because the daemon it hosts answers every call. A standalone `od --no-open` daemon is the headless fallback.
- The `od mcp --help` text undercounts the live tool set. The real surface is roughly 18 tools and must be verified live before relying on a tool.
- Some verbs need credentials. Local reads work without a cloud account, but generation, media, research, and plugin-publish may require a `vela login` or configured providers.
- The `references/` directory and `graph-metadata.json` are delivered separately from these supporting docs.
