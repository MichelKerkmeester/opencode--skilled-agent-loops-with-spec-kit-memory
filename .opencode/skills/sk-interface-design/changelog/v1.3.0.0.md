# v1.3.0.0 - Parity protocol re-centered on mcp-open-design

**Released:** 2026-06-14

The sibling `mcp-magicpath` skill is retired now that `mcp-open-design` drives the installed Open Design app from the terminal, so the shared Claude Design parity protocol re-centers on a single generation transport. The protocol's two members are now this skill (the judgment) and `mcp-open-design` (the Open Design terminal transport). The fidelity check moves off the MagicPath hosted-canvas mechanism (`previewImageUrl` plus the `design_fidelity.py` helper) onto Open Design's real `previewUrl`: the rendered design from a completed multi-turn run, with the written files read through `get_artifact`. The loop, the grounding, the reuse-before-generate discipline, the quality floor, and the anti-default mandate are all unchanged.

---

## What Changed

### claude_design_parity.md re-centered

- The protocol description and the two-member framing drop `mcp-magicpath`. The judgment stays here and the transport is `mcp-open-design`.
- Section 5 fidelity check now uses the completed run's `previewUrl` and the build's written files, read via `get_artifact` once `get_run` reports completed. Because Open Design is local-first, that preview is local and not gated behind a remote sign-in.
- The run self-heal replaces the MagicPath editable-source retry. A run left `awaiting_input` has produced no design, so answer the discovery form to fire the build, then poll and re-run if the build does not complete, capped at two retries.
- The element-target revision grammar routes a targeted edit as a follow-up message on the run conversation or an `od ui respond` answer, rather than a MagicPath component revision.
- The guardrails drop the hosted-canvas and write-themes-back-to-MagicPath lines and restate the no-unsanctioned-write-back rule for Open Design's gated verbs.

### Reference sweep

- SKILL.md drops the `mcp-magicpath` related-skill line and the magicpath mention in the parity reference summary. Version moves to 1.3.0.
- README, `references/design_inventory.md`, and the feature catalog (`feature_catalog.md`, `03--design-grounding`, `05--integration-boundary`, `07--claude-design-parity`) re-point every magicpath mention to `mcp-open-design`.
- The manual testing playbook re-centers the ID-009 fidelity scenario on the `previewUrl` mechanism and updates its source anchors and negative control. The negative control is now a finished-design claim from a run left awaiting input or an artifact-only file-add, in place of a session-gated canvas screenshot.

### Routing and discoverability

- `graph-metadata.json` drops the `mcp-magicpath` `prerequisite_for` edge and removes it from `related_to`. The `mcp-open-design` prerequisite edge stays.
