CONTEXT
HfLocalProvider (TypeScript, transformers.js-backed) and cocoindex_code/shared.py both hardcode prefix conventions for one specific embedding model family (Nomic). Any model swap silently inherits the wrong prefix → ~5-8% recall loss. We are migrating from Voyage to local embeddings (Qwen3-Embedding-4B for code, EmbeddingGemma for memory). The new models use DIFFERENT prefix conventions. We need a model-keyed registry + env-var override so future swaps are config-only.

  • ✓ Registry + env override hybrid → chosen
- **Consequences**: extending support = add registry entry (one-line PR); users with custom models use env vars; default fallback (empty) is safe.
