# Hallmark Design Skill Reuse Research

## 1. Executive Verdict

Hallmark is valuable as a source of granular quality probes, structural reasoning patterns, and compact design-DNA vocabulary, but it should not replace sk-design or become a preset catalog. The highest-value path is clean-room adaptation into the existing Interface, Foundations, Motion, Audit, and MD-GENERATOR owners. No standalone redesign, variant, study, theme, chart, tactile, imagery, or preview command is justified.

The strongest immediate adoptions are: correct Interface's hero/media and authored-depth contracts; add multi-page locks/deltas; add selected evidence-backed Audit probes; add a provider-neutral asset receipt; expose measured provenance and Motion in MD-GENERATOR; strengthen semantic motion/interruption/async proof; and add fallback-font metrics plus target-derived responsive verification.

Hallmark's curated themes can complement the 1,290-bundle extracted corpus only as independently authored, post-selection explanation or evaluation lenses. They must never select taste, emit values, bypass source rights, or become bundles/presets.

## 2. Licensing Verdict

Hallmark is MIT-licensed. The checked-in license permits use, copying, modification, merging, publication, distribution, sublicensing, and sale, but requires the copyright and permission notice in all copies or substantial portions. It is supplied without warranty. [SOURCE: .opencode/specs/sk-design/014-hallmark-design-skill-research/external/hallmark/LICENSE:1-20]

Practical verdict:

- `COPY` is legally available, but no direct-copy candidate is recommended by this research.
- `ADAPT` and `LEARN` should normally use independently authored sk-design terminology and preserve stronger native contracts.
- Copying substantial Hallmark prose, tables, schemas, examples, code, recipes, catalogs, or assets requires the Hallmark MIT notice.
- Hallmark's MIT license does not establish rights for hosted imagery, third-party quotations, fonts, logos, reference sites, or generated outputs. Those require separate provenance and rights evidence.
- The meaning of "substantial portions" and final repository notice placement require owner/counsel confirmation; this is source-text analysis, not legal advice.

## 6. Audit Findings

Hallmark's current slop checklist contains 58 checks: integer gates 1-57 plus gate `38a`; the README's 57 count is stale. Shipped Audit should keep its stronger evidence labels, user-impact severity, owner routing, and five-dimension scoring. The useful additions are selected structural fingerprints, fabricated-content/fake-chrome checks, input geometry, wrapped-affordance checks, intrinsic grid/long-word/section-head probes, and sticky/focus interaction edge cases. Every promoted tell needs positive, clean, and legitimate-exception fixtures. [SOURCE: .opencode/specs/sk-design/014-hallmark-design-skill-research/001-research/research/lineages/sol-opencode/iterations/iteration-002.md:14-57]

## 7. Interface And Foundations Findings

The shipped modes already own build, redesign, directions, layout, responsive adaptation, color, typography, and data visualization. The best Hallmark-derived changes are multi-surface redesign scope, site locks/page deltas, authored-depth provenance, hero media abstention/deletion, fallback font metrics, target-derived responsive proof, stylesheet-entrypoint preservation, and controlled custom-media proof. Fixed macrostructure, component, font, palette, voice, and theme catalogs should not be imported.

## 8. Motion Findings

Adopt a semantic motion character/rationale envelope in shared and Foundations artifacts; keep actual durations, easings, interruption, reduced-motion behavior, and runtime proof in Motion. Propagate interruption/reversal/retrigger semantics into pattern cards and add safe async feedback checks. Reject Hallmark's ten-theme duration multipliers, global reduced-motion recipe, universal state table, and fixed recipes. [SOURCE: .opencode/specs/sk-design/014-hallmark-design-skill-research/001-research/research/lineages/sol-opencode/iterations/iteration-005.md:16-88]

## 9. MD-GENERATOR Findings

Measured Playwright/computed-CSS extraction remains canonical. Add a report-level provenance/coverage envelope, neutral measured structural summaries, a conditional Motion section from captured evidence, and optional detector-backed texture/risk observations only after precision tests. Keep uppercase `DESIGN.md`; do not put authored or screenshot-estimated values in `tokens.json`. Optional DTCG/shadcn files must be separate, traceable post-validation outputs. [SOURCE: .opencode/specs/sk-design/014-hallmark-design-skill-research/001-research/research/lineages/sol-opencode/iterations/iteration-006.md:16-129]

## 10. Theme And Corpus Findings

The styles corpus contains exactly 1,290 canonical bundles and retrieval-manifest records. Hallmark's twenty themes and four genres are generative packages; sk-design's corpus is rights-gated evidence. If evaluation shows value, independently authored many-to-many archetype lenses may explain why a mode-selected source is useful or rejected. They must run after rights eligibility, remain advisory, emit no token values, preserve `no-fit`, and never become source bundles, selectors, rank authority, or presets. [SOURCE: .opencode/specs/sk-design/014-hallmark-design-skill-research/001-research/research/lineages/sol-opencode/iterations/iteration-007.md:13-74]

## 12. Phased Plan

- Phase 1, surgical existing-mode corrections: hero signature role, authored-depth origin, multi-page locks/deltas, fallback metrics, responsive proof, selected Audit probes, stylesheet preservation, emotion intake, chart cautions, controlled imperfection, and sound applicability. Add no command or mode.
- Phase 2, shared evidence envelopes: asset receipt, semantic motion character, interruption/async card propagation, measured provenance, and conditional MD-GENERATOR Motion section. Require cross-file schema/fixture parity.
- Phase 3, bounded reasoning/evaluation: abstract structural decision cards and optional independently authored archetype lenses. Promote only after no-preset, no-rights-bypass, no-value-emission, and `no-fit` fixtures pass.
- Phase 4, optional new capabilities: separately validated authored brand-system artifact, optional DTCG/shadcn exporters, then a project-path render adapter. Each needs demonstrated demand and explicit artifact-authority tests.
- Deferred: preview normalization, haptics, automatic corpus promotion, and screenshot diagnosis remain closed until concrete user pain/runtime evidence exists.

## 13. Acceptance Gates

1. Typography-only heroes pass when brief-grounded; decorative filler still fails; authored values carry origin.
2. Multi-page fixtures preserve shared type/color/spacing/navigation while retaining distinct page composition.
3. Every new Audit tell has positive, clean, and legitimate-exception fixtures with registry/reference/playbook parity.
4. Asset receipts record origin, rights, hosting, dimensions, accessibility, unresolved risk, and generation metadata; unknown rights block exact reuse.
5. Motion fields cannot override register ceilings, keyboard/frequency gates, or reduced-motion semantic parity; no named themes/fixed multipliers appear.
6. MD-GENERATOR provenance/Motion sections are deterministic projections of captured evidence; fabricated, missing-source, and estimated-token fixtures fail.
7. Derived lenses do not change retrieval eligibility, ranking authority, target values, locks, or `no-fit`.
8. Authored brand artifacts use a distinct name/schema and cannot enter measured `DESIGN.md`, `tokens.json`, or styles ingestion without reviewed conversion.
9. Project-path study measures only a launched canonical render; static code evidence remains separately labeled.
10. Substantial Hallmark expression triggers notice verification; external assets require independent rights evidence.
