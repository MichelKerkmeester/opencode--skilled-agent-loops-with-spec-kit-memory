# v1.0.0.4 — Rust language support

Rust joins the OpenCode surface's language slice as a first-class language alongside TypeScript, Python, shell, JavaScript, and config. A Rust-touching task now pulls the Rust standards; multi-language tasks pull every touched language's slice instead of only the first detected one.

## New

- `references/rust/` — Rust standards for the OpenCode surface: the system-code alignment gate applied to Rust plus the napi-rs / wasm-bindgen / WASI / cdylib FFI-boundary rules.
- `assets/checklists/rust_checklist.md` — the per-language Rust quality gate.
- `manual-testing-playbook/language-standards/009-rust-standards.md` (`OC-009`) — the Rust routing scenario for the surface benchmark.
- Rust detection: the `.rs` extension establishes the language within the OpenCode surface; `Cargo.toml`/`Cargo.lock` markers select it when no `.rs` file is touched. napi-rs and wasm-bindgen vocabulary are intent signals, not stand-alone surface detectors.

## Changed

- Language slicing moved from a single first-match language to the full touched-language set — a task touching two languages now pulls both slices. This supersedes the v1.0.0.0 "one language per task" standard with "touched-language set, not one-per-task."
- Surface routing, the parent language union, and the shared router-replay now register Rust as the sixth OpenCode language.

## Fixed

- Removed a duplicate Rust playbook feature file that collided with the config-schema scenario on id `OC-004`; the Rust scenario is now `OC-009` and the surface benchmark loads nine unique scenarios.

## Notes

Builds 1.0.0.1–1.0.0.3 were repository-sweep and frontmatter maintenance with no surface-behavior change, so this is the first surface changelog entry since the v1.0.0.0 establishment. The version is reconciled to 1.0.0.4 across `SKILL.md`, `README.md`, and this changelog.

# v1.0.0.0 — opencode surface established

First release of the `opencode` surface packet. The system-code evidence that previously lived under the workflow modes (`code-implement/references/opencode/`, `code-quality/assets/opencode-checklists/`, `shared/references/opencode-shared/`) is now consolidated under one read-only surface packet on the sk-code hub's surface axis.

## New

- `SKILL.md` — the surface contract: when the hub bundles this evidence, the language-sliced reference map, the non-negotiable system-code standards (plugins never write to the TUI, descriptors are load-bearing, alignment drift is a verification gate, one language per task), and the on-demand authoring checklists.
- `README.md` — packet at-a-glance and layout.
- `references/` — TypeScript/Python/shell/config/JavaScript standards plus the language-agnostic `shared/` tier (universal patterns, code organization, hooks, alignment verification).
- `assets/checklists/` — component authoring (skill/agent/command/mcp-server) and per-language quality gates.

## Notes

This packet is `packetKind: surface`, `backendKind: evidence-base`, read-only (`toolSurface` ⊆ {Read, Bash, Grep, Glob}), and advisor-invisible (`routingClass: metadata`). Spec-folder authoring is out of scope — it lives in system-spec-kit. The hub bundles this evidence alongside a workflow mode and slices it by the detected language; it never routes as a primary and mutates nothing.
