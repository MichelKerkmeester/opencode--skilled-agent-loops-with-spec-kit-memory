## What Changed

### D4-R grader fidelity (the should-fix cluster)

- **The grader is now dimension-aware end-to-end.** `composeGraderPrompt`, `parseGraderResponse`, the mock dispatcher, the cache key, and the cache metadata all thread a `dimId` (default `'D4'`, so every existing caller is unchanged). The user-prompt tail is derived from the dimension instead of hardcoding "Score D4 Hallucination only," so it no longer contradicts the D4-R task-outcome system prompt.
- **Fallback parsing no longer mislabels or accepts malformed payloads.** A new `normalizeParsedPayload` helper backs every parse path (strict, fenced, regex, score-only): it stamps the expected `dimId` when one is missing, normalizes a *mismatched* `dim_id` back to the requested dimension with capped confidence and an annotated rationale (new `*_dim_mismatch` parse statuses), and the score-only last resort uses the threaded `dimId` instead of a hardcoded `'D4'`.
- **The real grader passes the rubric as a system prompt.** `dispatchReal` now sends the system half via the verified `--append-system-prompt` flag and the candidate output via `-p`, instead of concatenating both into one `-p` argument.

### D4-R answer fairness

- **Longer task-outcome answers are graded in full.** The pre-grading truncation in `live-executor.parseLiveResult` moved from a magic `2000` to a named `GRADED_RESPONSE_MAX_CHARS = 8000`, so a thorough skill-on patch-plan is no longer clipped (and under-scored) before grading.
- **Robust routing-JSON extraction + model default.** `extractRoutingJson` now uses a string-aware brace-balanced scanner (`collectBraceBalancedObjects`) that handles nested-brace routing payloads the old bare-object regex rejected, and `runDispatch` backfills a missing model to `DEFAULT_MODEL`.

### Hardening

- **No shell-injection surface in the resume hint.** `dispatch-model.buildResumeHint` POSIX-shell-quotes the sentinel and loop-host paths via a `shellQuote` helper.
- **Config-load failures are visible.** `loadConfig` stays silent on a missing optional file (ENOENT) but warns on a malformed-JSON file that does exist, instead of silently returning `{}`.
- **Criteria-exec gate is explicit.** `score-model-variant` routes profile-command execution through `criteriaExecAllowed`, which warns when the gate is not explicitly set (preserving the trusted-profile default so the backward-compat test stays green) and documents the `DEEP_AGENT_ALLOW_CRITERIA_EXEC=0` opt-out.

### Maintainability + doc reconciliation

- **`scoreScenario` decomposed (byte-identical math).** The 153-line function is split into named helpers (`normalizeScenarioInput`, `computeSurfaceMatch`, `scoreD1Intra`, `scoreD2`, `scoreD3`, `scoreAssetRecall`, `firstFailingStage`, `modeAScore`, `buildLiveEvidence`); the D1-intra weights and surface-mismatch cap are named constants; mode-A scoring reads `WEIGHTS`; the negative-scenario `wastedCount` semantics are clarified. No score changes. Duplicated grader-base construction (`d4-ablation`) and event-stream parsing (`sweep-benchmark`) were de-duplicated into shared helpers.
- **Docs match the code.** `SKILL.md` gains the `SKILL_BENCHMARK runtime_assets` router branch and the 6 previously-missing scripts in §11; `README.md` reconciles the references count (14→17), the scripts table, the structure block, and the trigger phrases (6→9); `scoring_contract.md` completes the funnel stages and documents `advisorySignals`; the v1.11.0.0 changelog ties the D4-R scenarios to the `DEFAULT_D4R_SCENARIOS` constant.
