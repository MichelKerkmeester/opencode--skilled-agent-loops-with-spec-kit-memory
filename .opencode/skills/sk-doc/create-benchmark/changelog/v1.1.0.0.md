# v1.1.0.0 — behavior benchmark authoring added as a second benchmark family

`create-benchmark` now authors two distinct benchmark families. The original MCP-promotion workflow (a `mcp-server/benchmarks/benchmark-<YYYY-MM-DD>/` folder with a ten-section `benchmark_report.md` plus `SOURCE.md`) is unchanged. This release adds a second family: authoring a `behavior_benchmark` package for a deep-loop mode — an index, per-scenario machine contracts, and a Claude baseline — that specifies executor-model behavior at the mode's invocation surface and instantiates the shared measurement framework.

The measurement contract itself (five-dimension rubric, terminal buckets, budget formula, entry-surface and clarity enums, per-package ID-prefix table, and the runner) is NOT owned by this packet. It stays in `system-deep-loop/shared/behavior-benchmark/framework.md`; this packet authors packages that instantiate it.

## What Added

- **`assets/behavior-benchmark/behavior-benchmark-index-template.md`** — fillable scaffold for a `behavior_benchmark.md` package index (OVERVIEW with a declared-extensions subsection, SCENARIO TABLE, AXIS COVERAGE, EXECUTION, RELATED RESOURCES).
- **`assets/behavior-benchmark/behavior-benchmark-scenario-template.md`** — fillable scaffold for one `<PREFIX>-NNN-<slug>.md` scenario contract: the runner-parsed JSON machine contract plus Rationale / Pass shape / Failure modes prose. The shipped scenario carries no frontmatter and no OVERVIEW heading; the template's own frontmatter, usage comment, and Overview are stripped on copy.
- **`assets/behavior-benchmark/behavior-benchmark-baseline-template.md`** — fillable scaffold for `baselines/claude-baseline.md` (Overview, Baseline Table, Capture Provenance, Notes), shipping `pending` / `not_captured` cells until a real Claude leg lands.
- **`references/behavior-benchmark/behavior-benchmark-guide.md`** — end-to-end authoring guide: what a behavior benchmark measures, how it differs from MCP promotion, package layout, naming conventions, the eight-step authoring workflow, scenario-matrix design rules, and validation.
- **`SKILL.md` §8 BEHAVIOR BENCHMARK PACKAGES** — the in-contract summary of the new family: package shape, template map, authoring workflow, naming, and its own ALWAYS/NEVER plus success criteria. §1 gains a Benchmark Families router table and behavior activation and keyword triggers; the description, keywords, and Family Boundary now cover both families.
