## Findings
1. **[P1] over-claim** — Executive Summary claims phased hardening 'kills both hallucination classes — value fabrication via extraction bug fixes, prose fabrication via removing fabrication mandates and adding evidence gates.' Research §7 #5 openly concedes that even citation gating 'proves provenance, not truth' and that class-(b) and class-(c) sections carry [INFERRED] labels. Tier-2 C2 (SELF-REVIEW step) is itself labeled unreliable at §7 #6. The fixes mitigate, they do not eliminate prose fabrication. 'Kills' is marketing-level framing unsupported by the research's own evidence.
   - Recommendation: Replace 'kills both hallucination classes' with 'substantially reduces value fabrication and structurally gates prose fabrication.' Add explicit caveat that class-(b) and class-(c) [INFERRED] prose remains subject to LLM invention.
   - Evidence: research.md:15-16 ('kills both hallucination classes') vs research.md:263 (#5 'citation proves provenance, not truth') and research.md:264 (#6 'SELF-REVIEW step reliability — low')
2. **[P1] over-claim** — Section 2.3 claims 'Prose fabrication is not just undetected — it is structurally invisible to the existing validation infrastructure.' True for validate.ts. However, the research then claims in §5.2 Phase B that the doc-as-view architecture would 'structurally eliminate[] the LLM from value-table generation.' This is only true for class-(a) sections (§2,§3,§5,§6,§6.5,§9,§10,§12,§17 — which are value-table sections). Class-(b) and class-(c) sections (13 of the 17+4 sections) still involve AI prose. The framing conflates 'value-table generation' with 'all value-bearing sections.'
   - Recommendation: Clarify that doc-as-view eliminates LLM from value-TABLE rendering (hex, px, ratio values in class-(a) sections), not from interpretive prose about those values in class-(b) and class-(c).
   - Evidence: research.md:55 ('structurally invisible') and research.md:165-173 (three-class partition) — class-(a) is deterministic, class-(b) has token-cited AI notes, class-(c) has [INFERRED] AI prose
3. **[P1] error** — A3 (deltaE threshold raise from <3 to <10) is classified as 'SOUND, regression risk on visually distinct colors' in the verdict column. The regression risk is understated. OKLCH is a perceptually linear colorspace; CIEDE2000-based deltaE of 1 is approximately a JND. Raising the merge threshold 3.33× without calibration-study justification (a single constant, no adaptive algorithm) could catastrophically merge intentionally distinct brand colors. The regression guard at §6 Regression Guards #1 recommends testing against gold-standard examples but gives no acceptance criterion. This is not a safe 'slam-dunk.'
   - Recommendation: Either (a) calibrate the threshold against the gold-standard color palettes with a quantitative pass/fail criterion (e.g., 0% false-merge of known distinct brand colors), or (b) implement an adaptive threshold based on OKLCH distance distribution analysis within each site, or (c) reclassify to TIER-2 with explicit 'needs calibration study' dependency.
   - Evidence: research.md:81 (A3: '<3→<10', 'regression risk on visually distinct colors') vs cluster.ts:708 `deltaE(color.oklch, existing.oklch) < 3` / cluster.ts:1667 `< 3` — both have the same threshold. OKLCH is a perceptually uniform space; the jump to 10 is unsupported by any distribution analysis in the research.

The ~40 LOC estimate covers only `validate.ts:241-285` + `design_md_format.md:27-48`. In reality, the change requires: (a) validator logic changes in `checkSectionCompleteness` to distinguish always/conditional/optional sections, (b) format spec table changes, (c) prompt template changes to match the new conditional section behavior, (d) `checkSectionCoverage` (B9, +60 LOC) as a dependency for the data-driven condition test, and (e) integration testing across v1 and v2 format paths. True scope is likely 80-120 LOC across 3+ files with B9 as hard dependency.
   - Recommendation: Update LOE to 80-120 LOC across validate.ts + design_md_format.md + prompt template + B9 dependency. Make B9 a hard prerequisite of B1 in the ship-order table.
   - Evidence: research.md:82 (B1: '~40 LOC') vs validate.ts:241-285 + design_md_format.md:27-48 + B9 at research.md:90 as separate item. The B1 row's 'Deps' column is blank — B9 is not listed as a dependency despite checkSectionCompleteness needing checkSectionCoverage data to make conditional decisions.

   - Evidence: research.md:188 ('Kills ~60% of value-fabrication surface') is unsourced; research.md:264 (#6 'SELF-REVIEW step reliability' admits unreliability); research.md:263 (#5 'citation proves provenance, not truth')
