# Deep Start AI System Improvement Loop Presentation Contract

Presentation source of truth for `/deep:ai-system-improvement`. The command router owns only mode selection, the two HARD-BLOCK gates, the self-target fork, and asset loading. The workflow YAML dispatches the self-contained guarded refine loop host. This file owns the visible setup prompt, dashboard layout, result displays, and next-step suggestions. All guarded-loop logic — frozen scoring surface, kill-switches, worktree promote-N, resume, grades — is owned by the packaging and never comes from this router.

## 1. Startup Presentation

This command resolves setup through the command router's Mandatory Input Gate and the self-target fork, then loads the workflow YAML. For `:confirm` or no suffix, present the resolved inputs and wait for confirmation before dispatch. For `:auto`, resolve confidently from arguments and defaults, or fail fast naming the missing inputs.

### `:auto` Setup Resolution

Under `execution_mode = AUTONOMOUS` (from the `:auto` suffix), resolve the Mandatory Input Gate values from `$ARGUMENTS` and defaults:

1. **Resolve confidently**: bind `packaging_root` (first positional, or the `--self-target` profile-resolved root), `live` (default `false` = dry-run), and any of `--max-iters` / `--fixtures` / `--variants` / `--held-out` / `--samples` / `--proposer-model` / `--grader-model` the operator set. When `packaging_root` is bound and all self-target guards pass, load `.opencode/commands/deep/assets/deep_ai-system-improvement_auto.yaml`.
2. **Fail fast**: when `packaging_root` cannot be resolved, or a `--self-target` profile fails schema validation or the frozen-surface/allowed-diff cross-checks, emit the named-missing-inputs error with `/deep:ai-system-improvement:auto` as the command name and exit non-zero. Do not load YAML.

Dry-run is the default. `--live` requires a clean tree; in `:auto` with `--live`, fail fast rather than dispatch a live run on a dirty tree, an active lock, or an expired credential.

Use this block when `execution_mode = "INTERACTIVE"` or no suffix was supplied.

```

   |-- ":auto"    -> execution_mode = "AUTONOMOUS"
   |-- ":confirm" -> execution_mode = "INTERACTIVE"
   +-- No suffix  -> execution_mode = "ASK" (include Q-Mode)

2. RESOLVE the Mandatory Input Gate values from $ARGUMENTS and defaults:
```

**Phase Output:**

### Self-Target Fork Display

When `--self-target <profile>` is present, setup forks before dispatch and presents the resolved profile, packaging root, frozen surfaces, and allowed diff paths for confirmation (unless `:auto` with all checks green). The `--self-target` flag is router-owned setup and is never forwarded to `loop-host.cjs`; it compiles to `--packaging-root <resolved-root>` after the guard passes. Multi-candidate execution stays blocked unless the operator explicitly passes `--parallel`.

## 2. Dashboard / Checkpoint Layout

### Deep AI System Improvement

Lane D of the `deep-improvement` skill. Benchmarks an AI-system packaging (one prompt system shipped as CLI runtime, claude.ai Project, and native skill), re-grades outputs with an independent different-family grader, and runs a guarded autonomous refine loop: propose a bounded technique-doc edit, verify it against held-out fixtures inside an isolated git worktree, and promote or roll back. Self-reported quality scores are never the optimization target (the pilot measured them inflated ~+6/25 versus independent graders).

### Workflow Overview

| Phase | Name | Purpose | Outputs |
|-------|------|---------|---------|
| Setup | Resolve | Validate inputs + self-target guards; for --live, enforce clean-tree / lock / auth pre-flight | Ready-to-run inputs |
| Run | Dispatch | Run `loop-host.cjs --mode=non-dev-ai-system-refine` once; it spawns the packaging's guarded loop (contract validation, independent re-grade, promote-N, kill-switches) | Appended `loop-journal.jsonl` |
| Report | Surface | Read and present the packaging-owned journal | Per-sample grades, stop reason, promotion decision |

### Dry-Run vs Live

- **Dry-run (default)**: gates + grader-family guard + gap analysis, zero model dispatches, zero mutations.
- **`--live`**: guarded model dispatch and worktree promotion. Requires a clean tree, acquires the single-writer lock, and probes provider auth before the batch. Promotion only ever lands in a kept detached worktree — never the live tree.

### Kill-Switches (halt without promoting)

Scoring-surface drift, derived-copy drift, grader-family violation, hard-blocker lint failure, new floor breach, held-out regression (or below `LOOP_ACCEPT_MARGIN`), iteration ceiling, concurrent-run lock.

## 3. Results Display

### Return Status

```
ai-system-improvement: <packaging_root> mode=<dry-run|live> stop=<canonical_stop_reason>
  decision: <promote_accept|declined|killed:<reason>>
  journal -> <packaging_root>/benchmark/_loop/state/loop-journal.jsonl
```

The append-only `loop-journal.jsonl` is the machine record (per-sample grades, promotion decisions, canonical stop reasons). On `promote_accept`, the candidate edit lives in a kept worktree detached at the candidate state for deliberate operator merge; the loop never writes the live tree. This router never synthesizes a grade or a promotion decision.

### Error Handling

| Error | Action |
|-------|--------|
| Loop host STATUS=FAIL | Report the loop host's error verbatim; do not synthesize a grade or decision |
| Kill-switch fired | Report the canonical stop reason from the journal; no promotion |
| `--live` on a dirty tree / active lock / expired credential | Halt at pre-flight; do not dispatch |

### Presentation Boundary

The following router-owned display must render verbatim when triggered: the Phase 0 general-agent-required failure block with `STATUS=FAIL ERROR="Must be invoked directly, not pasted as inline sub-agent instructions"`, and the Mandatory Input Gate blocked-state wording, resolved-input confirmation, and missing-input failure summary.

The following content must not come from this router: loop-host progress, grades, promotion decisions, kill-switch details, journal rows, candidate worktree details, final benchmark/refinement report wording, and any packaging-owned fixture, variant, held-out sample, or operator-guide description beyond the setup fields named here.
