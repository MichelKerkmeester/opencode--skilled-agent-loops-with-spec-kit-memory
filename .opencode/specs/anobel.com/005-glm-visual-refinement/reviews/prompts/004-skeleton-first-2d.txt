## EXECUTIVE SUMMARY

GLM-5.2 cannot self-resolve 2-D constraint layout: tiles needing coordinate placement plus collision avoidance under a height budget (matrix / node / routing / funnel / popover) score 35-58, while linear-flow tiles score 86-94 ŌĆö a ~41-point primitive gap (`../../004-bento-visuals/research/research.md` ┬¦Root cause). This phase implements the **geometry kernel** (research angles A5 + A7, pipeline steps 7/8/9): compute a 480├Ś480 coordinate/safe-zone skeleton UPSTREAM (regions, boxes, row caps, anchors, connector routes, AABB collision gaps), pass it to GLM-5.2 as a hard rendering contract so the model never invents coordinates, recompute best-of-3 skeletons on first verifier failure, and downgrade to linear/stacked/compact primitives on second failure.

**Key Decisions**: Geometry is owned by a deterministic upstream module, not GLM (ADR-001); best-of-3 is upstream skeleton recompute, not three GLM calls (ADR-002).

**Critical Dependencies**: Phase 001 (deterministic gate + per-tile failure-JSON) and Phase 002 (primitive routing that labels 2D-positioned tiles).

## 2. PROBLEM & PURPOSE

### Problem Statement
2D-positioned bento tiles fail because GLM-5.2 owns the geometry: it places N rows/nodes at fixed pixel heights with no constraint solver, so cumulative height exceeds the 480px budget (RC-1), nodes/pills/text collide (RC-2), and the bottom title band is overrun when the diagram claims the whole canvas (RC-3) (`../../004-bento-visuals/research/research.md` ┬¦Root cause, rows RC-1..RC-3). Concrete failures: `accountbeheer-4 matrix` scored 35 (4th row spills onto the title), `oci-4 node` scored 58 (6 `position:absolute`, "Verbonden" pill overlaps the SAP card), `goedkeuringssysteem-4` scored 55 (`title_at_bottom:false`). These are hard-fact, geometry-checkable defects ŌĆö not taste.

### Purpose
Move 2D layout out of token prediction and into a deterministic skeleton plus a rendered verifier, so the ~41-pt diagram-vs-linear gap closes to roughly 16-23 pts and weak 2D tiles (35-58) move into the 65-82 band (`iter-r2-A5.md` ┬¦Predicted effect; `iter-r3-A5.md` ┬¦Expected Deltas).

## 3. SCOPE

### In Scope
- A deterministic **skeleton-compute module** that, for 2D-positioned treatments (matrix, node, routing, funnel, popover), measures Dutch copy and emits a 480├Ś480 skeleton: fixed regions (eyebrow / diagram / reserved title band), node/row boxes, row caps, connector anchors + routes, and AABB collision gaps (`iter-r2-A5.md` skeleton JSON; `iter-r2-A7.md` BENTO_GEOMETRY_KERNEL_V2).
- A **preflight AABB validator** that rejects a skeleton before any GLM call (no diagram/title intersection, no node/row collision, padding budget present, connectors terminate on declared anchors) ŌĆö pipeline step 7.
- A **skeletonŌåÆGLM render contract**: GLM receives the skeleton as canonical geometry, may choose copy/emphasis/variant/palette-token, and MUST NOT invent x/y/w/h, transforms, absolute positions, row heights, connector endpoints, or extra rows (`iter-r2-A5.md` glm_rules).
- An **A5 geometry verifier hook** after browser render that measures DOM/SVG bboxes for overflow, diagramŌł®title, node collisions, text padding, connector-anchor error, and title-at-bottom, extending the 001 ship-gate (`iter-r3-A5.md` `auditA5Geometry`).
- **best-of-3 recompute** (pipeline step 8): on first verifier failure, recompute up to 3 skeleton candidates upstream and pick the best geometry/AABB score ŌĆö never a free GLM coordinate retry (`iter-r4-pipeline.md` step 8; AdaCoder).
- **Downgrade-to-linear** (pipeline step 9): on second 2D failure, downgrade to linear-flow / stacked-list / compact-matrix preserving semantic intent via "+N more", not crowded geometry (`iter-r4-pipeline.md` step 9).

- The GPT-5.5 skeleton author ŌĆö that is the 005 escalation path; this phase ships the deterministic skeleton only.

## 8. EDGE CASES

### Data Boundaries
- Empty/short copy: skeleton still reserves the title band; diagram region may shrink but never overlaps the title.
- Maximum rows: matrix capped at 4 (with legend) / 5 (without); excess routes to "+N more" or downgrade.
- Over-budget diagram: nodes>5 / rows>3 / edges>6 auto-linearizes before code (`iter-r2-A7.md`).

### Error Scenarios
- GLM emits invalid JSON / over-dense plan: reject and recompute skeleton (plan-only retry), never accept model coordinates.
- GLM injects coordinate text despite the contract: verifier flags it; treated as a verifier FAIL ŌåÆ best-of-3.
- Connector endpoint > 4px from declared anchor: `connector_anchor_error_count > 0` ŌåÆ FAIL.

### State Transitions
- First verifier FAIL ŌåÆ best-of-3 recompute (step 8).
- Second verifier FAIL ŌåÆ downgrade (step 9).
- Downgrade success ŌåÆ ship downgraded primitive with semantic-intent preserved.

## 3. ARCHITECTURE

### Pattern
Deterministic upstream geometry + model-as-renderer. The model is demoted to a mechanical renderer; all placement is computed and audited outside it (`iter-r2-A7.md` BENTO_GEOMETRY_KERNEL_V2).

### Key Components
- **compute-skeleton.mjs**: measure copy ŌåÆ reserve title band ŌåÆ allocate diagram region ŌåÆ place node/row boxes within caps ŌåÆ compute connector anchors + orthogonal routes ŌåÆ emit schema-valid skeleton JSON.

- **verify-skeleton.mjs**: preflight AABB validator (step 7) + best-of-3 candidate scorer (step 8).
- **downgrade.mjs**: step-9 linearization to linear-flow / stacked-list / compact-matrix with "+N more".
- **gen-tile.mjs (modified)**: routes 2D tiles through the contract, forbids GLM coordinate text, orchestrates best-of-3 ŌåÆ downgrade.
- **_audit.mjs (modified)**: `auditA5Geometry` DOM/SVG bbox checks + extended ship-gate.

### Data Flow
002 labels tile `2d_positioned` ŌåÆ compute-skeleton emits skeleton ŌåÆ verify-skeleton preflight (step 7) ŌåÆ buildA5Prompt(skeleton) ŌåÆ GLM renders ŌåÆ browser ŌåÆ auditA5Geometry ŌåÆ ship-gate. On FAIL#1 ŌåÆ recompute best-of-3 (step 8). On FAIL#2 ŌåÆ downgrade (step 9). Skeleton JSON is canonical; renderer emits inline-style/CSS-vars; `data-a5-*` are audit-only.
