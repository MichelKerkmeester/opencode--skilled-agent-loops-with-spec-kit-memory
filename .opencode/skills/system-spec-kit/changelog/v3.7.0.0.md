# v3.7.0.0, Honest Verdicts, Earned Flags and a Search That Surfaces

This release is about whether you can trust what the system tells you, on three fronts that turned out to be the same front. A search verdict has to be earned before it reads good. A dark flag has to be measured before it graduates. And a search that should surface many spec folders has to actually surface them instead of quietly returning one. Each of those was held to the same rule, that nothing ships on faith, and each was caught at least once by something adversarial reading the real path rather than trusting a green test.

The work falls into three movements. First the honesty of the verdict and the cleanliness of the records the framework keeps about each spec folder. Then the live surfacing fix, where a real user query that should have matched many folders returned only one, traced to the surfacing and budget paths and fixed so the same query now returns six. Last the reckoning where eight dark-flag families faced a benchmark and the four winners earned a production-readiness pass, with the bitemporal edge history caught corrupting live reads before it could ship and then repaired.

The discipline did not change across any of it. A flag earns its place on real measurement or it stays off, a structural protection that only ever makes the system safer ships on for everyone, and nothing here flips a default on the path you use every day except where a correctness fix demands it. The dark flags now carry verdicts instead of question marks, the records are type-safe and reproducible, and a search that under-surfaced now degrades overflow into compact rows instead of deleting it.

## Search Intelligence

This release is mostly about one thing: a search verdict should be earned, it should tell you the truth about its own confidence, and a search that finds many folders should surface them instead of throwing the surplus away.

#### A Result Must Earn Its Verdict

A high similarity score is no longer enough to call a result good. The lexical-grounding floor requires a real keyword signal behind the top hit (a term match in the title or the trigger phrases or the body, or a BM25 keyword score already on the row) before it can reach a good verdict. So a fluent but absent term like "kubernetes," which once earned a confident citation from a single spurious high-cosine hit, now correctly reads weak. The floor shipped quietly earlier and graduated to on for everyone this cycle after the benchmark earned it.

&nbsp;

#### The Verdict Reflects True Signal, Not Background Hum

Every relevance score now sheds a measured corpus noise floor (an embedder-specific baseline, keyed to the nomic-embed-text-v1.5 model) before the system decides whether a match is real. A score that sits genuinely above the floor reads as a real match, and one that sinks into the background is handled honestly, so the verdict band reflects true signal rather than the embedder handing fluent but unrelated text a high background cosine. Graduated to on this cycle.

&nbsp;

#### Thin Evidence Is Capped, and the Warning Agrees

When the evidence behind a result is thin, search caps a good verdict down to weak so a confident answer cannot ride out on a gap, and the evidence-gap warning that sits beside the verdict now always agrees with it. This had been broken twice over. The cap was wired to a signal nothing ever set, so it never fired and a known gap could surface beside a confident verdict. And the warning itself was driven by a peakedness statistic (how far the top result stood above the pack) rather than by relevance, which got it backwards: a strong, tightly clustered set of good matches looked suspicious and got the warning, while a flat spread of weak off-topic matches slipped through clean. Both are fixed. The cap is connected, and the warning now keys off the same noise-floor-subtracted relevance the verdict uses, so a search for a real term like "graph" stays quiet while a genuine gap is flagged once, consistently. A benchmark of the old warning found it raised a false alarm on two of every three strong queries, the redesign brought that to zero.

&nbsp;

#### One Honest Score, Everywhere

A semantic result reports the same number the system ranks on. The score a caller sees now resolves through the calibrated absolute relevance (a 0-to-1 cosine scale) rather than the compressed fusion magnitude (RRF, which clusters low near 0.03), fixing a residual path that was still reporting the wrong figure for semantic rows. Separately, result lines from the relationship and degree channels that had rendered a blank dash now show their relevance number like any other row.

&nbsp;

#### A Cleaner Result Display

The result dashboard adds up. The header count now matches the rows actually shown rather than a pre-trim total, a long folder path renders as its leaf title instead of being cut off mid-string, and a status field that had reported an unrelated mechanism now reports the thing a reader expects, so a dense result set reads cleanly and the status you read is the status you get.

&nbsp;

#### Search Keeps Answering When the Embedder Is Down

A missing or empty embedding no longer collapses a search into nothing. When the embedding service is unavailable the pipeline falls back to keyword matching and carries an explicit flag that the dense channel was skipped, so an outage returns the best lexical answer rather than an empty result. The healthy path is unchanged.

&nbsp;

#### The Budget Trim Keeps the Best Result

Tail-appended rows are now exempt from the response-serialization token budget, so a backfill row added past the limit is not the first thing cut. The review found the sharp edge in that exemption: under the hardest squeeze the trim could drop every primary row before touching an exempt backfill, returning a response made entirely of backfill with the top-scored result evicted. The trim now reserves at least one primary row and keeps a pinned constitutional rule ahead of any additive backfill, so the squeeze can thin the result set without ever sacrificing the answer you asked for.

&nbsp;

#### A Guard Against Confident Hallucination

A build gate now refuses to pass when search returns good verdicts on terms it should know nothing about. Six bare off-corpus queries (kubernetes, oauth, kafka, terraform, graphql and webpack) run through the production verdict path, and the gate fails if too many false confirms slip through, with the ceiling set at zero. This is the regression net under the grounding and noise-floor work above.

&nbsp;

#### Queries Route by Shape

Every query is classified once by its retrieval shape before the pipeline runs. When shape routing is enabled a simple single-hop lookup skips the graph-relation expansion so it stays light and fast, while a multi-hop question keeps its graph-aware ranking, so the two diverge at the right seam without changing how you call search. The five-class classifier is on for everyone, but the shape routing that acts on it — the single-hop expansion skip — stays opt-in behind `SPECKIT_RETRIEVAL_CLASS_ROUTING` (default-off), as do the per-class weight profiles behind it.

&nbsp;

#### The Multi-Hop Appends Reach the Deep Reader

The deterministic multi-hop and lane-champion backfill appends were measured and graduated for deep-K readers. On the labelled multi-target queries, complete recall at twenty rose from just over half to almost all while the shallow ranks stayed flat, so a reader that asks for a wide result set now gets the related matches that the pipeline used to leave on the floor, and a reader that asks for the top few sees no change. The win is deep-K only and is recorded as such rather than dressed up as a blanket recall gain.

&nbsp;

#### A Density Probe for the Citation Ledger

The true-citation emitter stays at refine, sound but gated on real session-carrying traffic rather than more code. This release adds a density probe that surfaces through the health check and reports when the ledger has accumulated enough session-scoped pairs, with a per-class floor and a minority-ratio floor so a lopsided ledger cannot graduate the reranker on a count alone. The probe observes, it does not flip anything on.

&nbsp;

#### Citation Holds Across Models

A 144-cell benchmark drove `/memory:search` with four models (MiMo, Kimi, DeepSeek and gpt-5.5 fast) across twelve deliberately vague queries. The verdict held up where it should. On clearly-aligned and clearly-empty queries every model agreed, and the citation rule was obeyed wherever a model emitted a full verdict. The divergence is on the borderline queries, two of the twelve split the verdict because each model refined the vague search differently and landed on different results, not because the verdict logic wavered. The benchmark also isolated the off-corpus false-confidence that the grounding and noise-floor graduations then closed. DeepSeek came out the most efficient and stable driver, and Kimi over-explores expensively on the vaguest queries.

## Search Surfacing

The honesty work proved the verdict could be trusted. The surfacing work, added late in the cycle on 2026-06-26, proved the results actually arrive. A live `memory_search` that should have matched many spec folders was returning a single result, and the cause was not the ranking but two paths that quietly threw matches away.

#### A Query That Found Six and Showed One

A user query that should match many spec folders returned only one. Two faults compounded. Trigger-lane matches, the folders that match on their declared trigger phrases, were excluded from the surfaced results even when they ranked, so a whole lane of real matches never reached the reader. And the envelope token-budget enforcement deleted overflow rows outright instead of degrading them, so anything past the budget vanished rather than shrinking. The fix promotes trigger-lane matches into the surfaced results, holds a display floor of ten so a thin-looking page never collapses below a usable count and materializes budget overflow as compact rows at both the pipeline level and the envelope level, so a row past the budget renders as a compact line instead of being deleted. The folder-discovery listing also gained a z_future exclusion so future-staged folders stop diluting the live result set. With those in place the same live `memory_search` went from one result to six, one full row plus five compact rows.

&nbsp;

#### Filters That Stop Rejecting Valid Input

Two validation guards were over-eager. An empty-string filter was being rejected as malformed rather than read as a no-op, and the `retrievalLevel` parameter was being rejected outright, so a caller passing either could fail a search that should have run. Both guards now accept the input, so an empty-string filter and the `retrievalLevel` parameter pass through cleanly and the query runs as asked.

## Data Quality

The metadata the framework generates about each spec folder (the `description.json` and `graph-metadata.json` files that feed memory search and graph traversal) was hand-driftable and noisy. This release makes it type-safe, idempotent and reproducible, then migrates the whole tree onto the new rules, and a late-cycle pass cleaned up the lineage damage a bad backfill had left behind.

#### Type-Safe Records

A folder's generated status is now a closed set of values (planned, draft, complete and the rest) instead of free-running prose, and a first-class validator checks both generated files before a bad value can propagate. Legacy hand-edited statuses are re-derived into safe values and surfaced for review rather than left to leak through.

&nbsp;

#### Reads That Stay Clean

Reading spec metadata stopped rewriting it. The generator skips a write when the real content is unchanged (ignoring volatile timestamp fields), a targeted cache upsert updates one folder without rebuilding its neighbors, and access and freshness telemetry moved out of the generated JSON into an index-layer store so a read or a resume no longer dirties an unchanged file.

&nbsp;

#### Scoped Operations, No Side Effects

Refreshing one packet touches only that packet. The backfill tool now takes a folder argument and defaults to that single scope, so a one-folder intent no longer quietly dirties the whole tree, and a repo-wide run takes an explicit `--all`. Archived and future-staged folders are kept out of the descriptions cache by one shared policy, with archive memory behavior intact.

&nbsp;

#### Consistent Lineage and Synced Summaries

A folder's parent and children reconcile across partial rescans, so a valid parent relationship survives when one subscan misses it and child lists grow safely unless an operator runs an explicit prune. Separately, the description and causal-summary fields now derive from one shared synopsis rule so they stop drifting apart, and a drift checker re-derives them on read against a stored source-doc hash and reports staleness without writing anything. The generator also persists a source fingerprint that proves a field is fresh without re-reading the doc.

&nbsp;

#### Lineage Repaired After a Bad Backfill

A late-cycle cleanup undid the damage a wrong-basePath backfill had left in the records. Eleven `description.json` files had been regenerated with a polluted `parentChain`, traced to the backfill having run against a wrong base path, and were de-polluted back to a correct lineage. Four false claims that had crept into the prior remediation docs were corrected against the real state, and a phase parent that had lost four `children_ids` had them restored, so the tree reads its own lineage truthfully again.

&nbsp;

#### A Clean Migration and a Flag Reckoning

The whole tree was regenerated onto the new contracts, 2,049 spec folders including archives, with a byte-stable second run proving the output settles deterministically and zero validation violations afterward. A graduation benchmark then measured every default-off flag against the migrated tree and applied the verdicts: twelve flags earned their way to default-on or enforcing (the metadata integrity gates and the search-honesty suite above), one purely informational flag was deleted for earning no measured win, and the two that needed a freshness field the first migration had not written were graduated after a re-run wrote it.

## Skill Advisor

The advisor that routes a request to the right skill learned to tell a lane that genuinely found nothing apart from a lane that could not run at all, learned to distrust its own cached judgment when the embedding model underneath it has changed, and graduated its ranking spine while closing the self-recommendation hole the graduation exposed.

#### A Down Lane Is Not an Empty Lane

When a scoring lane is runtime-degraded (for example the graph is unavailable) the advisor no longer counts that against a skill the way it counts a lane that ran and honestly found no match. Degraded lanes drop out of the confidence denominator so a skill is not punished for an outage it had nothing to do with, while a lane that ran and matched nothing still counts so non-matching skills are not over-credited. The advisor output and its abstention explanations now name the degraded state plainly instead of hiding it inside a low score.

&nbsp;

#### Stale Embeddings Fail Loud, Not Silent

The advisor records which vector model wrote its semantic projection at write time and compares that against the active embedder on every read. If the two disagree the semantic shadow lane is marked degraded (and then follows the same lane-health rule above) rather than quietly trusting a projection built by a different model. Legacy projections that carry no model signature now fail closed instead of being assumed safe, so a model swap can no longer leave the advisor reading from a stale map without anyone noticing.

&nbsp;

#### RRF Fusion Graduates With Its Safety Net

The reciprocal-rank-fusion spine was measured on a widened forty-two-prompt set and graduated, lifting top-1 by one with zero regressions and deterministic, near-total agreement. It ships paired with the conflict-rerank seam because the seam is RRF's own regression net, repairing a case that plain fusion would otherwise break. The separate self-recommendation guard was cut in the same pass, measured as behaviorally redundant because the generic explainer floor and an un-flagged penalty already did its work.

&nbsp;

#### The Self-Recommendation Penalty, Documented and Widened

Cutting that guard left an implicit penalty as the only thing stopping the advisor from recommending itself on an audit or explainer prompt, and that penalty was undocumented, so a future refactor could have removed it without anyone noticing. It now carries a durable explanation and a contract test, validated two ways so the lock is real. The review then found the penalty matched only the advisor's exact id and missed its alias, which still recommended itself to the top in the default configuration. The predicate now uses the canonical id set, so both the exact name and the alias are demoted, and a pre-existing test that had encoded the alias bug was corrected to the right behavior. The penalty value itself is unchanged.

## Code Graph

The structural index that powers code search and impact queries got the same determinism and resilience treatment as memory search, and then it earned the headline of the flag reckoning. A dark experiment to give every edge a readable history was wired up, caught broken and rebuilt, and the staleness repair that bounds re-parsing got a real number behind its default.

#### Transient Parser Failures Get a Second Chance

A parser hiccup no longer condemns a file to permanent invisibility. Transient parse failures now retry up to a configurable ceiling (`SPECKIT_PARSER_SKIP_LIST_MAX_RETRIES`, default 5) before they are treated as fatal, and a successful reparse clears the skip-list entry entirely. Genuinely fatal entries (the ones that need a human to look) keep skipping immediately as before, so the retry budget is spent only where it can actually help.

&nbsp;

#### Stable Answers, Every Time

Equal-trust context edges (the relationships the graph walks when it answers a query) now sort by stable content-derived keys (the related content hash, the symbol identity, the file path, the edge type and the endpoints) instead of by whatever order the database happened to return. So two identical queries against the same data come back in the same order, even across separate scan rebuilds, and a diff of two runs no longer shows phantom reordering.

&nbsp;

#### The Doc Lane Sees Structure

The documentation lane now indexes the structure inside your files as real graph symbols. Markdown headings become `heading` nodes and config keys become `key` nodes (for JSON, JSONC, YAML and TOML), all generated from local parsing so nothing leaves your machine. Config files exercise the lane by default and markdown extraction is opt-in through globs, so you can point it at the docs that matter without indexing every stray note.

&nbsp;

#### A Generation Watermark You Can Read

Every promoted scan now advances a monotonic generation counter (a number that only ever goes up, stored in the graph metadata and surfaced on freshness envelopes as `metadata.freshness.generation`). A failed promotion or a no-op scan leaves it untouched, so the number you read is an honest stamp of how current the index is. Reads still serve from whatever generation is present (the hard gate that would reject a stale-generation read stays deferred for a later cycle).

&nbsp;

#### The Staleness Cap, Chosen by a Cost Sweep

The reverse-dependency degree cap decides how many importers get re-bound when a high-fan-in dependency is renamed. Before this release it defaulted to uncapped, so renaming a thirty-importer module re-parsed all thirty. The cap now defaults to fifteen, and that number is not a guess. A degree-sweep benchmark drove the real scan handler across importer counts from five to twenty-five and found the re-bind cost is linear with no knee, about a third of a millisecond to under a millisecond per importer over a seventeen-millisecond scan floor. Fifteen fully repairs the common-to-moderate refactor at negligible cost while bounding the unmeasured hundred-plus-importer tail. The honest cost is recorded in the docs: above the cap, a high-fan-in rename leaves the overflow importer edges stale until each one is next edited. The cap is read only inside the force-parse branch, which is off by default, so the value is inert until you opt in.

&nbsp;

#### History You Can Query

A preserved history is worthless if nothing can read it. The first wiring left the as-of reader with no caller, so the review held it as honestly out of scope rather than letting it claim a feature that did not exist. This release wires it through: code graph queries take an optional as-of generation parameter that routes edge reads through the temporal readers and a new inbound mirror, deliberately skipping the dangling-edge exclusion because a historical edge legitimately points at a node that has since been deleted. Ask for a past generation and you get the old target, ask without the parameter and you get the current graph exactly as before. The whole surface stays byte-identical when the flag is off.

&nbsp;

#### The Bitemporal Writer, Wired and Then Caught

The headline is a save from a silent corruption. A pre-graduation review had found the bitemporal close-and-insert writer was built but never wired into the reindex path, so it was held at refine. The production-readiness pass wired it in, and the test suite went green. A ten-iteration deep review on opus then read the code and failed it on three blockers, because the tests were green only for cases that never touched the real failure modes. Before the fix, turning the flag on would have corrupted every live read. The live edge readers had no filter for closed edges, so after a reindex they returned both the superseded edge and its replacement. The production prune that runs at the end of a full scan deleted dangling edges regardless of the flag, and node replacement deleted a symbol's edges before the close path could run, so there was nothing left to close and an as-of query came back empty. After the fix, node replacement and the dangling prune close instead of delete when the flag is on, the live readers return only the current edge and a generation-stamp correction gives every superseded edge a real readable window instead of a zero-width one. A second deep review confirmed the repair with no blockers remaining.

&nbsp;

#### The Algorithm Gets Its Gradient, and Loses Anyway

The seeded PageRank's tie had shipped with a specific caveat attached, that a uniform 0.8/INFERRED/heuristic confidence on every CALLS edge gave the algorithm nothing to differentiate on, so the tie read as a verdict on the substrate rather than a refute of PageRank itself. This cycle built that exact prerequisite. A new **per-edge confidence differentiation** for CALLS edges lands behind `SPECKIT_CODE_GRAPH_EDGE_CONFIDENCE_DIFFERENTIATION` (default off), reusing a resolution-quality classification `cross-file-edge-resolver.ts` had already computed and been discarding, so a cross-file resolved edge now reads 0.9/EXTRACTED, a cross-file ambiguous edge reads 0.3/AMBIGUOUS, a same-file single-candidate edge reads 0.75/INFERRED and a same-file multi-candidate edge reads 0.35/AMBIGUOUS, while the flag stays off by default and the existing code-graph vitest suite passes identically with it unset. The deleted `computeBoundedPersonalizedPageRank` module (introduced at `657a0f6a3e`, deleted at `277c35344c`) was recovered byte-for-byte from git history and rewired to consume that gradient through the existing `contextEdgeReliability` blend, and the recovery caught its own near-miss along the way: the first pass swapped the module's dynamic import of the Memory MCP's compiled `collectWeightedWalk` for a local reimplementation of the same walker, because the compiled dist output was not present in the isolation worktree, a substitution that would have stood up a second graph-walk engine against an existing decision that had already ruled one out. Diffing the recovered file's imports against the pre-deletion original surfaced the swap before it shipped, and building the missing compiled output and restoring the real shared-substrate import closed it out.

A fresh full-repo reindex with the flag on landed four distinct confidence values in place of the old uniform 0.8 (892 edges at 0.3, 2,267 at 0.35, 16,198 at 0.75 and 2,838 at 0.9), and the original, unmodified benchmark harness was rerun against the same twenty labeled queries, the same metrics and the same damping sweep from 0.5 to 0.95 with both flags on. The substrate objection no longer holds: given a real gradient to differentiate on, PageRank no longer ties the flat walk, it loses on every metric (precision@3 down 0.10, precision@5 down 0.04 to 0.06, precision@8 down 0.031 to 0.038, recall@3 through recall@8 down 0.01 to 0.05, nDCG@3 down 0.057, nDCG@5 down 0.04, nDCG@8 down 0.03), and the best damping value in the sweep (0.5) manages only to tie flat nDCG@5 while every other value tested is worse. Both the differentiation flag and the recovered PageRank flag stay default-off, no production behavior changed, and no further seeded-PageRank revisit is planned.

## Deep Loop

The deep-research and deep-review loops (the multi-iteration `/deep:*` workflows that fan work out across parallel model workers) got a wave of determinism, recovery and observability fixes, and then the finding dedup graduated and learned to read titles while the lag gauge stopped crying wolf.

#### Fresh Runs Reduce Cleanly

A brand-new deep-research run no longer crashes on its first pass. The strategy template now ships with the reducer anchors (the machine-owned points where the run writes its carried-forward state) already in place, so the first reduce has somewhere to land and the run can start threading state forward immediately instead of failing before it begins.

&nbsp;

#### Fan-Out Merges Are Reproducible

A fan-out (work split across parallel model workers) now produces the same merged result no matter which worker finishes first. The research and review registries sort lineage labels and worker-identity metadata into a stable order before merging, so the combined output is decided by content rather than by a race, and two runs over the same inputs agree.

&nbsp;

#### A STOP Has to Be Corroborated

A deep-research STOP decision now needs a second source of truth when a run claims it has run out of new ground. Convergence compares the run's self-reported novelty against graph-observed novelty (the actual changes to stored questions, findings and evidence) and blocks a STOP that the graph cannot back up, so a loop cannot talk itself into stopping early on a claim nothing else supports.

&nbsp;

#### Divergent Findings Survive the Merge

When two workers return findings that share an ID but disagree on content, the merge now keeps both side by side and marks them as conflicting variants instead of letting whichever arrived first silently erase the other. A real disagreement between workers stays visible for a human to resolve rather than vanishing into a merge.

&nbsp;

#### Recovery That Stays Within Budget

Fan-out failures are now sorted into fixed classes (timeout, exit and salvage miss, which means a recoverable output that went missing). Only genuinely transient failures retry, and the retry counts are read from durable run logs, so a crash mid-run cannot quietly reset the retry budget and send a worker into an endless loop.

&nbsp;

#### Recovery Refuses to Fake a Clean Slate

Recovery mode no longer pretends a missing or corrupt state file is just a fresh new run. It refuses to proceed on broken state, and any orphaned lineages (workers that started but never reported back) are marked explicitly in the ledger, so partial state is visible as partial state instead of being papered over as success.

&nbsp;

#### Live Gauges and Stopped Summaries

A running fan-out now reports live pool gauges (pending, failed and lag counters) read from real run state rather than guessed, so you can see where a slow run actually is. And a run that gets interrupted now leaves a partial summary behind, so the work that did finish is captured instead of lost when the run stops early.

&nbsp;

#### Open Questions Thread Forward

Deep-research now carries its unresolved questions into the next iteration without spending an extra model call to do it. The reducer builds a continuity thread (the open questions carried forward from the last pass) and derives the next focus from that thread or the latest finding, so a loop keeps its train of thought across iterations for free.

&nbsp;

#### Dedup That Respects the Title

The near-duplicate finding collapse was graduated on a clean benchmark, then scale-validated past its original handful of synthetic records on a larger free-text set. That scale test surfaced the exact risk the earlier validation had flagged: the dedup key ignored the finding title, so two genuinely-distinct findings that shared a body but differed only in their title collapsed into one, a measured one-in-two false collapse on that shape. The key now applies a title-overlap gate, so disjoint titles stay separate while a true restatement still merges. The title-only false collapse dropped from one-in-two to zero, while exact duplicates still collapse and distinct recall holds.

&nbsp;

#### A Lag Gauge That Means It

The lag-ceiling gauge graduated alongside the dedup, but its metric was measuring the wrong thing. It read time since a worker was queued, which is queue back-pressure, so any healthy fan-out wider than its concurrency tripped the alarm. The metric is now time since the last completion while work is still pending, a real stalled-worker signal. A healthy busy pool reads quiet, a genuinely hung one fires once and the recommended ceiling dropped from five minutes to two now that it no longer drifts on healthy load. The committed default stays off because turning the gauge on is the separate graduation step.

## The Flag Reckoning

This cycle the benchmark decided the fate of each flag rather than a hunch, twice over. The first reckoning measured the migrated-tree flags and the search-honesty suite, the second measured eight dark-flag families against the real graph and corpus on their production paths.

#### Graduated to Live

The search honesty suite (the lexical-grounding floor, the noise-floor subtraction and the evidence-gap cap), the relevance-aware gap warning, the absolute-relevance calibration that reports one honest score, the metadata integrity gates and the false-confirm build gate all measured well enough on the real corpus to become the default. The multi-hop tail appends graduated for deep-K readers, the code-graph staleness degree cap graduated with a benchmarked default, the bitemporal edge writer graduated after the review caught and the repair closed its corruption, and the advisor RRF fusion spine graduated paired with its conflict-rerank seam. The deep-loop near-duplicate dedup graduated with its lag and heartbeat gauges. All the dark-flag graduates ship behind their existing default-off flags, byte-identical when off.

&nbsp;

#### Kept Quietly On

Three flags ship on without a headline because they make the records safer rather than the verdicts smarter. Derived-id provenance gives every causal edge a content-addressed ID (an identifier computed from the edge's own content, so the same edge always gets the same name), retention-forgetting is a deletion safety guardrail that keeps a cleanup from removing more than it should, and the world-summary grounding prelude seeds a search with a short factual frame before it runs. None of them change a number you read, so they sit on quietly.

&nbsp;

#### Held at Refine

## The Review That Caught a Masked Corruption

This release is the clearest case yet for the project's shipping rule, that a result-affecting change is not done until something adversarial has read it. The production-readiness pass shipped with a green test suite. A cli executor ran the tests and they passed. By the old standard the bitemporal wiring was ready to graduate.

A ten-iteration deep review on opus, reading the code under rotating lenses rather than running the existing tests, returned a FAIL with three must-fix blockers, eight should-fix findings and four advisories. The blockers were not subtle once seen: the live readers returned closed edges, the production prune deleted history the feature existed to keep, and node replacement deleted the edges before the close path could run. The tests were green because every flag-on assertion went through the one reader that happened to filter correctly, while the readers production actually calls were only ever checked with the flag off. The gap between green and correct was exactly the gap the review was there to close. Each blocker was fixed, a second review on opus confirmed the repair at zero blockers, and a final pass closed every remaining recommendation and follow-up at zero as well.

## Upgrade Notes

No migration is required and nothing changed on the common path. The honest-verdict suite (the lexical-grounding floor, the noise-floor subtraction and the evidence-gap cap) and the relevance-aware gap warning are on for everyone and need no action. The determinism flag was removed, so if you had set `SPECKIT_DETERMINISTIC_RANKING` it no longer does anything and the system ranks by recency as it did by default. The spec-metadata integrity gates that graduated are on by default, the cite-with-caveat tier and the envelope-fidelity guard are on by default alongside the rest of the honesty suite, and the type-safe status validator runs in a non-blocking report mode during the migration window before it enforces.

Every dark-flag graduate ships behind its existing default-off flag and is byte-identical when off, so the production flip remains a separate, evidence-gated decision. If you opt into the bitemporal edge reads, the recommended degree cap is fifteen, the as-of query parameter is available on code graph queries, and the live readers return the current edge. If you opt into the deep-loop gauges, the recommended lag ceiling is two minutes against the new stall metric and the recommended heartbeat is thirty seconds. The advisor self-recommendation penalty change is the one deliberate behavior shift that is not flag-gated, because demoting the advisor's own alias on an audit prompt is a correctness fix that the un-aliased path already applied. The search-surfacing fix is a correctness change on the live read path, so a query that under-surfaced before now returns its full match set with overflow degraded into compact rows.

## Verification

This release follows the same shipping rule throughout. A structural protection that only ever makes the system safer or cleaner ships on for everyone, and anything that changes a result ships off by default behind a named flag until a benchmark earns it on. Here that rule was enforced by two graduation benchmarks and then by independent deep reviews on opus.

#### Earn or Cut, Measured

The honest-verdict suite, the relevance-aware gap, the score-scale correction and the type-safe metadata validator are live for everyone. The eight dark-flag families were decided by real measurement, not by argument. Four graduated on a measured win, the cut flags went back on a measured loss or a measured tie, and the true-citation ledger was held on a clear-eyed read that the signal is sound but the data is not there yet. The multi-hop recall gain, the RRF top-1 lift and the degree-cap cost curve were each read off the real path.

&nbsp;

#### The Review That Earned Its Keep

The bitemporal corruption is the proof that a green test is not a finished feature. The cli test pass was honest about what it ran and wrong about what that meant, because it never exercised the readers production calls. A deep review reading the code found the gap, the fix closed it, a second review confirmed the fix and a third confirmed that every remaining recommendation and follow-up was closed. The two residual advisories that survived are honest about their scope: they touch only the as-of read surface under the flag and never a live read. The follow-up twenty-iteration review then surfaced a fresh backlog and the P1 set was remediated area by area, with the degraded embedding provider and the deferred reconciliation preventive recorded as honest carry-overs rather than hidden.

&nbsp;

#### The Live Check That Caught a Masked Bug

Two live dispatches caught what green benchmarks had missed. The evidence-gap redesign passed its first benchmark green, but a full-handler live dispatch showed the warning firing on every single search, because the benchmark fed clean absolute-relevance scores while production fed a compressed one. The fix threaded the production-correct signal in and a second dispatch confirmed it. Later in the cycle a live `memory_search` returned one result where it should have returned many, and reading the real path showed trigger-lane matches excluded and budget overflow deleted. The surfacing fix promoted the trigger lane, held a display floor of ten and materialized overflow into compact rows, and the same live query then returned six, one full row plus five compact.

&nbsp;

#### Byte-Identical When Off, Proven

Every flag-gated change in this release was proven byte-identical with its flag off, by dedicated off-path tests rather than by assertion. The bitemporal readers run their original queries, node replacement and the prune delete as before, the search appends never mark a row, the lag gauge emits no field and the dedup keeps both distinct findings. The system you run today behaves exactly as it did, until you decide to turn one of these verdicts on.
