## 1. SUMMARY

### Technical Context

| Aspect | Value |
|--------|-------|
| **Language/Stack** | Node.js (CommonJS fixtures + JSON), Vitest (TypeScript) |
| **Consumes** | `code-task-scorer.cjs` (oracle contract), `profile-validator.cjs` (profile shape) |
| **Storage** | JSON fixture + profile assets under `model-benchmark/` |
| **Testing** | Vitest (`model-benchmark/tests/sweep-isolation.vitest.ts`) |

### Overview
Oracle-correctness-first. For each validator, write a reference implementation (the strict, fully-correct rule set) and a deliberately-lax implementation (skips exactly one real rule the invalid set exercises). Generate every oracle case's expected boolean by RUNNING the reference impl over the spec-listed valid/invalid input lists — never hand-author the booleans. Then score BOTH impls through the real `code-task-scorer.cjs`; require reference = 1.0 and lax strictly < 1.0 before writing the fixture JSON. This guarantees the baked expected-values are correct by construction and that the fixture actually discriminates.

<!-- ANCHOR:architecture -->
## 3. ARCHITECTURE

### Pattern
Additive fixture/profile assets mirroring the existing hard/harder fixture pack, consumed unchanged by the sweep scorer and profile validator.

### Key Components
- **Fixture JSON** (`validate-ipv4|date|semver.json`): `{id, category, tier, fn_name, signature, task, visibleSpec, scope, tests[], hidden_tests[], expectedDifficulty, saturation}`; each oracle case `{name, args, expect}` with boolean `expect`.
- **v3 profile** (`capability-m3-vs-mimo-v3.json`): `model-vs-model`, 4 fixtures, n=5, two models, correctness gate 1.0, groupBy model.
- **vitest** (`sweep-isolation.vitest.ts`): additive describe blocks asserting shape, case count, invalid-dominance, and profile validity.

### Data Flow
1. Reference impl generates each oracle `expect` from the spec input lists.
2. `scoreCodeTask` extracts the function and runs every case in an isolated child against `expect`.
3. Reference scores 1.0; lax (one rule dropped) scores <1.0 → fixture is written.
4. `profile-validator.cjs` validates the v3 profile shape.
5. Vitest re-reads the on-disk fixtures/profile and asserts the contract.
