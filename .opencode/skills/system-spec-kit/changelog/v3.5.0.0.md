# v3.5.0.0, Memory That Maintains Itself

The previous release was about honesty and hardening. This one is about the memory and MCP substrate no longer needing a babysitter. Checkpoints become real database files that survive a crash. The memory index learns to reconcile and heal itself instead of drifting from the filesystem. Many sessions now share a single memory daemon, and a front-proxy hides a daemon recycle from every client so a restart stops surfacing as an error. And memory gets smarter on its own: after a save, enrichment, reconsolidation and causal relation inference now run in the background by default.

Alongside the runtime work, the deep-loop engine learned to fan out, the skill toolkit became forkable, each session can run isolated in its own worktree, and the long-running 026 program documented every one of its roughly 634 phases into one canonical changelog tree before closing itself out. A cleanup pass also removed the vendored research repos that never belonged in the shipping tree, about 2.1 million lines of code that was never ours to ship.

## Durable, Self-Maintaining Memory

#### Checkpoints that survive a crash

The old checkpoint path took an in-process snapshot, and a restore that failed partway could leave the live database half-applied. Checkpoints are now real files. Create writes a complete database file with `VACUUM INTO`. Restore swaps a finished file in rather than mutating the live one in place. A durable journal records the swap so a crash mid-restore is recoverable, and a `.needs-rebuild` sentinel forces a clean rebuild on the next boot if a restore cannot finish. The schema moved to v30 to carry a replay-enrichment marker with repair-on-replay.

#### The memory index maintains itself

`memory_index_scan` graduated from a manual trigger to a surface that keeps itself honest. Redundant concurrent scans collapse into one. Orphans get swept. A relocated spec folder is re-pointed instead of orphaned. A long scan runs async behind a drain circuit guard so a failing provider cannot wedge the queue, and the index now reports its state through `memory_health`. An active-row uniqueness guard and multi-tenant scope isolation close the door on duplicate rows and cross-scope bleed.

## One Shared Daemon, Debugged Hard

A large part of this release was spent in the launcher and daemon lifecycle, because that surface was behind most of the prior release's friction and a hard-to-pin disconnect.

#### Many sessions, one daemon

Multiple sessions now share a single `mk-spec-memory` daemon instead of each spawning its own and contending on the database. A lease-holder launcher owns the daemon child, and other launchers bridge their own client to it through a session proxy. The owner stores its real IPC socket path in the lease, so a secondary launcher pointed at a different socket directory still finds the live daemon instead of failing with no bridge socket.

#### A front-proxy that hides a recycle

A front-proxy sits between clients and the daemon. When the daemon recycles, a connected client no longer sees a hard error. The launcher reconnects, re-handshakes on a protocol-version drift and keeps each connected client independently transparent. It is also hardened against an externally killed or slow-booting daemon.

#### The debugging that backed it

A boot-time full-text-search auto-heal plus a launcher clean-close barrier fixed the daemon-lifecycle root cause behind the recurring corruption-looking failures. The launcher-lease integration suite, which had been failing every run because the fixture copied the launcher but not its library tree, so every spawned launcher died before it could write a lease, was repaired and un-skipped. It now passes nine tests across three runs with no temp-directory orphans.

## Memory That Gets Smarter on Its Own

#### Enrichment runs in the background

Post-insert enrichment, reconsolidation and quality auto-fix are now on by default, and enrichment runs async. A save returns immediately with `enrichmentStatus: deferred` while a bounded background scheduler enriches graph and entity data after the commit. Reconsolidation stays opt-in. A `SPECKIT_POST_INSERT_ENRICHMENT_SYNC` escape hatch restores the old synchronous path when you need it.

#### Causal relation inference

Spec-document chains and lineage links are now promoted into typed `created_by='auto'` causal edges. The backfill is bounded, dry-run by default and reversible. Two opt-in collectors add a similarity `supports` signal and a structural `contradicts` signal. A conflict guard refuses to invalidate an existing valid edge, which closes a case where a committed `contradicts` run over a reciprocal lineage pair could silently undo the valid `caused` edge it had just created. The production backfill raised relation coverage from 39.91% to 43.59% and `caused` edges from 3 to 103, with zero conflicting edges skipped.

#### Local-first embeddings

The embedding stack runs locally first and only reaches the cloud when you ask it to. Resolution is your explicit `EMBEDDINGS_PROVIDER` when set, then a persisted Ollama embedder, then a Python-free local fallback server, and cloud providers like OpenAI or Voyage are never auto-selected. This release closed the last place that rule was not honored: the primary resolver used to pick a cloud provider when Ollama was down and an API key happened to be set, the opposite of the intended priority. The local foundation underneath this, the Python-free local server and a consolidation onto a single nomic model, matured in the prior release. This release made the resolver match the design.

## The Deep-Loop Runtime Learns to Fan Out

The engine behind deep-research and deep-review gained an opt-in fan-out layer. One loop can now drive several executor lineages, each an isolated sub-packet with its own state directory, behind a concurrency-capped pool with a status ledger. A consumer-specific merge folds the lineages back together: research dedups and attributes findings, review rolls up severity and fails closed if any lineage reports a P0. The single-executor sequential path stays the default and is byte-identical to before. In this release the fan-out runs under the cap but still issues its lineages serially. The genuine-concurrency remediation was specced here and lands in the next release.

Two parsers behind the loops also got real fixes. The review transcript parser was rewritten so a model echoing the prompt no longer concatenates fenced blocks into invalid JSON and zeroes out the findings, and the convergence snapshot writer now refuses to coalesce every snapshot to one id and silently overwrite history.

## A Forkable Toolkit

A run of skilled-agent packets reshaped the skills themselves.

#### Three-layer prompt knowledge

The prompt toolkit was re-architected into three layers: framework craft in `sk-prompt`, per-model craft in `sk-prompt-small-model` and executor mechanics in the `cli-*` skills. The model registry and benchmarks moved into `sk-prompt-small-model` so the generic `sk-prompt` skill can be lifted out of this repo without dragging the small-model registry with it. The seam between the hub and every CLI skill was de-drifted, and every skill's playbook got global sequential numbering.

#### A reusable model-benchmark framework

The one-off prompt-framework rigs were generalized into one config-driven, model-agnostic benchmark framework. A single profile runs a framework bake-off or a model-versus-model comparison with no mode branches, correctness is a gate so a saturated score cannot crown a winner, and the verdict is WINNER, TIE or INCONCLUSIVE against a noise floor. The deep-improvement skill also grew a skill-benchmark lane that measures how a real agent discovers, routes to and uses a skill in situ.

#### More skills, sharper routing

A new `mcp-click-up` skill routes task work to a CLI for daily operations and the official ClickUp MCP for advanced ones. `sk-code` gained surface-aware routing plus a set of animation and frame-verification principles mined from a web-motion skill. `sk-git` gained a numbered `wt/{NNNN}-{name}` worktree convention with advisor routing. New model providers were wired in, with MiniMax defaulting to its Token Plan and the Xiaomi MiMo-V2.5-Pro added as a selectable executor. Feature-catalog standards were defined once and applied retroactively across roughly 370 catalog files.

## Worktree-Per-Session Isolation

The multi-session contention that caused much of the prior release's daemon friction now has a real prevention. Each session can run in its own worktree, reaped on close. The isolation core landed first, then the guard wired into the Claude session-start chain, and child sessions launched through the CLI dispatchers correctly share the parent's worktree.

## The 026 Program Closed Itself Out

The 026 graph-and-context-optimization program produced roughly 634 phase packets across 8 tracks, and their changelogs had scattered and drifted. This release made the history canonical.

#### One canonical changelog tree

Every live changelog was centralized into one tree with a per-program index and a regenerated timeline that links each spec folder to its changelog. Legacy-format entries were canonicalized with a fidelity guard. Stale spec-folder paths were repaired deterministically, dangling links and symlinks were cleaned up, and phase-parent rollups were added. An honest record was kept for removed work: the rerank-sidecar changelogs carry a later-removed note, because the sidecar runtime had already been deleted in earlier cleanup.

#### Structural reorg

Eight sibling packets were renested into themed 026 tracks with their path references rewritten, the 027 phases were renumbered so cleanup is phase zero, and several packets were relocated into the program proper.

#### The big cleanup

A reorg baseline removed the vendored `z_future/hybrid-rag-fusion-upgrade` research repos, which is where most of the 2.1 million deleted lines came from. This is housekeeping, not feature removal. The MCP server version was realigned to its package version, benchmark fixture lockfiles stopped being tracked to clear a dependabot alert, and the runtime configs had their MCP environment notes organized.
