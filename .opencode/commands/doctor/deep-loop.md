# SpecKit Doctor - Deep Loop Coverage Graph

Diagnose and rebuild `mcp_server/database/deep-loop-graph.sqlite`, the research/review coverage graph used by deep-research and deep-review loops for convergence signals, gap detection, provenance, contradictions, and coverage momentum.

```yaml
role: Expert Operator running deep-loop coverage graph diagnostics + rebuild
purpose: Detect empty/stale coverage graphs and rebuild nodes/edges from iteration folders
action: Discovery -> analysis -> (run: snapshot -> upsert graph rows -> verify convergence -> rollback-on-regression)
```

## 2. PURPOSE

Restore the deep-loop coverage graph to match current research and review iteration artifacts. The graph can drift when deep-loop runs predate graph events, when iteration folders move during packet consolidation, or when graph writes were skipped because reducers produced no `graphEvents`.

This command exposes drift through the interactive workflow and reports the actions needed for rebuilding from the packet-local iteration set. It uses `deep_loop_graph_upsert` for node/edge writes and validates that `deep_loop_graph_convergence` returns a non-empty signal for any packet with at least three iterations.

## STALENESS SIGNALS

The diagnostic phase detects:

1. **Empty graph** - graph status reports no nodes while `.opencode/specs/**/{research,review}/iterations/*.md` exists
2. **Stale graph** - graph database last write is older than 7 days or older than the newest iteration file
3. **Orphan nodes** - questions without claims, claims without evidence, findings without dimensions, or files without findings
4. **Missing convergence signal** - active packets with at least 3 iterations return empty convergence signals
5. **Coverage gaps** - `deep_loop_graph_query` returns uncovered questions or unverified claims for active packets
6. **Namespace mismatch** - research/review session IDs do not align with discovered iteration folders

Phase 3 first activity (`validate_targets_and_policy`) runs canonical-path validation before any write: resolved targets via `realpath`, exact-match against allowed_targets, no `..` escapes, no symlinks outside repo root, and no forbidden operation strings. Iteration markdown files are read-only inputs, never mutation targets.

## WORKFLOW PHASES (delegated to YAML)

```
Phase 0 - Discovery
  Purpose: detect empty graph, stale graph, iteration folders, and sampled status
  Tools: Glob, Bash(stat), deep_loop_graph_status, deep_loop_graph_query
  Lazy-init detection: if graph is empty and iteration folders exist, mark init_from_iteration_folders_available=true
  Output: { graph_status, iteration_inventory, empty_graph, stale_graph, orphan_signals[] }

Phase 1 - Analysis
  Purpose: map iteration files to packet/session namespaces and propose rebuild plan
  Tools: Read, Grep, deep_loop_graph_query, deep_loop_graph_convergence
  Output: { rebuild_plan, candidate_packets[], convergence_baseline, recommended_action }

Phase 2 - Snapshot
  Purpose: VACUUM INTO snapshot of deep-loop-graph.sqlite before any graph write
  Tools: Bash(sqlite3 ... 'VACUUM INTO ...'), disk-free preflight
  Output: { snapshot_path, snapshot_bytes }
  Halt-on: disk-free < 2x current DB size

Phase 3 - run
  First activity: validate_targets_and_policy
  Then: deep_loop_graph_upsert() per iteration node + edge from the source packet's iteration files
  Output: { packets_processed, nodes_upserted, edges_upserted, skipped }

Phase 4 - Post-verify
  Purpose: gold-battery convergence check
  Tools: deep_loop_graph_status, deep_loop_graph_query, deep_loop_graph_convergence
  Gold-battery: convergence signal must be non-empty for every rebuilt packet with >= 3 iterations
  Output: { gold_battery_pass, convergence_results, regressions[] }

Phase 5 - Rollback
  Purpose: restore snapshot on interactive regression, or prompt in interactive workflow
  Tools: Bash(cp -p)
  Output: { rolled_back, restored_at, rollback_reason }

Phase 6 - Report
  Purpose: emit final status + state log
  Output: { status, state_log_path, recommendation }
```
