# sk-design v1.6.0.0 - Indexed Styles And Interface Commands

> Adds an opt-in indexed style-retrieval backend and makes `/interface:*` the canonical creation-command surface without cutting over the authoritative flat-file path or renaming internal modes.

---

## Added

- A persistent style database under `styles/_db/` using SQLite and FTS5, a rebuildable vector projection, an incremental `DISCOVER` through `PUBLISH` indexer and eligibility-first weighted-RRF retrieval.
- Five canonical creation commands: `/interface:design`, `/interface:foundations`, `/interface:motion`, `/interface:audit` and `/interface:design-reference`.
- One shared nine-stage creation contract under `shared/creation-contract.md` for the canonical command surface.

## Changed

- Style-library access now passes through a `legacy|shadow|persistent` adapter in `styles/_engine/`.
- The canonical public creation namespace is `/interface:*`; each command still resolves to one of the existing `interface`, `foundations`, `motion`, `audit` or `md-generator` mode IDs.

## Preserved

- The adapter defaults to `legacy`, and flat style files remain authoritative. Persistent retrieval requires explicit opt-in.
- `/design:interface`, `/design:foundations`, `/design:motion`, `/design:audit` and `/design:md-generator` remain thin compatibility aliases.
- The five internal workflow mode IDs are unchanged; the alias surface is additive.

## Verification

- The persistent database implementation passed 24/24 database tests, while the legacy engine passed 20/20 tests.
- The command surface checker reported five canonical commands, five compatibility aliases and zero drift.
- The AI fingerprint fixture, AI fingerprint registry and procedure-card schema checks remained passing.
