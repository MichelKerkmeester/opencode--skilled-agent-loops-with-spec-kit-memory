# v1.0.1.0 - Smart-Routing Conformance

**Released:** 2026-07-12

Structural normalization so `SKILL.md` satisfies the canonical section contract enforced by `create-skill/scripts/package_skill.py`. No workflow behavior changed; all prose was preserved.

## What Changed

- Split the merged `## 1. WHEN TO USE + SMART_ROUTING` header into two distinct H2 sections — `## 1. WHEN TO USE` and `## 2. SMART ROUTING` — so the routing section matches on the canonical `SMART ROUTING` token (the underscored form did not). The decision-rule block and the family-boundary note moved under SMART ROUTING as routing content.
- Added a `Keyword triggers:` line inside WHEN TO USE for discoverability.
- Renamed the resources section to `## 9. RESOURCES FOR DEEP DETAIL & REFERENCES` so it satisfies the required REFERENCES section match while keeping the original intent.
- Added a `## 8. SUCCESS CRITERIA` section describing done-conditions for a playbook package.
- Renumbered all H2 headings contiguously from 1 to 9. No intra-file numeric cross-references needed adjustment.

## Why

The section matcher strips a leading `N.` and substring-tests the uppercased H2 text. A heading containing `SMART_ROUTING` (underscore) does not contain the `SMART ROUTING` substring, and a heading with no `REFERENCES` token fails that requirement. Splitting the header and renaming the resources section makes both required sections resolve without altering the packet's guidance.
