# Iteration 5: Deep-Review Destination Map for Portable `deep-context` Capabilities

## Focus
This iteration mapped portable standalone `deep-context` capabilities into `deep-review` outputs and verdict/report surfaces. The selected interpretation was narrow: use existing `deep-review` command, YAML, agent, state, report, resource-map, severity/verdict, and coverage/gap contracts to decide where context snapshot fields belong, and explicitly mark fields that should not migrate into review. Deferred alternatives: whole-repo `deep-context` reference cleanup and implementation edits.
