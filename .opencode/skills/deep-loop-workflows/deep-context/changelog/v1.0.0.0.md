First release of `deep-context`, the fourth deep loop. It gathers the existing-codebase context relevant to a feature and synthesizes a reuse-first Context Report, the "understand" step that runs before planning and implementation. It is built as the third consumer of `deep-loop-runtime` and runs a heterogeneous pool of models in parallel over a shared scope, using cross-executor agreement as the confidence signal.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/134-deep-context-gathering/` (Level 3)

---

## Why This Release Matters

Before this release there was no convergence-gated way to map the existing code before planning a feature. `/speckit:plan` discovered context ad-hoc (four one-shot exploration agents) and `/speckit:implement` relied on whatever the plan captured, so reusable code was missed (violating the "reuse, avoid new code" principle), stale references and context bloat crept in, and there was no signal for "enough context gathered".

`deep-context` fills that gap. It seeds a frontier from the code graph, sweeps slices with an operator-composed pool of models in parallel (for example two native agents plus MiMo, gpt and deepseek at once), merges their findings by `file:symbol` with agreement-weighted confidence, and stops on relevance-gated coverage saturation. The output is a Context Report whose top section is a REUSE catalog of verified pointers, consumable directly by planning and implementation.

---

## New Features

#### The deep-context loop

New skill `deep-context`, command `/deep:context` (`:auto` and `:confirm`), and a read-only `@deep-context` LEAF analyzer agent. The loop is host-driven: the host owns the frontier, dispatch, merge, state, convergence and synthesis, while each executor seat is a read-only analyzer that returns structured findings and writes nothing.

&nbsp;

#### Heterogeneous parallel by-model sweep

Each iteration dispatches the whole executor pool over the same focus concurrently. Native seats run as a parallel Task batch and CLI seats run via the council `dispatchCouncilSeats` scaffold, started together and barrier-joined. Cross-executor agreement (a finding confirmed by two or more distinct models) drives confidence, so a candidate three models agree on outranks a single-model find.

&nbsp;

#### Coverage-graph context support

`deep-loop-runtime` gained `loop_type='context'`: new node kinds and relations, a `computeContextSignals` function (slice coverage, reuse-catalog coverage, agreement rate, relevance floor, dependency completeness), and an `evaluateContext` stop contract with agreement and relevance as blocking guards. A `promptFramework` lineage field carries per-model prompt framing (for example COSTAR for MiMo, TIDD-EC for MiniMax).

&nbsp;

#### Reuse-first Context Report

The report ships verified pointers (`file:symbol` plus signature), never raw source bodies, which avoids context rot (degraded model attention on oversized inputs) and stale-reference failure. Sections are ordered by value: REUSE catalog, integration points, touch list, conventions, pruned dependency subgraph and gaps.

&nbsp;

#### sk-doc documentation package

SKILL.md (quality score 99) with the resilient smart-router, two references, the Context Report and config assets, a human-facing README and a scripts code-README. A `feature_catalog/` (20 features across 6 categories) and a `manual_testing_playbook/` (20 runnable scenarios) bring the skill to parity with the sibling deep skills.

&nbsp;

#### Speckit integration

`deep-context` is an AI-offered optional add-on (`:with-context`) on `/speckit:complete` and `/speckit:plan`, mirroring the existing `:with-research` pattern. When enabled it runs before specification and feeds the Context Report into planning.

## Upgrade

No migration required. Run `/deep:context` with a target feature, or pass `:with-context` to `/speckit:complete` or `/speckit:plan` to be offered the optional pre-planning step. The default executor pool (two native plus MiMo, gpt and deepseek) lives in `assets/deep_context_config.json` and is operator-composable.
