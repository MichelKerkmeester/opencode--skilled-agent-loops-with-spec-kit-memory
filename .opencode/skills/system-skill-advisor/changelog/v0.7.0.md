This release lands the system-skill-advisor portion of the 028 MCP-to-CLI dual-stack program. The `mk_skill_advisor` daemon gains a daemon-backed CLI front door — `node .opencode/bin/skill-advisor.cjs` — exposing all nine tools over the same daemon the MCP registration uses, with the registration unchanged. Because the advisor surface includes mutations, the CLI ships with a fail-closed trust gate: CLI calls are untrusted by default, mutation tools require an explicit grant, and native MCP callers receive trust only through an env-block declaration that callers cannot forge. Warm-only hook fallbacks for Claude Code and Codex plus a CLI fallback route in the OpenCode plugin complete the pairing, and runtime permission allowlists let the runtimes invoke the CLI without prompting.

> Spec folder: `.opencode/specs/system-spec-kit/028-mcp-to-cli-tool-transition/003-skill-advisor-cli/` (with program-wide phases under the same parent)

## What Changed

#### The `skill-advisor` CLI

- **Nine tools, one daemon** -- the four `advisor_*` tools, the four `skill_graph_*` tools and the internal propagation tool are all reachable as CLI commands over the same IPC socket the MCP registration uses. `list-tools` enumerates them offline; everything else validates arguments against the existing schemas and speaks JSON-RPC to the daemon. Exit taxonomy is shared with the sibling `spec-memory` and `code-index` CLIs: `0` success, `1` runtime error, `64` usage/schema error, `69` protocol/dist mismatch, `75` retryable daemon error.
- **Warm-only prompt path** -- `--warm-only` (with the `MK_SKILL_ADVISOR_CLI_WARM_ONLY` / `MK_SKILL_ADVISOR_CLI_PROMPT_TIME` env defaults) probes the advisor socket first and exits `75` instead of cold-starting the daemon from a hook. The dist-freshness guard refuses a stale build with exit `69`; `MK_SKILL_ADVISOR_CLI_DEV_ALLOW_STALE=1` is the development override.
- **Output integrity** -- natural exit with drain-awaited writes, closing the 64KB pipe-truncation hazard shared across the three CLIs.

#### The fail-closed trust gate

- **Untrusted by default** -- CLI calls carry no trust. The mutation tools `advisor_rebuild`, `skill_graph_scan` and apply-mode `skill_graph_propagate_enhances` require `--trusted` (or `MK_SKILL_ADVISOR_CLI_TRUSTED=1`), which is the maintainer path; an untrusted mutation attempt is refused with exit `64`.
- **MCP trust is declared, not asserted** -- the daemon fails closed when transport `_meta` carries no trust context. Native MCP registrations grant trust via `MK_SKILL_ADVISOR_TRUST_DEFAULT=trusted` in their env block — a deployment-time declaration callers cannot forge at request time.
- **Fuzz-tested** -- a dedicated trust-gate fuzz scenario joined the manual-testing playbook, and a hardening suite covers the gate alongside the dual-client tests.

#### Runtime integrations

- **Hook fallback** -- the shared `hooks/lib/skill-advisor-cli-fallback.ts` helper gives the Claude and Codex `user-prompt-submit` advisor hooks a warm-only CLI path: socket probe first, CLI when the daemon is warm, fail-open in about a millisecond when it is not. Gate 2 routing keeps working with the MCP transport down.
- **Plugin fallback** -- `mk-skill-advisor.js` gains a CLI fallback route through `plugin-bridges/mk-skill-advisor-bridge.mjs` while the primary bridge path stays untouched.
- **Allowlists** -- `.codex/settings.json` allowlists the CLI invocations for Codex; the Claude allowlist lives in local-only settings by decision.

## Upgrade

No migration required. The MCP registration and the Python facade (`skill_advisor.py`) keep working unchanged. `node .opencode/bin/skill-advisor.cjs list-tools --format text` answers offline; mutations need `--trusted`. Verify `MK_SKILL_ADVISOR_TRUST_DEFAULT=trusted` is present in the MCP registration env block if native mutations are rejected as untrusted. The new env flags are documented in system-spec-kit's `mcp-server/ENV-REFERENCE.md`.
