## Analysis: Newer 2025-2026 coding-agent loops with novel loop-control

I found two genuinely novel systems and one patterns repo, none in the prior catalogue. The standout finding is that the **novel loop-control math is migrating in from optimization theory**, not from the coding-agent ecosystem itself.

**1. SICA (MaximeRobeyns/self_improving_coding_agent)** — 368★, MIT, arXiv:2504.15228 (ICLR 2025 Workshop). A coding agent that runs an iterative improvement loop *on its own codebase*: (1) evaluate current version on benchmark tasks → (2) store results in an append-only archive (`results/run_<id>/agent_<i>/benchmarks/results.jsonl`) → (3) agent edits its own code → (4) repeat. Reported 17%→53% on SWE-Bench Verified. **Crucially, its own "Things to work on" flags that self-improvement runs are high-variance and path-dependent** ("early features often influence subsequent features"). That archive-as-convergence-signal and the path-dependence admission are both directly transferable.

**2. SkillOpt (microsoft/SkillOpt)** — ~6.4K★ (per openagentskill.com; repo verified), arXiv:2605.23904 (2026). A *text-space optimizer*: it treats a Markdown `skill.md` as the sole trainable parameter of a frozen agent and learns it through scored trajectory rollouts → reflection-driven edits → **held-out validation gates**. An edit is accepted *only* when it strictly improves a held-out validation score; bounded to add/delete/replace operations. This is a principled acceptance/termination gate that prevents overfitting to the in-loop trajectory — exactly the convergence criterion deep-improvement lacks.

**Transfer to system-deep-loop:**
- **Validation-gate as convergence criterion** (SkillOpt): deep-improvement's promotion step should gate each candidate on a *held-out* eval split, not on the in-loop trajectory that generated it. This operationalizes "convergence" as a monotonic-improvement guarantee over an unseen set rather than a fixed iteration cap.
- **Append-only performance archive** (SICA): the JSONL state should carry a per-iteration benchmark ledger as the convergence signal source (it's both `runtime/state-jsonl-checkpointing` and a `runtime/gauges-observability` input).
- **Bounded edit vocabulary** (SkillOpt): constraining the mutation space (add/delete/replace) to make search tractable and convergence measurable applies to deep-improvement candidate generation.
- **Path-dependence contradiction** (SICA): iterative self-improvement is non-monotonic and seed-dependent — convergence claims require multi-seed averaging, not a single trajectory.

A divergent signal: nearly all 2025-2026 "loop engineering" discourse (cobusgreyling/loop-engineering, lushbinary, the "Ralph Loop") is *patterns + a degenerate stop policy* (fixed `--max-iterations` cap + test-pass exit). The formal convergence machinery is being imported from optimization, not invented by coding-agent builders.
