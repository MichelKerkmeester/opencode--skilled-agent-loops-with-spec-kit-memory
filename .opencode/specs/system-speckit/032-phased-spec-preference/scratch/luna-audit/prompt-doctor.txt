CONTEXT
This repository ("Public") defines slash commands under .opencode/commands/<family>/. Each command typically has a *_presentation.txt asset (the visible prompt text shown to the user, listing lettered options like "A) foo  B) bar  C) baz") and one or more *_confirm.yaml / *_auto.yaml / *_routes.yaml assets (the machine-readable workflow logic that actually handles each answer, usually via keys like on_A/on_B/accepted_responses/choices, or an equivalent conditional-handling block). Some families also have compiled/*.contract.md mirrors of the presentation text.

A recent human audit of .opencode/commands/speckit/assets/speckit_resume_presentation.txt found a real bug: a Q0 line listed "D) Phase folder" as a selectable option, but tracing the matching speckit_resume_confirm.yaml showed phase-folder handling is actually a fully automatic downstream step with no on_D-style branch anywhere — "D) Phase folder" was never a real choice a user could select and have handled. It was removed. The same audit also fixed several "Skip" options that were sandwiched before a real processing choice instead of sorting last (e.g. "B) Skip C) Set goal: <objective>" was reordered to "B) Set goal: <objective> C) Skip").
