# Research Report: Spec-Kit Memory MCP — External Mining

> Canonical synthesis of a 7-iteration deep-research loop mining two external memory systems — **AIONFORGE** (Rust; `docs/retrieval.md`, `bi-temporal-model.md`, `consolidation.md`, `decay-and-importance.md`) and **GALADRIEL** (Python; `CACHING.md`, `README.md`, `harness/palace.py`) — for evidence-backed, code-mapped improvements to the Spec-Kit Memory MCP at `.opencode/skills/system-spec-kit/mcp_server/`.

## Executive Summary

The headline finding inverts the strategy's priors: the internal Memory MCP is **much closer** to the mined external models than "none today / likely a gap" assumed. Most high-leverage wins are **PROMOTE-the-off-state-flag, not build-new** — the target machinery (edge-based bi-temporal currentness, idempotency receipts, deferred/async save, channel-gated degrade, pure rank-time decay) already exists in code but is flag-gated OFF or structurally half-built. Of the top-6 ranked candidates, **5 are S-effort and 4 are PROMOTE-off-state**. The single cheapest high-leverage win is **C2-C** (graph-expansion gating per retrieval-class, extending an existing router primitive). For the parent roadmap, **code-graph is the highest-priority sibling**: it is touched by all four cross-cutting spines (determinism, bi-temporal currentness, query-class routing, graceful degradation) and shares memory's exact "indiscriminate graph expansion hurts single-hop precision" failure mode. Galadriel was ~70% saturated against aionforge and contributed exactly one strong net-new candidate (C-G1, the cadence/clock for the C4-C cursor). All 10 key questions have evidence-backed, code-mapped answers; the only residuals are empirical calibration items owned by the implementation packet.

## Internal Baseline

The live Memory MCP modules located across iterations 1–4, with file:line anchors:

- **5-channel RRF fusion** — core module `shared/algorithms/rrf-fusion.ts`. Default smoothing constant `DEFAULT_K = 40` (NOT 60), "optimized for ~1000-memory corpus," overridable via `SPECKIT_RRF_K` [CONFIRMED: shared/algorithms/rrf-fusion.ts:37-38, 150-164]. Source labels `vector, fts, bm25, graph, degree, keyword`; fusion reads rank position only (`1/(k+i+1)`), accumulates per-source, applies a convergence/overlap bonus for multi-source hits [CONFIRMED: rrf-fusion.ts:12-19, 308-388]. Channels are runtime-gated, not fixed at 5: `['vector','fts5']` always, then `bm25`/`graph`/`degree` if their flags are on — "5-channel" is the maximal active set [CONFIRMED: mcp_server/context-server.ts:911-914]. Graph channel gets a `GRAPH_WEIGHT_BOOST = 1.5` default ("curated causal edges are higher-signal") [CONFIRMED: rrf-fusion.ts:40-42, 302]; degree channel weight capped at `DEGREE_BOOST_CAP = 0.15` [CONFIRMED: lib/search/graph-search-fn.ts:53-56, 567].
- **Stage-2 fusion (the real ranking brain)** — `mcp_server/lib/search/pipeline/stage2-fusion.ts` runs a fixed **13-step signal-application order** with the hard invariant "every score modification happens exactly once here" (the G2 double-weighting guard) [CONFIRMED: stage2-fusion.ts:9-11, 21-45]. Pipeline stages: `stage1-candidate-gen → stage2-fusion → stage3-rerank → stage4-filter`.
- **Query-router seam** — `lib/search/query-router.ts` fuses two orthogonal classifiers into `RouteResult {tier, channels, classification, queryPlan, qualityFallback}`. The two existing axes are *complexity tier* (`simple|moderate|complex`, `query-classifier.ts:13`) and *task intent* (`add_feature|fix_bug|…`, `intent-classifier.ts:13`) — **neither is a retrieval-shape query-class router** [CONFIRMED: query-router.ts:36-52, 68-75; query-classifier.ts:13; intent-classifier.ts:13]. `ChannelName = 'vector'|'fts'|'bm25'|'graph'|'degree'` [CONFIRMED: query-router.ts:36, 68]. An embryonic graph-expansion gate already exists: `preserved`/`includeDegree` flags set by intent (`find_spec|find_decision`) or entity density `>= ENTITY_DENSITY_ACTIVATION_THRESHOLD` [CONFIRMED: query-router.ts:238-254, 86].
- **Save / index path** — handler set in `handlers/save/` (`create-record.ts`, `embedding-pipeline.ts`, `response-builder.ts`, `types.ts`). Already splits write from enrichment: `OperationStatus` includes `'deferred' | 'partial'`, `create-record.ts` logs "Using deferred indexing," `embedding-pipeline.ts` logs "Async embedding mode" behind an `asyncEmbedding` flag [CONFIRMED: handlers/save/types.ts:138, 304; create-record.ts:297; embedding-pipeline.ts:152]. `memory_index_scan` supports `background?: boolean` and reports `complete_with_pending_vectors` with a `deferred` count [CONFIRMED: handlers/memory-index.ts:153, 293-294, 1062-1064, 1376-1377]. `content_hash` (sha256) exists as a dedup signal, but identity is the sequential `memory_index.id` autoincrement (NOT content-addressed) [CONFIRMED: handlers/memory-index.ts:281-284; handlers/pe-gating.ts:407].
- **Recall serialization** — flows through `lib/response/envelope.ts`. Both `serializeEnvelope` (:99-101) and `serializeEnvelopeWithTokenCount` (:136-139) are a bare `JSON.stringify(envelope, null, 2)`; `wrapForMCP` (:284-295) emits an MCP `text` block. Render order = upstream score order; there is no content-derived re-sort stage [CONFIRMED: lib/response/envelope.ts:99-101, 136-139, 284-295]. Render site is `formatters/search-results.ts:782 formatSearchResults`.
- **Causal graph** — store `lib/storage/causal-edges.ts` with `RELATION_TYPES = {CAUSED, ENABLED, SUPERSEDES, CONTRADICTS, DERIVED_FROM, SUPPORTS}`, traversal weights `supersedes:1.5` (strongest), `contradicts:0.8` (dampened) [CONFIRMED: causal-edges.ts:21-43]. Table schema `causal_edges` at `lib/search/vector-index-schema.ts:1107`; tombstones `causal_edge_tombstones` at `lib/causal/sweep.ts:70`; traversal/boost (BFS, intent edge priors, hop decay) at `lib/search/causal-boost.ts`. Bi-temporal edge-close machinery exists in `lib/graph/temporal-edges.ts` (`valid_at`/`invalid_at` columns, `invalidateEdge()`) and `lib/graph/contradiction-detection.ts` (queries only live edges, auto-invalidates on `supersedes`/conflicting pairs) — **all gated OFF** by `isTemporalEdgesEnabled()` [CONFIRMED: temporal-edges.ts:35-80; contradiction-detection.ts:75-77, 85-90, 99-110].
- **Decay (FSRS)** — two parts. Rank-time *structural freshness* `lib/search/fsrs.ts` is a **pure write-free function** `computeStructuralFreshness(stability, centrality) = clamp(stability) * clamp(centrality)` [CONFIRMED: fsrs.ts:40-47, 63-82]. The only write-back is `strengthenOnAccess()` (`UPDATE memory_index SET stability, last_review, review_count+1, access_count+1, last_accessed`), gated by `trackAccess` (opt-in) — it is FSRS *reinforcement* (the "testing effect"), not decay computation [CONFIRMED: stage2-fusion.ts:574-618, 602-610, 876-877]. The rank-time `R(t)` (`calculateRetrievability(stability, elapsedDays)`) is already computed *before* the write fires, in `applyTestingEffect` [CONFIRMED: stage2-fusion.ts:897-910].
- **Idempotency receipts** — `lib/storage/idempotency-receipts.ts` defines aionforge-shaped receipts keyed by `{operation, contentHash, requestFingerprintHash, payloadHash}` over `memory_save`/`memory_update` with `miss | replay | conflict` states; client tokens stripped before hashing (content-derived). Gated OFF by `SPECKIT_MEMORY_IDEMPOTENCY` unless explicitly set; expired receipts already pruned on the server lifecycle [CONFIRMED: idempotency-receipts.ts:9-29, 43-79, 54-57; context-server.ts:1831].
