## [**3.4.1.0**] - 2026-05-05

This release adds validator coverage for generated metadata shape and removes a bare-directory routing entry from the system-spec-kit skill router. It closes the 077 validator-cleanup findings for graph metadata drift, description metadata drift, and the `ROLLOUT_FLAGS` markdown-only resource-map violation.

> Spec folder: `specs/skilled-agent-orchestration/078-opencode-authoring-recipe/004-validator-cleanup` (Level 1, phase child)

---

## What Changed

#### Validation

- **GRAPH_METADATA_SHAPE** -- New bash rule validates `graph-metadata.json` parseability, packet identity shape, derived status shape, and phase-parent `last_active_child_id` pointer integrity.
- **DESCRIPTION_SHAPE** -- New bash rule validates `description.json` parseability plus identity, `description`, and `level` field types while allowing extension keys.
- **Validator registry** -- Registers both new structural rules with aliases for targeted validation and help output.

#### Router Cleanup

- **ROLLOUT_FLAGS** -- Removed the bare `feature_catalog/19--feature-flag-reference/` directory entry from `RESOURCE_MAP`; the remaining `references/config/environment_variables.md` markdown route covers feature flag lookup without violating `_guard_in_skill`.

## Files Changed

| File | What changed |
| --- | --- |
| `scripts/rules/check-graph-metadata-shape.sh` | New graph metadata shape rule |
| `scripts/rules/check-description-shape.sh` | New description metadata shape rule |
| `scripts/lib/validator-registry.json` | Registered `GRAPH_METADATA_SHAPE` and `DESCRIPTION_SHAPE` |
| `mcp_server/lib/validation/orchestrator.ts` | Added the same shape checks to the default Node validation path |
| `SKILL.md` | Version 3.3.1.0 -> 3.4.1.0; fixed `ROLLOUT_FLAGS` resource map |

## Upgrade

No migration required. The new rules accept the current generated metadata schema and the newer nested packet identity form, so existing valid packets continue to pass strict validation.
