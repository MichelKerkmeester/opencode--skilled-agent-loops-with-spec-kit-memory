## [**3.2.0.0**] - 2026-05-05

This release adds the OpenCode authoring recipe foundation: 5 new authoring checklists (skill, agent, command, mcp-server, spec-folder) and a spec-folder-write recipe. Restores the machine-readable STACK_FOLDERS contract that regressed to inline strings. Fixes a stale relative link in references/opencode/shared/universal_patterns.md. Closes 077 findings F-001-005, F-006-002, F-007-001, F-007-002, F-008-001, F-008-002, F-008-004, F-010-001, F-010-004.

## What Changed

#### New Features

- **OpenCode authoring checklists** -- Added first-class checklists for skill, agent, command, MCP server, and spec folder authoring so sk-code's OpenCode scope has matching canonical assets.
- **Spec folder write recipe** -- Added a step-by-step spec folder recipe with scaffold, metadata, anchor, continuity, validation, and handoff gates.

#### Architecture

- **STACK_FOLDERS contract restored** -- Added the machine-readable Python-style stack folder dictionary back to SKILL.md for copy-pasteable router detection.
- **OpenCode resource map updated** -- Added conditional OpenCode resource rows for all new authoring checklists and the spec folder write recipe.

#### Bug Fixes

- **Stale checklist link fixed** -- Updated the universal patterns checklist pointer to the real `assets/opencode/checklists/` path.
