This release moves the review checklists out of `references/` and into `assets/`, where reusable checklist artifacts belong under the sk-doc skill convention, and aligns each one to the asset template. The review doctrine is unchanged — what changed is where the checklists live and how their overview sections read. All routing, README, metadata, playbook, and cross-skill references were updated in the same pass, so no load path breaks.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/146-sk-code-ponytail-based-refinement/008-sk-code-review-checklist-reclassification` (Level 1)

---

## New Features

Three changes: a reclassification, an asset-template alignment, and the router prose that points at the new home.

#### Review checklists reclassified references/ → assets/

Six files were `references/` but are really checklist artifacts a reviewer applies, not doctrine a reviewer reads: `code_quality_checklist.md`, `security_checklist.md`, `solid_checklist.md`, `test_quality_checklist.md`, `fix-completeness-checklist.md`, and `removal_plan.md`. They moved to a new `assets/` folder. The four genuine references stayed in `references/`: `review_core.md` (doctrine), `review_ux_single_pass.md` (single-pass UX behavior), `quick_reference.md` (index), and `pr_state_dedup.md` (the M-1 dedup spec). Every by-path reference was updated in lockstep — the §2 routing tables and `RESOURCE_MAP`, the README table, `graph-metadata.json`, the bidirectional sibling cross-links, nineteen manual-testing per-feature files, and the two cross-skill `sk-code/references/opencode/{python,shell}/quality_standards.md` pointers. The `.claude` and `.codex` runtime mirrors were re-synced. The rule canary keys on `pr_state_dedup.md` (which stayed), so it remains green.

&nbsp;

#### Checklists aligned to the asset template

The moved checklists carried a reference-flavored overview (`### Purpose` + `### Core Principle`). Each now also carries the asset template's `### Usage` subsection that names how a reviewer applies it, keeping the Core Principle. `fix-completeness-checklist.md` was the outlier — a lowercase `## 1. Overview` with Title-Case sections and no subsections — and was restructured to match its siblings: `## 1. OVERVIEW` with Purpose and Usage, upper-cased section names, dividers, and a Related Resources section. All six validate as assets.

&nbsp;

#### Router prose points at the new home

The `SKILL.md §2` Resource Domains block still described the checklists as living under `references/`. It now points at `assets/*_checklist.md` and `assets/removal_plan.md`, so the documented domain map matches the resolved load paths.

## Upgrade

The six checklists moved from `references/` to `assets/`. Any external reference to `sk-code-review/references/<name>.md` for one of the moved files must update to `sk-code-review/assets/<name>.md`. Historical changelogs (v1.1–v1.4) are left as point-in-time records and still cite the old paths. Run `node .opencode/skills/sk-code-review/scripts/check-rule-copies.js` to confirm the canary, and `validate_document.py --type asset` on `assets/*.md` to confirm conformance.
