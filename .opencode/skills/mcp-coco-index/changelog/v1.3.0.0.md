## [**1.3.0.0**] - 2026-05-05

This release adds canonical-resource-path opt-ins so explicit authoring checklists and skill references survive `**/.*` hidden-directory exclusion AND outrank unrelated matches in semantic search. Closes the fresh-clone portability gap that 077 packet F-001-002 surfaced for sk-code's OpenCode assets, plus the rank-priority gap surfaced by F-005-002 / F-007-005 / F-008-005.

> Spec folder: `specs/skilled-agent-orchestration/078-opencode-authoring-recipe/003-coco-priority` (Level 1, phase child)

---

## What Changed

#### New Features

- **Canonical resource paths** — `.opencode/skills/*/assets/opencode/**`, `.opencode/skills/*/assets/motion_dev/**`, and `.opencode/skills/*/references/**` are now ingested under defaults even though `**/.*` excludes hidden directories. Override via `project.canonical_resource_paths` in ccc_settings.yaml.

- **Canonical resource boost in ranking** — Search results matching canonical paths receive a +0.10 score boost and emit a `canonical_resource_boost` ranking signal. Ensures canonical authoring assets outrank unrelated matches that happen to mention similar terms.

#### Architecture

- **ProjectSettings.canonical_resource_paths** — New field with default factory pulling from `CANONICAL_RESOURCE_PATHS`. YAML round-trip preserves the field.

- **is_canonical_path() helper** — Module-level function in settings.py uses GitIgnoreSpec for consistent glob semantics with exclude_patterns.

## Upgrade

Existing indexes built before 1.3.0.0 do NOT include canonical-path-only files (because they were excluded at indexing time). To pick up the new defaults: `ccc reset && ccc index`.

Users with custom `exclude_patterns` are unaffected — canonical paths only bypass the default `**/.*` rule by being explicit opt-ins.
