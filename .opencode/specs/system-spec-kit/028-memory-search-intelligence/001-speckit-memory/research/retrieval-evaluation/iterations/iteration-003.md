## The duplication (concrete, already paid)
`cosineSimilarity` is defined **3× byte-equivalent** (`semantic-trigger-matcher.ts:174`, `memory-summaries.ts:29`, `save-quality-gate.ts:666`); BLOB↔Float32 converters **2×** (`memory-summaries.ts:61/68`, `semantic-trigger-matcher.ts:165`). Each semantic channel reimplements the *same loop* (embed→cache/BLOB→brute-cosine→gate): trigger (`semantic-trigger-matcher.ts:305-440`), summary (`memory-summaries.ts:98-200`); edges have none (`temporal-edges.ts` = `valid_at/invalid_at/strength`, zero vectors). The shared substrate **half-exists**: `embedding_cache` is already `input_kind`-partitioned (`embedding-cache.ts:151,157`) and the trigger channel already writes to it (`trigger-embedding-backfill.ts:274-292`).

A8's gate becomes reusable only once they share the interface.

## Key finding + a grounding correction
Unification is real leverage at the **bottom** (cosine/buffer math + the `input_kind`-partitioned `embedding_cache`) and **top** (one shadow gate over the 027 doctrine) — cheap wins neither prior pass reached. But the **middle retrieval/scoring layer must stay per-channel**: granularity (many high-threshold short trigger phrases vs one longer summary vs relationship triples), hit-semantics (document vs graph path), and promotion economics (trigger shadow-blocked, summary already live, edge LLM-dedup tail-risk) all differ where a single `retrieve()` would leak. Build `vector-math` + cache-backed `embed/store` + one shadow gate as **shared rails**; keep each channel's scoring its own **car**. Forcing one `retrieve()` across all three = premature abstraction.

**Grounding correction (fixes my own 06/07):** 028/007 #4's "summaries are fallback-only, not in the main blend" conflates two paths — the summary-*embedding* channel (`memory-summaries` → `stage1-candidate-gen.ts:1301`) is a **live default-on stage-1 candidate appender**, distinct from the community-summary weak-result fallback (`memory-search.ts:1158`). It is correctly absent from the *weighted RRF* list (`hybrid-search.ts:1310`) but it is **not inert**. 07's MEM-fused-summary-channel scope should distinguish the two.
