# v0.2.0, Production Isolation (Preview Track)

**Release date**: 2026-05-15
**Source directive**: user, "all 3 system skills need to be completely isolated. Users can delete system-spec-kit and expect a functioning system-code-graph and skill-advisor"
**Remediation packets**: 040 (active arc) + 040-followon (deferred items)
**Versioning note**: pre-1.0 preview track. v0.2.0 reflects the substantive nature of the isolation work. Runtime API ids (`advisor_recommend`, `advisor_rebuild`, `advisor_status`, `advisor_validate`, `skill_graph_*`) remain stable from v0.1.0.

---

## Headline

system-skill-advisor was hard-coupled to system-spec-kit at the import level, 6 production .ts files imported cross-skill helpers (shared-payload types + unicode-normalization). v0.2.0 closes all 6 by duplicating the small types locally into `mcp_server/lib/shared/` and routing the heavyweight embeddings stack through a visible symlink (user's chosen pattern: "make the dependency visible to repo users").

**Smoke test verdict** (script at `018-three-way-isolation-finalize/smoke-test/`, log committed):

```
skill-advisor tsc exit BEFORE removing system-spec-kit:  1 (TS5101 baseUrl warning)
skill-advisor tsc exit AFTER removing system-spec-kit:   2 (NEW errors, symlinked embeddings dangled)
skill-advisor tsc exit AFTER restoring system-spec-kit:  1 (back to baseline)
```

Interpretation: skill-advisor's NON-embeddings code is fully isolated. Embeddings-touching files (semantic-shadow lane + skill-graph-db's embedding load) fail when system-spec-kit is removed, by design, via a dangling symlink that telegraphs the missing dependency.

## What changed

#### Local lib/shared/ created with 7 types + canonicalFold

- Need: 4 production .ts files imported `AdvisorEnvelopeMetadata`, `AdvisorEnvelopeFreshness`, `AdvisorEnvelopeStatus`, `SharedPayloadEnvelope`, `SharedPayloadSection`, `SharedPayloadSourceRef`, `SharedPayloadTrustState` from `system-spec-kit/mcp_server/lib/context/shared-payload.ts`. Two of them imported `canonicalFold` from `system-spec-kit/shared/unicode-normalization.ts`.
- Change: new `mcp_server/lib/shared/shared-payload.ts` carries 7 types + transitive helpers (~1099 lines). New `mcp_server/lib/shared/unicode-normalization.ts` carries the full `canonicalFold` function + `CANONICAL_FOLD_VERSION` + `HIDDEN_CHAR_PATTERN` + `COMBINING_MARK_PATTERN` + `CONFUSABLE_REPLACEMENTS` (74 lines).
- Why it matters: production .ts cross-skill imports go from 6 → 0. Both new files carry header comments naming the source-of-truth in system-spec-kit so future contributors know where to look for upstream changes.

&nbsp;

#### Heavy embeddings dependency made visible via symlink

- Need: 2 files (`lib/scorer/lanes/semantic-shadow.ts`, `lib/skill-graph/skill-graph-db.ts`) imported `createEmbeddingsProvider` from `@spec-kit/shared/embeddings/factory`, a 1253-line factory + 4 provider implementations + profile + availability + types (~3000-5000 lines if duplicated). Too heavy to duplicate. Too critical to drop.
- Change: `mcp_server/lib/shared/embeddings` is now a symlink to `../../../../system-spec-kit/shared/embeddings`. The symlink is visible in the file tree, repo users can see "this skill depends on system-spec-kit/shared/embeddings". Consumers import via the `@spec-kit/shared/embeddings/factory.js` workspace alias (resolved by tsconfig paths to the real location, bypassing the symlink-resolution edge cases). Both consumer files carry NOTE comments explaining the symlink contract.
- Why it matters: dependency is now self-documenting. Delete system-spec-kit → the symlink dangles loudly in the file tree → the missing dependency is impossible to misdiagnose.

&nbsp;

#### package.json @spec-kit/shared dependency removed

- Need: `mcp_server/package.json` declared `"@spec-kit/shared": "file:../../system-spec-kit/shared"` as a workspace dep. That declaration was the runtime "magic" that made the alias resolve via `npm install`.
- Change: declaration removed (and `package-lock.json` regenerated). The `@spec-kit/shared` workspace alias now resolves ONLY via tsconfig paths, type-check-only. Runtime imports resolve via the file system (through the symlink).
- Why it matters: cleaner dependency manifest. Anyone reading `package.json` no longer sees a phantom cross-skill dep.

&nbsp;

#### 1 test file isolated via vi.mock

- Need: `tests/handlers/advisor-recommend.vitest.ts` imported `buildErrorResponse`, `ErrorCodes`, `MemoryError` from `system-spec-kit/mcp_server/lib/errors.ts`.
- Change: `vi.mock` the cross-skill module with a small `__fixtures__/errors.ts` stub.
- Why it matters: test suite is now cross-skill-import-free. Vitest discovers 24 advisor tests post-refactor.

&nbsp;

#### CI guardrail covers skill-advisor

- Need: the existing CI workflow had no reverse-direction audit for skill-advisor (only for code-graph).
- Change: 2 new blocking audit steps added to `.github/workflows/isolation-check.yml`:
  - skill-advisor reverse-direction relative-path imports (excludes `lib/shared/embeddings/**` symlink and tests/stress_test)
  - skill-advisor `@spec-kit/*` workspace-alias imports (excludes `@spec-kit/shared/embeddings/*`, the single documented exception)
- Why it matters: any future regression introducing a new cross-skill dep fails CI with a precise error message.

&nbsp;

#### Deferred-items cleanup (followon)

- `mcp_server/tsconfig.json`: `ignoreDeprecations: "6.0"` added, silences the pre-existing TS5101 `Option 'baseUrl' is deprecated` warning at line 6.
- `mcp_server/tsconfig.json`: `rootDir: "."` removed, the alias resolution to `system-spec-kit/shared/embeddings/factory.js` lies outside the previous rootDir constraint. Removing it lets tsc walk the workspace boundary cleanly.

## Upgrade

No migration required. The runtime API surface is unchanged from v0.1.0:
- `advisor_recommend`, `advisor_rebuild`, `advisor_status`, `advisor_validate`, same signatures, same behavior.
- `skill_graph_scan`, `skill_graph_query`, `skill_graph_status`, `skill_graph_validate`, unchanged.

If you maintain a fork:
- Re-run `npm --prefix .opencode/skills/system-skill-advisor/mcp_server install` to refresh `package-lock.json` without `@spec-kit/shared`.
- The symlink at `lib/shared/embeddings` is git-tracked and will recreate on checkout (git stores it via mode 120000).

## Known limitations

- **Embeddings dependency on system-spec-kit is intentional**, by user choice, the symlink at `lib/shared/embeddings` makes this visible rather than hidden. Embeddings-backed features (`semantic-shadow` scoring lane + `skill-graph-db` embedding load) require system-spec-kit/shared/embeddings to be present. The dangling symlink is the failure signal.
- **`tsconfig.json` extends from `system-spec-kit/mcp_server/tsconfig.json`**, at the build-config level, skill-advisor still references system-spec-kit. Deleting system-spec-kit breaks `tsc --build` via the inherited config. Inlining the base config is a future cleanup if pursued.
- **tsc emits `TS6307 File is in the program because ...` informational notes** for files reached through the `@spec-kit/shared/embeddings/*` alias, these are NOT errors (exit code 0). They describe the cross-skill program structure, which is the architectural truth.
