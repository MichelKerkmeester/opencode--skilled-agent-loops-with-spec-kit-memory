This release expands the documented model roster from a single `gpt-5.5` lock to the full four-model GPT-5.5 / GPT-5.6 set, each with its own reasoning-effort ceiling. All twenty model×effort cells were live-verified callable through cli-codex before the docs were written.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/134-cli-codex-revival/008-cli-codex-model-roster-and-alignment/` (Level 3)

---

## What Changed

#### Model Roster

- **Four models documented** -- `gpt-5.5` (default), `gpt-5.6-luna`, `gpt-5.6-terra`, `gpt-5.6-sol`, replacing the "gpt-5.5 for every task" single-model contract. `gpt-5.5 medium` stays the skill default.
- **Reasoning-effort scale extended** -- added `max` and `ultra` to the documented `none…xhigh` set, with per-model ceilings: `gpt-5.5` ≤ `xhigh`; `gpt-5.6-luna` / `gpt-5.6-terra` ≤ `max`; `gpt-5.6-sol` ≤ `ultra`.
- **Selection guidance** -- default `gpt-5.5 medium`; escalate to `gpt-5.6-luna max` for deep implementation and `gpt-5.6-sol xhigh`/`ultra` for verification and review.

#### Verification & Alignment

- **Roster live-verified** -- all four models responded across their documented effort levels (20/20 cells) via `codex exec` on ChatGPT OAuth.
- **CX-002 playbook reframed** -- the "gpt-5.5 model lock" scenario now verifies the default pin plus the callable GPT-5.6 roster instead of asserting a single supported model.

## Upgrade

Default dispatch is unchanged (`--model gpt-5.5 -c model_reasoning_effort="medium" -c service_tier="fast"`). To reach deeper reasoning, name a GPT-5.6 model per dispatch within its ceiling — e.g. `--model gpt-5.6-sol -c model_reasoning_effort="ultra"`.
