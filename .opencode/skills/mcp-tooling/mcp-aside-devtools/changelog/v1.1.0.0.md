## [**1.1.0.0**] - 2026-07-16

This release brings the packet to registered-state truth and full inventory parity with its sibling transports. The `aside` UTCP manual is now live in `.utcp_config.json`, every "later phase" claim is flipped to the registered reality, and the packet gains the feature catalog and asset snapshot its siblings already carry.

#### Registered-state doc truth

- Need: The `aside` manual was registered in `.utcp_config.json`, but the packet still documented registration as a later phase — including a gated test scenario and a doctor that treated manual absence as expected.
- Change: Every stale claim across SKILL.md, README, INSTALL_GUIDE, references, server packages, and the playbook now states the registered truth: registration done 2026-07-16, discovery pending a fresh Code Mode session, `aside.aside_repl` unconfirmed until `tool_info()`. The ASD-011 discovery scenario is ungated (precondition: a session with the code_mode MCP loaded; SKIP-valid without one), and `doctor.sh` now treats manual absence as an error instead of expected-absent info.
- Why it matters: Consumers stop deferring the MCP lane for a phase that already landed, and a registration regression gets caught by the doctor instead of being read as normal.

&nbsp;

#### Feature catalog

- Need: Sibling packets carry a `feature_catalog/` capability inventory; this packet had none, leaving no single place to see what the surface can and cannot do.
- Change: A root catalog plus five per-intent domains (task, repl, mcp, install, troubleshoot — matching the SKILL.md router's intent keys) inventory 12 capabilities across the three lanes, each traced to existing packet docs or the completed research, with mutation class and UNKNOWN flags per capability.
- Why it matters: Capability questions get answered from one inventory that stays honest about version-pinned evidence and open unknowns instead of scattering claims across references.

&nbsp;

#### Registered manual asset

- Need: The registered manual object lived only in the live config and in prose mirrors; nothing carried a verifiable canonical snapshot plus the remaining confirmation steps.
- Change: `assets/utcp-aside-manual.md` holds the byte-true snapshot of the registered entry (verify with jq, do not re-add), the remaining post-registration checklist, and the open single-vs-dual-manual question; the MCP intent route now loads it.
- Why it matters: Anyone can verify the registered shape programmatically, and a future re-registration pastes a verified object rather than improvising one.
