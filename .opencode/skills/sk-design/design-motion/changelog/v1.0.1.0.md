# v1.0.1.0 - Router default fix and traceability closeout

**Released:** 2026-06-25

Patch release: Smart Router default fix plus traceability and coverage cleanup for the temporal-layer child.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/154-sk-design-parent`

## Fixed

- Smart Router default no longer collides with the STRATEGY intent resource. `DEFAULT_RESOURCE` now points at `references/corpus_map.md` (matching the foundations and audit siblings), so a strategy/timing prompt loads `motion_strategy.md` through its intent route instead of self-reporting a false "no keyed knowledge base" notice on the skill's most common temporal concern.

## Changed

- `SMART ROUTING` resource-loading table now describes the ALWAYS level as `corpus_map.md` plus the matching temporal-concern reference, matching the post-fix routing behavior.

## Added

- New manual-testing scenario `MOTION-MICRO-001` (`manual_testing_playbook/04--micro-interactions/`) covering press feedback, the required active/pressed state, morphing-icon rules, and delight boundaries.

## Traceability

- `Corpus Distilled` in `v1.0.0.0.md` now lists all eight sources from `references/corpus_map.md`, adding `external/fixing-motion-performance.md` and `external/make-interfaces-feel-better.md`.
