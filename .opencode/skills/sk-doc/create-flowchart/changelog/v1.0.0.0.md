# v1.0.0.0 - Initial Release

**Released:** 2026-07-06

Initial release of `create-flowchart`, one of the ten workflow packets in the `sk-doc` parent hub's workflow-only architecture. The packet authors and validates ASCII-style markdown flowcharts — decision trees, approval and review loops, parallel fan-out/fan-in pipelines, system swimlanes, and user journeys — grounded in a real source process, then runs a packet-local validator that checks box-width consistency, connector presence, decision branch labeling, nesting depth, and diagram size before delivery.

## What Included

- `SKILL.md` — the inline workflow contract: when to draw a flowchart versus a short bullet list, the six workflow-shape-to-pattern-asset routing table, the 12-step creation workflow (identify shape and source, read the target, select one pattern, extract real nodes, draft, validate, fix errors, report), notation rules for consistent box-drawing and labeled branches, pattern-specific build rules per workflow shape, and the Always/Never/Escalate rules.
- `references/` — deeper creation guidance beyond the inline contract, dissected into single-concern files behind a `README.md` route-map: `worked_example.md` (a validator-passing decision-tree example), `notation_and_validator.md` (validator mechanics, box-width notation, and common pitfalls), and `pattern_selection.md` (pattern-selection detail and split heuristics).
- `assets/flowcharts/` — six pattern assets used for visual guidance, one per workflow shape: `simple_workflow.md` (linear sequence), `decision_tree_flow.md` (conditional branching), `parallel_execution.md` (fan-out/fan-in), `approval_workflow_loops.md` (review and escalation loops), `system_architecture_swimlane.md` (service/layer swimlanes), and `user_onboarding.md` (user journeys).
- `scripts/validate-flowchart.sh` — the packet-local validator required before delivery for any generated or edited flowchart file; exits `0` on pass (including warning-only runs) and `1` on error.

## Notes

This packet has no packet-local `graph-metadata.json`; the single advisor identity and workflow registry live at the `sk-doc` hub root. When surrounding markdown quality is in scope, it reuses the shared `../shared/scripts/validate_document.py` validator rather than duplicating document-level checks.
