This release adds a design-generation prompt pattern so sk-prompt serves the framework's design-generation tools, not just text tasks. The skill was a generic text-prompt engine: seven frameworks, DEPTH, CLEAR, and three format guides. None of it knew how to write a brief for `mcp-magicpath` canvas authoring or `mcp-open-design` `start_run`, where a generic brief produces the median AI look the house design skill exists to resist. A new reference plus router wiring closes that gap without a new pipeline.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/145-mcp-open-design/006-sk-prompt-design-tool-usecases/` (Level 2)

---

## What Changed

#### Added

- **`references/design_generation_patterns.md`** -- a focused reference that layers design-specific slots onto the existing frameworks (COSTAR, CRISPE, CRAFT) rather than inventing a new one. It covers four things the generic frameworks did not: the grounded anti-default brief (subject and single job, design-system reuse-before-generate, an explicit avoid-list, the one aesthetic risk, a fidelity target), the String Seed of Thought anti-median variation technique (random 12-char string, ASCII sum mod N, index into brief-grounded directions before reasoning), pre-answering Open Design's multi-turn discovery form so a run does not drift to its recommended defaults, and a pointer to the `claude_design_parity.md` §6 handoff manifest for the move into `sk-code`. It owns the prompt only and defers the look to `sk-interface-design` and the run transport to the two MCP skills.

#### Changed

- **Smart router learns a `DESIGN_GEN` intent** -- `SKILL.md` gains a `DESIGN_GEN` entry in `INTENT_MODEL` (keywords: magicpath, open design, start_run, design generation, generate ui, canvas, design brief, variations) and a matching `RESOURCE_MAP` row that loads the new reference alongside `patterns_evaluation.md`. The Resource Domains block, the Resource Loading Levels table (a new CONDITIONAL row), the §1 use-case list, the §5 references, and the §9 resource list all name the new reference.

## Upgrade

No migration required. Existing modes, frameworks, and CLEAR scoring are unchanged. Design-generation routing is additive: it engages only when the request carries design-generation signals, and text-prompt behavior is identical to 2.1.3.0.
