Structural normalization of the `create-changelog` `SKILL.md` so it satisfies the canonical section contract shared across the `sk-doc` workflow packets. The previous layout merged its opening section as a single `WHEN TO USE + SMART_ROUTING` heading and carried no top-level references section, so the contract's separate `SMART ROUTING` and `REFERENCES` checks could not resolve. This release splits the header into two distinct sections and promotes the references list to its own section, with no change to the packet's workflow behavior.

---

## What Changed

#### Documentation

- **Section contract conformance** -- The opening `SKILL.md` heading combined when-to-use guidance and routing under one merged title, and the routing token used an underscore, so the shared section contract could not match a standalone smart-routing section. This splits the opening into a `WHEN TO USE` section and a `SMART ROUTING` section, keeping every existing activation trigger, exclusion, and family-boundary note, and adds an explicit output-mode routing summary that points at the detailed workflow sections.
- **References section promotion** -- The references list lived as a subsection of the rules and had no top-level heading, so the contract's references check could not resolve. This promotes it to its own section without altering any of the referenced paths.
- **Heading renumbering** -- All second-level headings were renumbered contiguously to absorb the new sections. No cross-references needed adjustment because the prose carries no section-number citations.
