# v1.16.0.0 - Lane D Kit 1.3.0 + Consolidated-Pilot Example

Both changes were proven live on the Copywriter and Barter deals instances during the 2026-06-11 skill consolidation (system_prompt dissolution) and are folded back per the lane's rule that guardrail-hardening fixes land in BOTH the kit and the live instances.

---

## 1. KIT 1.3.0 - MULTI-ROOT DERIVE (NEW)

- `packaging_config.schema.json`: `derived_copies[]` gains an optional `src_root` (packaging-relative; default stays `derive_source_root`). Existing configs stay valid and render byte-identically.
- `templates/derive.py.template`: `derive()` and `check()` resolve each entry against `entry.get("src_root", SOURCE_ROOT_LABEL)`; missing-source drift messages name the per-entry root. Enables the pilot's real mirror set: asset mirrors (`src_root: "skill/assets"`) and the SKILL.md byte-mirror (`src_root: "skill"`) that replaced the dissolved system_prompt mirror.
- `init_packaging.py` validates `src_root` (non-empty, packaging-relative, no leading `/`, no `..`).

## 2. KIT 1.3.0 - GAUNTLET LIVE-PACKAGING OVERLAY (FIXED)

- `templates/gauntlet.py.template`: the attack worktree is still created from git HEAD for isolation, then the LIVE packaging is rsync-overlaid (`-aL --delete`) so the battery tests current code against current data; `-L` materializes external symlinks (shared globals) as file content, keeping the sandbox self-contained even when symlink targets were renamed without being committed (proven by the 2026-06-11 workspace folder restructure: HEAD still had the old global-KB folder name and overlaid live symlinks broke inside the worktree until -L). Previously the battery was blind to uncommitted layout changes - the consolidation's relocated frozen scoring file (`meqt_scoring.md` / `deal_scoring.md`) did not exist at HEAD and attack A1 crashed with FileNotFoundError instead of exercising the guardrail.
- `reset_wt_file` restores attack-mutated files from the LIVE tree instead of `git checkout` - a checkout cannot restore files that do not exist at HEAD, and the worktree now mirrors the live packaging by contract.
- Both live instances run 10/10 with the overlay; attack semantics are unchanged (attacks still mutate only the isolated copy).

## 3. EXAMPLE CONFIG - CONSOLIDATED PILOT

- `frozen_surface`: the scoring region entry now points at `skill/references/meqt_scoring.md` anchor `## 2. MEQT SCORING SYSTEM` (frozen name `meqt_scoring_s2.frozen.md`) - the byte-identical relocation out of the dissolved `system_prompt.md`.
- `derived_copies`: the system-prompt mirror entry became the MEQT-scoring mirror; standards/brand_extensions mirror versions track the live pilot (v0.121 / v0.104); new entries for the two asset mirrors and the `Copywriter - System - Skill` byte-mirror exercise `src_root`.
- `synthesized_surfaces`: `skill/SKILL.md` removed - it remains hand-maintained at source but is now a mechanically mirrored surface, not synthesis-only.
- `kit_version`: 1.3.0.

## 4. VALIDATION

- `init_packaging.py --config packaging_config.example.json --check-only` renders clean; rendered `derive.py` and `gauntlet.py` byte-compile.
- Live-instance proof: Copywriter derive 15/15 + deals derive 20/20 mirrors green, both gates checks green with relocated anchors, both gauntlets 10/10 post-overlay.
