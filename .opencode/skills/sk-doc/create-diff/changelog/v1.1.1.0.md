# v1.1.1.0 - Multi-file Boundary Dividers

**Released:** 2026-07-20

Aggregate before/after reports now make every file transition unmistakable.
Validated file markers render as semantic start and end bands rather than
ordinary context lines that can disappear when long unchanged runs collapse.

## What Changed

- Added strict validation for pre-composed aggregate pairs with at least two
  unique, ordered, path-matched file envelopes.
- Preserved every recognized boundary through collapsed context.
- Added full-width `START FILE` and `END FILE` row groups to unified and
  side-by-side reports using the existing safe HTML dialect and design tokens.
- Added a 32px canvas-colored, accessibility-inert gap before every file after
  the first, masking the frame edges beside the whitespace so start bands no
  longer attach to the preceding diff.
- Reset Markdown section context at file transitions so one file's heading cannot
  label the next file's changes.
- Kept malformed or incidental marker text on the ordinary document path.
- Added regression coverage for both views, collapse behavior, malformed
  envelopes, hostile path escaping, and cross-file heading isolation.

## Compatibility

The CLI still compares one before document with one after document. Multi-file
review uses pre-composed aggregate inputs; native directory comparison and
repeated multi-file CLI arguments remain out of scope. Snapshot behavior, diff
statistics, extraction fidelity, the CSP, and command assets are unchanged.
