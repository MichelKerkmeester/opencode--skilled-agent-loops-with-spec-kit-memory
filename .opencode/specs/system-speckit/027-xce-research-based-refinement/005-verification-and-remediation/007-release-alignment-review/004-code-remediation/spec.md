## 2. PROBLEM & PURPOSE

### Problem Statement
The Track B review (`../002-code-vs-sk-code-opencode/review/findings-all.json`) surfaced 154 findings (66 P0 / 47 P1 / 41 P2) against the OpenCode code surface. The raw P0 count is inflated roughly 3x by false positives: `// Feature catalog:` comments that are gate-allowed, TSDoc `@example` illustrative values, and `dist/` build output. Fixing the raw set blindly would strip valid comments and produce wasted churn.

### Purpose
Remediate only the confirmed sk-code OpenCode-surface misalignments so the code aligns with sk-code standards (comment hygiene, shell strict-mode, no `any[]` in public types), behavior-neutral, via gpt-5.5-fast general + `sk-code` fixer seats.


<!-- /ANCHOR:problem -->

<!-- ANCHOR:scope -->
## 3. SCOPE

### In Scope
- Comment-hygiene ephemeral-id cleanup: drop the perishable label, keep the durable WHY.
- Shell `set -euo pipefail` strict-mode where missing.
- `any[]`→typed public DB row (`lib/storage/write-provenance.ts`).
- Module headers and opted-in P2 style nits (catch-typing, TSDoc) where behavior-neutral.

### Out of Scope
- `// Feature catalog: <name>` comments — NOT in comment-hygiene's forbidden list; gate-allowed; leave them.
- TSDoc `@example` values — illustrative API examples, not ephemeral pointers; leave them.
- `scripts/dist/**` build output — gitignored, regenerated by `npm run build`; rebuild instead of editing.
- Any finding that does not reproduce against the live file.

### Files to Change

| File Path | Change Type | Description |
|-----------|-------------|-------------|
| `lib/storage/write-provenance.ts` | Modify | `any[]`→typed public DB row |
| Handlers / lib / shared / context-server | Modify | Comment-hygiene ephemeral-id cleanup |
| Advisor + code-graph + deep-loop scripts | Modify | Comment-hygiene + shell strict-mode |
| `bin/` + hooks + scripts | Modify | Shell `set -euo pipefail` strict-mode |

## 8. EDGE CASES

### Data Boundaries
- **Finding does not reproduce**: skip and log refuted; do not edit.
- **Allowed comment matched as P0**: leave untouched (Feature-catalog comments, TSDoc examples).

### Error Scenarios
- **dist/ flagged as stale**: rebuild via `npm run build`, do not hand-edit compiled output.
- **Empty fixer seat output**: salvage from the seat transcript; do not fabricate a fix.

### Concurrent Operations
- **Multiple fixer seats**: file-disjoint scopes prevent write collisions; reduce centrally after.
