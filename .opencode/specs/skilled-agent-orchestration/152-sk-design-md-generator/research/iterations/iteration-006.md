# Iteration 006 — Track C (deepseek)

## Focus
Remove fabrication incentives from format-spec/style-guide (conditional named-principle, AP-29, ABSENT-stamp escalation)

## Findings
1. **[P0] C3: mandatory-philosophy solicits fabrication** — Three format-spec clauses demand named philosophy/narrative regardless of whether tokens.json supplies evidence: (a) §9 Depth & Elevation requires 'Named principle -- state the depth philosophy upfront using one of these categories' (line 542) unconditionally; (b) §10 Motion System is REQUIRED even when no motion data exists — the minimal fallback STILL demands a 'Motion philosophy' sentence (line 589-592); (c) §1 Visual Theme & Atmosphere requires 2-4 'Named design principles' (line 122-128) plus 'Comparative framing -- at least one sentence in the prose must use a comparative structure' (line 130). With zero tokens backing a 'chromatic depth' or 'whispered authority' claim, the model MUST fabricate to satisfy the mandatory clause.
   - Recommendation: In design_md_format.md: modify §9 line 542 from 'Named principle — state the depth philosophy upfront using one of these categories (or a custom name)' to 'Named principle — REQUIRED ONLY when depth-related tokens (shadow stacks, RGBA values, blur radii) exist in tokens.json. When absent, stamp: **Depth philosophy:** ABSENT — no shadow/elevation data captured.' Same pattern for §10 line 589-592 (Motion philosophy) and §3 line 122-130 (Named design principles + comparative framing): prefixed with 'CONDITIONAL ON EVIDENCE. If tokens.json contains no [domain] data, stamp ABSENT and omit the prose.' In design_md_prompt_template.md §2 TEMPLATE: add rule 7: 'Named principles, philosophy paragraphs, and comparative framing are CONDITIONAL: only include when directly grounded in a measurable token value from tokens.json. If no backing data exists, stamp the section ABSENT rather than fabricating.'
   - Evidence: design_md_format.md:§9 line 542; §10 lines 589-592; §3 lines 122-130; design_md_prompt_template.md:§2
2. **[P0] C1: missing anti-fabrication clause in SKILL.md** — SKILL.md §4 NEVER rules (lines 297-304) only prohibit numeric-value fabrication: 'NEVER estimate, round, or invent a hex, pixel, font-weight, shadow, or radius value.' The ALWAYS rules only mandate copying numeric CSS values verbatim. There is ZERO language prohibiting the model from asserting design relationships, causal claims, philosophical interpretations, named principles, or comparative claims about other systems — all of which are equally hallucinatory and equally unfalsifiable against tokens.json. The Cardinal Fidelity Rule (line 240) covers 'every hex, pixel, font-weight, box shadow...' — purely quantitative. Nothing covers qualitative assertions.
   - Recommendation: In SKILL.md §4 NEVER rules, add NEVER #6: 'NEVER assert a design relationship, causal explanation, named principle, design philosophy, comparative claim about other systems, or interpretive narrative that cannot be traced directly to a specific measurable value in tokens.json. If tokens.json contains only raw CSS values without intent/rationale, stamp those sections ABSENT rather than fabricating the WHY.' This clause creates a single, testable rule: every qualitative claim must cite at least one supporting token key.
   - Evidence: SKILL.md:§4 lines 240, 289, 297-304
3. **[P0] C11: style-guide/format-spec contradiction** — Style guide §10 Information Density (line 347) states: 'A paragraph with zero values is an opinion. A paragraph with one value is an observation. A paragraph with 2-4 values is documentation.' This is the 'deletion test' — prose without measurable values is opinion and should be deleted. The format spec (design_md_format.md) simultaneously MANDATES philosophy paragraphs (Depth §9, Motion §10, Visual Theme §1 named principles) that routinely contain zero token-extractable values — they are pure narrative. The result is a direct contradiction: the format spec requires paragraphs that the style guide classifies as 'opinion' and instructs the model to delete.
   - Recommendation: Resolution: The format spec wins for section existence (it defines the contract), but the style guide's density rule wins for CONTENT of those sections. Amend the style guide §10 deletion test to add: 'Exception: Required philosophy/narrative sections (Depth §6, Motion §6.5, Visual Theme §1) use an EVIDENCE-BACKED test instead of a value-count test: each claim must cite at least one specific token key from tokens.json. A paragraph with zero token citations is still opinion — stamp ABSENT.' Simultaneously amend design_md_format.md §9, §10, §3 to mark philosophy paragraphs CONDITIONAL ON EVIDENCE with stamp-ABSENT fallback (see C3). This eliminates the contradiction by making BOTH docs agree that philosophy requires evidence.
   - Evidence: writing_style_guide.md:§10 line 347; design_md_format.md:§9 line 542, §10 lines 589-592, §3 lines 122-130
4. **[P1] C2: writing_style_guide.md is the primary fabrication driver** — The writing_style_guide.md is ALWAYS-loaded (SKILL.md §2 Resource Loading Levels table, line 103). It is the most dangerous resource for fabrication because four sections actively solicit interpretive invention beyond tokens.json: §15 Comparative Framing Pattern (lines 529-586) provides sentence templates for claiming what 'most systems' / 'the conventional approach' does — knowledge the model does not have from a single-site crawl; §16 Named Principle Pattern (lines 589-631) declares 'Unnamed decisions get forgotten' and provides a naming formula with 10+ examples, encouraging the model to assign evocative names to patterns it merely observes; §17 Frequency Interpretation Pattern (lines 634-661) frames raw frequency as 'noise' and solicits 'narrative of intent' — i.e., fabricate WHY; §18 Intent Narration Pattern (lines 664-691) explicitly bridges 'the gap between WHAT a system does and WHY' — the WHY cannot be extracted from CSS.
   - Recommendation: Down-weight from ALWAYS to CONDITIONAL (EXTRACT_WRITE only — already the case, but reinforce). More critically: prepend a FABRICATION GATE preamble to the doc: 'ALL named principles, comparative framing claims, intent narratives, and frequency interpretations MUST cite at least one specific token key from tokens.json as the basis for the claim. When tokens.json provides raw values but no design rationale, document the values in their reference format (tables, component specs) and stamp qualitative sections ABSENT. Never assert what another design system or convention does without a reference.' This neutralizes the fabrication-incentivizing passages without removing the legitimate guidance on prose quality.
   - Evidence: writing_style_guide.md:§15, §16, §17, §18; SKILL.md:§2 Resource Loading Levels line 103
5. **[P1] C9: AP-29 Interpretive Fabrication anti-pattern** — New anti-pattern: asserting a design relationship, cause, design intent, or qualitative consistency that is not present in the raw tokens.json but is invented from the model's training data or the style guide's narrative-solicitation patterns. Two canonical examples: (1) Gradient-depth: tokens.json records a gradient value `linear-gradient(135deg, #1a1a2e, #16213e)` but the model writes 'The gradient creates atmospheric depth by transitioning from cool navy to deeper midnight tones' — the gradient endpoints exist in tokens, but the claim that it 'creates depth' or the description 'atmospheric' / 'navy' / 'midnight' is fabrication. (2) Focus-consistency: tokens.json records `outline: 2px solid #533afd` on buttons but the model writes 'The consistent 2px focus ring across all interactive elements ensures a unified accessibility experience' — the value exists but the claim of cross-element consistency and the intent ('ensures unified experience') is not verifiable from the data.
   - Recommendation: Append to anti_patterns.md after AP-28 (line ~721):

### AP-29: Interpretive Fabrication
**Description**: Asserting a design relationship, causal explanation, design intent, named principle, philosophical meaning, or comparative claim about other systems that is not present in tokens.json. The raw CSS values exist but the interpretive layer is invented.
**Bad Example**:
```markdown
The gradient creates atmospheric depth by layering cool navy tones over midnight undertones, establishing a premium, technical feel.
```
**Why It's Wrong**: tokens.json contains `linear-gradient(135deg, #1a1a2e, #16213e)` — the hex values exist, but 'atmospheric depth,' 'cool navy,' 'midnight undertones,' 'premium,' and 'technical feel' are all invented. The model has no evidence for any of these claims.
**Correct Approach**:
```markdown
- Hero background: linear-gradient(135deg, #1a1a2e 0%, #16213e 100%) [SOURCE: tokens.colorTokens.gradient.hero]
```
State the value and its source token key. Do not add interpretive intent, mood descriptors, or design rationale absent from the extraction data.
**Detection**: For every qualitative claim (narrative sentence not composed of CSS property: value pairs), check: does this claim cite at least one specific token key from tokens.json as the basis for the assertion? If not, flag AP-29.
   - Evidence: anti_patterns.md:append after AP-28 (line ~721-722)
6. **[P1] C10: escalate-if and stamp-ABSENT for missing backing tokens** — Currently SKILL.md §4 ESCALATE IF (lines 305-311) covers extract failures, dark-mode ambiguity, L2/L3 boundary straddlers, and extractor-bug hex mismatches — but has NO path for when required philosophy/narrative sections have zero backing token data. The prompt template (design_md_prompt_template.md §2) also has no 'if absent, stamp ABSENT' fallback. The result: the model fabricates because saying 'I have no data for this required section' is not a supported output path.
   - Recommendation: In SKILL.md §4 ESCALATE IF, add item 6: 'ESCALATE IF a required section (e.g., §1 named principles, §6 depth philosophy, §6.5 motion philosophy) lacks any backing token data in tokens.json. Offer to either: (a) stamp the section ABSENT with a note that extraction captured no data for this domain, or (b) manually provide the design rationale.' In design_md_prompt_template.md §2 TEMPLATE, add rule: 'Sections §1 (named principles/comparative framing), §6 (depth philosophy), and §6.5 (motion philosophy) are CONDITIONAL ON EVIDENCE. When tokens.json lacks data for a required philosophy subsection, write the heading then stamp: **ABSENT** — no [domain] data captured. Do not fabricate.' In design_md_format.md, add to §22 Global Formatting Rules: 'Philosophy and narrative sections: when no backing token data exists, stamp the subsection ABSENT rather than fabricating. The heading must still appear; the content is a single line: **ABSENT** — extraction captured no [domain] data.'
   - Evidence: SKILL.md:§4 lines 305-311; design_md_prompt_template.md:§2 lines 33-57; design_md_format.md:§22

## Questions Answered
- All mandatory philosophy/named-principle clauses in the format spec (Depth §9, Motion §10, Visual Theme §1) are unconditional — they demand narrative REGARDLESS of data presence, creating a systemic fabrication incentive.
- SKILL.md's NEVER rules and cardinal fidelity contract cover ONLY quantitative values (hex, px, weight, shadow, radius). No clause prohibits asserting qualitative relationships, causes, or design intent not in tokens.json.
- The writing_style_guide.md is the primary fabrication driver: always-loaded, and §15-§18 actively solicit interpretive claims (comparative framing, named principles, intent narration, frequency interpretation) beyond what CSS extraction can yield.
- Style guide §10's 'deletion test' (paragraph with zero values = opinion) directly contradicts format spec's mandatory philosophy paragraphs. Both are always-loaded. The contradiction is unresolved.
- A missing anti-pattern (AP-29) and a missing stamp-ABSENT/escalate path leave the model with no supported alternative to fabrication when data is absent.
