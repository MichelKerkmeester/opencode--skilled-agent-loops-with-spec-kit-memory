# v1.0.0.0 — create-benchmark packet established

Initial release of `create-benchmark` as one of the ten workflow-only packets under the `sk-doc` parent hub. The packet promotes a shipped spec packet's curated MCP benchmark evidence into a consuming skill's `mcp_server/benchmarks/benchmark-<YYYY-MM-DD>/` folder: a ten-section `benchmark_report.md`, a `SOURCE.md` wayfinding pointer back to the authoritative spec packet, and the supporting `results.csv` / `per-probe.jsonl` / `runtime-measurements.md` artifacts operators need to find the winner without leaving the skill tree.

## What Included

- `SKILL.md` — the primary contract: adoption gate, required package shape, the ten-step authoring workflow, the fixed ten-section `benchmark_report.md` report contract, date/naming conventions for dated folders, re-runs and retirements, authority hierarchy, and ALWAYS/NEVER/ESCALATE rules.
- `references/benchmark_creation.md` — deep overflow for benchmark promotion: case studies, sibling examples, and uncommon edge cases.
- `assets/benchmark/benchmark_report_template.md` — template for the ten-section `benchmark_report.md`.
- `assets/benchmark/source_template.md` — template for the `SOURCE.md` wayfinding file.

## Notes

This packet has no packet-local `graph-metadata.json` and no bound `/create:` slash command; the single advisor identity lives at the `sk-doc` hub root, and the shared create-quality-control backbone (validation, frontmatter templates, core standards) is loaded from `../shared/`.
