### P2, Suggestion

- **F006**: No explicit path boundary validation on file writes in frontmatter-version.mjs, `frontmatter-version.mjs:295`. The `applyVersion` function writes to `absFile` without checking that the path stays within the skills root directory. A caller passing `--paths` with a file containing absolute paths outside the skills tree could cause writes to arbitrary locations. Mitigated by: (1) the tool is a developer utility, not a network service; (2) `discoverInScope` constrains to the skills root; (3) the `--paths` flag reads from a file list that is typically generated by the tool itself. Low practical risk but worth a defensive check.
