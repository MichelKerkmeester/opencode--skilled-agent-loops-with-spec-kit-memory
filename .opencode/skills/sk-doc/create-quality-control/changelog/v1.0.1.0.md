# v1.0.1.0 - Smart-routing section conformance

**Released:** 2026-07-12

Normalized `SKILL.md` to the canonical section contract so the packaging check recognizes a discrete SMART ROUTING section.

## What Changed

- Split the merged `WHEN TO USE + SMART_ROUTING` header into two distinct H2 sections: `## 1. WHEN TO USE` and `## 2. SMART ROUTING`. The section matcher tests headings by substring, and the underscore form (`SMART_ROUTING`) did not contain the required `SMART ROUTING` token, so the packet read as missing its routing section.
- Moved the family-boundary prose (how this packet dispatches within the `sk-doc` hub and where advisor identity lives) under the new SMART ROUTING section; activation triggers and skip criteria stay under WHEN TO USE.
- Renumbered the remaining H2 sections contiguously (HOW IT WORKS through REFERENCES shift from 2-7 to 3-8). No intra-file cross-references depended on the old numbers.
- All prose, workflow steps, rules, success criteria, and references are preserved unchanged; this is a structural normalization only.
