# Iteration 7: Security Boundary and Untrusted Input Analysis

## Focus
Security analysis: untrusted document input, parser isolation, dependency audit, privacy guarantees.

## Findings

### F1: Untrusted Input Threat Model
| Threat | Vector | Mitigation | Severity |
|--------|--------|------------|----------|
| XSS in HTML report | Malicious `<script>` in source document | HTML-escape ALL output; CSP `script-src 'none'` | High |
| XXE (XML External Entity) | Malicious DOCX/XML with external entity refs | Disable external entity resolution in XML parsers | High |
| Path traversal | Filenames with `../` sequences | Resolve and validate all paths against allowed directories | High |
| ZIP bomb (DOCX) | Compressed docx expands to gigabytes | Size limit on decompressed content (100MB default) | High |
| PDF JavaScript | Embedded JS in PDF | Strip/ignore JavaScript in PDF metadata | Medium |
| `javascript:` URLs | Links in HTML/Markdown | Strip `javascript:` and `data:` URI schemes from links | Medium |
| Resource exhaustion | Very large documents | Configurable max file size (default: 50MB); streaming where possible | Medium |
| Symlink attacks | Symlink pointing outside snapshot directory | Resolve realpath; refuse to follow external symlinks | Medium |
| ReDoS (regex DoS) | Maliciously crafted input for regex engines | Avoid complex user-data regex; timeout on diff computation | Low |

### F2: Parser Isolation Strategy
- **cheerio**: Default parser (htmlparser2) does NOT execute scripts — safer than jsdom
- **mammoth**: Does NOT execute Word macros; converts to HTML only. External file access disabled by default.
- **remark**: Pure text→AST parser; no code execution
- **pdf-parse**: Text extraction only; no rendering engine
- No parser runs with elevated privileges; all operate on file content only

### F4: Privacy Guarantees (v1)
- Zero network requests generated by the core library
- No telemetry, analytics, or crash reporting
- No document content leaves the local machine
- Snapshot storage at user-controlled path
- No cloud dependencies or API keys required
- The HTML report is fully self-contained (no CDN assets)
