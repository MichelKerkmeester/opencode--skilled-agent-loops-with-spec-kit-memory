# system-skill-advisor v0.5.0 — Prompt-knowledge drift guard extended + CI/hook-wired

**Released**: 2026-06-03
**Status**: Feature — the `check-prompt-quality-card-sync.sh` guard gains three structural checks and is now enforced on commit + in CI.

---

## Summary

The duplication guard previously checked only the framework + CLEAR tables across the 5 cli-* cards, and ran manually. It now enforces the full "one home per fact" invariant for the 3-layer prompt-knowledge architecture and runs automatically:

- **CHECK 2 — Tier-3 pointer-only**: no `cli-*/SKILL.md` re-enumerates the canonical escalation triggers (catches the precedence-drift class); each must point to the canonical card.
- **CHECK 3 — registry/profile/_index completeness**: every adopted model (a `model-profiles.json` entry with `recommended_frameworks`) has a profile AND an `_index.md` row, and every profile maps back to a registry id (catches zero-hub-weight entries). Unadopted stubs are exempt.
- **CHECK 4 — discovery reachability**: every adopted model is reachable by name in each dispatching executor's `graph-metadata.json` (catches the model-unreachable-by-name class, e.g. qwen3.6).
- **Wiring**: the guard now runs as a blocking gate in the pre-commit orchestrator (`.opencode/scripts/git-hooks/pre-commit`, bypass `SPECKIT_SKIP_CARD_SYNC=1`) when a prompt-knowledge surface is staged, and in CI (`.github/workflows/prompt-card-sync.yml`).

All checks are structural (pointer presence, table absence, registry membership, trigger membership) — no semantic/NLP matching.
