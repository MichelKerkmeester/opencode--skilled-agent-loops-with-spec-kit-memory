# v1.0.1.0 — Top-Level Install Guide

**Released:** 2026-07-16

---

## What's New

### Top-level INSTALL_GUIDE.md (front door)

- Added `INSTALL_GUIDE.md` at the packet root, bringing mcp-click-up to install-guide parity with the sibling packets (`mcp-chrome-devtools/INSTALL_GUIDE.md`, `mcp-figma/INSTALL_GUIDE.md`), which already carry a top-level guide.
- The front door promotes and reorganizes the content of `references/install_guide.md`: AI-first install block, component overview and architecture, prerequisites + cupt install, authentication (token and OAuth), optional Code Mode MCP configuration, verification, troubleshooting, and a resources table.
- No content was invented: every command and configuration block derives from `references/install_guide.md`, `SKILL.md`, and the mcp-servers READMEs.

### Link direction (deliberate)

- `references/install_guide.md` keeps its FULL content and remains the smart router's INSTALL-intent target (`SKILL.md §2 RESOURCE_MAP`), so all existing inbound links — from `SKILL.md`, `README.md`, `examples/README.md`, `mcp-servers/clickup-cli/README.md`, `mcp-servers/clickup-mcp/README.md`, `scripts/doctor.sh`, and `references/mcp_tools.md` — keep resolving to the same step-by-step guide they describe.
- The reference now ends with a short pointer note to the new front door, and the front door links back into the reference for the phase-by-phase validation ladder.
