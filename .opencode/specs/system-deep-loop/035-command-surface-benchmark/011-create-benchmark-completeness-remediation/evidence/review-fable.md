## 2. MISSING COVERAGE

Judged conservatively (lane-local evaluator/scorer/runtime code is correct boundary, not a gap):

1. **`command_benchmark_matrix.json` has no authored shape.** It is a run-selection *input* (scenario ids, legs, samples, gate counts) — the same artifact class as a Lane B profile, which create-benchmark DOES template — yet no template, field table, or guide section covers it. An author can produce every other 066 artifact from create-benchmark; this one they cannot without reading `run-command-behavior-matrix.cjs`. (Confirmed: no `command_benchmark` string in any create-benchmark file except the behavior guide §7 table row that names the file.)
2. **Lane C fixture-pair authoring is unrouted from the authoring home.** `deep-improvement/references/skill_benchmark/scenario_authoring.md` (public/private pair schema, anti-circularity tiers, contamination rules) is genuinely an *authoring* doctrine, and SKILL §10's NEVER-clause even says "templates here are for the `benchmark/README.md` index and INPUT fixtures/profiles only" — implying fixture inputs are in-scope — but neither SKILL §10 nor the storage guide links `scenario_authoring.md`. Asymmetric with Lane B, whose fixtures are fully covered. An author can still do the job (the lane operator guide chain reaches it), so this is a missing cross-link, not a hard gap.
3. **No "command benchmark" routing key/trigger.** SKILL §1 keyword triggers include `sk-doc-command` (conformance) but not "command benchmark"/"command-surface benchmark"; the §2 unknown-fallback checklist enumerates six families and never mentions the two-axis composition, so the one benchmark family fronted by its own `/deep:*` command is the only one create-benchmark cannot route by name.
4. Nothing else is missing: every other authoring surface that exists in system-deep-loop (behavior packages ×5, conformance package inputs, Lane B fixtures/profiles, Lane C storage/index, Lane A setup docs, MCP-promotion folders) has a template or guide inside create-benchmark, verified against disk.
