## 2. Adjudication Note

Every P0 candidate and every P1 candidate was re-read at the cited file:line and adversarially re-tested before final severity. Two candidate P1s were downgraded to P2 under the deep-review false-positive discipline, and they are the most important adjudication outcomes of this pass.

**Downgrade 1: correctness-9-1 (Lane B is model-blind), P1 to P2.** Opus surfaced that the wired Lane B loop scores each fixture's own reference content against that fixture's own criteria, never invoking a model, so the aggregate is a model-independent constant (empirically reproduced: aggregateScore 100, benchmark-pass, all rows 100). On cross-check this is NOT a new class. The 014 gpt-5.5 review already raised it as P0 ("Model benchmark never dispatches a model", correctness-1-2). Packet 015 dispositioned it as DOCUMENT-ACCEPT, refuted as an arbiter-upheld intended deferral traceable to 122/007 (raised there as DR-001-P1-001 "model-benchmark route never invokes the generic dispatcher"). Re-opening an arbiter-upheld by-design deferral as a fresh blocker would violate the adversarial discipline. The behavior runs end-to-end, writes a valid report, and corrupts nothing. The legitimate residual that opus DID add is a traceability gap: the shipped docs still assert Lane B "benchmarks a model" (SKILL.md:14,34,42,273) and "score model variants across fixtures" (auto YAML:6) while the wired and 5dim+llm paths both score the materialized reference, not a model candidate. That residual is recorded as a P2 doc-traceability advisory. correctness-9-2 (the unreachable LANE_MODEL_BENCHMARK resolver branch for dispatch-model.cjs) is the structural footprint of the same accepted deferral and stays P2.

**Downgrade 2: security-6-1 (criteria-grep arbitrary file read), P1 to P2.** The grep and grep_absent acceptance branches build path.join(cwdAbs, a.file) with no containment check, and the DEEP_AGENT_ALLOW_CRITERIA_EXEC=0 gate does not cover them. On re-read the documented guarantee at SKILL.md:279 is worded "refuse criteria-driven shell execution." A file read is not shell execution, so the gate's literal scope does not claim to cover it. The read result is bounded to a pass or match-count surfaced into the score, not exfiltrated. The DOCUMENT-ACCEPT trusted-author boundary at SKILL.md:279 explicitly scopes profiles and fixtures as operator-authored same-trust-domain content. The finding's own confidence was 0.6 and its own downgradeTrigger names exactly this boundary. Held at P2 as defense-in-depth.

**Why security-2-1 stays P1 while security-6-1 drops.** They look adjacent but differ at the load-bearing word. The bundle-gate path is execSync, which IS "shell execution," so SKILL.md:279 literally promises to cover it and does not, a genuine fail-open against a documented control. The grep path is a file read, outside the literal text of the gate. Same scorer, different severity, by the wording of the guarantee each one is measured against.

No finding was inflated to reach a verdict, and no P0 survived.
