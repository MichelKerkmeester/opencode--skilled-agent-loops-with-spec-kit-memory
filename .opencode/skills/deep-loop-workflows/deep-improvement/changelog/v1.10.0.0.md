## What Changed

### Router-benchmarkability â€” make a real skill measurable (phase 009)

- **Machine-readable router for `sk-code`.** Lane C's Mode A needs an `INTENT_SIGNALS` + `RESOURCE_MAP` router to replay. `sk-code` keeps its router in a referenced doc (`references/smart_routing.md`), not inline in `SKILL.md`, so the benchmark couldn't read it. Added a machine-readable router block (DEFAULT_RESOURCE + 14 intents + a RESOURCE_MAP covering every routable content file) to `smart_routing.md`, with a drift-guard regression test that fails if any mapped path goes missing on disk or any routable doc stops being covered.
- **Harness now follows a referenced router.** `router-replay.cjs` `parseRouter()` resolves a skill's router doc when the dictionaries are not inline in `SKILL.md` (`findReferencedRouterDoc`), and stamps the `routerSource`. `d5-connectivity.cjs` and `contamination-lint.cjs` thread the `skillRoot` through so the reference-following resolves correctly.
- **`DEFAULT_RESOURCE` is now a list.** The replay parser, the route builder, and the D5 connectivity coverage check accept a list of always-loaded resources (string form still supported), so a skill's routing preamble â€” the docs consulted on every route, not mapped to one intent â€” is modeled honestly and counts as covered rather than orphaned.
- **sk-code router tuned from its own benchmark.** sk-code always detects stack, detects phase, and consults the router before code work, so those three docs moved into `DEFAULT_RESOURCE` (clearing three orphan references and lifting D5 to 100), and the language-standards / config intents gained file-extension signals (`.ts`/`.py`/`.sh`/`.json` â€¦) matching how sk-code routes by target path â€” not by copying benchmark scenario text. Router-mode aggregate 31 â†’ 40, D2 29 â†’ 44, routed-intra failures 12 â†’ 5; the residual failures are the documented surface-flattening replay artifacts that live mode resolves.

### Lane C "Mode B" â€” live playbook benchmarking (phase 010)

- **The playbook is the corpus.** `load-playbook-scenarios.cjs` parses a skill's `manual_testing_playbook/` into normalized gold scenarios, tagged `routing` / `advisor` / `browser` (sk-code â†’ 24 scenarios). Gold is captured as authored, which surfaces routerâ†”gold drift.
- **One corpus, two trace-modes.** `run-skill-benchmark.cjs` scores the same scenarios in `router` (deterministic, the CI gate â€” real-gold D1-intra/D2/D3 instead of the previous empty-gold placeholders) and `live` (the operator default). `score-skill-benchmark.cjs` keeps a back-compat adapter for the legacy fixture shape and adds Aâ†”B divergence; `build-report.cjs` adds coverage, per-classKind, routed-out, divergence, and contamination sections.
- **Live executor.** `live-executor.cjs` dispatches a scenario as a routing-analysis prompt through `cli-opencode`, parses the real NDJSON event schema, captures activation + observed reads, and extracts the model's stated routing JSON across fenced / bare / prose shapes (cross-model robust).
- **Browser executor.** `browser-executor.cjs` runs browser-gated scenarios (MR/CB) through `bdg` (Chrome-only) with honest verdicts â€” real PASS where automatable, `PARTIAL-NEEDS-ARTIFACT` / `SKIP-NO-BROWSER` otherwise; never a fabricated PASS.
- **D4 usefulness ablation (approximate).** `d4-ablation.cjs` grades skill-on vs skill-off (`MK_SKILL_ADVISOR_HOOK_DISABLED=1` + a "don't read the skill" preamble, verified by `observedReads==0`) via the existing `gradeD4` grader; attribution is stamped `approximate` by design.
- **Auto-CREATE generator (opt-in, staged).** `playbook-generator.cjs` can author scenarios for a skill lacking them â€” opt-in only, writes to a `_generated_staging/` dir (never the live playbook), behind four validation gates (contamination, parser round-trip, structural, self-consistency).
- **`loop-host.cjs`** gained `--scenarios`, `--executor`, and `--playbook-dir`; the internal default stays `router` for deterministic direct runs.

### Test lane organization (file org)

- **Every Vitest suite now lives beside the lane it covers** (`agent-improvement/tests/`, `model-benchmark/tests/`, `skill-benchmark/tests/`, `shared/tests/`) instead of one flat `scripts/tests/`. Cross-lane fixtures and the suite index moved to `shared/tests/`. `vitest.config.mjs` include simplified to `*/tests/**/*.vitest.ts`.
- **References repointed:** `scripts/README.md`, the relocated suite/fixtures READMEs, the feature catalog, the manual testing playbook anchors, and the `/deep:skill-benchmark` command + `deep-improvement` agent Lane C note. Historical changelogs and deep-research/review iteration logs were left as point-in-time records.

### Review hardening (independent gpt-5.5 pass)

An independent fresh-context review (gpt-5.5-fast) of the Lane C code drove four fixes, each with a regression test where behavior changed:
- **D5 path-escape is separator-bounded.** The skill-root containment check used a bare `startsWith`, which a sibling like `<root>-evil/â€¦` could slip past; it now requires a path-separator boundary.
- **Live surface mismatch fails routing.** A live result whose observed surface disagrees with the scenario's expected surface now caps D1-intra below the pass line and reports a `surface-mismatch` funnel stage, instead of passing on incidental resource overlap. Router mode (no observed surface) is unaffected.
- **`--playbook-dir` is honored.** The flag was forwarded by loop-host but dropped by the orchestrator; it now threads through to the parser, with usage text and a focused test.
- **Comment hygiene.** Scenario IDs were lifted out of code comments into behavioral descriptions (kept only in executable branch/data values, where they are the runtime contract).
