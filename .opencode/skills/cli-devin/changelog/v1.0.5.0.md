This release applies the empirical winners from the 003-eval-loop optimization run (42 SWE 1.6 dispatches across 6 iterations) to cli-devin's prompt scaffolding. The SWE-1.6 Prompt-Quality Contract introduced in v1.0.2.0 (sk-prompt + pre-planning block) is now data-driven: **RCAF (Role/Context/Action/Format) is the empirically-best framework for SWE-1.6**, beating the STAR baseline by 33% and the BUILD-with-dense-pre-plan variant by 35%. Medium pre-plan density beats dense; standard bundle-gate language beats strict (verbose constraints HURT scores by pushing SWE 1.6 toward defensive output).

## What Changed

### SWE-1.6 default framework: STAR → RCAF

The v1.0.2.0 contract listed STAR, RCAF, and BUILD as equally-valid frameworks for SWE-1.6, with STAR carrying the ★ canonical marker. The 003 run (42 dispatches, 6 iterations, 7 fixtures grounded in documented SWE-1.6 failure modes) showed RCAF + medium pre-plan scored 0.5796 — 33% higher than STAR's 0.4357. RCAF's role anchor gives SWE 1.6 immediate framing without burning tokens on situation-setting; the result is tighter, more focused output.

The contract still requires sk-prompt + pre-planning + CLEAR 5-check. What changed is the default framework recommendation: RCAF first, STAR for narrative-heavy tasks where role framing doesn't fit, BUILD for multi-file refactors where scope boundaries dominate the prompt design.

---

### Bundle-gate language: do NOT tighten beyond "standard"

The v-005-build-strict-bundle-gate variant in the 003 run combined BUILD framework + dense pre-plan + smoke-run-required wording + aggressive anti-hallucination. It scored 0.4846 — LOWER than the v-004-rcaf-medium winner (0.5796). The hypothesis was that more explicit constraint language would push SWE 1.6 toward more disciplined output; the empirical result was the opposite. Verbose constraints push SWE 1.6 toward defensive output: more disclaimers, more "I can't do this in this session" caveats, less direct code.

The new Rule #12 wording explicitly tells callers: do not tighten bundle-gate language beyond "standard". Trust the framework; don't pile on imperatives.

---

### Medium pre-plan density: do NOT go dense

v-002-build-dense-preplan (dense pre-plans with 4+ steps, full I/O contracts per step, stop conditions per step) scored 0.4293 — the WORST of all 6 variants tested. Dense pre-planning didn't help; SWE 1.6 followed the structure mechanically but didn't gain quality from it. Medium density (3 ordered steps, one acceptance criterion per step, one verification command per step) is the sweet spot.

---

### Anti-hallucination wording: framework dominates

v-003-anti-hallucination-strong (STAR + medium + aggressive anti-hallucination wording with explicit allowlist enforcement and "reject plausible-but-fake flags" language) scored 0.4961 — only 0.06 above STAR baseline. RCAF without any anti-hallucination tuning scored 0.5796 — 0.08 above v-003. Framework choice is **2.4× more impactful than anti-hallucination wording** for the cli-devin failure modes tested. Anti-hallucination wording is still useful as a backstop for high-risk fixture clusters (CLI flag generation, library symbol references), but it's not the primary lever.

## Upgrade

**No invocation breaking changes.** Existing v1.0.4.X dispatches keep working — v1.0.5.0 changes are framework-recommendation refinements within the existing contract. The contract's structural requirements (sk-prompt + CLEAR 5-check + pre-planning block) are unchanged.

**Operationally significant for SWE-1.6 callers**: when composing a new SWE-1.6 dispatch via sk-prompt, prefer RCAF unless the task clearly fits STAR (narrative-heavy) or BUILD (multi-file refactor). Use medium pre-plan density (3 steps). Do not pile on strict bundle-gate language. The 003-eval-loop synthesis.md is the canonical reference for which patterns won and why.

**Empirical scope caveat**: the 003 run measured prompt-text quality on 7 fixtures with documented SWE-1.6 failure modes. The fixtures had a known limitation — SWE-1.6 `--print` mode returns markdown text rather than files-on-disk, so the D1 Acceptance dimension was capped to 0.0 on 4/7 fixtures via D2 hard-gate. Variant differentiation came from D2 (bundle-gate text shape) + D3 (path/cwd discipline) + D4 (hallucination grader) + D5 (pre-plan structure). The RCAF win is robust on those dims; a follow-on packet adding file-extraction-from-markdown would unlock the full D1 signal and may revise the ranking. For now, RCAF + medium + standard is the data-driven default.

**No 4-runtime mirror required**: cli-devin is a `.opencode`-only skill, not an agent. The agent-mirror rule (memory: `feedback_new_agent_mirror_all_runtimes.md`) applies to top-level decision-making entities (agents), not to tooling extensions (skills). See `specs/skilled-agent-orchestration/114-cli-devin-swe16-prompt-optimization/004-skill-uplift/decision-record.md` ADR-001 for the full distinction.
