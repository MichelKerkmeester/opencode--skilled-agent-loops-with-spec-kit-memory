## Suggested Fix Order
1. Start with P0 containment: shared CLI supervisor, dispatch-time recursion guard, stale-lock/deep-review serialization. These are the controls that prevent new process storms while deeper fixes are being built.
2. Then stabilize state recovery: atomic JSONL, delta repair, executor `kind`, and reducer behavior. Without this, cleanup failures remain hard to diagnose.
3. Then add the expected-daemon and host-memory visibility layer. That gives operators a safe sweep command and a pressure model that matches Apple Silicon reality.
4. Finally harden sidecars, in-process caches/queues, hooks, and external tooling cleanup with targeted regression tests.

## Residual Risk
Most evidence is source-level, not live-process proof. That is appropriate for this research-only packet, but the implementation packet should treat verification harnesses as first-class work. The failures that matter are interruption paths: child ignores signal, parent dies, JSONL append is interrupted, stale lock survives, or cleanup runs while expected daemons are warm.
