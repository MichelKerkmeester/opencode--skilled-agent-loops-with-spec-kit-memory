## [**1.6.0.0**] - 2026-05-19

Benchmark folder creation becomes the sixth sk-doc mode. The benchmark format mechanics that lived across three locations in system-spec-kit and sk-doc consolidate into a single canonical reference at references/benchmark_creation.md. A new SOURCE.md scaffold joins the benchmark_report_template.md in assets/benchmark/. The smart router gains BENCHMARK intent routing, and both SKILL.md and README.md update to reflect the six-mode surface.

> Scope: benchmark folder creation mode (Level 2 packet — packet citation omitted per skill-docs-no-spec-refs convention).

---

## New Features

### Benchmark Folder Creation as Mode 6 (Packet 006)

The new mode scaffolds skill-local mcp-server/benchmarks/benchmark-<YYYY-MM-DD>/ folders that promote curated bake-off results from a spec packet into the consuming skill. Two templates drive the work: benchmark_report_template.md for the ten-section curated narrative with headline metrics and recommendations, and source_template.md for the SOURCE.md wayfinding pointer that maps each operator question to the right file in the originating spec packet. The smart router BENCHMARK intent routes to this mode when keywords like benchmark creation or benchmark folder appear.

&nbsp;

### Consolidated benchmark_creation.md reference

The trio of benchmark format docs collapsed into one canonical reference following the sk-doc *_creation.md pattern. The references/benchmarks/FORMAT.md file at 384 LOC, the references/benchmarks_format.md file at 205 LOC, and the assets template merged into references/benchmark_creation.md at 414 LOC with 10 numbered sections from OVERVIEW through RELATED RESOURCES. The consolidated doc generalizes the Section 5 example and Q7 to remove stack-specific references like Ollama, cat-24/409 and sentence-transformers so the guidance is portable across MCP skills. The mid-flight redesign was an operator decision after the first author pass.

&nbsp;

### New SOURCE.md scaffold (assets/benchmark/source-template.md)

A new fillable scaffold at 135 LOC provides the complete SOURCE.md structure for wayfinding from the skill-local benchmark folder back to the authoritative spec packet. The scaffold covers frontmatter, overview, spec packet location, a when-to-read-what table that maps questions to files, an evidence file map, follow-on packets and update rules. This is a wayfinding doc, not a duplicate of the benchmark report. It answers operator questions like which candidate won, is the production default safe, and what fixture was used by pointing to the right file in the originating spec packet.

&nbsp;

### Smart router updates

The BENCHMARK intent keywords extended to include benchmark creation, benchmark folder and source.md template while removing the deprecated benchmarks format keyword. The RESOURCE_MAP entry for BENCHMARK now loads three resources instead of two: references/benchmark_creation.md, assets/benchmark/benchmark-report-template.md and assets/benchmark/source-template.md. The Section 2 Resource Domains bullet list added assets/benchmark/ as a tracked domain for benchmark folder templates.

## Migration Notes

Operators with code that referenced the old benchmark format paths must update to the new canonical location. The references/benchmarks/FORMAT.md path, the references/benchmarks_format.md path, and both sibling skill FORMAT.md symlinks no longer exist. Update any references to `.opencode/skills/system-spec-kit/mcp-server/benchmarks/FORMAT.md` or `.opencode/skills/sk-doc/references/benchmarks/FORMAT.md` or `.opencode/skills/sk-doc/references/benchmarks_format.md` to point at `.opencode/skills/sk-doc/references/benchmark_creation.md`. There is no symlink fallback. Anything reading mcp-server/benchmarks/FORMAT.md directly will get a file-not-found error.
