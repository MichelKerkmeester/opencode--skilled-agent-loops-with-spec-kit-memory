# target-profiles: intentionally empty (dynamic-only)

---

## 1. OVERVIEW

This directory is **intentionally empty** — it holds only `.gitkeep`. Agent improvement runs exclusively in dynamic mode: `scripts/agent-improvement/generate-profile.cjs` derives a fresh target profile on the fly from the agent's own structure and rules, and `scripts/agent-improvement/score-candidate.cjs` applies the 5-dimension scoring against it. Because every run regenerates its profile, no static profile files are committed here. The directory exists so the `profileCatalog` (in `../target-manifest.jsonc`) and `targetProfiles` (in `../improvement-config.json`) path references stay consistent.

Current state:

- Contains only `.gitkeep`; no static profile files are shipped.
- `../target-manifest.jsonc` sets `dynamicProfileEnabled: true`, an empty `targets` array, and points `profileCatalog` here.
- A runtime profile would be generated by `../../../scripts/agent-improvement/generate-profile.cjs`, not written into this directory.
- Static benchmark profiles for the model-benchmark lane live elsewhere, under `../../model-benchmark/benchmark-profiles/`.

---

## 2. BOUNDARIES AND FLOW

| Boundary | Rule |
|---|---|
| Write policy | No static profiles. The catalog stays empty (`.gitkeep` only); dynamic mode never persists a profile here. |
| Ownership | Path is referenced by `target-manifest.jsonc` (`profileCatalog`) and `improvement-config.json` (`targetProfiles`) for consistency only. |
| Exports | None. Nothing imports from this directory. |
| Scope | Agent-improvement profiles only. Model-benchmark profiles belong to `../../model-benchmark/benchmark-profiles/`. |
