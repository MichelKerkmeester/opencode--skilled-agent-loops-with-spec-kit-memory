## New Features

#### Async file I/O with closure-scoped lazy-mkdir guard

The advisor's metrics persistence in `mcp_server/lib/metrics.ts` used synchronous `mkdirSync` plus `writeFileSync` to flush diagnostic and outcome records, which blocked the event loop on every advisor invocation. The hot path is now `await mkdir` plus `await writeFile` from `fs/promises`, gated by a closure-scoped `dirReady = false` flag that ensures the directory check runs exactly once per process. The two persistence entrypoints (`persistAdvisorHookDiagnosticRecord` and `persistAdvisorHookOutcomeRecord`) are now async functions that await the underlying writes. Bounded-retention semantics, read the file, slice the last N records, rewrite the file, are preserved per the council's section 10.1 mandate, because making the writes fire-and-forget would have allowed unbounded growth between rotations.

&nbsp;

#### Opt-in debug logging via `SKILL_ADVISOR_DEBUG`

Advisor diagnostics used to flush on every invocation, even when no one was reading them. The new `SKILL_ADVISOR_DEBUG=1` enable gate keeps the persistence hot path quiet by default and only writes diagnostic records when an operator has explicitly opted in. Outcome records remain unconditional because they are part of the bounded-retention metrics contract.

&nbsp;

#### 4-tier configuration precedence in `mk-skill-advisor.js`

The plugin used to read configuration only from `rawOptions` passed at construction. Operators who wanted to override advisor behavior outside of the OpenCode config had no surface to do so. The plugin now loads configuration through `loadConfig()` with a 4-tier precedence: `rawOptions` wins first, then a JSON config file at `~/.config/opencode/plugin/mk-skill-advisor.json`, then 7 new `MK_SKILL_ADVISOR_*` environment-variable fallbacks, then hardcoded defaults. The legacy `SPEC_KIT_PLUGIN_NODE_BINARY` env-var is preserved as a backward-compat fallback for the node-binary path. The existing env-var disable handling at lines 35 through 39 of the plugin is left untouched so operators who only ever used the disable switch see no behavior change.

&nbsp;

#### Awaited config promise at module init

The 4-tier config precedence is loaded as a `configPromise` at module init. The plugin factory now awaits that promise before option normalization, so configuration is fully resolved before the plugin reports its option surface to OpenCode. The council's section 10.2 review flagged the original race-prone path where a synchronous factory could return before the config file finished loading. That race no longer exists.

Both packets ran through a 5-seat deep-ai-council deliberation before any source change. ## Backward Compatibility

This release is backward-compatible for every existing user. Operators who passed configuration via `rawOptions` only continue to work, `rawOptions` still wins at the top of the precedence chain. Operators who used environment variables only continue to work, the legacy `SPEC_KIT_PLUGIN_NODE_BINARY` fallback is preserved and the new `MK_SKILL_ADVISOR_*` variables are additive. Operators who used neither continue to work, the hardcoded defaults are unchanged from v0.2.0. The metrics file format on disk is unchanged. Existing files are read and rewritten through the new awaited code path without schema changes. Source commits: 65e6f0479, 45c5a7118.

---

## Upgrade

No migration required. Pick up the new release and the plugin will discover any pre-existing `~/.config/opencode/plugin/mk-skill-advisor.json` on first invocation. Set `SKILL_ADVISOR_DEBUG=1` if you want diagnostic records persisted. Leave it unset for the quiet default.
