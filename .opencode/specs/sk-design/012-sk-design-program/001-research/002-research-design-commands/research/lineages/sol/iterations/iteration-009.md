# Iteration 9: `/interface:foundations` Creation Template

## Focus
Specialize the shared scaffold for color, typography, layout, spacing, hierarchy, responsive adaptation, themes, and token systems.

## Findings
1. **Brief intake:** target surface, Brand/Product register, visual problem, content density, existing tokens/components, accessibility target, viewport/theme matrix, preserve constraints, and requested output (direction, tokens, or implementation-ready spec). Route runtime animation, interaction feel, or empirical browser diagnosis elsewhere. [SOURCE: .opencode/skills/sk-design/design-foundations/SKILL.md:32-40]
2. **Exemplar grounding:** owned tokens and shipped system are primary evidence; an external exemplar is useful only for one named axis such as editorial hierarchy, density, or palette relationship. Never copy a whole style identity. Record transformed principle and rejected mismatch. [SOURCE: .opencode/skills/sk-design/design-foundations/SKILL.md:77-81]
3. **Scaffolded flow:** `Context Manifest -> axis classification -> static-authoring boundary -> foundations mode -> axis-specific reference load -> semantic token scaffold -> component/application mapping -> responsive/theme states -> contrast/readability/overflow checks -> optional rendered proof -> accepted handoff`. [SOURCE: .opencode/skills/sk-design/design-foundations/SKILL.md:48-69]
4. **Creative build:** require a compact semantic scaffold covering only applicable layers: primitive/semantic/component tokens, type roles, spacing rhythm, grid/measure, hierarchy, state/theme deltas. Explicitly connect each choice to subject/job instead of emitting generic token inventories.
5. **Visible output:** `Foundations Read`, `Resolved Constraints`, `Grounding Record`, `Token/Type/Layout Scaffold`, `Responsive and Theme Matrix`, `Validation Evidence`, `Open Decisions`, and accepted handoff. Runtime measurements are labeled unverified until produced by a browser/build executor.

## Prompt Template
```text
Create the static visual foundations for {target}. Resolve {register, visualProblem, density, existingTokens, accessibilityTarget, viewportThemeMatrix, preserve, output}. Classify requested axes and route dynamic behavior or measured diagnosis out of this command. Ground first in owned tokens; use an external reference only for a named axis and record the transformed principle. Load sk-design mode foundations and the minimum axis references. Produce a brief-specific semantic token/type/layout scaffold, map it to real content/components, define responsive/theme deltas, and run static contrast/readability/overflow checks. Distinguish authored direction from measured evidence. Return the eight output blocks and hand off only accepted static decisions.
```
