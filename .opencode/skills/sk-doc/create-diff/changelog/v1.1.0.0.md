# v1.1.0.0 - Functional Engine

**Released:** 2026-07-15

Turns `create-diff` from a preview-gated orchestration stub into a working, self-contained document diff mode. The comparison engine now ships with the packet as skill-owned scripts, so the mode produces real before/after reports without waiting on an external package.

## What Changed

- **Added `scripts/create_diff.py`** — the comparison engine (Python 3 standard library):
  - Format extraction for plain text and Markdown (full fidelity), HTML and DOCX (visible/structural text), and text-layer PDF (via `pdftotext`/`pypdf`/`pdfplumber` when available), each with explicit fidelity tiers and warnings.
  - Honest refusal of unsupported input: binary office/media/archive formats (`.xlsx`, `.pptx`, `.png`, `.zip`, …) and any file with binary content exit `3` rather than being raw-byte "text"-diffed into a meaningless report; unknown *text-like* extensions still fall back to text with a stated assumption.
  - Deterministic line-level diff with inline word-level highlighting and a cheap move heuristic.
  - Content-addressed baseline snapshots (`snapshot`/`compare`), an explicit-pair fallback (`compare-pair`), and lifecycle commands (`status`, `cleanup`, `capabilities`).
  - A self-contained, zero-JavaScript, accessible HTML report (unified and side-by-side views) with a restrictive CSP, escaped source content, and dark/light support. Honors `SOURCE_DATE_EPOCH` for byte-reproducible output.
- **Added `scripts/validate_report.py`** — asserts a report is safe and self-contained (doctype, `lang`, CSP meta, zero `<script>`, no inline handlers, no remote references); inspects real markup only, so escaped hostile text in document content does not false-positive.
- **Added `references/`** — capabilities/fidelity, workflow, CLI reference, accessibility contract, and a worked example behind a route-map.
- **Added `assets/fixtures/`** — a runnable before/after worked example.
- **Added `feature-catalog/`** — canonical capability inventory (root `feature-catalog.md` + 8 per-feature files across 3 categories), authored to the create-feature-catalog template.
- **Added `manual-testing-playbook/`** — operator-facing manual validation package (root `manual-testing-playbook.md` + 11 deterministic scenario files across 3 categories), authored to the create-manual-testing-playbook template.
- **Rewrote `SKILL.md` and `README.md`** — functional contract; removed the "engine pending" preview language; added INTEGRATION POINTS and RELATED RESOURCES sections; aligned to the create-* canon and wired in the feature catalog and playbook.

## Architecture Note

The engine is embedded in the skill as Python-stdlib scripts rather than a separate portable npm package (a documented amendment to the original wrapper-only design — see `.opencode/specs/sk-doc/999-create-diff-mode/001-research-and-requirements/decision-record.md`, ADR-002). This makes the mode work with zero install; richer formats degrade honestly instead of guessing.

## Notes

Still no packet-local `graph-metadata.json`/`description.json`. The `/create:diff` command remains optional and unbuilt; the mode routes via advisor aliases and direct script invocation. Scanned/image-only PDF OCR remains out of scope.
