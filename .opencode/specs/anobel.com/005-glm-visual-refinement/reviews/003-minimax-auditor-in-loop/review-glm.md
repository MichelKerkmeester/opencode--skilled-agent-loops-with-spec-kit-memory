- **Generator ≠ auditor is correctly settled and load-bearing.** GLM confabulates audits (the fake orange CTA / `#cccccc`), so an external critique channel is the only trustworthy signal. Keeping MiniMax as auditor and GLM as repairer is the right Self-Refine variant.

- **Failure-only routing (SKIP_SHIP on the 27 sentinels) + max 3 fixes/tile + env-switch rollback** are disciplined and keep the blast radius small. The rollback rule (≤1 conversion or false_fix > 2/18) is a real kill criterion, not theater.

## Gaps / risks / errors (specific)

1. **The gate is weakest precisely where A4's targets live (RC-2).** Your own ledger marks node-on-node collision as *refinement* ("no gate computes bounding-box overlap"), while RC-1 overflow is hard-fact. So the deterministic gate verifies the *overflow* fixes reliably but is blind to the *node-collision* fixes — and RC-2 is a primary failure on the 2D cohort (oci-4=6 absolutes, accountbeheer-4=4). The adopt-if gate therefore has a hole exactly over the defect class A4 exists to repair. A round-2 tile that "fixes" overflow by cramming nodes can pass geometry (no overflow) yet still be visually broken, and MiniMax is the known false-negative on subtle clips (your §3). **Net: your strongest gate is orthogonal to your hardest defect.** This undermines the `false_fix_rate ≤ 1/18` target's measurability, not necessarily its achievement.

2. **Same auditor finds *and* verifies — confirmation loop.** MiniMax diagnoses round-1 and then rescores round-2 as the SHIP signal. Any systematic MiniMax leniency (e.g., it scores tiles it "already explained" more charitably) is not caught by this design. You're using one model for both ground-truth and acceptance. Add either a second auditor spot-check on the converted cohort or a human spot-check — there is no human/ground-truth criterion anywhere in REQ-001…006 for an 18-tile batch where that's trivially affordable.

3. **`classifyFixes` regex has no precision/recall target — it's an unverified brittle NLP step load-bearing for the whole phase.** MiniMax issues are free-form English/Dutch prose ("matrix content spills past inner panel and overlaps bottom title/description"). A regex mapping that to `defect_type: vertical_overflow` will misclassify, and a *misclassified fix contract fed to GLM is worse than no contract* (it instructs the wrong repair). REQ-001 only requires "schema-valid ≤3 fixes," which says nothing about *correctness*. Add an acceptance criterion: classifier precision on a hand-labeled gold set of the 18 rows (e.g., ≥85%), measured *before* any round-2 generation. Without it you're flying blind on the input to the entire loop.

4. **T0 ("generic improve this") is a strawman and REQ-005 may prove nothing.** A fair generic arm would feed GLM the *rendered image + the raw MiniMax issue text as plain bullets* — no typed schema, but full information. If your T0 is literally "improve this" with no issue list, T1 wins trivially and the A/B is theater. Define T0 to share the evidence and differ *only* in structure; otherwise the ≥3-conversion delta is a measure of schema-vs-no-context, not schema-vs-prose.

5. **Sequencing: why A4 before 004?** The durable cure for the root cause is *don't ask GLM to do 2D placement at all* (decomposition/skeleton-first, phases 004/005) or *route 2D tiles to a linear primitive* (phase 002/A3). A4 is a reactive band-aid whose value the plan itself admits shrinks to +1–3 once 001/A1 capture overflow/title/contrast. The plan blocks correctly on 001, but doesn't justify ordering A4 ahead of 004. If 004 lands first, A4's remaining marginal target is the gate-blind RC-2 set — the hardest, least-verifiable slice. The "why now?" question is unanswered and is more important than the two open questions in §7.

6. **retry_depth=1 against a capability gap caps the ceiling.** GLM failed round-1 because it can't solve the constraint problem, not for lack of a hint. One more attempt with a typed contract may move the overflow-class (because the fix is "reserve 104px + use stacked-list," a *procedural* instruction GLM can follow) but is unlikely to move the genuine 2D-placement class (RC-2). The standalone 5–7 prediction is plausible *only* if most of the 18 FIX are RC-1/RC-3 (procedural-fixable); if a meaningful share are RC-2, expect 2–4. The prediction should be conditioned on the RC-mix of the 18 rows, which isn't broken down.

7. **One-shot vs recurring is never stated, and it changes the build/buy math.** If this is a one-time improvement of 45 tiles, hand-authoring fix-JSON for 18 rows (1–2h, zero classification error) dominates building + validating a regex classifier. The adapter is only worth its cost if the pipeline regenerates *new* tile sets repeatedly. The spec never says which. If one-shot, the adapter is over-engineered; if recurring, say so and the design stands.

## Strongest improvement or alternative
**Before building A4, decompose the 18 FIX rows by RC-id and split the phase in two by fixability:**
- *RC-1/RC-3 (procedural — reserve title band, switch to stacked/linear primitive):* these are already captured by **phase 001's deterministic-gate repair pass**. Fold them in; A4 adds ~nothing here. (This is your own open question — resolve it by measuring the RC-mix, not by deferring it.)
- *RC-2 (genuine 2D collision, gate-blind):* route these to **phase 004 skeleton-first** rather than asking GLM to re-attempt the same impossible 2D placement. A4's single retry will mostly fail here anyway, and you can't verify the few that "pass."

Then A4 shrinks to a *validation harness* for whatever 001/004 don't catch — or is deferred entirely. If you keep A4 as-is, make two hard additions: (a) a `classifyFixes` gold-set precision gate (≥85%) before any generation, and (b) a human or second-auditor spot-check on every adopted conversion, because the deterministic gate + MiniMax together are insufficient exactly on this cohort. This converts A4 from "optimistic repair loop" to "measured, gated loop with a known-blind-spot disclaimer."
