# Iteration 002 — Deepen + design (4 Opus sub-agents, parallel)

Each fix from iteration-1 was deepened into a concrete, code-grounded design. Source: workflow `wf_60883ac5-906`, phase "Iteration A". Verdicts on these designs are in `iteration-003.md`.

## F2 — Dispose the hf-local native ONNX session on provider invalidate
**Feasible: YES (the dispose chain is real).** `extractor.dispose()` → `model.dispose()` (transformers `pipelines.js:182-184`) → iterates `this.sessions` calling `session.handler.dispose()` (`models.js:1085-1092`) → onnxruntime-node `OnnxruntimeSessionHandler.dispose()` (`backend.js:33-35`) → native NAPI `binding.InferenceSession.dispose()` (`binding.js:8-10`), which frees the ORT graph/weights/arenas. The model is held at `HfLocalProvider.extractor` (`hf-local.ts:208`); `invalidateProviderSingleton()` (`embeddings.ts:445-452`) nulls the singleton with **no** dispose, orphaning native memory each swap (RC-1).
**Design:** add optional `dispose?(): Promise<void>` to `IEmbeddingProvider` (`types.ts:54-62`); `HfLocalProvider.dispose()` sets `disposed=true`, drains an in-flight refcount, then `await extractor.dispose()`. `invalidateProviderSingleton()` captures `outgoing` before nulling and fires `void outgoing.dispose().catch(...)` (stays sync for its two callers: `embeddings.ts:393` self-heal, `reindex.ts:472`).
**Self-flagged hazard:** use-after-free if dispose frees the native session while a `model(...)` run is in flight (native `run()` is synchronous on a libuv tick); the design proposes an `inFlight` refcount + drain barrier + bounded timeout.

## F1 — Launcher RSS-ceiling watchdog + supervised auto-respawn (mk-spec-memory-launcher.cjs)
**Feasible: partially.** Design: sample child RSS (`ps -o rss= -p <pid>` confirmed working on Darwin; `/proc` fast-path on Linux — mirrors `mk-code-index-launcher.cjs:289-307`) on an unref'd interval; on `SPECKIT_DAEMON_RSS_LIMIT_MB` breach for N consecutive samples, graceful SIGTERM→grace→SIGKILL **before** the OS OOM-killer; respawn with backoff + crash-loop guard; preserve the launcher-pid lease across recycle. New env: `SPECKIT_DAEMON_RSS_LIMIT_MB` (1536), `_SAMPLE_MS` (15000), `_BREACH_SAMPLES` (2), `_SIGTERM_GRACE_MS`, `_MAX_RESTARTS`/`_RESTART_WINDOW_MS`, `_WATCHDOG_DISABLED`.
**Self-flagged hazard:** the daemon serves a PRIMARY client over launcher-inherited stdio and SECONDARY clients over the child-owned `daemon-ipc.sock`; a respawn gives the new child a fresh stdin → the design proposes re-piping `process.stdin`→new child. (iteration-003 shows this is insufficient — see there.)

## F3 — Liveness-probe-before-bridge + respawn-on-dead-socket (launcher-ipc-bridge.cjs)
**Feasible: partially.** Today `maybeBridgeLeaseHolder` bridges on `fs.existsSync(socketPath)` alone (`:122`); a crashed daemon leaves a stale socket (daemon unlinks only on graceful close, `socket-server.ts:195-204`), and the lease records the **launcher** pid so `kill(pid,0)` liveness passes (`mk-spec-memory-launcher.cjs:189-194`). Design: a bounded connect-probe (tri-state alive/dead/starting) + a **STARTING grace window** keyed on lease `startedAt` (TOCTOU defense: lease is written before the daemon spawns + binds its socket), returning a `{action: bridged|reported|respawn}` verdict so the launcher (not the bridge) owns respawn. Decision table covers socket present/absent × probe alive/dead/timeout × lease age.
**Self-flagged hazard:** "accept-success = alive" and probe-then-bridge TOCTOU.

## RC-5 — IPC per-connection leak
**REFUTED (feasible: false, ~80% confidence).** Static reference-graph trace of both byte-identical socket-servers (`socket-server.ts:107-149`) + the MCP SDK: the per-connection `secondaryServer`, the `data` listener, the `socket.write` monkey-patch, and the `StdioServerTransport` are reachable only through the socket + `activeSockets`/`activeTransports` (the only module roots). The `close` handler (`:140-145`) deletes both map entries and calls `transport.close()`, which removes the transport's socket listeners and (via SDK `protocol.js:248-269`) sets `_transport=undefined`. The residue forms an unrooted island → GC-collectable. **No per-reconnect leak; do NOT build F5.** Residual ~20%: a socket that never emits `close` (half-open FD) would retain one connection — a different bug, not RC-5.
