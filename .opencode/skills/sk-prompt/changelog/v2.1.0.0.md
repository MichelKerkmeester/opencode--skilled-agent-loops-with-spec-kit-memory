# sk-prompt v2.1.0.0 — forkable: CLI quality card moved out to the hub

**Released**: 2026-06-03
**Status**: Boundary cleanup. `sk-prompt` is now purely the generic, forkable prompt-framework engine.

---

## Summary

`cli_prompt_quality_card.md` was a CLI-dispatch-specific asset living inside the otherwise-generic `sk-prompt` framework engine. It moved out to `sk-prompt-small-model` (the model-craft hub), leaving `sk-prompt` owning only the generic, fork-portable framework knowledge:

- the 7-framework definitions + CLEAR scoring (`references/patterns_evaluation.md`)
- DEPTH thinking (`references/depth_framework.md`)
- format guides + the framework registry (`assets/`)

`sk-prompt` no longer references any small-model or cli-dispatch specifics, so it can be forked standalone.
