# Iteration 1 (A1 — keystone): a memory-retrieval eval harness

> Model: **Opus 4.8 via claude2** (acct#2, read-only). Orchestrator-written. newInfoRatio **0.8**. **Headline: the eval harness is already ~80% built — the keystone is an extension, not a greenfield build.**

## EXISTING-INFRA (the premise-changing finding)
A live retrieval-eval harness already exists: a **110-query graded golden set** (290 relevance judgments, grades 1/2/3, 93 real production memory IDs, 0 placeholders; `ground-truth-data.ts`/`-generator.ts`, mapped via FTS5 by `map-ground-truth-ids.ts`); **12 metrics** (`eval-metrics.ts:586 computeAllMetrics` — MRR/nDCG/recall/precision/F1/MAP/hitRate + 5 diagnostic); a **live end-to-end runner** `eval_run_ablation` driving real `hybridSearchEnhanced` against the golden set with per-channel recall deltas + paired sign-test p-values (`eval-reporting.ts:238-339`, `ablation-framework.ts:554`, gated `SPECKIT_ABLATION=true`); a **baseline runner** (`bm25-baseline.ts:579 runBM25Baseline`); a **dashboard** (`eval_reporting_dashboard`); a **5-table eval DB** (`eval-db.ts`); an **alignment guard** (`assertGroundTruthAlignment` `ablation-framework.ts:314-352`); and **traffic-feedback + LLM-judge + agreement** scaffolding (`ground-truth-feedback.ts`). Only 3 genuine gaps remain (below).

## Candidates
1. **gate-accuracy-eval-lane → NET-NEW (H/S) — the keystone.** Add a verdict-scoring lane: per golden query, run live search, capture the `requestQuality` verdict, score vs golden expectation. **Label binding:** graded relevance (0-3) → gate's 3-class verdict (max-grade-3 ⇒ expect `good`/citable; 1-2 ⇒ `weak` ok; `hard_negative` ⇒ expect `gap`). **Metrics:** gate-accuracy + a confusion (false-cite / false-suppress) — NOT recall@k. Evidence: `assessRequestQuality` `confidence-scoring.ts:355-398` (thresholds :38-39,67) reads `resolveAbsoluteRelevance` `pipeline/types.ts:89`; the 12 existing metrics are all *ranking* — none reads a verdict. Maps to a new fn in `eval-metrics.ts` + reuse the `eval_run_ablation` wiring capturing `requestQuality`.
2. **embedding-coverage-precondition → EXTENDS `assertGroundTruthAlignment` (M/S).** The guard throws on golden-ID parenthood mismatch but checks *presence*, not *embedding reachability*. Post-reindex, vector recall moves; a pre-reindex recall delta reads against a quarter-dark index. Add an embedding-coverage assert so no recall number is trusted until coverage is whole. Confirms "reindex is gate-zero" is *already partially enforced in code*, not just prose.
3. **baseline-snapshot-rerun → EXTENDS (H/S) — first action, not a build.** `runBM25Baseline` + `recordBaselineMetrics` already write mrr@5/ndcg@10/recall@20/hit_rate@1 to `eval_metric_snapshots`; `eval_reporting_dashboard` reads them back by sprint/channel. Gate-zero step = reindex → run existing baseline → snapshot → re-run the recall/saturation checks the campaign couldn't trust. Resolves the "no benefit numbers" caveat by **execution, not construction**.
4. **traffic-selection-wiring → EXTENDS (M/S).** The live-traffic label bootstrap is fully scaffolded (`recordUserSelection` → `generateLlmJudgeLabels` → `computeJudgeAgreement` ≥80%) but `recordUserSelection` is wired at exactly ONE site (checkpoint restore `checkpoints.ts:831`), NOT the `/memory:search` cite path. Wire it at the cite path → the golden set self-expands with real labels, feeding the gate-accuracy lane + the calibration model (A2).

## Key finding
The minimal harness that unblocks A2/A3/A5 is **one net-new metric lane (gate-accuracy) bolted onto the already-live harness, run once behind the reindex precondition** — NOT a new build. The instrument 015 repaired (the cosine gate) is the exact dimension the existing harness was blind to (it measures *ranking*, never the `good/weak/gap` verdict). Two consequences: (a) metrics must **split by candidate class** — ranking candidates use the existing ablation; gate/calibration candidates are invisible to recall and need the new gate-accuracy lane; (b) gate-accuracy lane + traffic-wiring together **produce the real labeled set 017's isotonic calibration (A2) is explicitly blocked on**. Gate-zero ordering is firm: **reindex → embedding-coverage assert → baseline snapshot → gate-accuracy lane** (never build-a-candidate first).
