# v1.0.0.0 - Initial Release

**Released:** 2026-06-26

Initial release of `sk-design`, the parent hub for the design family. The hub is routing-only: `Skill(sk-design)` loads it, and its router reads the request and loads the matching nested mode packet.

## What's Included

- `SKILL.md` with the section 2 smart-routing table over the five design modes.
- `mode-registry.json` mapping each workflow mode to its nested packet.
- A single `graph-metadata.json`, the one advisor identity for the whole family.
- `shared/` baselines that every mode loads: `anti_slop_principles.md`, `cognitive_laws.md` and `design_token_vocabulary.md`.
- Five nested mode packets: `design-interface`, `design-foundations`, `design-motion`, `design-audit` and `design-md-generator`.

## Routing

A design request resolves through the hub to exactly one mode packet; a request that spans modes stays at the hub for disambiguation. Each mode keeps its own design logic, references and assets.
