# v1.0.0.0 - Initial Release

**Released:** 2026-07-06

Initial release of `create-manual-testing-playbook`, one of the ten workflow packets in the `sk-doc` parent hub's workflow-only architecture. It authors `manual_testing_playbook/` packages: a root `manual_testing_playbook.md` plus numbered `NN--category-name` folders of per-feature files, each carrying a deterministic scenario contract (exact prompt, exact command sequence, expected signals, evidence, pass/fail criteria, failure triage) so another operator or agent can reproduce the verdict.

## What's Included

- `SKILL.md` -- the packet contract: activation triggers and non-triggers, the canonical package shape (root playbook plus category folders and per-feature files), the 18-step authoring workflow, scenario design rules (determinism, natural-human vs RCAF prompt voice, the prompt synchronization gate), and the validation/release gates.
- `references/manual_testing_playbook_creation.md` -- standards and rationale for package structure, root-vs-per-feature responsibilities, prompt quality, and validation.
- `assets/testing_playbook/manual_testing_playbook_template.md` -- root playbook scaffold.
- `assets/testing_playbook/manual_testing_playbook_snippet_template.md` -- per-feature scenario scaffold.

## Notes

This packet has no packet-local `scripts/`; validation runs through the shared `../shared/scripts/validate_document.py` and `extract_structure.py`. It also carries no packet-local `graph-metadata.json` -- the single advisor identity and mode registry live at the `sk-doc` hub root.
