This release hardens the advisor persistence and routing layer against the deep-review P1 findings, closing an unsanitized injection surface on the highest-weight routing fields, a record-losing metrics race, two unhandled-rejection paths in the hook diagnostics and a rollback that erased lifecycle state. The fixes came out of the 20-iteration deep review of the 028 playbook-findings remediation and were verified per fix with non-gamed tests, the routing change the dispatch attempted was reverted as incomplete, and the full advisor typecheck stayed clean.

> Spec folder: `.opencode/specs/system-spec-kit/028-memory-search-intelligence/006-review-remediation` (the deep-review P1 advisor batch)

---

## What Changed

#### The derived routing fields get sanitized

- **The injection surface was wide open.** BEFORE: `sanitizeDerivedMetadata` only rewrote the path-like derived fields `source_docs` and `key_files`, so `derived.trigger_phrases`, `derived.key_topics` and `derived.entities` passed through verbatim into the projection. Those are the highest-weight routing match-terms, so an instruction-shaped phrase in a skill's graph metadata reached the scorer unfiltered. AFTER: the sanitizer runs `sanitizeMetadataStringArray` over all three fields, so instruction-shaped, control-char and over-length values are stripped before they become routing triggers.
- **Proven by a real round-trip.** A new test indexes a skill whose `trigger_phrases` carry an `ignore previous instructions` style injection, runs the real index-then-project path and asserts the safe phrases survive while the injection is dropped from `derivedTriggers`. It is an end-to-end round-trip, not a mock that only checks a call fired.
- **Known remaining gap, named not hidden.** The separate `skill_docs` upsert path still writes and reads `title`, `description` and `trigger_phrases` raw. It is documented as an open gap rather than claimed sanitized.

#### Durable metrics stop losing records under concurrency

- **The write was a lock-free read-modify-write.** BEFORE: `writeBoundedJsonl` read the whole file, pushed one line in memory and rewrote the file with no lock, so two concurrent advisor invocations for one workspace could clobber each other and lose records. AFTER: the add is an atomic `O_APPEND` write of a single sub-`PIPE_BUF` line, serialized per path through an in-process queue, so concurrent writers no longer lose records. Bounding is retained under the same queue, trimmed only past 1.5x the cap so the common path stays a cheap append.

#### Hook diagnostics can no longer surface an unhandled rejection

- **Fire-and-forget inside a sync try.** BEFORE: the claude and codex `user-prompt-submit` and `prompt-wrapper` hooks called the async diagnostic persist without awaiting it inside a synchronous try, so a rejected write became an unhandled rejection and the diagnostics-never-affect-hook-behavior guarantee did not hold for the async path. AFTER: each persist gets a `.catch` that swallows the failure deliberately, and `emitDiagnostic` is exported and parameterized so the swallow is testable.

#### Rollback stops erasing lifecycle state

- **Rollback over-stripped.** BEFORE: the derived-block rollback destructured away `lifecycle_status` and `redirect_to` alongside the derived block, silently un-deprecating skills and dropping supersession redirects. AFTER: it strips only the `derived` block, so the author-space lifecycle and redirect fields are preserved.

## Files Changed

| File | What changed |
| ---- | ------------ |
| `mcp_server/lib/skill-graph/metadata-sanitizer.ts` | `sanitizeDerivedMetadata` now sanitizes `trigger_phrases`, `key_topics` and `entities` alongside the path-like fields |
| `mcp_server/lib/metrics.ts` | `writeBoundedJsonl` is an atomic per-path-serialized `O_APPEND` add with bounding retained, plus tighter bounded-field validation |
| `hooks/claude/user-prompt-submit.ts`, `hooks/codex/prompt-wrapper.ts`, `hooks/codex/user-prompt-submit.ts` | the fire-and-forget diagnostic persist gets a `.catch`, `emitDiagnostic` exported and parameterized for testing |
| `mcp_server/lib/lifecycle/rollback.ts` | rollback strips only the `derived` block, preserving `lifecycle_status` and `redirect_to` |
| Advisor test suites | a real index-then-project round-trip test proves an injected `trigger_phrases` value is dropped, plus hook-diagnostic, lifecycle and observability coverage |

## Upgrade

No migration required. Adopt via a fresh session so the daemon respawns with the new dist. The sanitizer change only affects skills whose derived metadata carries instruction-shaped or over-length values, so a clean corpus is byte-identical. Verify by indexing a skill with an instruction-shaped `trigger_phrases` value and confirming it does not appear in the projection's `derivedTriggers`. The pre-existing python-versus-TS parity and divergence-ledger tests fail on a separate baseline and are unrelated to this batch.
