### PROPOSALS
1. **Tag** simplify. **Change** Replace the default GPT autonomous command prelude with a compact profile: command contract first, read-only/write mode, allowed tools, `Read before edit`, `do not modify out of scope`, `verify only if code changed`, `do not ask spec-folder questions for read-only tasks`, and `load skills/docs only when command explicitly requests them`. **Expected effect** Reduces Gate-3 latch, template partial rendering, and silent protocol stalls in GPT/opencode command-run cells. **Effort** S. **Risk** Some spec-kit governance will be deferred until mutation or completion paths.

2. **Tag** optimize. **Change** Deduplicate `CLAUDE.md` and `AGENTS.md` at injection time by hashing content and injecting one canonical copy plus a one-line “AGENTS.md mirrors CLAUDE.md” note. **Expected effect** Cuts roughly half of root policy token load in all GPT session-start cells without changing rules. **Effort** S. **Risk** Low, if the hash check is exact and fail-closed to current behavior.

3. **Tag** re-approach. **Change** Move Gate 3, Memory Save, Completion Verification, phase-folder documentation, and deep-loop workflow locks behind intent triggers rather than unconditional startup text for autonomous executors. **Expected effect** Targets command-contract override failures, especially read-only and one-command benchmark cells where these gates are irrelevant. **Effort** M. **Risk** Medium; mutation commands must still receive the full gate packet before first write.

4. **Tag** optimize. **Change** Keep OpenCode plugin injections terse by default: continuity brief <= 600 chars, advisor brief <= 300 chars, no fallback hard-block directive unless the prompt is code-writing or comment-editing. **Expected effect** Reduces prompt-distance and hard-rule salience in GPT executor cells while preserving routing hints. **Effort** M. **Risk** Medium; less context may reduce first-turn routing quality.

5. **Tag** simplify. **Change** Make the autonomous execution profile explicitly outrank repo-wide conventions for command-scoped runs: “If this command says read-only/no questions/no writes, do not run documentation gates, do not ask Gate 3, and do not invoke write-time completion workflows.” **Expected effect** Directly addresses Gate-3 halt and workflow-latching cells. **Effort** S. **Risk** Low for read-only runs; must not apply to ambiguous write prompts.
