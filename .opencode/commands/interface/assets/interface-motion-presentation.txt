# Interface Motion Presentation Contract

## 1. Progressive Intake

Resolve trigger, intent, before/after state, affected elements, interruption/reversal, runtime, performance budget, material, accessibility policy, and proof surface. Reject motion with no state or attention purpose.

```text
Interface Motion setup
Which transition, before/after states, intent, interruption behavior, runtime/library, performance budget, reduced-motion policy, and proof surface apply?
```

## 2. Common Visible Output

```text
Route Proof
Resolved Brief
Context Manifest
Grounding Record
Creation/Remediation Artifact - Motion Design Spec
Critique/Validation
Evidence Ledger
Next Action/Handoff
```

## 3. Results Display

```text
STATUS=OK PRODUCES="Motion Design Spec" NEXT=/interface:design,/interface:foundations,/interface:audit PROOF=componentState,motionPurpose,timingModel,reducedMotionPath
STATUS=ASK MISSING=<input>
STATUS=FAIL ERROR=<named-cause>
STATUS=DEFER ROUTE=<hub|sibling>
```

## 4. Next-Step Suggestions

| Condition | Suggested target |
|---|---|
| motion choices change or depend on the overall interface direction | `/interface:design` |
| motion constraints affect static layout, token, or responsive decisions | `/interface:foundations` |
| the motion spec is ready for performance, accessibility, or quality review | `/interface:audit` |
| accepted motion moves to implementation | `sk-code` via `.opencode/skills/sk-design/shared/sk-code-handoff.md` |

`HANDOFF_REQUIRED=false`. Recommendations never silently chain.

## 5. Example Usage

```text
/interface:motion modal-open-close --library framer-motion
```

Returns: a motion plan with purpose, timing, easing, interaction states, performance notes, and reduced-motion behavior.

## Grounding & Proof Requirements (research-backed)

- **Executable grounding, owned-system first.** Before creating, discover the owned styles system and emit a **Reuse Report** (which existing tokens/components/reference styles apply) and a **Violation Scan** (any proposed value not backed by the owned system) inside the **Grounding Record**. Do not use unverified tokens or components. This is the primary lever for on-brand, non-generic output.
- **Brand constraints.** Capture and honor a `must-use` / `must-preserve` / `must-avoid` policy for the target in the **Resolved Brief**; non-generic output comes from concrete constraints plus verified source, not adjectives.
- **Proof-tier honesty.** In the **Evidence Ledger**, label every proof at its true level — `authored` / `observed` / `measured` / `validated` / `verified`. A read-only command (Read/Glob/Grep only) caps at `validated` and must NOT claim `measured` or `verified`, which require runtime or render inspection. Surface `PROOF_TIER=<level>` in the results.
- **Artifact-first output.** Lead the visible result with the **Creation/Remediation Artifact**, decisions, proof, and **Next Action/Handoff**; the full context envelope (Route Proof, Context Manifest, Grounding Record, Evidence Ledger) is retained for audit and shown in full only on `--evidence full`.
- **Mode-specific.** Specify duration, easing, and trigger per interaction plus explicit **reduced-motion** behavior; label runtime-observed timing at its true proof tier.
