# v1.0.0.0 — quality mode established

First release of the `quality` workflow mode on the `sk-code` hub. Born in the parent-hub refactor that collapsed sk-code from eight sub-skills to four surface-primary packets, the author-side quality gate is now a single mode child that runs after the surface skill (`code-webflow` / `code-opencode`) implements changes and before the surface's verification workflow or done-claim.

## New

- `SKILL.md` — the mode contract: when to run the gate, consuming the parent shared surface router, selecting a checklist by detected surface and target path, applying P0/P1/P2 author-side checks, checking comment hygiene per modified file, fixing gate failures in place, and the boundary against `code-review` (findings-only output stays there).
- `README.md` — mode at-a-glance and layout.
- `assets/code_quality_checklist.md` — the shared quality-gate checklist.
- `scripts/` — the comment-hygiene checker (`check-comment-hygiene.sh` + test), `check-dist-staleness.sh`, and the Claude PostToolUse post-edit hook and router (`hooks/`, `lib/post-edit-router.cjs`).
- `manual_testing_playbook/` — the `quality-gate` routing scenario for the mode benchmark.
- `benchmark/` — the router (`router-mode-a`) and live (`live-mode-b`) skill-benchmark baselines.

## Notes

This mode consumes the parent `sk-code` surface detection, edits only files the surface skill already changed to clear gate failures, creates no new files, and dispatches no subagents. Findings-first review lives in `code-review`; spec-folder authoring lives in `system-spec-kit`. The packet is currently at 1.0.0.1 following a post-establishment metadata sweep with no behavior change.
