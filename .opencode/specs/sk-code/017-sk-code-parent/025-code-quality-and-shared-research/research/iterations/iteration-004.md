# Iteration 4: Deep-loop, hook, benchmark, and quality-standard integration for code-quality

## Focus
This iteration investigated the required focus: how `code-quality` should feed or reuse deep-review/deep-research outputs, repo hook signals, behavior-benchmark scenarios, and durable software-quality standards. The selected interpretation is an evidence-handoff design for the existing `sk-code` quality mode, not turning `code-quality` into a deep-loop controller or formal review agent. The explicit dispatch focus overrides the stale reducer `NEXT FOCUS`, which still points at the original baseline.

## Findings
1. Deep-loop integration should keep `code-quality` as an upstream evidence producer, not a deep-loop mode. The deep-loop hub has distinct `research` and `review` modes, keeps per-mode convergence/state/artifact contracts inside each packet, and requires exactly one hub-level advisor identity; `code-quality` already states that it owns the author-side quality gate but not implementation planning, verification evidence, or formal review reporting. Therefore the integration seam is a packet-local handoff summary that deep-research/deep-review can consume, not a new `code-quality` loop. 3. Hook signals are useful early warnings but not sufficient quality-pass evidence. `code-quality` documents three comment-hygiene enforcement layers: write-time warning, pre-commit block, and CI block; the PostToolUse hook runs only on `Write|Edit`, invokes comment-hygiene and dist-staleness checkers, prints warnings, and always exits 0. ## Reflection

- What did not work and why: Looking for a delta artifact path in the leaf contract did not yield permission; the allowed-write list intentionally omits `research/deltas/*`, so this iteration records the omission rather than inventing a state surface.
- What I would do differently: In the synthesis pass, convert the handoff model into a prioritized implementation backlog with exact target docs/scripts/checklists and scenario IDs, separating low-risk documentation fixes from benchmark and hook test additions.
