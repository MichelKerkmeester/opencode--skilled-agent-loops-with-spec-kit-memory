## What Changed

#### Permission-mode taxonomy now matches the binary

The v1.0.0.0 docs claimed three permission modes (`normal` / `dangerous` / `bypass`). The real binary has two: `auto` (default — auto-approves read-only tools, prompts on write or exec) and `dangerous` (auto-approves all tools — requires explicit operator approval). Every flag value, prose mention, slash command reference, and cross-CLI mapping was rewritten to the real two-tier taxonomy.

---

#### Model preset spelling corrected and OS-level sandbox added

The model ID `deepseek-v4-pro` does not exist; the binary accepts `deepseek-v4`. The four-model preset is now `swe-1.6` (default for context gathering, tool use, and simple-to-medium well-defined tasks), `deepseek-v4` (primary for complex tasks), `glm-5.1` (complex-task fallback — agentic / tool-use), and `kimi-k2.6` (complex-task fallback — large context). The full binary supports 19 models in total; the four documented are the cli-devin routing baseline. The `--sandbox` flag was missing from v1.0.0.0 — it is now documented as a Research Preview OS-level sandbox (macOS seatbelt / Linux bwrap+seccomp) that layers on top of `--permission-mode dangerous` for defense in depth.

---

#### Auth provider, missing flags, missing subcommands, and slash commands

Devin authenticates through Cognition's Codeium / Windsurf bridge (`server.codeium.com` API, `app.devin.ai` webapp). The v1.0.0.0 token-source URL was wrong and is corrected. New flags documented: `--sandbox`, `--respect-workspace-trust`, `--agent-config`, and `-p` / `--print` (the canonical non-interactive flag for cli-devin dispatches). New subcommands documented: `cloud` (Devin Cloud resources — environment setup, sandbox sessions, builds), `sandbox` (Research Preview), and `uninstall`. The slash-command list dropped `/normal` and `/bypass` (invented in v1.0.0.0 — not present in the binary); the verified list is now `/mode`, `/plan`, `/clear`, `/fork`, `/revert`, `/steps`, `/ask`, `/model`, `/context`, `/help` plus whatever the live `/help` surfaces in your installed version.

---

#### Playbook DV-007 re-purposed; root playbook re-aligned

DV-007 was titled "bypass mode (DESTRUCTIVE)" in v1.0.0.0, validating a permission-mode value that does not exist. The scenario is re-purposed at the same DV-007 ID to validate the real `--sandbox` flag — OS-level process sandboxing layered on top of `--permission-mode dangerous`. The renamed scenario file (`003-sandbox-flag.md`) keeps the destructive-approval discipline from the original and adds OS-level boundary checks. The Cat 2 scenario file `001-normal-mode.md` is renamed to `001-auto-mode.md` to match the real default. The root playbook §8, §17 cross-reference index, and Cat 2 file inventory all reflect the rename.

## Upgrade

**Invocation breakage** for operators using v1.0.0.0 documented flags:

- `--permission-mode normal` → use `--permission-mode auto` (the real binary default)
- `--permission-mode bypass` → use `--permission-mode dangerous` (binary has no `bypass`; `dangerous` is the strongest mode)
- `--model deepseek-v4-pro` → use `--model deepseek-v4` (the `-pro` suffix never existed)

Scripts and automation embedded with v1.0.0.0 flag values must be updated to the v1.0.1.0 values. Auth flow stays the same (`devin auth login`), but the documented token URL is corrected — Devin authenticates via Cognition's Codeium / Windsurf bridge (`server.codeium.com` API, `app.devin.ai` webapp), not the `preview.devin.ai/settings` URL v1.0.0.0 cited.

The four-model preset spelling is now binary-accurate: `swe-1.6` / `deepseek-v4` / `glm-5.1` / `kimi-k2.6`. The full Devin binary supports 19 models in total (see `devin --help` for the live enumeration); the four cli-devin documents are the routing baseline.
