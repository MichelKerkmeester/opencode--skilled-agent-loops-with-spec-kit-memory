# sk-design v1.2.0.0 - Mode-Local Procedure Operating Model

> Wires private procedure-card support into the existing five public mode packets without adding public modes or a second advisor identity.

---

## Added

- Mode-local procedure selection in all five mode packets, with each mode citing the selected private card or an explicit no-procedure fallback before substantial output.
- Context and proof expectations for procedure-backed output, including the evidence each mode must name before ready or handoff claims.
- Direct no-subagent fallback language so mode packets can run inside the current session without requiring subagent dispatch.
- `mode-registry.json` `proceduresPath` pointers for the five existing modes. These are non-routing metadata only; they do not change `workflowMode`, `toolSurface`, or `backendKind`.

## Changed

- The four advisory modes now keep their runtime tool declarations aligned with their read-only registry surface: Read, Glob, and Grep only.
- `README.md` now describes the operating model as private mode support rather than a user-facing procedure catalog.

## Preserved

- The public mode taxonomy remains `interface`, `foundations`, `motion`, `audit`, and `md-generator`.
- The advisor identity remains the single `sk-design` hub with exactly one graph metadata file at the hub root.
- `design-md-generator` remains the only mutating mode and keeps the Playwright extract-write-validate backend boundary.
- The existing benchmark baseline under `benchmark/baseline/` remains untouched.
