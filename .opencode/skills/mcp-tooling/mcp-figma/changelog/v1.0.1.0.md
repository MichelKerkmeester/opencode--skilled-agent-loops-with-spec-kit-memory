# v1.0.1.0 - Worked Example Walkthroughs

**Released:** 2026-07-16

---

## What's New

### examples/ directory (new)

The skill now ships worked walkthroughs mirroring its manual-testing-playbook scenarios, bringing it to examples parity with the sibling packets (`mcp-chrome-devtools/examples/`, `mcp-click-up/examples/`). The set is deliberately SAFE: no destructive verb, no yolo patch, no `init-agent` anywhere in the directory.

- `examples/README.md`: directory overview, safety properties (binary discipline, gating fidelity, command provenance), prerequisites, per-walkthrough usage and exit codes, troubleshooting, and see-also links.
- `examples/safe-connect-daemon-health.sh`: binary detection with the naming-trap refusal, `connect --safe` (plugin bridge, no patch), and read-only `daemon status` / `daemon diagnose`. Mirrors DETECT-001, CONNECT-001, and DAEMON-001. The daemon token is never printed.
- `examples/inspect-export-readonly.sh`: read-only inspect (`node tree` / `find`) plus `export screenshot -f svg -o <path>` to an explicit, non-existing output path with an overwrite refusal. Mirrors INSPECT-001 and EXPORT-001.
- `examples/optional-mcp-context.md`: discovery-first walkthrough for the opt-in Framelink `figma` MCP via Code Mode (`list_tools` / `tool_info` before any call, the prefixed `figma_FIGMA_API_KEY` requirement, read-only context pull). Mirrors MCP-001.

Every command derives from the skill's own SKILL.md, references, feature catalog, and playbook. Gated and destructive operations are marked gated exactly as those documents define, and none run in the examples.
