## [**1.1.2.0**] - 2026-06-02

This release standardizes how named feature worktrees are named and sharpens how the skill is discovered by the skill advisor. Named feature worktrees now use a single unified, numbered branch namespace — `wt/{NNNN}-{name}` — with a matching `.worktrees/{NNNN}-{name}` directory, replacing the prior `type/short-description` branches and unnumbered `.worktrees/short-desc` directories. The advisor-facing description and trigger lists were strengthened so git, worktree, branch, commit, PR, merge, and finish intent routes reliably to sk-git, while explicitly ceding spec-folder/memory intent to system-spec-kit and code-implementation/test intent to sk-code.

---

## What Changed

#### Worktree Naming Convention

- **Unified numbered branch namespace** -- All named feature worktrees now use `wt/{NNNN}-{name}` (e.g. `wt/0001-add-oauth`). The `wt/` prefix groups every feature-worktree branch under one folder in Git UIs. `{NNNN}` is a 4-digit zero-padded global counter computed as `max(existing NNNN under .worktrees/) + 1` (first is `0001`).

- **Standardized directory home** -- Named feature worktrees live at `.worktrees/{NNNN}-{name}` under the repo-local, already-gitignored `.worktrees/` home. Directory selection no longer juggles `worktrees/` or a global `~/.config` location for feature worktrees.

- **Lifecycle strategies kept, renamed** -- Fast-merge (default), long-running, and detached experiment replace the old `main_temp` / `feature_branch` / `main_detached` labels. The branch name is identical for both branch-creating lifecycles; only post-creation management differs. Detached experiments create no branch, so no number is assigned.

- **Two distinct lanes documented** -- A note now distinguishes named feature worktrees (numbered `wt/{NNNN}-{name}`) from the launch wrapper's ephemeral per-session worktrees (`work/{runtime}/{slug}` + `.worktrees/{runtime}-{slug}`, auto-reaped, intentionally not numbered). The wrapper's behavior is unchanged.

#### Advisor Routing

- **Sharpened discoverability** -- The SKILL.md frontmatter `description`, the keyword comment block, the keyword-trigger list, and the smart-router intent and on-demand keyword lists now own: git worktree, create worktree, numbered worktree, restructure worktrees, worktree prefix, wt/ branch, branch, commit, conventional commits, pull request, PR, merge, rebase, finish work, integrate changes, git workflow.

- **Reduced cross-skill overlap** -- The skill now explicitly does NOT claim spec-folder / memory / continuity / save-context intent (owned by system-spec-kit) or code-implementation / test-writing intent (owned by sk-code).

## Upgrade

No migration required. Existing worktrees keep their current names; the new `wt/{NNNN}-{name}` convention applies to worktrees created from this release onward. The launch wrapper's ephemeral per-session worktrees are unaffected.
