`mcp-tooling` gains its second design transport: `mcp-refero` brings read-only real-app UI design-reference research (Refero MCP via Code Mode) into the hub, paired with `sk-design` for judgment.

---

## Why

The hub's only design-side bridge was the Figma Desktop transport. Refero's MCP surface (8 read-only tools over real-app screens, flows, and styles) was already registered in `.utcp_config.json` but had no skill packet, no routing vocabulary, and no transport contract — design-reference research had no advisor-reachable home.

## What Changed

#### New transport packet (`mcp-refero/`)

- 29-file nested packet authored from a converged 10-iteration deep-research fan-out (3 lineages, all first-attempt); `packetKind: "transport"`, `backendKind: "code-mode-remote-mcp"`, `mutatesWorkspace: false`, Write/Edit/Task forbidden.
- Doubled-prefix Code Mode callable documented from live evidence with a mandatory post-registration `tool_info` confirmation; plan gating (Pro 8,000 calls/month, Free has no MCP) and OAuth-Inferred posture carried honestly.

#### Hub surfaces

- `mode-registry.json`: fifth `modes[]` entry; `transport-axis` extension now declares two transports with `crossHubPairing` to `sk-design` for both; new `backendKind: code-mode-remote-mcp`.
- `hub-router.json`: `mcp-refero` signal (weight 4; `refero-aliases` + `design-reference-research` + `hub-identity`), mirrored vocabulary, tie-break appended after `mcp-figma`.
- `SKILL.md` (v1.2.0.0), `description.json`, `graph-metadata.json`: transport prose generalized (pairing rules now cover all transports), mode table row, keywords, trigger example, intent signals, domains, packet entity.
- `manual-testing-playbook/hub-routing/refero-design-reference.md`: routing scenario guarding the Refero-vs-Figma vocabulary boundary.
- `.utcp_config.json`: UNCHANGED — the `refero` manual was already registered and is verified byte-identical by the packet's asset snapshot.
