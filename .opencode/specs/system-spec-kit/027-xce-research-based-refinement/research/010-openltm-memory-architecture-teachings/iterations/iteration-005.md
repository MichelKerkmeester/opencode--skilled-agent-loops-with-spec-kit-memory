## Mechanism
OpenLTM defines exactly six relation types in both TypeScript and SQLite: `supports`, `contradicts`, `refines`, `depends_on`, `related_to`, `supersedes` (`packages/openltm-core/src/db.ts:29`, `packages/openltm-core/src/db.ts:30`, `packages/openltm-core/src/schema.sql:95`, `packages/openltm-core/src/schema.sql:100`). Relations have optional `note` and bounded `weight`, and migration text says effective weight is “base × decay” at read time (`migrations/018_relation_semantics.sql:1`, `migrations/018_relation_semantics.sql:7`). In the inspected traversal path, however, `getNeighbors()` selects only neighbor id and relation type, not weight or note (`packages/openltm-core/src/graph.ts:51`, `packages/openltm-core/src/graph.ts:56`). `relate()` validates source/target existence, then `INSERT OR IGNORE`s only source, target, and type, so it creates but does not update edge metadata (`packages/openltm-core/src/db.ts:587`, `packages/openltm-core/src/db.ts:603`).

Graph reasoning is an undirected BFS over `memory_relations`: outgoing and incoming edges are unioned (`packages/openltm-core/src/graph.ts:51`, `packages/openltm-core/src/graph.ts:56`), traversal starts from an active seed and stops at `depth` (`packages/openltm-core/src/graph.ts:69`, `packages/openltm-core/src/graph.ts:93`), and visited nodes are queued once (`packages/openltm-core/src/graph.ts:111`, `packages/openltm-core/src/graph.ts:115`). It classifies `contradicts` and `supersedes` as conflicts, and `supports` and `refines` as reinforcements (`packages/openltm-core/src/graph.ts:39`, `packages/openltm-core/src/graph.ts:40`, `packages/openltm-core/src/graph.ts:105`, `packages/openltm-core/src/graph.ts:109`). `depends_on` and `related_to` are traversed but not surfaced as conflict/reinforcement buckets.

Implicit relation inference is bounded: only pairs among the first six visited nodes are considered, capping inference at 15 pairs (`packages/openltm-core/src/graph.ts:161`, `packages/openltm-core/src/graph.ts:164`). If enabled, OpenLTM classifies missing edges and persists them via `relate()` (`packages/openltm-core/src/graph.ts:122`, `packages/openltm-core/src/graph.ts:143`). Separately, `learn()` fire-and-forgets auto-relation detection after embedding, using top-5 similar memories at threshold `0.6` unless `autoRelate` is disabled (`packages/openltm-core/src/db.ts:273`, `packages/openltm-core/src/db.ts:284`, `packages/openltm-core/src/db.ts:400`, `packages/openltm-core/src/db.ts:405`). The classifier can auto-create only `supports`, `contradicts`, `refines`, or `related_to`; `supersedes` and `depends_on` are excluded from auto-inference (`packages/openltm-core/src/embeddings.ts:306`, `packages/openltm-core/src/embeddings.ts:315`).

Ordinary `recall()` does FTS5 query expansion, optional semantic fallback at similarity threshold `0.5`, filters to `status = 'active'`, enriches rows with direct relations, and returns them; it does not build the BFS conflict summary in the inspected body (`packages/openltm-core/src/db.ts:417`, `packages/openltm-core/src/db.ts:462`, `packages/openltm-core/src/db.ts:510`, `packages/openltm-core/src/db.ts:562`, `packages/openltm-core/src/db.ts:584`). Optional context graphing exists via `getContextMergeWithGraph()`: when `ltm.graphReasoning` is enabled, it seeds BFS from two globals plus one scoped memory at depth 2 and emits up to five reasoning lines (`packages/openltm-core/src/db.ts:682`, `packages/openltm-core/src/db.ts:702`). Conflict lines explicitly warn “verify before applying” (`packages/openltm-core/src/graph.ts:235`, `packages/openltm-core/src/graph.ts:239`). The conflict inspection view also treats only `contradicts` and `supersedes` as conflicts (`migrations/004_graph_reasoning.sql:5`, `migrations/004_graph_reasoning.sql:18`). Supersession has schema support via `status='superseded'`, `superseded_by`, and `superseded_at`, but `relate({relationship_type:'supersedes'})` does not itself update those fields (`packages/openltm-core/src/schema.sql:50`, `packages/openltm-core/src/schema.sql:58`, `packages/openltm-core/src/db.ts:587`, `packages/openltm-core/src/db.ts:603`).

## Negative knowledge
Do not copy `relate()` as-is: `INSERT OR IGNORE` prevents metadata updates and does not enact supersession state.

Do not copy the full cluster/layout subsystem into retrieval: label propagation and pinned coordinates are graph-UI concerns, not continuity quality (`src/cluster.ts:8`, `migrations/016_memory_layout.sql:2`).

Do not assume OpenLTM auto-detects supersession: its classifier excludes `supersedes` and `depends_on` (`packages/openltm-core/src/embeddings.ts:306`, `packages/openltm-core/src/embeddings.ts:317`).
