This release cleans up the benchmark layer so the folder names, the benchmark contents and the citations that point at them all agree. The renumbered benchmark folders are now staged in git, two near-duplicate benchmarks are merged into one and roughly forty stale citations are repointed. The result is a benchmark layer where every reference resolves to the right place.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/130-prompt-knowledge-layering/011-model-profiles-and-benchmark-merge/` (Level 1)

---

## What Changed

#### Architecture

- **Benchmark rename reconciled** -- the benchmark folders had been renamed on disk to a clean `00N` scheme (`113-003` to `001` through `127-006` to `006`) without git ever recording the move, so git saw them as deleted-and-added. This version stages the rename so history tracks it correctly.
- **006 merged into 005** -- benchmarks 005 (capability discrimination) and 006 (strict-validation fixtures) were the same MiniMax-M3-vs-MiMo comparison split across two folders. We folded 006 into `005/eval/strict-validation/`, moved 005's original data to `005/eval/capability-discrimination/`, added a merged top-level `synthesis.md` that ties both subsets together and removed the `006-` folder. The merge structure was verified and 006 confirmed gone, so the comparison now lives in one coherent place.

&nbsp;

#### Documentation

- **Citations repointed** -- roughly forty benchmark citations across the profiles, `_index.md`, `pattern-index.md` and `model-profiles.json` still used the old ids (`120/003`, `126/004`, `127/004`, `127/006`, `113/003`, `113/005`), which no longer resolved. They now read `benchmark 00N`, verified free of stale benchmark or phase ids in active docs, so readers can follow them.
- **Annotation hygiene** -- `model-profiles.json` annotation fields now carry durable WHY wording with refreshed ids and still validate as JSON, and `pattern-index.md` lost its remaining ephemeral spec-phase references (the `Phase NNN` column became `Status`, and the arc name, deleted-phase line and spec-folder links were removed), so the docs stay accurate as phases churn.
