## Creation/Remediation Artifact - Interface Direction Spec

### Chosen direction: “The first signal”

The empty state treats the dashboard as a calm control room awaiting its first signal. The screen explains the future value of the workspace, shows the actual metrics the team will receive, and gives one decisive setup action.

#### Composition

- Full dashboard shell remains visible:
  - left navigation
  - workspace switcher
  - date-range control disabled until data exists
  - main content area
- Main panel uses a wide editorial empty state rather than a centered illustration.
- Left column:
  - eyebrow: `NORTHSTAR OPERATIONS`
  - heading: `Your team’s first signal starts here`
  - body: `Connect Stripe to bring revenue, customer growth, and conversion into one shared view.`
  - primary button: `Connect Stripe`
  - secondary text action: `See what will be available`
- Right column:
  - three compact metric cards with muted “awaiting data” treatment:
    - `Monthly recurring revenue`
    - `New customers`
    - `Trial conversion`
  - Each card includes a thin horizontal baseline and a small signal marker, but no fake values or chart lines.
- Bottom reassurance strip:
  - `You can change the source later`
  - `Northstar Operations · 8 members`

#### Palette intent

Use a dark ink interface foundation with one owned accent reserved for action and signal.

- Background: deep blue-black, not pure black
- Primary text: warm near-white
- Secondary text: cool desaturated blue-gray
- Action accent: electric mint or blue-green
- Metric-card surfaces: slightly lifted blue-black
- Borders: low-contrast cool gray
- Loading shimmer: same accent family at low opacity

Exact color values remain unassigned until the owned token system is available.

#### Type intent

- Heading: compact, high-x-height grotesk with a confident but practical tone
- Body: neutral sans-serif optimized for short explanatory copy
- Metric labels: medium-weight small text with strong contrast
- Workspace name: uppercase or small-caps treatment with increased tracking

Avoid oversized marketing typography. The product should feel operational, not promotional.

#### Layout intent

- Comfortable density with generous separation between explanation, action, and preview cards
- Main empty-state content aligned to the dashboard’s existing content grid
- Primary action stays above the fold
- Preview metrics remain visibly secondary to the setup action
- Mobile collapse order:
  1. heading and explanation
  2. connect action
  3. metric cards
  4. reassurance strip

#### State definitions

Empty:

```text
Your team’s first signal starts here

Connect Stripe to bring revenue, customer growth, and conversion into one shared view.

[Connect Stripe]  See what will be available
```

First-run after action:

```text
Connecting Stripe to Northstar Operations

We’re checking permissions and preparing your first metrics. This usually takes a moment.

[Cancel connection]
```

Loading:

```text
Preparing your dashboard

Your first metrics are being calculated from Stripe. You can leave this page open while we finish.

Monthly recurring revenue     Loading
New customers                 Loading
Trial conversion              Loading
```

Error recovery:

```text
Stripe could not be connected

Your workspace is still empty. Check the connection permissions and try again.

[Try again]  Choose another source
```

#### Motion intent

- Restrained entrance: content fades and rises a few pixels once on first render
- CTA hover: slight luminance increase, no scale bounce
- Connection transition: button changes label and shows a small inline progress indicator
- Loading cards: low-contrast shimmer moving left to right, slow and continuous
- Reduced motion: remove entrance and shimmer; retain state changes through text and color only

### Contrastive directions considered

1. “The first signal” — editorial, calm, explanatory. Chosen for trust and clarity.
2. “Team cockpit” — denser card grid with setup embedded in a command-center panel. Rejected because it makes an empty workspace feel prematurely complex.
3. “Guided setup” — sequential onboarding with three setup steps. Rejected because it adds ceremony before the user has seen the product’s core value.
