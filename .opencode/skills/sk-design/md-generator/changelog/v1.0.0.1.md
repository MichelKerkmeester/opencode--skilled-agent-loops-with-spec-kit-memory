# v1.0.0.1 - Documentation Contract Repair

**Released:** 2026-06-25

Documentation-accuracy and contract-alignment patch following a two-model deep review. The embedded engine behavior is unchanged; this release makes the documented operating contract match the shipped runtime so every documented invocation succeeds verbatim and every load-bearing claim traces to code. No extraction behavior changed.

---

## Fixed

### Extraction command contract (P1)

- Established a single canonical contract: run the one-time setup (`npm install`, `npx playwright install chromium`) from `backend/`, but run every pipeline script (`extract.ts`, `validate.ts`, `report-gen.ts`, `preview-gen.ts`, `proof.ts`) from the **repo root** with the full script path. `extract.ts` refuses any `--output` that resolves inside the skill, so a relative `--output .opencode/specs/<track>/<packet>/output` only resolves correctly from the repo root. The previous docs told operators to `cd backend` first, which made the documented `--output` resolve inside the skill and be rejected before the crawl.
- Aligned `SKILL.md` (§3 Invocation, §7 tool usage), `README.md` (Quick Start, Extract Flags, Post-Extraction Scripts), `INSTALL_GUIDE.md` (first extraction + troubleshooting), `backend/README.md` (Quick Start + Usage Examples), and every `manual_testing_playbook/**` scenario to the repo-root invocation.
- Clarified the `extract.ts` refusal-hint and usage text to state the repo-root requirement (error-hint and `printUsage` only; no change to the guard or extraction logic).

### Detector and interaction schema documentation (P1)

- Reconciled the detector catalog and `DETECT-001` to the real output schema: framework lands in `tokens.json` `meta.framework`, icons in `iconSystem`, motion in `motionSystem`, and the design boundary in the sibling `extraction-report.json` `designBoundary` (not a `tokens.json` field). Replaced the non-existent top-level `framework` / `motion` / `designBoundary` references and the inspect command.
- Reconciled the interaction catalog and `INTERACT-001` to actual behavior: captured interaction diffs are folded into `tokens.json` `components[].variants[]` (`hoverChanges` / `focusVisibleChanges` / `focusChanges` / `activeChanges` / `disabledStyle`), not a top-level `tokens.interaction.captures` block. Removed the stale v2 "§11 State Matrix" framing from `feature_catalog/07`.

### Front-door doc accuracy (P2)

- `README` no longer claims Typography is deterministically pre-rendered; Typography is authored as prose from the FACTS block (the pre-rendered set is Colors, Spacing & Shapes, Surfaces, Quick Start).
- Fixed the README `--fast` flag row (it still captures interaction) and removed the false `--output` default; documented `--fast-no-interaction` and `--insecure` in the README, SKILL.md, and the flag tables.
- Reconciled the Node floor to **20 or newer** across README, INSTALL_GUIDE, SKILL.md, and the playbook, and removed the false "package.json declares the version range" claim.
- Noted that the L1 stability class is emitted as the `infrastructure` layer string in `tokens.json` (with `system` / `campaign` / `content` for L2–L4); "Permanent" is the human label.
- Corrected the playbook test count (50 → 68 across 7 files) and the stale `scripts/__tests__/**` vitest include (actual: `tests/**/*.test.ts`); fixed stale `stabilityClass` field references to `stability.layer`.
- Replaced stale `/design-md` command references in `cli.ts` and `report-gen.ts` (the skill is advisor-routed; there is no such command).
- Refreshed `graph-metadata.json` to drop the stale `output/<domain>/` path (output is the required `--output` spec folder).

### Validator hardening (P2)

- `validate.ts` `checkSectionCompleteness` now anchors required headings to line-start, so a deeper heading (e.g., `### Layout`) can no longer satisfy a `## Layout` requirement via substring match. Added a SKILL.md caveat documenting exactly what validation re-traces (hex + Quick Start values) versus what upstream pre-rendering guarantees (pixel/weight/shadow/radius values).
