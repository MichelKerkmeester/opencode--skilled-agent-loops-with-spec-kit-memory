## What Was Built

A deterministic compiled contract represents all six `sk-design` modes. The compiler preserves one
named UI-build bundle, compiles the legal two-axis grammar, reference-follows into six nested mode
routers, and manifests ninety authored leaves. The candidate remains shadow-only.
<!-- /ANCHOR:what-built -->

---
<!-- ANCHOR:how-delivered -->
## How It Was Delivered

Raw hub, registry, packet, and router bytes are hashed and identity-checked before the shared
compiler receives normalized destinations, signals, leaves, composition, and authority edges. The
router uses authored weights, tie-break, null default, and explicit-axis joining. Selected nested
defaults and intent resources flow through the shared projector into the real read-only scorer.
<!-- /ANCHOR:how-delivered -->

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Keep one named authored bundle | Preserve `ui-build-bundle` exactly. |
| Compile all two-axis pairs | The real parser requires every legal ordered target list in policy data. |
| Give transport evidence-only authority | Transport stays routable without acquiring commit authority. |
| Project nested selected leaves | Preserve each mode's authored resource router instead of stopping at packet SKILL.md. |
| Keep null default hard-locked | Zero evidence cannot manufacture a design axis. |
| Keep legacy authority | Prove the candidate and rollback without a live flip. |

3. The candidate is not live serving authority and document replay cannot commit effects.
