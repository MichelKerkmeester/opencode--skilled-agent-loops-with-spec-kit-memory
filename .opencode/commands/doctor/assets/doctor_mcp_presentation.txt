# /doctor:mcp Presentation Contract

This file is the single source of truth for user-facing presentation in `/doctor:mcp`: startup questions, MCP assessment dashboards, install/repair prompts, verification summaries, final reports, troubleshooting, and next-step text. The command router owns sub-action routing only. The workflow YAML owns execution behavior.

## 1. Startup Presentation

When `$ARGUMENTS` lacks a positional sub-action, ask this prompt first.

```text
What do you want to do with MCP servers?
   1) Install fresh    (all 5 MCP servers from install guides)
   2) Debug + repair   (diagnose failures, apply guided fixes)
   X) Cancel
```

Accepted answers:

| Answer | Result |
|--------|--------|
| `1`, `I`, `install` | sub_action = `install` |
| `2`, `D`, `debug` | sub_action = `debug` |
| `X`, empty, `cancel` | `STATUS=CANCEL` |

### Unknown Sub-Action Failure

```text
Unknown sub-action '[sub_action]'. Valid: install, debug. Try `/doctor:mcp install` or `/doctor:mcp debug`.
STATUS=FAIL ERROR="unknown_sub_action"
```

### Cross-Sub-Action Flag Failures

```text
Flag '--fix' is only valid for `debug`. Did you mean `/doctor:mcp debug --fix`?
STATUS=FAIL ERROR="cross_sub_action_flag_injection"
```

```text
Flag '--runtime' is only valid for `install`. Did you mean `/doctor:mcp install --runtime <name>`?
STATUS=FAIL ERROR="cross_sub_action_flag_injection"
```

## 7. Troubleshooting Display

| Situation | Suggested Action |
|-----------|------------------|
| Install just finished | The install workflow runs the health check automatically |
| Install prerequisites missing | Install Node >= 20.11.0, Python >= 3.11, npm, and npx, then rerun |
| Debug repair did not take | Read the cited install guide and apply the manual install section |
| MCP is OK but subsystem doctor fails | Use `/doctor <target>` for subsystem diagnostics |
| Need full spec-kit rebuild | Run `/doctor:mcp debug --fix`, then `/doctor:update` |

End interactive presentations with: `What would you like to do next?`
