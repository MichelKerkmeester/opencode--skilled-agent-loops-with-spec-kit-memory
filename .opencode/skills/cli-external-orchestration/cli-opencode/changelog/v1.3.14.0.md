# cli-opencode v1.3.14.0 — Xiaomi Direct API + MiniMax M2.7 removal

**Released**: 2026-06-07
**Status**: Documentation update. Adds the Xiaomi Direct API provider (`xiaomi`) as a pay-per-token alternative to the Token Plan, and removes all MiniMax M2.7 references in favor of MiniMax-M3 as the sole model for both Token Plan and Direct API.

---

## Summary

Two changes in this release:

1. **Xiaomi Direct API** — The `xiaomi` provider is now documented as a pay-per-token alternative to `xiaomi-token-plan-ams` (Token Plan), mirroring how MiniMax has both `minimax-coding-plan` (Token Plan) and `minimax` (Direct API). Model: `xiaomi/mimo-v2.5-pro`. Auth: `opencode providers login xiaomi`.

2. **MiniMax M2.7 removal** — All references to MiniMax-M2.7 (including `-highspeed` variants) have been removed. MiniMax-M3 is now the sole model for both `minimax-coding-plan` (Token Plan) and `minimax` (Direct API). The 120/003 benchmark (originally run on M2.7) TIDD-EC + dense contract is carried forward to M3.

## Xiaomi Direct API Additions

- **Provider**: `xiaomi` (pay-per-token). Auth via `opencode providers login xiaomi`.
- **Model**: `xiaomi/mimo-v2.5-pro` (same model as Token Plan, different billing).
- **Pre-flight detection**: `XIAOMI_DIRECT_OK` via `grep -qE "xiaomi([^-]|$)"` (skips `xiaomi-token-plan-ams`).
- **Routing**: 4-row MiMo routing table — Token Plan (default), Token Plan missing, Direct API requested, Direct API missing.
- **Invocation**: Same flags as Token Plan (`--variant high`, omit `--agent`).

## MiniMax M2.7 Removal

- All `minimax-coding-plan/MiniMax-M2.7-highspeed` fallback references removed.
- All `minimax/MiniMax-M2.7` Direct API references changed to `minimax/MiniMax-M3`.
- `-highspeed` variant notes and `m3-highspeed` / `minimax-m3-highspeed` trigger phrases removed.
- The 120/003 benchmark TIDD-EC + dense contract is explicitly carried forward to M3 until re-benchmarked.

---

## Caveats

- The MiniMax TIDD-EC + dense contract was benchmarked on M2.7; it carries forward to M3 pending re-benchmark.
- The Xiaomi Direct API endpoint is provider-managed (no documented base URL — do not fabricate one).
- Confirm live model ids with `opencode models xiaomi` before dispatch.
