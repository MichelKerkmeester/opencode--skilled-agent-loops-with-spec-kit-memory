# v1.4.0.0 — conformance-benchmark authoring family

`create-benchmark` now authors five benchmark families. The new
`conformance_benchmark` family provides stable input scaffolding for deterministic
artifact-conformance checks run later through a deep-alignment peer adapter.

## What Added

- **Conformance family routing** — activation triggers, the guarded `FAMILIES`
  discriminator, a family-table row, and a concise package contract in SKILL.md.
- **Authoring-command branch** — `/create:benchmark
  --family=conformance_benchmark` fills the family index, benchmark contract,
  lane config, and fixture manifest, validates them, and stops before execution.
- **Family-parity regression** — validates family directories, table and README
  registration, required conformance resources, and the lane-owned template
  boundary.

## What Changed

- Routing projections in the packet README and sk-doc hub now include the
  conformance family and its peer-adapter and deep-alignment trigger vocabulary.
- The duplicated behavior template/workflow/naming block moved out of SKILL.md;
  the existing behavior authoring guide is now authoritative for that procedure.
- SKILL.md version advanced from `1.3.0.0` to `1.4.0.0` while remaining below the
  5,000-word packaging ceiling.

## Boundary Held

The family templates stable inputs only. Adapter implementation, dimensions,
severities, discovery, convergence, scoring, reduction, reports, and scorecards
remain lane-owned and are never invoked by the authoring command.
