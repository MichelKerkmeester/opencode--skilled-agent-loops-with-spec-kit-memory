## 3. Cross-Cutting Themes

| Theme | Synthesis |
| --- | --- |
| Propose, do not mutate | OpenLTM’s strongest transferable pattern is staging uncertain promotion, transcript extraction, git learning, relation inference and dedup as proposals. system-spec-kit should keep shadow-first reducers and extend them to memory admission and duplicate handling. |
| Explain every retrieval path | The recurring gap is not retrieval power. It is observability: actual-ranker contribution, channel availability, provenance source, and contradiction/supersession warnings should be visible when requested. |
| Write-path safety beats retrieval cleverness | Redaction, provenance, actor requirements, atomic idempotency and deletion tombstones matter before FSRS, causal edges or co-activation can help. Bad writes become durable bad memory. |
| Derived surfaces must be rebuildable and auditable | FTS, vector shards, retrievability caches and migrations should be treated as derived or versioned surfaces with checksums, model/dim metadata, repair reports and pair-aware backups. |
