# cli-external v1.0.0.0

## Parent hub creation (fold-in of cli-opencode + cli-claude-code)

- Created the `cli-external` parent hub: one advisor identity routing to two `packetKind: "workflow"` modes, `cli-opencode` and `cli-claude-code`, through `mode-registry.json` and `hub-router.json` (ADR-001: zero extensions — no surface-axis, no transport-axis).
- Relocated both skills via `git mv` (ADR-002 full rename): `cli-opencode/*` -> `cli-external/cli-opencode/*`, `cli-claude-code/*` -> `cli-external/cli-claude-code/*`. Both packets carry `grandfatheredFolderMismatch: false`.
- No command created for the hub or either mode (ADR-003) — both stay advisor-routed and cross-skill-referenced only, matching their pre-merge shape.
- Dissolved both children's independent `graph-metadata.json` identities into one hub identity (`skill_id: cli-external`, `family: cli`), folding the union of their edges, domains, and intent signals (ADR-004).
- Rewrote `executor-delegation.ts` to source its executor alias table from this hub's `mode-registry.json` `packetSkillName` values instead of the top-level `family === 'cli'` projection filter, so delegation prompts keep resolving to `cli-opencode` / `cli-claude-code` and never to the non-executor `cli-external` hub identity (ADR-005).
- Repointed the fail-open PreToolUse dispatch-preflight hook, the constitutional CLI-dispatch skill-preload path template, the reciprocal `sk-prompt` advisor edge, and other live functional/prose referrers to the nested hub layout.

See `.opencode/specs/skilled-agent-orchestration/125-cli-external-parent/` for the full decision record and phase history. Each mode's own version history continues unchanged in its packet-local `changelog/`.
