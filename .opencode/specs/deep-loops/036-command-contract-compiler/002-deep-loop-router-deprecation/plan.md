# Plan: Deep-Loop Router Agent Deprecation

<!-- SPECKIT_LEVEL: 1 -->

---

<!-- ANCHOR:summary -->
## 1. SUMMARY

Retire the `deep-loop` primary router agent, which the compiled-command-contract mechanism rendered dead. A dependency sweep confirmed it is off every live path (no dispatchers, no registry entry, commands dispatch leaves directly, orchestrate already forbade dispatching it). The change is a scoped deletion of the three mirror agent files plus a reword of the two `@deep-loop`-specific boundary lines in each `orchestrate.md` mirror, keeping their positive leaf-dispatch rule.
<!-- /ANCHOR:summary -->

---

<!-- ANCHOR:quality-gates -->
## 2. QUALITY GATES

- Zero live references to the router remain outside historical spec artifacts (grep sweep).
- `.opencode`↔`.claude` agent-set parity holds; the mirror-sync checker passes on the changed paths (exit 0).
- No leaf agent, `mode-registry.json`, `/deep:*` command, or compiled contract is touched.
<!-- /ANCHOR:quality-gates -->

---

<!-- ANCHOR:architecture -->
## 3. ARCHITECTURE

Before: `/deep:*` request → (optional) `@deep-loop` router → resolve mode-registry → dispatch leaf. After (already live): `/deep:*` command body renders a compiled contract that names its own leaf + dispatch rules, and orchestrate resolves `/deep:*` leaves directly from its registry-backed priority table. The router node is removed from the graph; the two orchestrate boundary lines that existed only to forbid routing through it are generalized to a no-intermediary-router guardrail.
<!-- /ANCHOR:architecture -->

---

<!-- ANCHOR:phases -->
## 4. IMPLEMENTATION PHASES

1. **Verify vestigial** — sweep for every dispatch shape targeting the router; confirm registry + command-body absence.
2. **Delete** — `git rm` the two tracked mirrors (`.opencode`, `.claude`) in one commit; `rm` the untracked `.codex` local mirror.
3. **Reword** — drop the `@deep-loop` illegal-chain example and rewrite the prohibition paragraph in all three `orchestrate.md` mirrors, keeping the positive rule.
4. **Verify** — agent-set parity + mirror-sync checker on changed paths.
<!-- /ANCHOR:phases -->

---

<!-- ANCHOR:testing -->
## 5. TESTING STRATEGY

No unit tests (documentation/agent-definition change). Verification is the mirror-sync checker (exit 0 on the changed paths), a direct `.opencode`↔`.claude` basename-parity diff, and a repo-wide reference sweep proving the router is dereferenced.
<!-- /ANCHOR:testing -->

---

<!-- ANCHOR:dependencies -->
## 6. DEPENDENCIES

Depends on the compiled-command-contract rollout (phase 001 + the review/research/context/ai-council wiring) already being live — that is what makes the router redundant. No downstream consumer depends on the router.
<!-- /ANCHOR:dependencies -->

---

<!-- ANCHOR:rollback -->
## 7. ROLLBACK PLAN

`git revert` the deprecation commit restores both tracked mirror files and the original orchestrate lines; the `.codex` mirror is regenerated by the local mirror-sync hook from `.opencode`. No data or state migration is involved.
<!-- /ANCHOR:rollback -->
