# v0.1.0.0 - Initial Release

**Released:** 2026-06-14

Initial (pre-1.0) release of the `mcp-figma` skill. It teaches an agent to drive **Figma Desktop from the terminal** through the silships **figma-cli** (published as `figma-ds-cli`), with an **optional** Figma MCP via this project's Code Mode. The CLI is the primary surface — inspect, export, render, manage tokens/variables/components, and extract/import a design system; the MCP is opt-in and used only to pull Figma design data into the agent for codegen. The spine of the skill is the read-only / mutating / **destructive** gating taxonomy and the `sk-interface-design` handoff that keeps design judgment where it belongs. Modeled on the sibling `mcp-chrome-devtools` (CLI-primary + MCP-via-Code-Mode) and `mcp-open-design` (CLI + optional MCP, daemon model, gated verbs).

Version is **0.1.0** (pre-1.0) until live external verification (real figma-ds-cli install + connect + MCP discovery) is complete.

### Design decisions

- **Canonical binary `figma-ds-cli`** (npm), with explicit warnings against the unrelated `figma-cli` npm package (unic/figma-cli, bin `figma`).
- **Safe connect is the default**; the yolo `app.asar` patch is consent-gated with an `unpatch` rollback.
- **Framelink `figma` is the only documented MCP path** this release; the official Figma Dev Mode MCP is a future option.
- Minimal runtime `allowed-tools` (`Read, Bash, Grep, Glob, mcp__code_mode__call_tool_chain`); figma-cli is referenced, **not vendored** (MIT).
