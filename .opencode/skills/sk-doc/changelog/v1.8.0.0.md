sk-doc now defines and enforces a 4-part frontmatter versioning standard, and ships a deterministic engine that stamps a `version` into every in-scope skill doc.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/154-frontmatter-versioning` (phase parent)

---

## What Changed

#### Versioning standard

- **New `references/frontmatter_versioning.md`** -- the single source of truth for the 4-part `X.Y.Z.W` format, the changelog-anchored derivation (`anchor = max(frontmatter, changelog)`), the numstat-gated build segment, the line-wise insertion rule, and the enforcement rollout.
- **`assets/frontmatter_templates.md` and the nine doc templates** now carry `version` as a required field, so every doc generated by `create:*` is born versioned.

#### Engine

- **New `scripts/frontmatter-version.mjs`** -- a self-contained Node engine with `compute` (dry-run manifest), `apply` (idempotent line-wise insert), `verify` (value match), and `gate` (presence + format) modes. It resolves each skill's anchor, normalizes 3-part versions to 4-part, and counts real edits via a single `git log --follow --numstat` per file gated on `numstat > 0`.
- **New `scripts/tests/test_frontmatter_version.mjs`** -- 21 fixture-based assertions covering derivation, insertion placement, idempotency, conflict handling, and normalization.

#### Enforcement

- **`quick_validate.py` and `package_skill.py` now require a 4-part `version`** for skills (commands keep it optional).
- **New `scripts/check-frontmatter-versions.sh`** -- the corpus-wide CI / pre-commit gate; exits non-zero on any in-scope doc missing or malformed.

#### Rollout

- The standard was applied across the whole skill corpus: 2,210 in-scope docs now carry a 4-part version (12 frontmatter-less docs are skipped, not synthesized).

#### Hardening (phase 6 deep-review remediation)

- **`scripts/frontmatter-version.mjs`** -- removed a dead variable, raised the git-history `maxBuffer` from 64 to 256 MB so a pathological history cannot silently yield a zero edit count, and added a fail-closed path-containment guard so `apply` only ever writes inside the skills root even when a `--paths` manifest lists a stray path.
- **`scripts/check-frontmatter-versions.sh`** -- now fails fast with a clear message when `node` is missing from PATH.
- **`references/frontmatter_versioning.md`** -- documents the one intentional reconcile asymmetry: `SKILL.md` is reconciled to its anchor without `--update`, because the `SKILL.md` version is the anchor of record.
