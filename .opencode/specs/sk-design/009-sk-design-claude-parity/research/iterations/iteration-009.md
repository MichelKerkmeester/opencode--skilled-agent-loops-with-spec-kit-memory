## Migration Plan
1. **Phase 0 - Baseline and ownership gate:** Capture `git status`, current diff, benchmark baseline hash/report, and the five-mode routing contract; ask the human whether existing staged/uncommitted sk-design changes are intended baseline or must be isolated before edits.
2. **Phase 1 - Compatibility shell:** Preserve public `workflowMode` keys, `backendKind`, `advisorRouting`, one `graph-metadata.json`, and the hub's no-per-mode-logic rule while adding any Claude-like operating-contract prose.
3. **Phase 2 - Private procedure layer:** Add/move Claude-like procedural content inside mode packets or non-discoverable references/assets; do not create new public mode identities or packet-local graph metadata.
4. **Phase 3 - Mode-by-mode refactor:** Refactor interface/foundations/motion/audit/md-generator separately; keep `md-generator`'s Playwright extract-write-validate pipeline and mutating tool surface isolated.
5. **Phase 4 - Benchmark and parity proof:** Run router benchmark against the frozen baseline, add an after/live report as sibling output, and only then consider advisor/live parity checks.

## Compatibility Safeguards
- Keep `sk-design` as the only advisor-routable identity; mode packets remain nested implementation choices.
- Keep `mode-registry.json` as the single source of truth; do not duplicate router maps in hub prose.
- Keep `hub-router.json` vocabulary changes additive unless a human approves route-breaking alias changes.
- Keep shared references as vocabulary/reference base only, not a sixth workflow owner.
- Keep external Claude/Codex files read-only and do not copy embedded instructions blindly.

## Human Decision Points
- Decide whether existing staged/uncommitted sk-design changes are the baseline to build on, work to preserve separately, or changes to revert before implementation.
- Approve whether Claude parity means private procedure cards behind five modes or a public taxonomy expansion; evidence supports the private-procedure option.
- Approve benchmark acceptance thresholds: router-only parity, live/advisor parity, or both.
- Approve whether regenerated metadata/description files are in scope for the implementation phase.
