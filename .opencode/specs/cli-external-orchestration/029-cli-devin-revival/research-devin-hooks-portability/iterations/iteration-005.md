# Iteration 5 — Complete per-hook and per-plugin Devin port verdict matrix

## Focus

Consolidate the complete Claude hook inventory, all OpenCode plugin entrypoints, and the seven runtime-neutral guard cores into one ADR-001-ready matrix. Every row carries a Devin verdict, target event/matcher, native Claude-import disposition, rationale, and repository evidence.

## Actions Taken

- Re-read the deep-research state, strategy, configuration, phase 001 Devin contract summary, and prior iteration findings before researching the requested focus.
- Re-checked `.claude/settings.json`, `.opencode/plugins/README.md`, the 15 non-test OpenCode plugin entrypoints, the Codex hook-contract guard table, and the Claude/runtime-neutral hook documentation.
- Cross-checked the Devin event set, matcher semantics, stdin/output envelope, `DEVIN_PROJECT_DIR`, `run_subagent`, and `read_config_from.claude` facts against the phase 001 summary and iteration 2 evidence.
- Produced the consolidated matrix below. No production hook, plugin, adapter, or configuration file was changed.

## Findings

### Inventory correction and Devin baseline

The live `.claude/settings.json` contains seven top-level hook keys — `PreToolUse`, `UserPromptSubmit`, `PreCompact`, `SessionStart`, `Stop`, `SessionEnd`, and `PostToolUse` — with 19 command handlers (`.claude/settings.json:14-180`). The earlier iteration's phrase “eight event keys” was a counting error: `PermissionRequest` is one of Devin's eight lifecycle events, but it is not registered in this repository's Claude settings; the matrix includes that absence explicitly as C-20. Devin's eight-event contract, entry shape, stdin envelope, `DEVIN_PROJECT_DIR`, and top-level decision semantics are pinned in `.opencode/specs/cli-external-orchestration/029-cli-devin-revival/001-devin-contract-pin/implementation-summary.md:55-59`, with the `run_subagent` mapping confirmed in iteration 2 (`research/iterations/iteration-002.md:17-41`).

Verdict meanings are strict: **portable 1:1** means the source behavior can use the named Devin event without changing its policy semantics, subject to the stated cwd/import condition; **needs adaptation** means the policy or behavior is reusable but the transport, matcher, payload, output, or lifecycle boundary changes; **cannot port 1:1** means Devin has no equivalent hook/plugin surface for the source behavior as registered.

### Native import decision for ADR-001

Native Claude import is a compatibility baseline, not the adapter layer. It is a reasonable first path for C-10 through C-13 and C-16 because those commands are shell-only lifecycle checks; it is a conditional smoke-test candidate for C-06 and C-07 because `UserPromptSubmit` and prompt context align. It does not solve changed matchers (`Bash`, `Write`, `Edit`, `Task` versus `exec`, `edit`, `run_subagent`), Claude-specific cwd wrappers (`CLAUDE_PROJECT_DIR` versus Devin's `DEVIN_PROJECT_DIR`), undocumented `async`, nested Claude deny envelopes, missing `last_assistant_message`, `PreCompact` timing, or any OpenCode plugin registration. These limits are documented by the phase 001 contract and iteration 2/4 evidence (`implementation-summary.md:55-59`; `research/iterations/iteration-002.md:49-68`; `research/iterations/iteration-004.md:20-29,58-84`).

ADR-001 should therefore make `.devin/hooks.v1.json` authoritative for Devin-specific matchers and output, while optionally enabling `read_config_from.claude:true` for the compatible shell baseline. The seven guard cores should be wrapped by explicit Devin adapters; OpenCode plugins should be selectively rebuilt as Devin lifecycle commands or explicitly omitted when their host-only surface has no equivalent.
