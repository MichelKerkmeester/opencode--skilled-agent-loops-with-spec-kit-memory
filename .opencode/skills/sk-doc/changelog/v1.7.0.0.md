sk-doc's skill validator now recognizes CommonJS and ESM scripts and multi-extension templates, so it stops flagging valid skill files as problems.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/116-deep-skill-evolution/008-deep-skill-doc-evolution` (Level 2)

---

## What Changed

#### Validator

- **package_skill.py recognizes `.cjs` and `.mjs` scripts** -- the script-extension allowlist held only `.py`/`.sh`/`.bash`/`.js`, so every skill that ships CommonJS or ESM helpers drew false "unexpected file type in scripts/" warnings. `.cjs` and `.mjs` are now valid extensions, and a `README.md` inside `scripts/` is allowed as documentation rather than flagged.
- **Asset snake_case check handles multi-extension templates** -- the check read `file.stem`, which left the intermediate `.md` on names like `prompt_pack_round.md.tmpl` and falsely reported them as non-snake_case. It now evaluates the base name before any extension, so `.md.tmpl` templates pass.
