# v1.1.0.0 - De-vendor to Apache-2.0 with live Open Design grounding

**Released:** 2026-06-14

This release removes the skill's vendored MIT design data and makes it Apache-2.0 only. The frozen, copied design CSVs and the search engine over them are gone, and the aesthetic inventory now comes live from a real Open Design design system through the new `mcp-open-design` skill. The skill keeps its Apache-2.0 spine unchanged: `references/design_principles.md` is the same vendored Anthropic `frontend-design` guidance, and `LICENSE.txt` carries the only license the skill ships under.

---

## What Changed

### Removed the vendored MIT data and search

- Deleted the nine vendored MIT design CSVs and their directory README. They were a frozen, copied snapshot, and copying licensed data into the skill carried a license and notice burden the skill no longer needs.
- Deleted the zero-dependency BM25 design-search scripts that ranked over those CSVs. They served only the deleted data and were themselves MIT-derivative, so removing them removed the last MIT-licensed code.
- Deleted the vendored MIT license file and the third party notices file. With no MIT-covered material left on disk, the notice obligation lapses, in that order: data and code first, attribution second.

### Apache-2.0 only

- The skill now ships zero MIT-covered material. The only licensed content is the vendored Apache-2.0 guidance from Anthropic's `frontend-design` skill.
- `references/design_principles.md` is unchanged Apache content, and `LICENSE.txt` (Apache-2.0) is retained as the skill's single license.
- `SKILL.md` frontmatter declares `license: Apache-2.0; see LICENSE.txt`, and the version is now 1.1.0.

### Live Open Design grounding (the replacement)

- The aesthetic inventory is now sourced live from a real Open Design design system via the new `mcp-open-design` skill, read only when that app is installed.
- A matched system serves one of two anti-default-safe roles, chosen by the brief: reuse-ground (reuse its tokens and components before generating) or critique-against (name the closest realized default, then deviate from it).
- It is never a chooser. Exactly one system is resolved from the subject, never a pick-a-vibe menu, and the read is live and never cached into the skill. Keeping the read live is also what keeps the skill free of any new license burden.
- `references/design_inventory.md` is reframed from the vendored-CSV critique-against catalog to this live Open Design grounding, and `references/claude_design_parity.md` carries the Open Design intake and reuse bullets.

### Quality floor stays authored

- `references/ux_quality_reference.md` remains the authored quality floor. Its rules are paraphrased non-copyrightable facts (accessibility, motion, touch, responsive, forms, data visualization) and stay verbatim. Only the links to the deleted data and the MIT-provenance lines were removed.

### sk-doc surfaces realigned

- `feature_catalog/` drops the design-data-sets and design-data-search entries and replaces the section with a single Open Design grounding feature. Feature count moves from 13 to 11.
- `manual_testing_playbook/` rewrites the critique-against scenario to read a live Open Design system as the default to deviate from, with a no-chooser and no-cache negative control, and rewrites the licensing scenario to assert the Apache-2.0-only state where a clean grep is the pass signal. The folder topic is renamed to `04--system-as-critique-against/`.
- `graph-metadata.json` drops the removed data and search from `key_files`, rewrites the causal summary to describe an Apache-2.0-only skill that sources design systems live from Open Design, and refreshes the data-and-search trigger phrases and topics.

## Notes

- The design process, rules, and anti-default mandate are unchanged. `design_principles.md` stays the authority.
- The Open Design grounding is optional and gated on `mcp-open-design` plus the installed app. When no real system fits, the free-axis anti-default process governs exactly as before.
- v1.0.0.0 stays as historical record of the data-backed arrangement this release supersedes.
