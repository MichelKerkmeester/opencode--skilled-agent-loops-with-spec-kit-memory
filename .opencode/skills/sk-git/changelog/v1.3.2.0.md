## [**1.3.2.0**] - 2026-07-14

Documentation-surface enrichment and a hyphen-case filesystem migration. No script behavior, branch grammar, or command surface changed. The skill adopts the hyphen-case naming convention ahead of the repo-wide program, rewrites the README to the create-readme canon, adds code READMEs for the executable surfaces, and reconciles every documentation claim surfaced by a fresh independent review.

## What Changed

#### Hyphen-case migration

- **The `references/`, `assets/`, `feature-catalog/` and `manual-testing-playbook/` trees are now hyphen-case.** 66 files and 13 directories moved from snake_case to hyphen-case via `git mv` (history preserved), and every path reference — the SKILL.md Smart Router `RESOURCE_MAP`/`LOADING_LEVELS`/`DEFAULT_RESOURCE`, relative cross-links, link labels, and per-feature metadata — was repointed to the new names. `package_skill.py --check` snake_case findings on `references/`/`assets/` are advisory-only until the repo-wide naming program flips the checker.

#### Documentation surface

- **`README.md` rewritten to the create-readme canon.** A HOW IT WORKS section now conveys the skill's unique surface: the owner-first worktree grammar, the clone-wide number allocator and its lock, the ask-first workspace rule, launch-wrapper autosync, the reaper contract, deterministic commits, and the safety refusals.
- **Four code READMEs added** for `scripts/`, `scripts/tests/`, `.github/workflows/` and `.github/hooks/scripts/`, each accurate to the files it documents (including the Copilot hook wrappers' file-writing fallback).
- **feature-catalog and manual-testing-playbook** verified against the sk-doc `create-feature-catalog` / `create-manual-testing-playbook` templates and `template_rules.json`; the optional Root-vs-Feature rule was added for fuller scaffold alignment.

#### Review remediation

- **A fresh independent review reconciled several stale documentation claims.** The README's worktree-lane framing now marks `wt/{NNNN}-{slug}` as legacy/non-conformant (owner-first is current), the commit-type priority order matches the runtime rule, the allocator script docs carry the correct `create-detached <slug> [base]` and `validate-pair <branch> <dir>` signatures, the test-harness expected output reads `FAIL=0`, and the verification commands point at gates that actually pass.
