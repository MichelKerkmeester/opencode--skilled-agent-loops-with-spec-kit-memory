# v3.6.0.0, The Memory System Earns Its Trust Back

The 026 release gave the daemons a second door. This one goes through that door and reworks the rooms behind it. Spec-027 is a research-derived refinement pass over the memory and search system — the questions came out of the peck, gem-team, memclaw and OpenLTM study lanes, and the answers land here as a single theme: retrieval that is more relevant, more resilient, and safer to trust. A corrupted vector shard no longer silently degrades search; the lexical fallback no longer leaks memory or drops scoped results; automated writers can no longer quietly clobber a human's edits; and a daemon whose transport drops mid-session can be adopted instead of duplicated.

None of it changes the everyday call shape. `memory_search`, `memory_save` and the rest behave exactly as before for the common path — the work was in the failure paths, the corpora that don't fit the happy case, and the guards that were missing.

## Vector Resilience, From Detection to Durability

#### What went wrong

A live malformed vector shard was observed silently degrading search — the read path trusted a shard that could not answer. And even once detection and rebuild existed (013), the "this shard needs repair" intent lived only in memory: a restart before the rebuild finished lost it, and the next boot attached a fresh empty shard while health still reported success — permanent recall loss with nothing flagged.

#### The fix

Phase 013 added detection, quarantine and auto-rebuild with hardened dimension discovery. Phase 020 made it durable: a repair-pending sentinel is persisted, and boot uses a completeness check — vector rowcount against the `memory_index` success count, not a probe that returns OK for a missing file — to decide between resuming a real repair and clearing a stale sentinel on an already-rebuilt shard. The quarantine rename is itself a durable marker, so repair resumes even when no sentinel write could land. An in-flight guard stops a double boot-attach from scheduling the same rebuild twice.

## The Lexical Fallback Grew Up

The packed in-memory BM25 engine (014) replaced a memory-hungry fallback with typed-array postings and restored BM25F field weighting, so title and trigger matches rank above body noise. The realistic-corpus warmup then exposed a 743MB RSS spike against a 150MB budget; phase 017 cut it to 136MB with no-copy chunked postings and Uint8→16→32 width promotion, ranking byte-identical, and re-enabled the hard RSS gate with peak sampling. Phase 021 fixed the last relevance bug in the lane: it applied the caller's limit before resolving spec-folder and deprecated-tier filters, so a scoped search could return fewer results than existed — now it over-fetches the candidate set (with batched metadata resolution that stays under SQLite's parameter limit), filters, then truncates.

## Trust, Provenance, and Safe Writes

Phase 018 tracks adoption of the shipped 027 features across the system — observability, the provenance guard, BM25, BFS, feedback calibration, tombstones. The provenance work closes a real gap: automated reducer and feedback writers now inject provenance context, so the write-ingress guard enforces the constitutional rule that automated writers never overwrite manual or constitutional fields, and the publication gate no longer excludes machine-authored rows for missing provenance.

---

## The Advisor Joins the Resilient Three

Phase 019 brought the mk-skill-advisor launcher up to the resilience bar the spec-memory and code-index launchers reached in 026/027: an owner lease, a reconnecting session proxy, and — the piece that mattered most — acting on a dead-socket respawn decision instead of discarding it, so a hung daemon is reaped and replaced under a bootstrap lock rather than leaving a session stranded or spawning a second writer.

## Internal Seams (No User-Facing Change)

- **011 — command presentation/router separation.** Memory, speckit, create and doctor command families had their workflow routing split from their Markdown presentation contracts. A structural refactor; behavior and output are unchanged. **No changelog entry warranted beyond this note.**
- **012 — causal traversal BFS.** Two recursive-CTE graph traversals whose join condition defeated index use were replaced by a shared app-level BFS helper. Same traversal results, fewer full scans — a performance internal. **No behavior-facing entry warranted beyond this note.**
- **015 — storage adapter ports.** The five-port seam over better-sqlite3 is hygiene and testability groundwork; no caller-visible behavior changed.

---

## Memory-Write Safety, Hardened

Two late phases tightened the write path's integrity. **022 — provenance injection**: automated reducer and feedback writers now tag their writes with source-kind provenance (`lib/storage/write-provenance.ts`), so the constitutional "automated writers never overwrite manual" guard and the publication gate can rely on it; two pre-existing guard-coverage gaps (same-path reindex-retire, feedback auto-promotion lacking a source_kind ingress check) are documented as follow-ons that this tagging unblocks. **023 — idempotency flag-ON correctness** (`SPECKIT_MEMORY_IDEMPOTENCY`, default OFF): replay now returns the original response verbatim, receipts are immutable first-writes (`ON CONFLICT DO NOTHING`), the receipt-write guard never leaves a retry-blocking receipt, and a concurrent first-write loser replays the winner instead of diverging — fixed as a prerequisite to any future enablement, which stays gated pending a dist rebuild.

## Verification

Every behavior-changing phase shipped under its own deep review: five independent review seats (three for the single-function 021 fix), a Fable 5 adversarial adjudication, remediation of confirmed findings, and a scoped commit. Ranking parity for the BM25 work was held byte-identical against the existing oracle suites; the vector durability and advisor lifecycle work was verified against temp-fixture, sandboxed-socket tests that never touch a live shard or host daemon. The dual-stack CLI surface (`spec-memory` 37 / `code-index` 8 / `skill-advisor` 9) stayed green throughout.
