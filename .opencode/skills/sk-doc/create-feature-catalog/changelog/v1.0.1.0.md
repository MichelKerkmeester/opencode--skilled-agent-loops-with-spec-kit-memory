# v1.0.1.0 - Smart-routing section conformance

Structural normalization so `SKILL.md` satisfies the canonical section contract enforced by the shared `package_skill.py --check`. No workflow, rule, or authoring guidance changed — this is a heading and reference-surface fix only.

## What Changed

- Split the merged `WHEN TO USE + SMART_ROUTING` header into two conformant sections: `## 1. WHEN TO USE` (activation triggers and when-not-to-use boundary) and `## 2. SMART ROUTING` (creation decision rule, decision pseudocode, and family boundary). The prior underscore-joined heading did not match the required `SMART ROUTING` section token.
- Added a dedicated `## 11. REFERENCES` section so the required references surface is a first-class heading rather than only an inline resource list.
- Added a `## 10. SUCCESS CRITERIA` section capturing the completion bar for a catalog package.
- Added the top-of-file `<!-- Keywords: ... -->` discovery comment.
- Renumbered all H2 headings contiguously and updated the one intra-file cross-reference that the renumber shifted.

## Why

The section headings, not the content, blocked packaging validation. Making the routing surface, references surface, and success bar explicit headings keeps the packet legible and machine-checkable without altering the feature-catalog authoring contract.
