This release adds the missing sk-doc section dividers to cli-devin's reference and asset docs. Several files ran their numbered H2 sections together with no `---` separator between them, and two reference docs also lacked the leading divider before section 1 that the reference template uses. The separators are in now, so the docs scan cleanly and match the rest of the skill set. No dispatch behavior changed.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/130-prompt-knowledge-layering/014-doc-divider-sweep-and-sk-prompt-cleanup/` (Level 1)

---

## What Changed

#### Documentation

- **Section dividers added** -- `references/context-budget.md`, `references/output-verification.md`, `assets/confidence-scoring-rubric.md`, `assets/deep-loop-iter-template.md` and `assets/prompt_quality_card.md` were missing the standalone `---` thematic break that the sk-doc templates require before each numbered H2 section. We inserted the missing blank-padded dividers before every non-first H2, and added the section-1 leading divider to the two reference docs that lacked it (reference docs carry a `---` before section 1, after the title). The edits add only blank and `---` lines, so a content-skeleton diff against the prior version is byte-identical on every non-blank, non-divider line, verified.
- **Two docs deferred** -- `references/cli_reference.md` and `references/quota-fallback.md` carried unrelated uncommitted content edits at sweep time, so their divider fixes were held back to avoid bundling that work. They will be swept alongside the owner review of those content edits.
