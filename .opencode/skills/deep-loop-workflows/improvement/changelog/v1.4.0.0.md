# v1.4.0.0 - Skill Rename: sk-improve-agent to deep-improvement

Date: 2026-05-06

## 1. OVERVIEW

This changelog records the active skill-family rename from `sk-improve-agent` to `deep-improvement` and the migration boundary for paths, metadata, and runtime references.

## 2. SUMMARY

This release renames the skill from `sk-improve-agent` to `deep-improvement` so the skill folder, skill identity, metadata, and active documentation use the `deep-*` naming convention while preserving the existing agent and command surfaces. Git history around the rename shows the pre-v1.5 family still using `sk-improve-agent` in changelog/docs commits and the later v1.5 packet completing the separate `@improve-agent` agent rename.

## 3. WHAT CHANGED

- Skill folder: `.opencode/skill/sk-improve-agent/` moved to `.opencode/skill/deep-improvement/` in the then-current singular OpenCode layout. The later plural-layout migration moved it again to `.opencode/skills/deep-loop-workflows/improvement/`.
- Skill identity: skill frontmatter, trigger phrases, key files, and derived entities now use `deep-improvement`.
- Advisor scoring tables: skill advisor routing and scoring references target the new skill name.
- Command surfaces in 3 runtimes: OpenCode/Copilot, Claude, and Codex command surfaces keep `/deep:agent-improvement` while routing references move to the new skill path.
- Agent skill-matrix lines in 3 runtimes: OpenCode/Copilot, Claude, and Codex agent matrices keep the then-current `@improve-agent` identity while pointing at `deep-improvement`; v1.5.0.0 later renames the agent itself.
- Root docs: top-level references describe the new skill name and path.
- Install guides: installation and packaging instructions install `deep-improvement`.

## 4. MIGRATION

The old `sk-improve-agent` skill name no longer resolves. Use `.opencode/skills/deep-loop-workflows/improvement/` for current skill files, references, assets, scripts, feature catalog entries, and manual testing playbooks. The command `/deep:agent-improvement` is unchanged. Agent identity migration is documented separately in v1.5.0.0.

## 5. PRECEDENT

This rename follows the sister rename in `specs/skilled-agent-orchestration/070-sk-deep-rename/`, which moved `sk-deep-review` and `sk-deep-research` to `deep-review` and `deep-research`.
