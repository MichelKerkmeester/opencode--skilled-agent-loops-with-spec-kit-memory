This release brings sk-code's smart router and its bundled assets up to the sk-doc skill-authoring standard, a follow-on alignment pass after the ponytail-based refinement. Nothing about surface detection, routing precedence, or the Iron Law changes; the work is conformance and consistency: the router section now shows the two canonical elements it was missing, the authoring checklists read to the asset template, and the always-loaded routing references shed stale link labels and duplicates.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/146-sk-code-ponytail-based-refinement/007-sk-code-asset-router-alignment` (Level 2)

---

## New Features

Three conformance additions, none of which change routing behavior, the STACK_FOLDERS surface map, or either Iron Law line.

#### Router §2 made canonical (loading levels + fallback)

The `SKILL.md §2 SMART ROUTING` section gained the two elements the canonical `sk-doc/assets/skill/skill_smart_router.md` standard makes visible inline: a **Resource Loading Levels** table (ALWAYS / CONDITIONAL / ON_DEMAND) and the **UNKNOWN_FALLBACK** disambiguation checklist. Both mirror `references/smart_routing.md`, which keeps the heavy `INTENT_SIGNALS` / `RESOURCE_MAP` pseudocode factored out, so the surface-first two-axis design and the deferral are preserved — only the canonical surface is now readable in SKILL.md. The four router guards stay green (STACK_FOLDERS parses, both Iron Law lines intact, SKILL.md validates, the ladder gate is unchanged).

&nbsp;

#### Authoring checklists aligned to the asset template

The five `assets/opencode/checklists/*_authoring.md` files (agent, command, mcp_server, skill, spec_folder) failed `validate_document.py --type asset` because they led with `## 1. PURPOSE` instead of the template's required `## 1. OVERVIEW`. Each now opens with `## 1. OVERVIEW` carrying a `### Purpose` and a `### Usage` subsection (the old WHEN TO USE content), with `---` dividers and renumbered sections. The checklist content is preserved verbatim; all eleven checklists in the folder now validate as assets.

&nbsp;

#### Always-loaded reference cleanup

`stack_detection.md` and `phase_detection.md` are part of the always-loaded `DEFAULT_RESOURCE` preamble, so accuracy there is paid on every route. Both carried stale link labels left over from a past rename (`code_surface_detection.md`, `intent_classification.md`, `resource_loading.md`) and duplicate links. The labels now match the real filenames (`stack_detection.md`, `smart_routing.md`) and the duplicates are gone. `smart_routing.md` was left as-is — it already validates and its machine-readable `RESOURCE_MAP` is drift-guarded, so resurfacing it would risk the guard for no gain.
