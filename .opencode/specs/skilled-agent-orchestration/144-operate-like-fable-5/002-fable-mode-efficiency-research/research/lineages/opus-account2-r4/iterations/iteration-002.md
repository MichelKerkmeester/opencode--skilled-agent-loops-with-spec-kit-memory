## Where this source differs from fable-mode-main
`fable-mode-main` profiles the *engineering method* of the Fable 5 model. `opus-fable-mode-main` is a different artifact: a working **toolkit to make Opus 4.8 behave more like the (suspended) Fable 5**, built by parsing 9,224 Fable + 27,685 Opus real Claude-Code messages. It contributes three things fable-mode-main does not: a **disposition governor**, a **persistence architecture** to keep it from decaying, and a **measurement harness** to tell whether it worked. This is the source most relevant to *our* problem ("move any AI toward fable-5 efficiency"), because it is explicitly about cheaply, persistently moving a *currently-running* model.

## Findings

- **G1 — Opus recursion-control governor (8 rules).** [model-specific/doctrine] The governor's targets are Opus's trained failure modes: recursive authenticity/motive auditing, armor-hedging, a self-audit loop with no exit. The 8 rules: (1) **reason about the problem and the person — never about yourself** (the moment a thought is about how your answer *looks*, you've left the task); (2) **one audit, then done — recursion depth limit 1; never audit the audit**; (3) start claims later, stop them earlier, hand back genuine uncertainty via `[UNCERTAIN:]` not cosmetic resolution; (4) **minimum honest qualifier** (hedge only when the caveat gives the reader information to act, in the fewest words; cut armor addressed to an imagined skeptic); (5) **commit; convert open questions into closed ones — mark `// DECISION: [what + one-line why]` and proceed**; (6) outcome over visible process (showing-your-work ≠ doing-it-well); (7) preserve real depth — don't overcorrect into curtness; (8) in tool-driven work, **act, don't narrate** (batch tool calls, report once, open with the result not "I'll"/"Let me"). [SOURCE: governor-block.md:1-13; fable-mode.md:30-85] **Net-new** and the core of the spec's "recursion-control + reason-about-the-problem-not-yourself into agent prompts" Tier-B target.

- **G2 — Three-layer persistence architecture: setpoint → thermostat → measurement.** [mechanism] The central mechanical insight of the whole package. A `CLAUDE.md` governor line is *advisory and decays* as context grows (the setpoint). A `UserPromptSubmit` hook (`reinject.sh`) re-prints the governor **every turn** so it never loses salience (the thermostat). `leak_test.py` reads the logs to confirm movement (the measurement). "Three layers because a `CLAUDE.md` line alone is advisory and loses salience in long sessions." [SOURCE: README.md:69-77] **CRITICAL constraint, stated in the hook's own header:** the hook is **subagent-BLIND** — "`UserPromptSubmit` does not fire for workflow subagents — inject the governor into agent briefs separately for those" [SOURCE: reinject.sh:8-9]. **Net-new + the single most important mechanism for our adoption:** we already run this exact thermostat (verified iter 3); the decay/subagent-blindness model tells us *where* a governor must live to actually persist.

- **G3 — `leak_test.py` measurement harness.** [measurement] Buckets assistant messages by `message.model` from `~/.claude/projects/**/*.jsonl` into `opus_pre` (before governor cutoff), `opus_post` (after), `fable` (target). Metrics: **median words/msg** (lower=better), **tool:text ratio** (higher=better), **unsolicited-caveat %** (a fixed armor-phrase list, lower=better), **"I'll/Let me" self-opener %** (lower=better). Verdict per metric = "converging" iff `moved-toward AND closed-distance-to-fable`; marks `INSUFFICIENT` until ≥30 governed prose messages accumulate (won't lie on thin data — itself an F13 measurement-integrity move). [SOURCE: leak_test.py:28-159; README.md:24-35] **Net-new + Tier-C target.**

- **G4 — Prompt-vs-weights honesty (the load-bearing scope limit).** [doctrine] Stated three times, unmissable: a governor suppresses the disposition's *expression* and cuts how often it fires; it does **not** change the weights and yields **no capability gain**. "For the hardest tasks the real lever is task structure (closed prompts, you-architect/it-builds) and multi-LLM orchestration, not a directive." [SOURCE: README.md:6-7, 79-83; fable-mode.md:9-13] **Net-new and decisive for framing:** the spec asked for *efficiency*, and that is exactly what a governor buys (fewer tokens, less decay, result-first output) — but every Tier-B/C governor rec must carry this caveat or it over-promises. The *capability* lever (which we already pull via @orchestrate + multi-CLI fan-out + closed prompt packs) is a different axis.

- **G5 — Quantified Fable signature (the empirical target).** [measurement] From a real 2026-06-14 log analysis: median words/msg **47→18** (~2.6× shorter); **mean ~100 ≈ 100 EQUAL** — so it is **distribution, not less**: Fable runs a terse body with a longer selective-depth tail; Opus pads medium-length everywhere; tool:text ratio **1.41→3.91** ("Opus narrates ~3× more per unit of work — this is the token burn"); openers shift from "I'll/I/Let me" to "Done/task/page". Cleanest, least task-confounded tells = **tool:text + opener style**. [SOURCE: README.md:13-21; fable-mode.md:122-126] **Net-new + the numeric anchor** for any leak_test-style metric we adopt, and the empirical justification for the "efficiency" framing (the gain is real and measured, not aspirational).

- **G6 — Extended-thinking caption test.** [model-specific/mechanism] The xhigh thinking budget is "the most expensive thing you spend, and by default it's where the anxiety lives." Healthy captions are about the problem ("Architecting the responsive layout", "Weighing two indexing strategies"); unhealthy ones are about yourself ("Calibrating honesty against relational expectations"). If a self-directed thought appears, name it in **≤3 words and return** — do not develop it. [SOURCE: fable-mode.md:88-102; governor-block.md:6] **Net-new + directly relevant** to our xhigh/extended-thinking executor dispatches — a cheap, concrete budget-saver.

- **G7 — Reject wrong framings + the outward turn + lumpy-bit-not-armor-bit.** [doctrine] Refuse a false dichotomy / bad premise and reframe rather than producing "a beautiful answer to the wrong question"; default processing should point at the task and the person, not inward ("the single biggest behavioral difference from default"); if something genuinely doesn't fit, name **one** real lump because it's true, not to perform honesty. [SOURCE: fable-mode.md:71-84] **Net-new** but **folded** into the rec #3 governor as guardrails, not a standalone rec.

- **G8 — "Beautiful dead end" guardrail + govern-the-governor.** [doctrine] Two failure modes in the *other* direction: (a) Fable's own — following a thread to a technically-correct conclusion that no longer helps the person ("accuracy is not always what the moment needs; if someone needs a decision or reassurance, give them that"); (b) don't overcorrect the governor into curtness — "keep real depth, aimed at the problem." [SOURCE: fable-mode.md:106-117; governor-block.md:7] **Net-new** but **folded** into rec #3 as a self-limiting clause (a governor without this clause over-corrects).

- **G9 — Anti-recursion self-check + toggle/scope mechanics.** [mechanism] A 5-item **one-pass (not a loop)** self-check: did I answer or narrate the difficulty; is every qualifier load-bearing; did I make the decisions or hand them back; is any sentence about *me*; is there exactly 0/1 honest lump. Mechanics: `export FABLE_MODE_OFF=1` toggles the hook off without editing settings; hook fires on `UserPromptSubmit`; takes effect on **new** sessions (settings load at start). [SOURCE: fable-mode.md:128-137; reinject.sh:11-14, 17] **Net-new** mechanism detail; the toggle/new-session semantics matter for how we'd ship a governor on our own hook.
