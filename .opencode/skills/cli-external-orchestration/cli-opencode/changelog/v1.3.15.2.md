# cli-opencode v1.3.15.2 — GPT-5.6 model-roster update (full Sol/Terra/Luna catalog)

**Released**: 2026-07-09
**Status**: Logic update. Replaces every live GPT-5.5 reference with the GPT-5.6 family, documents the full twelve-slug GPT-5.6 catalog wherever the OpenAI roster is enumerated, adopts `gpt-5.6-sol` as the flagship default, and fixes a model-tier-vs-reasoning-effort wording conflation.

---

## Summary

OpenAI's GPT-5.5 three-tier family (`gpt-5.5` / `gpt-5.5-fast` / `gpt-5.5-pro`) is superseded by GPT-5.6, whose live catalog (confirmed via `opencode models openai`) is twelve slugs: a bare tier plus three named families — **Sol**, **Terra**, **Luna** — each offered in three model tiers (base / Fast / Pro):

- Bare: `gpt-5.6`, `gpt-5.6-fast`, `gpt-5.6-pro`
- Sol: `gpt-5.6-sol`, `gpt-5.6-sol-fast`, `gpt-5.6-sol-pro`
- Terra: `gpt-5.6-terra`, `gpt-5.6-terra-fast`, `gpt-5.6-terra-pro`
- Luna: `gpt-5.6-luna`, `gpt-5.6-luna-fast`, `gpt-5.6-luna-pro`

`gpt-5.6-sol` is the adopted flagship default. Every documented single-option default/premium reference points at the Sol tier of the matching old tier (`gpt-5.5`→`gpt-5.6-sol`, `gpt-5.5-fast`→`gpt-5.6-sol-fast`, `gpt-5.5-pro`→`gpt-5.6-sol-pro`); every full-roster enumeration lists all twelve slugs.

## Why

GPT-5.5 is no longer the live OpenAI premium option this skill should recommend — GPT-5.6 Sol is the operator's adopted flagship default, and the model family expanded from one line of three tiers into a bare tier plus three named families. Leaving GPT-5.5 slugs in place would route dispatches at a superseded family; documenting only Sol/Terra would hide the live Luna family and the bare tier.

## Model Family: Before → After

- **Before**: `openai/gpt-5.5`, `openai/gpt-5.5-fast`, `openai/gpt-5.5-pro` (one line, three tiers)
- **After**: twelve live GPT-5.6 slugs — bare (`gpt-5.6`/`-fast`/`-pro`) plus Sol, Terra, Luna (each base/Fast/Pro)
- `gpt-5.6-sol` is the flagship default OpenAI model for this skill; the other families and the bare tier are documented alongside it wherever the full roster is enumerated.

## Coherence Fix (model tier vs reasoning effort)

The prior draft of the Model Selection paragraph called base/Fast/Pro "three reasoning tiers", conflating two independent axes:

- **Model tier** — base / Fast / Pro — is chosen by the model slug (e.g. `gpt-5.6-sol-fast`).
- **Reasoning effort** — `--variant none`/`low`/`medium`/`high`/`xhigh` — is a separate lever applied on top of any tier.

The wording now names base/Fast/Pro as "model tiers" and states they are orthogonal to the `--variant` reasoning-effort lever, matching the `--variant` table in `references/cli_reference.md` §5.

## Caveats

- **Per-tier `--variant` ranges are carried over, not re-measured.** The Pro-tier reasoning-effort restriction (`medium`/`high`/`xhigh`, vs the full `none`..`xhigh` for base/Fast) was a documented fact for `gpt-5.5-pro`; it is generalized here to all four GPT-5.6 Pro slugs (`gpt-5.6-pro` / `gpt-5.6-sol-pro` / `gpt-5.6-terra-pro` / `gpt-5.6-luna-pro`) as a documentation default. `opencode models openai` lists slugs but not variant ranges, and the source `gpt-5.5-pro` slug is no longer in the live catalog to re-check, so this generalization is inferred — smoke-test a Pro slug at `--variant low` before relying on the restriction. This caveat is now flagged **inline** in `references/cli_reference.md` §5 (the point of use SKILL.md directs readers to), matching the file's convention for its other unverified mappings.
- **The RM-8 Layer 4 safety recommendation names a Copilot-unverified model.** Both `references/destructive_scope_violations.md` §Layer 4 and SKILL.md ALWAYS rule 15 recommend `cli-copilot` + `gpt-5.6-sol --reasoning-effort high` for multi-phase deep-review dispatches. Whether the GitHub Copilot surface exposes a `gpt-5.6-sol` slug is UNVERIFIED — `opencode models openai` does not cover Copilot, and the only Copilot-catalog check on record was for `gpt-5.5`. Because this is the destructive-scope mitigation layer, a nonexistent model there would silently fall back to the risky default; both locations now carry a "verify availability on the Copilot surface" annotation instructing the reader to pick an available Copilot model instead. (Pre-existing and out of scope for this change: no `cli-copilot` skill exists under `.opencode/skills/` at all.)
- The GPT-5.5-era note that "the model cache lists the fast variant as the base slug + `experimental.modes:['fast']`" was an implementation quirk specific to that model's cache representation and is not carried forward for GPT-5.6 without independent verification; the `-fast` model tiers are documented as first-class slugs. Confirm live slugs and any cache-representation quirks via `opencode models openai`.
