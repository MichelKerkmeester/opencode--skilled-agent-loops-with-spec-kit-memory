This release gives the detailed frontmatter on skill reference/asset docs a real runtime consumer. With `SPECKIT_ADVISOR_DOC_TRIGGERS=true`, every `skill_graph_scan` harvests `title`, `description`, `trigger_phrases`, `importance_tier` and `contextType` from each skill's `references/` and `assets/` markdown into a new `skill_docs` table, the daemon watcher tracks those docs for freshness, and the scorer turns their phrases into dampened routing signal inside the existing `derived_generated` lane. Doc-specific prompts now rank the owning skill AND point at the exact file to open via an optional `matchedDocs` response field. The flag defaults off, and flag-off behavior is byte-identical to v0.7.0 â€” that invariance is the hard contract in the new test file.

## What Changed

#### Doc harvest into the skill graph

- **`skill_docs` table** -- additive DDL (schema + migration), UNIQUE per skill+doc path, FK-cascade with skill deletion. The harvest runs for every skill on every scan â€” doc edits never touch `graph-metadata.json`, so node-level hash skips cannot gate doc work â€” and stays cheap via per-doc content hashes. Stale rows are swept per skill; unreadable docs warn (`DOC-READ-FAILED`) and continue.
- **Capped walker, list-aware parser** -- references/assets recursion (READMEs excluded, depth 6, 200 docs/skill), block and inline `[a, b]` list forms, 12 phrases/doc, 300-char field clamp.
- **Watcher freshness** -- each harvestable doc becomes a `doc-frontmatter` watch target when the flag is on, so editing a reference re-indexes its skill without a manual scan.

#### Dampened scoring with matched-doc pointers

- **Rides the derived lane** -- no sixth lane, no weight redistribution. Per skill: top-3 strongest doc matches, each multiplied by its tier weight (constitutional/critical 1.0 down to deprecated 0.2), summed contribution capped at 0.45 â€” below what one curated skill-level phrase earns, so doc volume assists ranking but cannot hard-route alone (~0.68 confidence for a doc-only match at normal tier).
- **`matchedDocs` on the strict schema** -- max 3 skill-relative paths, extracted from `doc:` evidence behind a sanitization allowlist (references/assets prefix, markdown only, traversal rejected); new `doc_reference_signal` evidence type.

#### Parity and rollout hardening

- **Python Gate-2 parity** -- `skill_advisor.py` harvests the same doc phrases from disk under the same flag. The rollout smoke exposed that the source-metadata signal merge only ran on the legacy JSON graph path; the SQLite loader (which always wins in this repo) now merges it too.
- **Launcher env allowlist** -- the same smoke caught the daemon child never receiving the flag: `mk-skill-advisor-launcher.cjs` forwards only `CHILD_ENV_ALLOWLIST` keys, and `SPECKIT_ADVISOR_DOC_TRIGGERS` is now on that list. Any future advisor env flag needs the same entry to reach the daemon.

## Upgrade

No migration required: the table is created additively on open, and with the flag unset nothing changes. To enable, set `SPECKIT_ADVISOR_DOC_TRIGGERS=true` in the advisor env block of each runtime config (this repo sets it in `opencode.json`, `.claude/mcp.json` and `.codex/config.toml`), then adopt via fresh session â€” the advisor launcher exits if its daemon child is SIGTERMed, and launchers spawned before the flag flip keep their old env, so every advisor-attached session must end before one fresh session respawns the daemon with the flag. Verify with a trusted `skill_graph_scan` (expect `docs` counters) and an `advisor_recommend` for a doc-specific phrase (expect `matchedDocs`). Doc coverage grows with the skill-frontmatter authoring campaign (84 of ~355 docs carry `trigger_phrases` at release). Rollback: unset the flag (rows persist, ignored) or drop the `skill_docs` table.
