The ONE loop unifying the 3 bespoke shadow mechanisms (feedback=gate+cycles; trigger=stats-no-gate; calibration=flag-no-metric). Found a stranded shadow: the learned Stage2 combiner "computed but NOT used, shadow-only" with no promote path (`search-flags.ts:517`).

Bridges the unified gate ↔ the eval harness.

## Key finding
The system has exactly ONE promotion gate (`evaluatePromotionGate`, `shadow-scoring.ts:547`), shaped for ONE class (ranking levers) because its only metric is rank-delta and `compareRanks` scores only the live∩shadow intersection. The reusable gate keeps the **spine** (deterministic 20% holdout · ≥2 non-regressing cycles · promote/wait/rollback · one generalized ledger with `candidate_id`+`candidate_class`+metric-JSON) and **swaps a metric panel per class**: **(R) ranking** [A3-S5, learned combiner] → nDCG@10 + **nDCG@1** + a **demotion-count guard** (catches S5 demoting correctly-fused non-vector hits) + latency; **(G) gate/calibration** [A1, A2] → **ECE+Brier+precision/recall/FP** (net-new), golden-set labels not `adaptive_signal_events`; **(C) new-channel/recall** [A6 trigger-union, #4 fused-summary] → **recall-delta@k over the UNION** + FP-rate + cost. Promote-on-evidence = the flag-symbol lifecycle (`isOptInEnabled`→`isFeatureEnabled` on `promote`). **The §13 isotonic calibration is frozen at opt-in *precisely because no CLASS-G gate exists to manufacture its promote evidence* — A8 supplies the missing gate, not a new flag.**
