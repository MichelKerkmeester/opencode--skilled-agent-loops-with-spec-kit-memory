# v1.2.0.0 - Seed-of-thought variation diversity

**Released:** 2026-06-14

This release adds one mechanism for the moment the skill is most likely to fall back to a default: when a brief asks for several directions at once. A median-biased model returns N safe versions of the same layout no matter how the prompt is worded. The new `references/variation_diversity.md` adapts the string seed-of-thought technique to the skill's grounded, anti-default philosophy: a committed random seed sets a non-median starting point inside a subject-grounded option space, then the remaining directions are spread to be genuinely distinct. Grounding, the critique against AI-default looks, and the quality floor stay primary. The seed only breaks the tie the median would otherwise win, and it is never surfaced as a style chooser.

---

## What Changed

### New reference: variation_diversity.md

- A debiasing mechanism that activates only when two or more directions are requested. For a single design the existing critique against defaults already does the job, so the mechanism stays out of scope there.
- The seed is a debiaser, not a decider. It changes where reasoning starts, never what survives. Every direction it points at is still grounded in the subject, justified, and run through the anti-default critique, and a direction that fails the critique is dropped rather than kept.
- The grounded option space is built before the seed touches anything. The agent enumerates candidates on one diversity axis (layout, signature, palette family, or type pairing), each already defensible for the subject, with the AI-default named at index 0 as the critique baseline. The seed works over the non-median set only, so a produced direction can never be the median.
- The procedure is documented end to end: commit a random 12-character string, sum its ASCII values, take the start as the sum modulo the non-median set size, spread the rest with a coprime stride, and report the trail for auditability. A worked letterpress-studio example carries real numbers through the whole flow.

### SKILL.md hook

- Version moves to 1.2.0.
- SMART ROUTING gains a keyword trigger ("give me N variations", "show me options", "multiple directions"), a conditional resource-loading row for the new reference, and a router pseudocode branch that loads it when the brief asks for two or more directions.
- A sixth ALWAYS rule requires the seed-of-thought debias for multiple directions and restates the no-style-chooser guardrail.
- Section 5 lists the new reference with a consult-only-when-more-than-one-direction note.

### Routing and discoverability

- `graph-metadata.json` adds `references/variation_diversity.md` to `key_files`, adds a variation-diversity trigger phrase, and extends the causal summary to record the new mechanism.
- The README Related Documents table lists the new reference so it is not orphaned.

|---|---|---|
| Multiple directions | Standard critique, no explicit debias | Seed-of-thought debias over a grounded, median-excluded option space |
| Median pull on N variations | Unaddressed; the model could return N safe copies | A committed seed forces a non-median start and a distinct spread |

- The design process, rules, and anti-default mandate are unchanged. `design_principles.md` stays the authority and grounding stays upstream of the seed.

- v1.1.0.0 stays as historical record of the de-vendor to Apache-2.0 that this release builds on.
