This release makes cli-claude-code authenticate through the Claude subscription OAuth only, and adds the current-generation Claude models to the CLI reference. The `ANTHROPIC_API_KEY` path is removed from the dispatch flow, the auth pre-flight, the reference tables, and the troubleshooting guidance. The two OAuth paths that remain are `claude auth login` (interactive) and `claude setup-token` (a non-interactive OAuth token for CI/CD).

---

## What Changed

#### Auth Model

- **Claude subscription OAuth only** -- `claude auth login` (interactive browser flow) or `claude setup-token` (non-interactive CI/CD OAuth token). The `ANTHROPIC_API_KEY` env-var method is no longer offered.
- **Auth pre-flight simplified** -- the two-variable (`ANTHROPIC_KEY_OK` / `CLAUDE_OAUTH_OK`) decision tree collapses to a single OAuth check via `claude auth status`. When not authenticated, the skill asks the user to run `claude auth login` and does not dispatch.
- **Reference and troubleshooting reframed** -- `cli_reference.md` §3 (Authentication), §11 (Environment Variables), and §12 (Troubleshooting) drop the API-key method, the `ANTHROPIC_API_KEY` env var, and the Anthropic Console key-management link.

#### Models

- **Current generation added to the CLI reference** -- `claude-opus-4-8` (Opus 4.8), `claude-sonnet-5` (Sonnet 5), and `claude-fable-5` (Fable 5) are listed alongside the prior-generation `claude-opus-4-6` / `claude-sonnet-4-6` / `claude-haiku-4-5-20251001` IDs, selectable via `--model` where the target CLI supports them.

#### Unchanged

- The skill default pin (`claude-sonnet-4-6`), routing, permission modes, and every dispatch rule are untouched.
- The generic NEVER rule against sending secrets (API keys, passwords) in prompts stays -- that is secret hygiene, not an auth method.
- Security-audit playbook fixtures that seed a hardcoded `sk-ant-...` secret into a deliberately vulnerable test file are unchanged -- they are test inputs, not auth config.

## Upgrade

If a machine relied on `ANTHROPIC_API_KEY` for Claude Code, run `claude auth login` once (or `claude setup-token` for CI/CD) to establish the OAuth session. To use a current-generation model, name its ID explicitly, e.g. `--model claude-opus-4-8`. Dispatch flags and the skill default are otherwise unchanged.
