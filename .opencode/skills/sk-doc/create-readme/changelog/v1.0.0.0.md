# v1.0.0.0 — README and install-guide authoring packet established

First release of `create-readme` as one of the ten workflow packets in the `sk-doc` parent hub's workflow-only architecture. It writes current-state folder `README.md` files (general project, skill, feature and component READMEs, plus source-code folder orientation) and five-phase install guides (sections 0 through 10, prerequisites through verification) from local evidence only, routing by artifact type and folder purpose rather than a single fixed template.

---

## What's Included

- `SKILL.md` — the primary workflow contract: route-by-artifact-type decision trees for README vs install guide, the README authoring workflow, the general and code-folder README output shapes, the folded five-phase install-guide workflow with validation checkpoints and STOP blocks, the validation and audit workflow, and the writing rules (current-state only, no fabricated commands or metrics, Human Voice Rules).
- `references/readme_creation.md` — extended README quality criteria, progressive-disclosure guidance, analogy patterns and table patterns beyond what the inline SKILL.md covers.
- `references/install_guide_creation.md` — extended install-guide examples, validation-checkpoint patterns and troubleshooting standards.
- `assets/readme/readme_template.md` — fillable scaffold for general project, skill, feature and component READMEs.
- `assets/readme/readme_code_template.md` — fillable scaffold for source-code folder READMEs, including diagram examples.
- `assets/readme/install_guide_template.md` — fillable scaffold for the 11-section install guide.
- `scripts/audit_readmes.py` — repository README audit script that checks template alignment, broken local references and key artifact coverage.

---

## Notes

This is a nested workflow packet: it owns README and install-guide authoring only. It does not carry `graph-metadata.json`; advisor identity, skill graph metadata and cross-packet routing live at the `sk-doc` hub root.
