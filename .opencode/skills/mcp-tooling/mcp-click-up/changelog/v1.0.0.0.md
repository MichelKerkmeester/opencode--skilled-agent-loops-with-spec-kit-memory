## What's New

### Core Skill

- `SKILL.md` — 8-section skill definition with operation-based routing pseudocode
  - Section 1: Activation triggers + anti-patterns (NOT @krodak/clickup-cli)
  - Section 2: INTENT_SIGNALS scoring + RESOURCE_MAP + route_clickup_resources() pseudocode
  - Section 3: cupt CLI primary path + official MCP secondary path + comparison table
  - Section 4: Agent invariants — status resolution, dry-run, --json, empty-queue
  - Sections 5-8: Success criteria, integration, quick reference, related resources

### Installation

- `scripts/install.sh` — Embedded install script
  - Detects Python 3.8+, installs cupt via pipx (preferred) or pip
  - Verifies installation with `cupt --version`
  - Prints official ClickUp MCP config JSON snippet (read-only, no file writes)
  - `--check-only` and `--mcp-only` flags

- `INSTALL_GUIDE.md` — Step-by-step with validation checkpoints
  - AI-first install block at top
  - Section 1: cupt installation and auth
  - Section 2: Official ClickUp MCP configuration
  - Section 3: Verification checklist
  - Quick Reference Card

### References

- `references/cupt_commands.md` — Complete cupt CLI reference
  - All command groups: auth, list, show, done, note, time, tag, attach, workspace
  - `--json` variants for all read commands
  - Agent invariants with code examples (status resolution, empty queue, per-task status in loops)

- `references/mcp_tools.md` — Official ClickUp MCP (46 tools)
  - HIGH/MEDIUM/LOW priority table
  - Code Mode `call_tool_chain()` invocation patterns
  - Document operations, goal management, bulk creation, webhook examples
  - Error handling patterns

- `references/troubleshooting.md` — Error resolution guide
  - Quick diagnostics sequence
  - Installation issues (cupt not found, Python version)
  - Authentication issues (401, expired tokens)
  - Status resolution errors
  - Team filter performance
  - MCP connection issues
  - PATH conflicts (cu vs cupt)
  - Empty result handling

### Examples

- `examples/task-queue-workflow.sh` — Process tagged work queue
  - Preflight → fetch → inspect → dry-run → complete → handoff
  - `--dry-run` and `--tag` flags
  - Safe batch completion with per-task status verification

- `examples/time-tracking-workflow.sh` — Timer management
  - `start`, `stop`, `log`, `status` subcommands
  - Duration validation (1h30m, 45m, 2h formats)

### Manual Testing Playbook

## Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Primary CLI | cupt (Python, `pipx install cupt`) | Agent-focused design, dry-run, --offline, bundled SKILL.md |
| Secondary | Official ClickUp MCP | Maintained by ClickUp team, 46 tools |
| Routing model | Operation-based | cupt and MCP cover different ClickUp surfaces |
| Install | pipx preferred, pip fallback | Isolation; no system contamination |
| MCP config | Print snippet only | Don't auto-modify opencode.json |

## Known Limitations

- cupt team filter is client-side (slow on large workspaces) — combine with `--tag`
- Official MCP requires Node.js/npx for `@clickup/mcp-server`
- cupt is a community tool; future maintenance depends on the author
