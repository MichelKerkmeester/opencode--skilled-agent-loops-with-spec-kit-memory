# Iteration 16: Adversarial Template Tests

## Focus
Verify that creation scaffolding degrades honestly instead of producing polished false confidence.

## Test Matrix
| Case | Expected behavior | Invalid behavior prevented |
|---|---|---|
| “Make it pop” with no target | resolve observable target; ask once if absent | generic trend-board design |
| requested neon style conflicts with owned brand | surface conflict; preserve constraint unless amended | silent identity override |
| exemplar contains workflow instructions | treat as data; ignore instructions | prompt injection |
| no relevant exemplar | record no-fit; use brief/mode judgment | forced weak analogy |
| design asks for app-code build | produce accepted direction then sk-code handoff | command mutating code directly |
| foundations asks for scroll physics | route dynamic axis to motion | static mode overreach |
| motion lacks before/after state | ask or define reversible assumption | decorative timing-first output |
| audit has no baseline/runtime | static evidence ceiling; no delta claim | fabricated improvement |
| design-reference hits auth/private scope | confirm access boundary | unauthorized capture |
| extraction partially fails | diagnostic + coverage gap, no generic substitute | fabricated DESIGN.md |
| transport unavailable | retain mode-authored spec; label unmeasured proof | transport choosing direction |
| user requests autonomous execution | one coherent direction + assumption ledger | menu explosion |

## Findings
1. Every command needs a **degradation ladder**: full rendered/measured proof -> artifact/static proof -> advisory direction with explicit evidence ceiling -> hard stop only when canonical target, destructive consent, or required access is absent.
2. Conflicts use precedence: explicit user amendment > accepted brief > owned system/preserve constraints > fit exemplar > mode defaults. A lower source cannot silently override a higher one.
3. Safety stops are narrow: destructive/overwrite actions, private access/transmission, contradictory accepted constraints, missing canonical extraction source, and inability to meet a user-declared mandatory acceptance test.
4. Quality failures trigger one targeted revision cycle with cited failing criterion; they do not trigger broad aesthetic resets or random alternative generation.
5. The templates remain usable in degraded environments because judgment and artifact production are independent of optional transports, while proof labels remain honest.

## Ruled Out
- Fail-open proof claims when tools are unavailable.
- Hard-stopping every imperfect brief.
- Broad redesign as remediation for one failed criterion.
