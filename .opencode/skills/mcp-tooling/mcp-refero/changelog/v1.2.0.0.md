## [**1.2.0.0**] - 2026-07-16

Kebab-case naming migration: every file and directory name in this packet now uses hyphens only, per the repo-wide hyphen-naming convention (sk-doc/017). 32 paths were renamed via `git mv` (4 scenario/category directories, 28 files), and every in-packet reference — SKILL.md smart-router RESOURCE_MAP/DEFAULT_RESOURCE/ON_DEMAND lists, playbook root index, feature-catalog tables, scenario `expected_resources` mirrors, scripts, fixtures — plus the `.opencode/specs/mcp-tooling/009-mcp-refero` packet record was updated to the new paths.

#### Kebab-case renames

- Need: The packet shipped with snake_case reference/asset/scenario names (`references/tool_surface.md`, `manual_testing_playbook/read_only/`, ...), which the hyphen-naming convention now bans for filesystem names.
- Change: All snake_case directories (`discovery_setup/`, `intra_routing_recall/`, `read_only/`, `safety_gate/`) and files (e.g. `tool_surface.md` → `tool-surface.md`, `mcp_wiring.md` → `mcp-wiring.md`, `utcp_refero_manual.md` → `utcp-refero-manual.md`, `discovery_fixture_2026-07-16.json` → `discovery-fixture-2026-07-16.json`, feature-catalog tool files like `search_styles.md` → `search-styles.md`) were renamed with `git mv`; scenario frontmatter `category:` labels and `expected_resources` paths follow the new names. Note the FILE names changed while the MCP tool identifiers they document (`refero.refero_refero_search_styles`, dotted registry names) are code identifiers and remain snake_case in prose by design.
- Why it matters: One naming dialect across the packet; router path lists, benchmark gold resources, and on-disk names can no longer drift apart on case style.

&nbsp;

#### Tool-mandated carve-out (NOT renamed)

- Need: An evidence check before renaming the two top-level scaffold directories.
- Change: `feature_catalog/` and `manual_testing_playbook/` (the top-level directories) intentionally stay snake_case — they are hardcoded in `sk-doc/create-skill/scripts/package_skill.py` (VERSIONED_DOC_SUBTREES + scaffold checks), `.opencode/commands/doctor/scripts/parent-skill-check.cjs` (6 refs), and the skill-benchmark loaders (`load-playbook-scenarios.cjs`, `playbook-generator.cjs`, `run-skill-benchmark.cjs`). The sk-git pilot renamed them and its package gate fails. Additionally, the two root index FILES `manual_testing_playbook/manual_testing_playbook.md` and `feature_catalog/feature_catalog.md` also keep their names: `load-playbook-scenarios.cjs:375` loads `manual_testing_playbook.md` by exact filename as the playbook root index, and both filenames sit in tool skip-lists (`load-playbook-scenarios.cjs:307`, `validate-playbook-topology.cjs:136`).
- Why it matters: Renaming any of these four names silently changes benchmark-loader shape detection and package-gate scaffold checks; the carve-out keeps the tooling contract intact until the tools themselves migrate.
