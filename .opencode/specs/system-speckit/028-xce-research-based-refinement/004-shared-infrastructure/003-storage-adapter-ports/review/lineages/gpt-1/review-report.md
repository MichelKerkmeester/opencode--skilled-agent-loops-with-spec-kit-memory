## Remediation Workstreams

| Workstream | Findings | Action |
|------------|----------|--------|
| VectorStore identity contract | F001, F003 | Either persist caller IDs in the port adapter or rename/narrow the port contract to generated memory IDs; update fake and tests accordingly. |
| VectorStore clear semantics | F002 | Split vector-payload clearing from metadata deletion or update the interface docs and fake to match destructive clear behavior. |
| Fake parity | F004 | Mirror real retry attempt derivation and abort handling in `FakeContentionPolicy`, then add a contract case using `retryDelaysMs` without `attempts` and a `shouldAbort` case. |

## Spec Seed

- Add an acceptance criterion clarifying whether `VectorStore.id` is caller-owned or generated by the backing memory index.
- Add an acceptance criterion for destructive scope of `VectorStore.clear()`.
- Add fake parity requirements for retry delays and abort semantics.

## Plan Seed

1. Update `VectorStore` tests to assert `get(record.id)` and `delete(record.id)` after `upsert(record)` for both fake and better-sqlite subjects.
2. Choose and implement the identity behavior that satisfies the port contract.
3. Add a test that `clear()` does not delete non-vector metadata, or explicitly document and assert the destructive behavior.
4. Align `FakeContentionPolicy` with real retry and abort semantics.
