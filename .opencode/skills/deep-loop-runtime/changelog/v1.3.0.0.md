# deep-loop-runtime v1.3.0.0

> Structural cleanup follow-on. Renames storage/ to database/, untracks + gitignores the runtime sqlite, adds the 4 missing test-subfolder READMEs, and confirms sk-code opencode alignment.

---

## Why This Release Matters

Two structural issues remained after v1.2.0:

1. The runtime SQLite directory was named `storage/` while the sibling `system-spec-kit/mcp_server/` uses `database/`. The inconsistent name made the convention harder to follow and the sqlite inside was committed to git, so external repo users shipped a stale binary database instead of regenerating a fresh one on first run.

2. Of the 5 test subfolders, only `helpers/` carried a README. `unit/`, `integration/`, `lifecycle/` and `council/` had none, leaving the test layout under-documented.

v1.3.0.0 aligns the directory name with the sibling convention, makes the sqlite a regenerated artifact (gitignored), and brings every test subfolder up to the documented sub-readme standard.

---

## What Changed

### storage/ to database/

`git mv storage database` preserves history. The only load-bearing code change is the `COVERAGE_GRAPH_DATABASE_DIR` constant in `coverage-graph-db.ts` (the four `.cjs` scripts and the writer-lock path all derive from it, so one constant covers the runtime). The lifecycle test's hardcoded lock path was updated to match. Current-state docs (SKILL.md, README, references, graph-metadata) now point at `database/`. Historical changelogs (v1.0.0.0, v1.1.0.0, v1.2.0.0) keep their `storage/` references as accurate point-in-time records.

### sqlite gitignored

`git rm --cached database/deep-loop-graph.sqlite` stops tracking the runtime DB. New `.gitignore` rules ignore `database/*.sqlite` (+ `-shm`, `-wal`, `.bak`). `database/README.md` stays tracked. External users get a clean checkout and the DB lazy-initializes on first script run via `initDb(COVERAGE_GRAPH_DATABASE_DIR)`.

### test subfolder READMEs

Added `unit/README.md`, `integration/README.md`, `lifecycle/README.md`, `council/README.md`, each enumerating its test files + the module/surface under test, matching the existing `helpers/README.md` style. `tests/README.md` §2 links each.

### sk-code opencode alignment

`verify_alignment_drift.py --root .opencode/skills/deep-loop-runtime` reports PASS (54 files scanned, 0 findings). Test files (`.vitest.ts`) are exempt from the `MODULE:` header requirement per the opencode TS conventions; all carry LF endings and conformant import order.

## Upgrade

No consumer action required. The script argv contract, stdout JSON shape, exit codes and TS import paths are unchanged. The DB directory moved from `storage/` to `database/`; consumers that hardcoded the old path (none found in-repo) should switch to `COVERAGE_GRAPH_DATABASE_DIR`. On a fresh checkout the sqlite is absent and regenerates on first script run.
