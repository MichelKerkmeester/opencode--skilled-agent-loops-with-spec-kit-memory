## Questions Answered
- **KQ4 — GPT slowness mechanism:** The dominant confirmed mechanism is not the 66-line rendered prompt pack and not full YAML carriage to the leaf. It is role-resolution overhead at the generic/custom-agent and CLI positional-message boundary: native deep dispatch is named but still backed by generic `subagent_type`, while CLI OpenCode has no agent flag and relies on prompt text. State-read and validation overhead are bounded secondary costs. [SOURCE: .opencode/agents/orchestrate.md:162] [SOURCE: .opencode/commands/deep/assets/deep_research_auto.yaml:853] [SOURCE: .opencode/commands/deep/assets/deep_research_auto.yaml:916] [INFERENCE: based on findings 1-4]
- **KQ5 — Pre-route vs negotiate:** Add a compact `Resolved route` contract to the deep-research prompt template, prepend it in CLI OpenCode dispatch, and add a `Deep Route` field to orchestrator task packages for deep routes. These are minimal prompt-structure edits that reduce mode inference without removing existing Claude-friendly context.
