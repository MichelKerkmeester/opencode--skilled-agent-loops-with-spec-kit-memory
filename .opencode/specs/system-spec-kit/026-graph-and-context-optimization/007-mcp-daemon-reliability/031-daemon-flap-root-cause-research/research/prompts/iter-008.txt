A layered fix for the daemon-flap root cause (daemon liveness is session-coupled; advisor/spec-memory die with stale sockets; only a fresh non-warm invocation resurrects them):
- **Layer 1 (immediate, pure-Node, cross-platform):** C2 cold-start self-heal (warm-only probe fires a detached `kickBackgroundColdStart` then still returns 75; also kick on SessionStart) + B's uniform release/adopt for advisor+code-index (replace kill-on-shutdown with release/adopt; backend-only + launcher-side session proxy) + a proactive pre-bind stale-socket sweep + HF-sidecar refcounting + active-client fail-open + long/no idle for main daemons.
