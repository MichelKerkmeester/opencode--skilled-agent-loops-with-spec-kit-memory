Initial release of `create-changelog`, one of the ten workflow packets in the `sk-doc` parent hub's workflow-only architecture. It turns a spec folder, component hint, or recent git history into either a global component changelog under `.opencode/changelog/{component}/v{VERSION}.md` with four-part semantic versioning, or a packet-local nested changelog written through the spec-kit nested generator, formatted from the shared compact or expanded changelog template.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/125-sk-doc-parent` (Level 3)

---

## What Changed

#### New Features

- **create-changelog packet** -- The `sk-doc` family had no dedicated changelog-authoring contract. This release adds the inline seven-step workflow (analyze context, resolve output target, determine version, generate content, validate, write, report), topology detection between global and packet-local nested output, dynamic component discovery under `.opencode/changelog/`, auto/major/minor/patch/build version-bump detection, and compact vs expanded format selection driven by the shared template.

## What Included

- The inline `SKILL.md` workflow -- the primary contract: activation triggers, required inputs, topology and target resolution, the changelog format contract, the seven-step creation workflow, validation checks, and always/never/escalate rules.
- `references/changelog_creation.md` -- the supplementary creation reference with worked examples and edge cases for both global and packet-local output.
- `README.md` -- packet overview and quick start.
- No `assets/` or `scripts/` ship in this packet. It reuses the shared template at `../shared/assets/changelog-template.md` and the shared validators under `../shared/scripts/`.
