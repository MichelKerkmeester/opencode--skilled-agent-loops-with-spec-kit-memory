This release records an informal, single-sample visual-design comparison between the three active small-model profiles, so anyone picking a model for design or illustration work has one early data point to lean on. The notes are explicitly informal (n=1) and are not benchmark evidence. No dispatch behavior, no design judgment in `sk-design-interface`, and no executor mechanics in `cli-opencode` changed.

---

## What Changed

#### Documentation

- **Design bake-off observation recorded** -- there was no captured signal on how the small models differ on visual-design taste, so a quick informal pass was run. Three models (DeepSeek-v4-pro, MiMo-v2.5-pro, Kimi-k2.7-code) were each dispatched once through `cli-opencode` applying `sk-design-interface` against one fixture (the Budgetteren bento page), one sample per model. The takeaways: Kimi-k2.7-code showed the strongest brand restraint (blue-led, orange held back for a single alert, gold kept rare, cohesive overall). MiMo-v2.5-pro over-accented (orange and gold spread across most cards, weak color restraint despite clean shapes). DeepSeek-v4-pro produced the richest individual drawings but ran gold-heavy while starved of brand blue. This is one anecdote, not a verdict, and it gives a starting hypothesis to test properly later.
- **Caveats kept front and center** -- a one-line, anecdotal note can quietly harden into "fact" if it is not fenced, so each model reference now carries a new "Design & Illustration Tasks" section flagged as an informal observation with sample size one. Every note cites `model_profiles.json` as its data anchor and points at `/deep:model-benchmark` (at least three fixtures times at least two samples) as the path to canonical, repeatable evidence. The matching `model_profiles.json` entries are tagged as illustration caveats rather than measured strengths or weaknesses.
