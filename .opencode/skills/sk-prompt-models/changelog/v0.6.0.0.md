# sk-prompt-small-model v0.6.0.0 — Owns model registry + all model benchmarks (hub consolidation)

**Released**: 2026-06-03
**Status**: Ownership expansion — model registry (`assets/model-profiles.json`) moved in from `sk-prompt`; 6 benchmark run-data sets relocated here under `benchmarks/`.

---

## Summary

`sk-prompt-small-model` is now the **single source of truth** for both model-profile registry data and all model-benchmark evidence. Two previously distributed asset classes are consolidated here:

1. **Model registry:** `assets/model-profiles.json` moved in from `sk-prompt`. All 8 active profiles resolve at the new hub path. The deleted `sk-prompt/references/model-profiles.md` schema and adoption content are now covered by this skill's pattern-index §4.
2. **Benchmark data:** A new `benchmarks/` directory holds the relocated run-data, harness files, and synthesis for 6 sub-phases previously scattered across spec sub-phases:

| Benchmark folder | Origin |
|---|---|
| `benchmarks/120-003-minimax-m3-framework/` | Spec 120/003 — MiniMax-M3 framework bakeoff |
| `benchmarks/126-004-mimo-v2-framework/` | Spec 126/004 — MiMo-V2.5-Pro framework benchmark |
| `benchmarks/113-003-swe16-framework/` | Spec 113/003 — SWE-1.6 framework eval |
| `benchmarks/113-005-swe16-rerun/` | Spec 113/005 — SWE-1.6 rerun |
| `benchmarks/127-004-capability-discrimination/` | Spec 127/004 — M3 vs MiMo capability discrimination |
| `benchmarks/127-006-strict-validation/` | Spec 127/006 — M3 vs MiMo strict-validation fixtures |

Each origin sub-phase was gutted to its spec-kit document shell and a `BENCHMARK-RELOCATED.md` pointer. The benchmark harness and run-data themselves live here.
