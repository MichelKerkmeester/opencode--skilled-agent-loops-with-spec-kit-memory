# v1.0.0.0 — code-review mode established

First release of the `code-review` workflow mode on the `sk-code` hub. The standalone `sk-code-review` skill was folded into the sk-code parent hub as its findings-first review mode, and every `sk-code-review` name reference was repointed to the sk-code hub. This entry establishes the mode on the hub; the v1.1.0.0–v1.5.0.0 entries below it were inherited from the standalone `sk-code-review` skill and continue its lineage.

## New

- `SKILL.md` — the mode contract: a stack-agnostic findings-first review baseline paired with `sk-code` surface-standards evidence for the detected surface, severity-ranked findings with `file:line` evidence, and mandatory security/correctness minimums (unknown surfaces review against the baseline and disclose uncertainty).
- `README.md` — mode at-a-glance and layout.
- `references/` — the review doctrine: `review_core.md`, `review_ux_single_pass.md`, `pr_state_dedup.md`, and `quick_reference.md`.
- `assets/` — reusable review checklists: security, code quality, SOLID, test quality, fix completeness, and removal plan.
- `manual-testing-playbook/` — routing scenarios covering baseline review flow, scope/precedence, severity-and-evidence discipline, security/correctness minimums, structural-impact preflight, re-review with stale context, efficiency/restraint, cross-CLI orchestration, and intra-routing recall.
- `benchmark/` — the router (`router-mode-a`) and live (`live-mode-b`) skill-benchmark baselines.
- `scripts/check-rule-copies.js` (+ test) — guards against divergence between copied review rules.

## Notes

This mode produces findings-first output and does not edit the code under review. Author-side correction before review lives in `code-quality`; applying accepted fixes lives in the surface skills (`code-webflow` / `code-opencode`). The pre-existing v1.1.0.0–v1.5.0.0 changelog entries predate this establishment entry chronologically but were authored under the standalone `sk-code-review` skill; they are retained unchanged.
