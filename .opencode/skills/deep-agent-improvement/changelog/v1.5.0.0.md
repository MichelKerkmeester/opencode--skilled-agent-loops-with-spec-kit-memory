## Summary

Packet 079 renamed the skill folder `sk-improve-agent` → `deep-agent-improvement` but kept the agent name `@improve-agent` stable, citing the original 070-sk-deep-rename precedent that agents already used modern non-`sk-` names. That left an asymmetry: skill `deep-agent-improvement` operated by agent `@improve-agent`. This packet renames the agent to complete naming-family alignment.

After v1.5.0.0 the family is consistent:
- **Skill**: `deep-agent-improvement` (folder under `.opencode/skills/`)
- **Agent**: `@deep-agent-improvement` (4 runtime files: `.opencode/`, `.claude/`, `.gemini/`, `.codex/`)
- **Slash command**: `/improve:agent` (UNCHANGED — `agent` is a generic sub-name, parallel to `/improve:prompt`)

This is a pure semantic rename. No behavior change, no dispatcher contract change, no skill change.
