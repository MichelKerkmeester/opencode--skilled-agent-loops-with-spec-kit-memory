# v1.0.0.0 - First Stable Release

**Released:** 2026-06-15

First stable release of the `mcp-figma` skill. It teaches an agent to drive **Figma Desktop from the terminal** through the silships **figma-cli** (published to npm as `figma-ds-cli`), with an **optional** Figma MCP via this project's Code Mode. The CLI is the primary surface: inspect, export, render, manage tokens/variables/components, and extract or import a design system. The MCP is opt-in and used only to pull Figma design data into the agent for codegen. The spine of the skill is the read-only / mutating / **destructive** gating taxonomy and the `sk-design-interface` handoff that keeps design judgment where it belongs. Modeled on the sibling `mcp-chrome-devtools` (CLI-primary plus MCP-via-Code-Mode) and `mcp-open-design` (CLI plus optional MCP, daemon model, gated verbs).

This release promotes the skill to a stable 1.0.0.0 line. Live external verification is complete: the real `figma-ds-cli` (1.2.0 full repo build) was installed and exercised, and an embedded install scaffold now mirrors the other mcp-server skills.

## What's New in 1.0.0.0

### Embedded CLI install scaffold

- Added `mcp-servers/figma-cli/` as the canonical install entry point for the silships `figma-ds-cli`, mirroring the embedded-server layout used by `mcp-click-up`. `mcp-servers/figma-cli/setup.sh` delegates to the skill's canonical `scripts/install.sh`, which selects the full silships repo build over the minimal npm build and never installs the unrelated `figma-cli` npm package (unic/figma-cli, bin `figma`).
- Added `mcp-servers/figma-mcp/` as a pointer to the optional Framelink `figma` Code Mode MCP. Nothing is installed there: the manual is already registered in the repo `.utcp_config.json`, so the README documents the opt-in path rather than vendoring a server.

### Live verification

- Verified against `figma-ds-cli` 1.2.0, the full repo build that exposes safe connect, the local daemon, design-system extract, and the complete command surface. The minimal npm `figma-ds-cli` 1.0.0 build is still detected, and the installer upgrades from the silships repo when only the minimal build is present.

### Install and doctor parity

- The skill sits in the shared mcp-skill install-and-doctor standard alongside `mcp-open-design`, `mcp-chrome-devtools`, `mcp-click-up`, and `mcp-code-mode`: an `INSTALL_GUIDE.md`, a non-mutating `scripts/install.sh`, and a read-only `scripts/doctor.sh`.

### House voice and hardening

- The skill's documentation was conformed to the house voice (no em dashes, no prose semicolons) across SKILL.md, the references, the install guide, the feature catalog, and the manual testing playbook.

## Carried from the initial build

The capabilities below formed the skill's first build and remain the core of the 1.0.0.0 surface.

### Core Skill

- `SKILL.md`: lean house-template runtime instructions
  - Section 1: WHEN TO USE (CLI vs optional-MCP triggers, use cases, anti-patterns)
  - Section 2: SMART ROUTING (direction plus binary detection, phase flow, six-intent guarded router)
  - Section 3: HOW IT WORKS (binary verification, Figma Desktop plus daemon model, connect modes, command classes, optional Framelink MCP)
  - Section 4: RULES (ALWAYS / NEVER / ESCALATE IF), centered on the `figma-ds-cli` naming trap, safe-default connect, and gating every destructive verb
  - Sections 5-8: references, success criteria, integration points, related resources

### References

- `references/figma_cli_reference.md`: binary identity, connect/daemon model, grouped command examples
- `references/tool_surface.md`: read-only / mutating / destructive / arbitrary taxonomy plus the destructive set with rollbacks
- `references/mcp_wiring.md`: the optional Framelink `figma` Code Mode manual, prefixed `.env` token, discover-first flow, `call_tool_chain` example (official Figma Dev Mode MCP noted as a future option)
- `references/troubleshooting.md`: symptom to cause to fix table (including the `figma-cli` npm collision)

### Scripts

- `scripts/install.sh`: install plus verify only (npm `figma-ds-cli` or silships repo, never `npm i -g figma-cli`, never connects or patches)
- `scripts/doctor.sh`: read-only diagnostics (binary identity, Figma Desktop, daemon, ports, Code Mode `figma` manual)
- `scripts/connect-safe.sh`: guided safe (plugin) connect, refuses to patch
- `scripts/connect-yolo.sh`: gated yolo connect, requires `--i-understand-this-patches-figma`
- `scripts/daemon.sh`: daemon status/diagnose/start/stop/restart/reconnect wrapper
- `scripts/unpatch.sh`: roll back the yolo patch
- `scripts/print-utcp-snippets.sh`: prints the optional Code Mode wiring, edits nothing
- `scripts/_common.sh`: shared binary-resolution plus helpers

### Docs

- `README.md`: narrative house-voice overview
- `INSTALL_GUIDE.md`: install/connect/daemon/MCP/safety guide
- `feature_catalog/feature_catalog.md`: capability inventory by area
- `manual_testing_playbook/manual_testing_playbook.md`: safe operator validation scenarios

### Design decisions

- **Canonical binary `figma-ds-cli`** (npm), with explicit warnings against the unrelated `figma-cli` npm package (unic/figma-cli, bin `figma`).
- **Safe connect is the default**, and the yolo `app.asar` patch is consent-gated with an `unpatch` rollback.
- **Framelink `figma` is the documented MCP path**, and the official Figma Dev Mode MCP is a future option.
- Minimal runtime `allowed-tools` (`Read, Bash, Grep, Glob, mcp__code_mode__call_tool_chain`). The figma-cli is installed through the scaffold and referenced, **not source-vendored** (MIT).
