deep-improvement gained a first-class model-benchmark mode and, on top of it, became a genuine two-lane skill end to end: Lane A (improve an agent file) and Lane B (benchmark a model or prompt framework) are now co-equal at the command, doc, agent, and on-disk layers. The capability was built from a designed mode-selector + pluggable seams, then hardened by two independent deep-review -> remediation cycles (gpt-5.5 and Opus 4.8) before Lane B shipped.

## What Changed

### Arc 1 — Build the model-benchmark mode (phases 001-007)

#### Model-benchmark mode (build) (phases 001-003)

- **Designed the mode selector + three pluggable seams** before writing code (phase 001: ADRs + build plan), then deep-researched the build with MiniMax M2.7 to converge a verified, build-ready delta (phase 002: seam contracts + build-delta, 5 evidence iterations).
- **`loop-host.cjs` gained a mode-switch entry point.** It now dispatches between `agent-improvement` (the existing path, untouched) and `--mode=model-benchmark`, so the same loop shape can drive either an agent definition or a model/prompt benchmark.
- **New `dispatch-model.cjs` generic dispatcher** runs a model against fixtures under prompt-framework variants — the Lane B equivalent of dispatching a Claude agent, but for arbitrary models across the 5 executors.
- **Ported the decoupled 120/003 scorer** into `scripts/model-benchmark/scorer/` (deterministic checks under `scorer/deterministic/`, the LLM grader harness under `scorer/grader/`, and `score-model-variant.cjs`), making benchmark scoring a first-class part of the skill rather than a packet-local one-off.
- **Mode-aware state records:** loop state now carries a `mode` field so a benchmark run reaches `benchmark-complete` and is distinguishable from agent-improvement runs in the journal and registry.
- **TST-1 byte-identity gate:** the agent-improvement branch is held byte-identical through every build and reorg, so the existing lane provably never regresses.

#### Opt-in 5-dim scorer (phase 005)

- **`run-benchmark.cjs` gained `--scorer pattern|5dim` (default `pattern`) and `--grader noop|mock|llm`.** The default `pattern` path stays byte-identical; the 5-dim scorer is lazy-loaded only on the `5dim` path, and the choice is recorded as `scoringMethod` in the report and state log. This closed the hardening-review deferral (F-P2-1/F-P2-2) that flagged the ported 5-dim rubric as built-but-unwired.
- **SKILL.md documented the model-benchmark mode for the first time** (WHEN-TO-USE case + scorer options), with the README structure table refreshed for the new scripts and the `scorer/` subtree. DQI moved 94 -> 97 (excellent band). Regression tests cover both scorer paths plus unknown-scorer fallback.

#### Hardening review + remediation (phases 007 -> 004)

- **An independent tri-model hardening review** (gpt-5.5 + MiniMax M2.7 + Opus 4.8 arbiter; CONDITIONAL verdict) audited the Arc-1 build for correctness, security, traceability, and maintainability. It was originally run as separate packet 122 and later folded into 121 as phase 007.
- **Remediated the 3 unique confirmed P1 defects** (phase 004), each with a regression test: the dispatcher dropped the requested working directory for 4 of 5 executors (cwd now propagates to `spawnSync` for all 5), the D3 path guard misclassified sibling-prefix paths (now a separator-bounded prefix guard), acceptance criteria ran un-gated shell commands (now a criteria-exec gate, default ON with a documented opt-out), and the D4 grader accepted unbounded model scores (now clamped to `[0,1]`).
- **Closed the actionable P2 advisories** in the same pass: Atomics-based backoff replacing the busy-wait, a grader-cache raw-input gate, dependency-injected `fs` in `dispute.cjs` (removing a global monkey-patch), a deterministic-scoring regression test, delta-units documentation, and a reconciled REQ-004 `--cwd` wording. The two non-defects (intended deferral, dedup artifact) were recorded as no-change with rationale.

#### Maintenance (phase 006)

- **Deep-loop init no longer creates empty `{mode}_archive` directories.** All 4 deep-loop restart branches now archive lazily and guarded, with regression tests, and pre-existing empty archive dirs were swept.

### Arc 2 — Two co-equal lanes (phases 008-018)

#### Command layer asks the lane (phase 008)

- **`/deep:start-agent-improvement-loop` now resolves a use-case lane.** When an agent path is supplied it auto-resolves to Lane A and runs exactly as before (CMD-1 behavioral identity), so existing invocations are untouched. A `--lane=model-benchmark` flag or a `--profile` argument routes to Lane B, and a lane question appears only when the lane is genuinely ambiguous.
- **New dedicated `/deep:start-model-benchmark-loop` command** plus its gemini mirror and two Lane B workflow YAMLs (`deep_start-model-benchmark-loop_{auto,confirm}.yaml`) that run `loop-host.cjs --mode=model-benchmark`, reduce, journal, and offer mode-aware promotion.
- **Advisor registration:** a `deep-model-benchmark` alias group plus bounded command-level disambiguation so benchmark phrasing routes to the new command rather than the Lane A one.
- **Latent fix:** `loop-host.cjs` never forwarded `--scorer` and `--grader` to `run-benchmark.cjs`, so selecting `5dim` or a `mock`/`llm` grader through any command path silently fell back to `pattern`/`noop` while the report disagreed with the journal. Both flags are now forwarded in the model-benchmark branch only, with a regression test. The agent-improvement branch is byte-identical so TST-1 stays green.

#### SKILL.md two co-equal lanes (phase 009)

- **The skill doc presents Lane A and Lane B as parallel sections** instead of modes 1 to 3 plus a trailing Mode 4. A WHEN TO USE two-lane table was added, and the smart router gained a `MODEL_BENCHMARK` intent at weight 5 with a matching RESOURCE_MAP entry. DQI stayed in the excellent band at 97.

#### references and assets lane-separated (phase 010)

- **`references/` and `assets/` are now organized by lane** (`agent-improvement/`, `model-benchmark/`, `shared/`) rather than by function. This supersedes the v1.8.0.0 function grouping (scoring, promotion-gates, integration, workflow), which was an intermediate step. Every literal was repointed: SKILL.md RESOURCE_MAP, DEFAULT_RESOURCE, RUNTIME_ASSETS, both lanes of command YAMLs, both command docs, README, graph-metadata, scripts, the feature catalog and playbook, and the benchmark profile's internal fixtureDir.

#### agent lane awareness (phase 011)

- **The `@deep-improvement` awareness note went from "Mode awareness" to "Lane awareness"** across all 4 runtime mirrors, byte-identical (mirror-drift 0). The note makes clear there is one agent and Lane B dispatches models through `dispatch-model.cjs`, never a second Claude agent.

#### catalog, playbook, and dashboard visibility (phase 012)

- **The feature catalog and manual testing playbook label each entry by lane.**
- **F-P2-5 closed:** `reduce-state.cjs` now surfaces the `mode` metadata it already recorded. The registry carries per-profile and global `modes`, and the dashboard prints a "Lane (mode) mix" line in the global summary and each profile section. This is display only, routing is untouched, and a missing mode counts as agent-improvement so legacy ledgers still total. A new regression test covers it.

#### scripts lane-separated (phase 013)

- **The 16 scripts moved into `scripts/{agent-improvement,model-benchmark,shared}/`** (with `scorer/` under model-benchmark), while `lib/` and `tests/` stay at the scripts root. `loop-host.planInvocation()` was kept byte-identical and a new `resolveScriptPath()` helper does lane resolution at spawn time, so TST-1 holds. `dispatch-model.cjs` path joins were re-depthed (its config load is positively asserted to resolve to a real file, closing a silent-fallback risk). All references were rewritten, including the vitest path literals (13 suites at that phase).

#### Code-folder READMEs (phase 016)

- **Every source script subfolder gained a local code-folder README** (sk-doc code template), and existing READMEs were audited, so each lane subdir documents its own contents.

#### Hardening reviews + remediations (phases 014 -> 015, 017 -> 018)

- **First two-lane deep review (phase 014):** a 10-iteration cli-codex gpt-5.5 review with Opus 4.8 adjudication audited the full two-lane program (008-013). CONDITIONAL verdict; active registry of 1 P0 + 16 P1 + 16 P2.
- **Remediated all of them (phase 015):** the 1 P0 + 16 P1 + 16 P2 findings were closed before Lane B shipped.
- **Independent second-opinion review (phase 017):** a second deep review with Opus 4.8 agents cross-checked the post-015 code to confirm the fixes held and hunt new issues. CONDITIONAL verdict with 4 active P1 + 13 P2.
- **Remediated the Opus findings (phase 018):** the 4 P1 + 13 P2 were closed with one disposition each — including a materializer fixture-id guard, a bundle-gate criteria-exec gate, removal of dead Mode-4 anchors, and an executable Lane B benchmark-mode promotion path matched to its documented truth, with regression tests.

## Upgrade

No migration required for callers of `/deep:start-agent-improvement-loop` with an agent path, which behave exactly as before. New work that benchmarks a model or prompt framework should use `/deep:start-model-benchmark-loop` or pass `--lane=model-benchmark`; opt into the 5-dim rubric with `--scorer=5dim` (default stays `pattern`). Internal scripts moved into lane subdirs, so any external caller hardcoding `scripts/<name>.cjs` should update to `scripts/<lane>/<name>.cjs`.
