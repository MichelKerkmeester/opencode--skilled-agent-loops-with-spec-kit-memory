# v3.7.0.0, Honest Verdicts, Clean Records

The center of this release is whether you can trust a search verdict. A suite of honesty checks that had been proving themselves quietly are now live for everyone. A result needs real keyword evidence behind its score before it earns a good verdict, the background hum of the embedder is subtracted before the verdict is read, and a thin result is downgraded rather than dressed up. The evidence-gap warning that sits beside that verdict was rebuilt so it fires only on a real gap and never contradicts the answer.

Around that, the records the framework keeps about each spec folder got type-safe and reproducible. The generated metadata that powers memory search and graph traversal stops drifting, stops dirtying the tree when you read it, and the whole tree was migrated onto the new contracts.

The through-line is that none of this was assumed. A graduation benchmark measured every candidate against the real corpus and applied the verdicts: twelve flags earned their way on, one was deleted, and two search options were measured and declined. The evidence-gap fix itself was caught mid-flight when a live check disagreed with a green benchmark.

None of this changes how you call the system. `memory_search`, `memory_save` and the `/deep:*` commands all behave as before. The work happened in the honesty of the verdict and the cleanliness of the records, not on the path you use every day.

## Search Intelligence

This release is mostly about one thing: a search verdict should be earned, and it should tell you the truth about its own confidence.

#### A Result Must Earn Its Verdict

A high similarity score is no longer enough to call a result good. The lexical-grounding floor requires a real keyword signal behind the top hit (a term match in the title, the trigger phrases or the body, or a BM25 keyword score already on the row) before it can reach a good verdict. So a fluent but absent term like "kubernetes," which once earned a confident citation from a single spurious high-cosine hit, now correctly reads weak. The floor shipped quietly earlier and graduated to on for everyone this cycle after the benchmark earned it.

&nbsp;

#### The Verdict Reflects True Signal, Not Background Hum

Every relevance score now sheds a measured corpus noise floor (an embedder-specific baseline, keyed to the nomic-embed-text-v1.5 model) before the system decides whether a match is real. A score that sits genuinely above the floor reads as a real match, and one that sinks into the background is handled honestly, so the verdict band reflects true signal rather than the embedder handing fluent but unrelated text a high background cosine. Graduated to on this cycle.

&nbsp;

#### Thin Evidence Is Capped, and the Warning Agrees

When the evidence behind a result is thin, search caps a good verdict down to weak so a confident answer cannot ride out on a gap, and the evidence-gap warning that sits beside the verdict now always agrees with it. This had been broken twice over. The cap was wired to a signal nothing ever set, so it never fired and a known gap could surface beside a confident verdict. And the warning itself was driven by a peakedness statistic (how far the top result stood above the pack) rather than by relevance, which got it backwards: a strong, tightly clustered set of good matches looked suspicious and got the warning, while a flat spread of weak off-topic matches slipped through clean. Both are fixed. The cap is connected, and the warning now keys off the same noise-floor-subtracted relevance the verdict uses, so a search for a real term like "graph" stays quiet while a genuine gap is flagged once, consistently. A benchmark of the old warning found it raised a false alarm on two of every three strong queries, the redesign brought that to zero.

&nbsp;

#### One Honest Score, Everywhere

A semantic result reports the same number the system ranks on. The score a caller sees now resolves through the calibrated absolute relevance (a 0-to-1 cosine scale) rather than the compressed fusion magnitude (RRF, which clusters low near 0.03), fixing a residual path that was still reporting the wrong figure for semantic rows. Separately, result lines from the relationship and degree channels that had rendered a blank dash now show their relevance number like any other row.

&nbsp;

#### A Guard Against Confident Hallucination

A build gate now refuses to pass when search returns good verdicts on terms it should know nothing about. Six bare off-corpus queries (kubernetes, oauth, kafka, terraform, graphql and webpack) run through the production verdict path, and the gate fails if too many false confirms slip through, with the ceiling set at zero. This is the regression net under the grounding and noise-floor work above.

&nbsp;

#### Queries Route by Shape

Every query is classified once by its retrieval shape before the pipeline runs. A simple single-hop lookup skips the graph-relation expansion so it stays light and fast, while a multi-hop question keeps its graph-aware ranking, so the two diverge at the right seam without changing how you call search. The five-class classifier is on for everyone, the per-class weight profiles behind it stay default-off.

&nbsp;

#### Search Keeps Answering When the Embedder Is Down

A missing or empty embedding no longer collapses a search into nothing. When the embedding service is unavailable the pipeline falls back to keyword matching and carries an explicit flag that the dense channel was skipped, so an outage returns the best lexical answer rather than an empty result. The healthy path is unchanged.

&nbsp;

#### Citation Holds Across Models

A 144-cell benchmark drove `/memory:search` with four models (MiMo, Kimi, DeepSeek and gpt-5.5 fast) across twelve deliberately vague queries. The verdict held up where it should. On clearly-aligned and clearly-empty queries every model agreed, and the citation rule was obeyed wherever a model emitted a full verdict. The divergence is on the borderline queries, two of the twelve split the verdict because each model refined the vague search differently and landed on different results, not because the verdict logic wavered. The benchmark also isolated the off-corpus false-confidence that the grounding and noise-floor graduations then closed. DeepSeek came out the most efficient and stable driver, and Kimi over-explores expensively on the vaguest queries.

&nbsp;

#### A Cleaner Result Display

The result dashboard adds up. The header count now matches the rows actually shown rather than a pre-trim total, a long folder path renders as its leaf title instead of being cut off mid-string, and a status field that had reported an unrelated mechanism now reports the thing a reader expects, so a dense result set reads cleanly and the status you read is the status you get.

## Data Quality

The metadata the framework generates about each spec folder (the `description.json` and `graph-metadata.json` files that feed memory search and graph traversal) was hand-driftable and noisy. This release makes it type-safe, idempotent and reproducible, then migrates the whole tree onto the new rules.

#### Type-Safe Records

A folder's generated status is now a closed set of values (planned, draft, complete and the rest) instead of free-running prose, and a first-class validator checks both generated files before a bad value can propagate. Legacy hand-edited statuses are re-derived into safe values and surfaced for review rather than left to leak through.

&nbsp;

#### Scoped Operations, No Side Effects

Refreshing one packet touches only that packet. The backfill tool now takes a folder argument and defaults to that single scope, so a one-folder intent no longer quietly dirties the whole tree, and a repo-wide run takes an explicit `--all`. Archived and future-staged folders are kept out of the descriptions cache by one shared policy, with archive memory behavior intact.

&nbsp;

#### Reads That Stay Clean

Reading spec metadata stopped rewriting it. The generator skips a write when the real content is unchanged (ignoring volatile timestamp fields), a targeted cache upsert updates one folder without rebuilding its neighbors, and access and freshness telemetry moved out of the generated JSON into an index-layer store so a read or a resume no longer dirties an unchanged file.

&nbsp;

#### Consistent Lineage and Synced Summaries

A folder's parent and children reconcile across partial rescans, so a valid parent relationship survives when one subscan misses it and child lists grow safely unless an operator runs an explicit prune. Separately, the description and causal-summary fields now derive from one shared synopsis rule so they stop drifting apart, and a drift checker re-derives them on read against a stored source-doc hash and reports staleness without writing anything. The generator also persists a source fingerprint that proves a field is fresh without re-reading the doc.

&nbsp;

#### A Clean Migration and a Flag Reckoning

The whole tree was regenerated onto the new contracts, 2,049 spec folders including archives, with a byte-stable second run proving the output settles deterministically and zero validation violations afterward. A graduation benchmark then measured every default-off flag against the migrated tree and applied the verdicts: twelve flags earned their way to default-on or enforcing (the metadata integrity gates and the search-honesty suite above), one purely informational flag was deleted for earning no measured win, and the two that needed a freshness field the first migration had not written were graduated after a re-run wrote it.

## Code Graph

The structural index that powers code search and impact queries got the same determinism and resilience treatment as memory search. Repeated queries now return stable answers, every scan stamps a version you can read, transient parser failures stop becoming permanent skips, and the doc lane learned to see structure inside your prose and config.

#### Stable Answers, Every Time

Equal-trust context edges (the relationships the graph walks when it answers a query) now sort by stable content-derived keys (the related content hash, the symbol identity, the file path, the edge type and the endpoints) instead of by whatever order the database happened to return. So two identical queries against the same data come back in the same order, even across separate scan rebuilds, and a diff of two runs no longer shows phantom reordering.

&nbsp;

#### A Generation Watermark You Can Read

Every promoted scan now advances a monotonic generation counter (a number that only ever goes up, stored in the graph metadata and surfaced on freshness envelopes as `metadata.freshness.generation`). A failed promotion or a no-op scan leaves it untouched, so the number you read is an honest stamp of how current the index is. Reads still serve from whatever generation is present (the hard gate that would reject a stale-generation read stays deferred for a later cycle).

&nbsp;

#### Transient Parser Failures Get a Second Chance

A parser hiccup no longer condemns a file to permanent invisibility. Transient parse failures now retry up to a configurable ceiling (`SPECKIT_PARSER_SKIP_LIST_MAX_RETRIES`, default 5) before they are treated as fatal, and a successful reparse clears the skip-list entry entirely. Genuinely fatal entries (the ones that need a human to look) keep skipping immediately as before, so the retry budget is spent only where it can actually help.

&nbsp;

#### The Doc Lane Sees Structure

The documentation lane now indexes the structure inside your files as real graph symbols. Markdown headings become `heading` nodes and config keys become `key` nodes (for JSON, JSONC, YAML and TOML), all generated from local parsing so nothing leaves your machine. Config files exercise the lane by default and markdown extraction is opt-in through globs, so you can point it at the docs that matter without indexing every stray note.

## Skill Advisor

The advisor that routes a request to the right skill learned to tell a lane that genuinely found nothing apart from a lane that could not run at all. It also learned to distrust its own cached judgment when the embedding model underneath it has changed.

#### A Down Lane Is Not an Empty Lane

When a scoring lane is runtime-degraded (for example the graph is unavailable) the advisor no longer counts that against a skill the way it counts a lane that ran and honestly found no match. Degraded lanes drop out of the confidence denominator so a skill is not punished for an outage it had nothing to do with, while a lane that ran and matched nothing still counts so non-matching skills are not over-credited. The advisor output and its abstention explanations now name the degraded state plainly instead of hiding it inside a low score.

&nbsp;

#### Stale Embeddings Fail Loud, Not Silent

The advisor records which vector model wrote its semantic projection at write time and compares that against the active embedder on every read. If the two disagree the semantic shadow lane is marked degraded (and then follows the same lane-health rule above) rather than quietly trusting a projection built by a different model. Legacy projections that carry no model signature now fail closed instead of being assumed safe, so a model swap can no longer leave the advisor reading from a stale map without anyone noticing.

## Deep Loop

The deep-research and deep-review loops (the multi-iteration `/deep:*` workflows that fan work out across parallel model workers) got a wave of determinism, recovery and observability fixes. A fresh run now reduces cleanly, a fan-out merge is reproducible regardless of which worker finishes first, an interrupted run leaves something behind, and a STOP decision has to be corroborated before it is trusted.

#### Fresh Runs Reduce Cleanly

A brand-new deep-research run no longer crashes on its first pass. The strategy template now ships with the reducer anchors (the machine-owned points where the run writes its carried-forward state) already in place, so the first reduce has somewhere to land and the run can start threading state forward immediately instead of failing before it begins.

&nbsp;

#### Fan-Out Merges Are Reproducible

A fan-out (work split across parallel model workers) now produces the same merged result no matter which worker finishes first. The research and review registries sort lineage labels and worker-identity metadata into a stable order before merging, so the combined output is decided by content rather than by a race, and two runs over the same inputs agree.

&nbsp;

#### Live Gauges and Stopped Summaries

A running fan-out now reports live pool gauges (pending, failed and lag counters) read from real run state rather than guessed, so you can see where a slow run actually is. And a run that gets interrupted now leaves a partial summary behind, so the work that did finish is captured instead of lost when the run stops early.

&nbsp;

#### Recovery That Stays Within Budget

Fan-out failures are now sorted into fixed classes (timeout, exit and salvage miss, which means a recoverable output that went missing). Only genuinely transient failures retry, and the retry counts are read from durable run logs, so a crash mid-run cannot quietly reset the retry budget and send a worker into an endless loop.

&nbsp;

#### Recovery Refuses to Fake a Clean Slate

Recovery mode no longer pretends a missing or corrupt state file is just a fresh new run. It refuses to proceed on broken state, and any orphaned lineages (workers that started but never reported back) are marked explicitly in the ledger, so partial state is visible as partial state instead of being papered over as success.

&nbsp;

#### A STOP Has to Be Corroborated

A deep-research STOP decision now needs a second source of truth when a run claims it has run out of new ground. Convergence compares the run's self-reported novelty against graph-observed novelty (the actual changes to stored questions, findings and evidence) and blocks a STOP that the graph cannot back up, so a loop cannot talk itself into stopping early on a claim nothing else supports.

&nbsp;

#### Divergent Findings Survive the Merge

When two workers return findings that share an ID but disagree on content, the merge now keeps both side by side and marks them as conflicting variants instead of letting whichever arrived first silently erase the other. A real disagreement between workers stays visible for a human to resolve rather than vanishing into a merge.

&nbsp;

#### Open Questions Thread Forward

Deep-research now carries its unresolved questions into the next iteration without spending an extra model call to do it. The reducer builds a continuity thread (the open questions carried forward from the last pass) and derives the next focus from that thread or the latest finding, so a loop keeps its train of thought across iterations for free.

## Also in This Release

#### Determinism, Tested and Declined

A reproducible-ranking option (it dropped the recency boost so a fixed query always returned the same order) was benchmarked against the real corpus. It was perfectly reproducible but reshuffled the top results on roughly 40% of real-match queries with no evidence the new order was better, so recency stays and the option was removed. The always-on result-ordering tie-break that shipped alongside it was kept, it is a pure determinism win that needs no flag.

&nbsp;

#### Two More Honesty Tiers, Now Live

A borderline citation tier (`cite_with_caveat`, a third option between cite and do-not-cite that marks a result as weak rather than forcing a binary choice) and a render-fidelity guard (a pre-rendered verdict fragment that a post-render check re-emits if a weak downstream model drops the verdict from its response) both graduated to on this cycle, so a caller is warned on a borderline result and the verdict survives the render layer intact.

&nbsp;

#### Confidence Calibration, Observing

Search confidence carries a dormant calibration fitter that harvests real evaluation labels and measures how far a claimed confidence drifts from actual correctness. It runs observe-only and changes no verdict, and it stays unpromoted until a held-out benchmark justifies flipping it on.

## Internal Seams (No User-Facing Change)

- **The determinism deletion.** The reproducible-ranking flag and all its gated branches were removed after the benchmark declined it, reverting to the default recency-aware ranking with the always-on id tie-break kept.
- **Generator fingerprints and the identity resolver.** The graph-metadata generator persists a source-doc fingerprint that survives an idempotent re-derive, phase-child detection moved behind one shared contract, and a shared spec-folder identity resolver reconciles parent and child relationships across re-derives.
- **The score-averaging path.** The semantic score-resolution path was routed through calibrated absolute relevance rather than averaged fusion magnitudes, a correctness fix with no interface change.

## The Flag Reckoning

This cycle added 41 feature flags across all four subsystems, and a benchmark decided the fate of each one rather than a hunch.

#### Graduated to Live

The search work that earned its way on is already detailed above: the search honesty suite (the lexical-grounding floor, the noise-floor subtraction and the evidence-gap cap), the relevance-aware gap warning, the absolute-relevance calibration that reports one honest score, the metadata integrity gates and the false-confirm build gate all measured well enough on the real corpus to become the default.

&nbsp;

#### Kept Quietly On

Three flags ship on without a headline because they make the records safer rather than the verdicts smarter. Derived-id provenance gives every causal edge a content-addressed ID (an identifier computed from the edge's own content, so the same edge always gets the same name), retention-forgetting is a deletion safety guardrail that keeps a cleanup from removing more than it should, and the world-summary grounding prelude seeds a search with a short factual frame before it runs. None of them change a number you read, so they sit on quietly.

&nbsp;

#### Built, Measured, and Cut

A large body of work was built in full, measured against the corpus, and then declined because it did not earn its place. The entire semantic-edge layer family went this way: the edge vector index, edge triplet search, edge semantic dedup and invalidation, edge presence currentness and bitemporal recall. So did agentic recall, sleeptime consolidation (a background pass that would have re-folded memory while idle), the procedural-reliability lane, the cardinality penalty, summary fusion, the grounding-signal field and deterministic ranking. Each was a real experiment with real code behind it, and each one the benchmark sent back rather than ship on a guess.

&nbsp;

#### Still Dark, Awaiting a Benchmark

The rest stay off until a benchmark has its say. On the search side that is the track-B appends (deterministic multi-hop, lane-champion backfill and the true-citation emitter), the per-class retrieval weights, the cosine-topN reorder and the archived-inclusion levers. Two of those search appends do not reach the production reader as they are currently wired, a structural gap rather than a quality verdict, so they wait on a rewire and not on the bench. The subsystem experiments wait too: on code graph the bitemporal edge reads and the closed-vocabulary edge governance, on the advisor the RRF fusion spine and the self-recommendation guard and on the deep loops the near-duplicate finding collapse.

## Upgrade Notes

No migration is required for the common path. The honest-verdict suite (the lexical-grounding floor, the noise-floor subtraction and the evidence-gap cap) and the relevance-aware gap warning are on for everyone and need no action. The determinism flag was removed, so if you had set `SPECKIT_DETERMINISTIC_RANKING` it no longer does anything and the system ranks by recency as it did by default. The spec-metadata integrity gates that graduated are on by default, the cite-with-caveat tier and the envelope-fidelity guard are on by default alongside the rest of the honesty suite, and the type-safe status validator runs in a non-blocking report mode during the migration window before it enforces.

## Verification

This release follows the same shipping rule as the last: a structural protection that only ever makes the system safer or cleaner ships on for everyone, while anything that changes results ships off by default behind a named flag until a benchmark earns it on. The honest-verdict suite, the relevance-aware gap, the score-scale correction and the type-safe metadata validator are live for everyone, the determinism option was measured and removed, and the confidence calibration fitter stays observe-only pending a held-out benchmark.

#### The Graduation Reckoning

The flags did not graduate on faith. A benchmark measured thirteen default-off flags against the freshly migrated tree and applied the verdicts in one pass: twelve earned default-on or enforcing, one was deleted for earning no measured win, and two that had failed for a missing freshness field were graduated after a re-run wrote it. The lexical-grounding floor and the noise-floor subtraction each independently drove an off-corpus false-confirm rate from 0.833 to zero in isolation.

&nbsp;

#### Three Search Benchmarks, Two Said No

The search work was decided by real-corpus measurement. The vague-query benchmark proved the citation surface model-robust. The determinism benchmark measured the reproducible-ranking option, found it reshuffled the top results without improving them, and the option was removed. The gap-threshold benchmark found the evidence-gap detector was measuring the wrong thing entirely (peakedness, not relevance) and that no threshold could separate a good search from a thin one, which drove the relevance-aware redesign rather than a tuning tweak.

&nbsp;

#### A Live Check That Caught a Masked Bug

The evidence-gap redesign nearly shipped a regression. Its first benchmark passed green, but a full-handler live dispatch showed the warning firing on every single search. The benchmark had fed the detector clean absolute-relevance scores while production fed it a different, compressed score, so the bug only appeared on the real path. The fix threads the production-correct relevance signal into the detector, the re-benchmark was rebuilt to run through the real pipeline so it could not mask the same class of bug again, and a second live dispatch confirmed the warning now fires only on a genuine gap.
