This release enriches the Motion.dev animation surface with the design-principle layer it was missing. sk-code's animation references were strong on API how-to (animate/timelines, scroll/gestures, integration, a decision matrix, performance pitfalls) but light on *when and why* — which easing to pick, concrete duration and stagger values, anticipation and arc guidance — and had no animation-specific visual-verification recipe. Those gaps were mined from the MIT-licensed `Schmandarine/web-motion-skill` repo and translated to the Motion.dev API, additive-only, with the source's GSAP-specifics and recording scripts deliberately left out.

## New Features

#### Animation principles reference

A new `references/motion_dev/animation_principles.md` adds the design-principle layer: a motion vocabulary table (feedback → emphasis timing ladder), easing-as-perceived-physics (cubic-bezier arrays, not GSAP eases), stagger direction and offset guidance, anticipation/overshoot/secondary-action ranges, arc-and-depth notes, and a visual-review checklist. Motion.dev-framed, cross-linked to the existing motion_dev refs, MIT-attributed.

#### Principled-reveal snippet

A new `assets/motion_dev/snippets/principled_reveal.js` composes the principles into one reusable reveal: anticipation (scale 0.98→1) + the timing vocabulary + `stagger(offset, { from })` direction + a `prefers-reduced-motion` fallback. The previous nine snippets each demonstrated a single API; this is the first that composes several principles into a production-style reveal. MIT-attributed in its `@source`.

#### Frame-level visual-verification recipe

`references/motion_dev/performance_and_pitfalls.md` gains a frame-level visual-verification recipe (capture → extract frames → contact-sheet → drill-in) for checking animation timing/easing by eye. It defers to the project's existing bdg / chrome-devtools / Playwright tooling rather than vendoring the source repo's recording scripts.

## Routing

`references/smart_routing.md` wires the new artifacts so they route on animation intent: the §2 Phase-1 synonyms gain `easing` / `stagger` / `motion principles` / `motion language` (ANIMATION) and `stagger()` / `animation principles` (MOTION_DEV); the §5 MOTION_DEV map and the §11 machine-readable `RESOURCE_MAP["MOTION_DEV"]` now include `animation_principles.md` and `principled_reveal.js`; the PERFORMANCE row notes the frame-level verification recipe. The snippet README gains the new entry and `webflow/implementation/animation_workflows.md` gains a cross-reference.
