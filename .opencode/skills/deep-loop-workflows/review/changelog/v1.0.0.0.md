Initial release of deep-review as a standalone skill, split from the unified deep-research skill (v1.2.2.0). This skill owns all iterative code review and quality auditing functionality with its own command, YAML workflows, reference documentation, and manual testing playbook. The goal is to make the workflow easier to follow and easier to trust.

---

#### Skill Split from deep-research

- Extracted all review-mode functionality from the unified deep-research into a dedicated deep-review skill package.
- New command `/speckit:deep-review` with `:auto` and `:confirm` modes (previously `/speckit:deep-research:review`).
- Renamed review YAML workflows: `spec_kit_deep-review_auto.yaml` and `spec_kit_deep-review_confirm.yaml` (previously `spec_kit_deep-research_review_*.yaml`).
- Review-specific assets moved to deep-review: `deep_review_strategy.md`, `deep_review_dashboard.md`, `review_mode_expected behavior.yaml`, `deep_review_config.json`.
- Skill advisor routing updated: review keywords now route to `deep-review` instead of `deep-research`.

---

#### Standalone Reference Documentation

- Created `references/loop_protocol.md` - review-specific loop lifecycle (init, iterate, synthesize, save) with scope discovery, dimension ordering, cross-reference protocols, and claim adjudication.
- Created `references/convergence.md` - review convergence algorithms with severity-weighted ratio, P0 override, dimension coverage signal, quality guards, and provisional verdict logic.
- Created `references/state_format.md` - review state file schemas for config, JSONL, strategy, iteration files, review report, dashboard, claim adjudication, and finding registry.
- Created `references/quick_reference.md` - review-only cheat sheet with numbered H2 sections aligned to skill_reference_template.md.

---

#### Manual Testing Playbook

- Created 28 review-specific test scenarios across 6 categories (mirroring deep-research structure):
- 01--entry-points-and-modes (3 scenarios: DRV-001 to DRV-003)
- 02--initialization-and-state-setup (4 scenarios: DRV-004 to DRV-007)
- 03--iteration-execution-and-state-discipline (7 scenarios: DRV-008 to DRV-014)
- 04--convergence-and-recovery (6 scenarios: DRV-015 to DRV-020)
- 05--pause-resume-and-fault-tolerance (4 scenarios: DRV-021 to DRV-024)
- 06--synthesis-save-and-guardrails (4 scenarios: DRV-025 to DRV-028)

---

#### Runtime Updates

- New `.agents/commands/speckit/deep-review.toml` runtime wrapper.
- Agent definitions updated across all runtimes (.claude, .opencode, .agents, .codex) to reference `/speckit:deep-review` and `deep-review`.
- `CLAUDE.md`, `AGENTS.md`, `AGENTS_example_fs_enterprises.md` updated with deep-review entries.
- Root `README.md`, `.opencode/skills/README.md`, `.opencode/README.md` updated with deep-review catalog entries.
- Barter repo `skill_advisor.py` updated with matching review keyword routing.
