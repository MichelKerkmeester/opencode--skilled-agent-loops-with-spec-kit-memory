# Questions Answered

- Q1/D2: `/design:*` specificity should be task-command metadata, not only richer prose in the five pinned mode wrappers. The portable grammar is typed input class, accepted forms, examples, rejects, produced artifact, and next options.
- Q2/D3: Parent-to-sub-skill utilization should be checked through ordered `choreography[]` plus context/proof fields, before ordinary resource recall can score.
- Q5/all: The enforceable core is metadata schema, generated wrapper drift checks, argument fixture replay, route/bundle gold, choreography proof fields, and no-silent-chain status grammar. Open-ended intent interpretation and design taste remain advisory.
