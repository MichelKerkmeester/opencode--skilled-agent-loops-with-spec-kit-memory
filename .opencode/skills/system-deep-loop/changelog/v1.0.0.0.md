# v1.0.0.0 - Initial Release

First changelog for `deep-loop-workflows` as a parent hub. The hub is routing-only: `Skill(deep-loop-workflows)` loads it, and the router reads the request and loads the matching mode packet. This brings the deep-loop family in line with the parent-skill pattern - one invokable hub, one advisor identity, and the loop logic kept in the nested packets.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/155-parent-skill-native-invocability/002-deep-loop-alignment` (Level 3)

---

## New Features

#### Invokable-hub routing

The hub SKILL.md now carries the smart-routing contract, so an invocation of the parent loads the hub and routes to one mode packet. The active mode packets are `deep-research`, `deep-review`, `deep-ai-council` and `deep-improvement`, each registered in `mode-registry.json` and each holding its own loop logic behind the hub's single `graph-metadata.json` identity.
