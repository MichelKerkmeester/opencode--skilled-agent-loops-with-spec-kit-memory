3. **The Stop hook’s user-visible severity is weaker than the writer failure.** A missing generator is classified `skipped`; a non-zero/timed-out generator is classified `failed` and logged as a warning; and the hook’s outer CLI always exits `0`. Therefore ambiguous-leaf and missing-target failures do not block Claude Stop and can look like a successful stop unless hook logs are inspected. [SOURCE: .opencode/skills/system-spec-kit/mcp_server/hooks/claude/session-stop.ts:104-152,743-749]

4. **The most subtle absent-alias outcome is successful child metadata plus a stale phase-parent pointer.** Runtime folder detection can recover the canonical child by unique basename, and workflow follow-ups update that recovered absolute packet, but `generate-context` subsequently rechecks its original nonexistent legacy-qualified argument and skips `updatePhaseParentPointersAfterSave()`. This is neither total failure nor root split-brain: the child is MCP-visible, while parent resume routing can remain stale. [SOURCE: .opencode/skills/system-spec-kit/scripts/dist/spec-folder/folder-detector.js:929-945] [SOURCE: .opencode/skills/system-spec-kit/scripts/dist/core/workflow.js:1293-1408] [SOURCE: .opencode/skills/system-spec-kit/scripts/dist/memory/generate-context.js:732-760] [INFERENCE: the three cited stages use different target variables]

5. **A distinct plain legacy directory is the only surveyed state that systematically converts writers into silent split-brain instead of fallback or hard failure.** It satisfies legacy existence checks, wins scripts-side order, and accepts direct untracked scaffolding; meanwhile MCP discovery remains on canonical whenever `.opencode/specs` exists. The symlink-disabled Git checkout case is different: `specs` is a plain file, so it cannot accept packet writes and direct untracked creation hard-fails. - What worked and why: Reusing the prior writer inventory avoided the blocked broad scan; tracing the configured Stop hook into compiled generator/detector/workflow artifacts exposed the previously hidden basename-recovery and phase-pointer-skip stages.
- What did not work and why: Treating “legacy-first falls back to canonical” as sufficient initially missed that the Stop hook first converts its target into an explicit absent legacy path. Additional narrow runtime reads were required and exceeded the iteration tool budget.
