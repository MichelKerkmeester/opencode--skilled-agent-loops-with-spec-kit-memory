## [**1.0.0.0**] - 2026-07-16

This first release brings the Aside AI browser into agent workflows as a routed skill packet. It covers natural-language browser tasks, deterministic evidence capture, and a documented path to MCP composition — with every capability claim traced to completed research.

#### Three lanes, one router

- Need: Browser work from an agent splits into outcome-oriented goals, deterministic proof steps, and multi-tool composition — and each wants a different surface.
- Change: This release routes between the Aside agent-task CLI, the deterministic JavaScript REPL, and the Aside MCP server via Code Mode, with a weighted intent scorer picking the lane.
- Why it matters: People get the simplest surface for the job instead of handing deterministic proof work to a nondeterministic agent.

&nbsp;

#### Honest capability boundaries

- Need: The Aside tool surface is version-pinned and partially unverified; a packet that overclaims would break users at the first console-capture or parallel-session request.
- Change: The packet mandates runtime tool rediscovery, marks unresolved research questions as UNKNOWN (binding procedure, permission inheritance, model-flag spelling, parallel manuals, console/network capture), and fails closed on unverified capabilities.
- Why it matters: Failures get diagnosed by their real cause — a browser-unbound state is never mistaken for missing auth.

&nbsp;

#### Setup, diagnostics, and validation

- Need: New users needed install steps, health checks, and testable scenarios before trusting the workflow.
- Change: The release ships an install guide and wrapper (macOS-only curl installer), a read-only doctor with an MCP handshake probe, three example workflow scripts, a manual testing playbook, and server-package pointers including the drafted UTCP manual for the later registration phase.
- Why it matters: Operators can verify the whole surface end-to-end without the packet ever installing, updating, or registering anything implicitly.
