# v1.0.0.0 - First Release

**Released:** 2026-07-16

First release of the `mcp-refero` skill, the read-only Refero design-reference TRANSPORT packet of the `mcp-tooling` hub (`packetKind: transport`, `mutatesWorkspace: false`). It teaches an agent to search **Refero's library of real shipped UI** (styles, screens, flows) through the `refero` Code Mode manual already registered in `.utcp_config.json`, and to hand every design-affecting conclusion to `sk-design`, the mandatory cross-hub judgment partner. The packet is grounded exclusively in the completed three-lineage deep-research synthesis of the Refero MCP developer surface, and it is a deliberately **simpler peer of `mcp-figma`**: MCP-only, read-only-only, no CLI machinery, and no mutating or destructive gating taxonomy, because the provider surface itself is read-only.

### Key design decisions (from the research synthesis)

- **Doubled-prefix callable** `refero.refero_refero_<tool>(...)`: a preserved cross-lineage conflict resolved toward the live-verified local evidence, with the losing single-prefix derivation documented as negative knowledge and a mandatory post-registration `tool_info` confirmation adopted regardless.
- **Existing manual preserved byte-for-byte**: the packet consumes and verifies the registered `refero` manual (`npx -y mcp-remote https://api.refero.design/mcp`), never edits it, never adds a second manual, and never moves to a direct Bearer base setup.
- **Plan gating stated honestly**: Free has NO MCP access (denial, not degradation); Pro is 8,000 tool calls/month; no per-second/burst/concurrency/`Retry-After` contract is published and none is invented.
- **`response_format` on the seven text tools only**, never on `refero_get_screen_image`, with per-tool availability treated as a runtime `tool_info` check.
- **Ten open questions carried as UNKNOWN**, including end-to-end OAuth completion (Inferred pending operator authorization) and the local flow-batch-of-10 observation (heuristic, not contract).

### Boundaries

- Minimal runtime `allowed-tools` (`Read, Bash, Grep, Glob, mcp__code_mode__call_tool_chain`); Write, Edit, and Task are forbidden for this transport.
- The official `referodesign/refero_skill` methodology repository is referenced, not vendored: it is a peer of `sk-design`, and duplicating it would collide with local design authority.
- No hub files are modified by this packet's release; hub registration (mode-registry, hub-router, hub docs) ships separately.
