## Findings

- **F1 — Mutation-as-epistemology (the "mutation check"), weaponized against written claims.** [mechanism/ritual] After a test goes GREEN, deliberately break the production code in place, confirm the *specific* test fails, restore, re-green — a self-contained one-liner that leaves the tree untouched (backup → mutate → run → restore). The apex move: a predecessor note claimed a test pinned a credential-zeroing ordering; a non-destructive overlay mutation proved the suite stays green when the ordering is violated, so the *claim* was false — "a green test is not evidence; a green test that goes red when you break the thing it guards is evidence." Vacuous green tests are hunted as *defects* (a lie about coverage). Distinguishes "compile-RED" from "true RED": a compile failure does not count as a satisfying RED. [SOURCE: fable-mode-profile.md:111-123, 348-349; fable-mode.md:17] **Net-new:** Fable5.md has "run the real thing"/"reproduce before cause" but the mutation check as a *named ritual* and as a *claim-falsifier* is absent from round-1's shipped set.

- **F2 — The verification ladder with pre-named blind spots.** [ritual/method] Five named layers, cheapest→most authoritative: unit/table → in-memory orchestration (fake server) → on-server integration → live production scripts → headless-browser smoke. "in-memory-green is not production-green." Each layer's blind spot is named *in advance* and you climb specifically to hit it. Proof case: a protocol path green against the in-memory fake but dead on the real server, caught only on the integration re-run, "not by the review." [SOURCE: fable-mode-profile.md:126-139; fable-mode.md:18] **Net-new:** round-1 shipped no explicit ladder.

- **F3 — Adversarial-review-at-scale with a forced evidence schema + a completeness critic.** [mechanism] Spawn a role-segregated fleet to *attack* an artifact: parallel claim-verifiers (one per cluster), an adversarial recheck of anything contradicted, and a **completeness critic asking what is *missing***. Output schema forces a `claim / verdict / evidence` triple with file:line + verbatim quote per claim. Bidirectional — it refutes its *own* reviewers on re-derivation ("2 reviewer claims refuted"). Fleet sizes grew 5 → 13–14 → 24 → 34 over the project. [SOURCE: fable-mode-profile.md:141-146, 352-360] **Net-new + directly maps** to round-1's carried "machine-checkable evidence contract" follow-up and to our @orchestrate/@deep-review fleets.

- **F4 — Scar-tissue curation ("Traps already hit — do not rediscover").** [ritual/mechanism] A handoff ledger that records (1) the **blast site**, not just the rule ("`UID` is readonly in bash — broke a cleanup script"); (2) **where the trap will bite next**, pre-applied to code that doesn't exist yet (naming the not-yet-built features that will need the same filter); (3) whether a guard is **load-bearing vs defensive**, with its **activation condition** ("goes load-bearing if the inputs widen"), so a successor neither deletes it as dead nor trusts it blindly. [SOURCE: fable-mode-profile.md:202-211] **Net-new:** richer than Fable5.md's "log why you ruled something out."

- **F5 — Cold-successor handoff as an engineered protocol.** [ritual] The handoff carries *only what cannot be re-derived* — "state, sequence, and scar tissue." It opens with a numbered **Read order** telling a cold session which docs to read *and what each settles*. It is judged by **role-playing the reader cold** ("as the next session would: cold, with nothing else"). Its most ambitious form ships the *next* unit's ~100-line brief, including the one decision the successor must make day one. [SOURCE: fable-mode-profile.md:200-216, 512-521] **Net-new** as a discipline; maps to our `handover.md` / `_memory.continuity`.

- **F6 — Engineer staleness out of artifacts (rotting list → self-auditing test).** [mechanism/ritual] When you notice a fact that will rot, don't write it as prose — convert it to machinery that fails loudly: **counts become greps, enumerations become table-walking tests, "today X is true" becomes an assertion.** Flagship: API routes became a data table + a test that walks it, so any future mutating endpoint that forgets the CSRF check fails the suite instead of shipping unprotected — proven by a mutation check ("Routes become a table; CSRF coverage is now a test, not a list"). [SOURCE: fable-mode-profile.md:218-232, 520] **Net-new + high-leverage:** directly relevant to our comment-hygiene / anti-staleness gates — and the iter-3 dead-pointer is exactly the rot this prescribes converting into a check.

- **F7 — Fail-closed by construction (structural, not disciplinary, invariants).** [doctrine/mechanism] Prefer construction-based invariants over discipline-based ones: poison the unsafe default so a stray caller fails safe (replace `http.DefaultTransport.DialContext` with one that always fails); make a schema's DDL *alone* prove its constraint; reject rather than silently strip ("rejecting is visible, testable, and loses nothing legitimate"); enforce a cap twice so a mutation killing one layer still satisfies the contract; build a second, code-disjoint verifier so a bug can't hide in the checker. [SOURCE: fable-mode-profile.md:243-248, 280-283; fable-mode.md:22] **Net-new** vs round-1.

- **F8 — Decision economy: scope-frugal, process-expensive, never a dead control.** [doctrine] Be explicit about *not* doing work a later change will rewrite; leave a **named seam, not a bare `// TODO`**; "a dead button is worse than a missing one." Distinguish **scaffolding the contract** (ship the wire key now, explicitly null, so the client contract holds when the feature lands) from **scaffolding the implementation** (defer with a seam). Resist threading a systemic fix opportunistically through an unrelated one. [SOURCE: fable-mode-profile.md:69-79, 250-256; fable-mode.md:21] **Net-new** nuance beyond Fable5.md's "record a one-line follow-up."

- **F9 — Brief-as-sovereign with the letter-vs-intent dialectic.** [doctrine] The brief is the contract; read it for **intent**; deviate from the *letter* only with a recorded argument that the intent is better served; **an undocumented deviation is the actual sin** ("We implement the brief, not the bug"). Explicit hierarchy: contract (immutable) > spec (amendable, dated notes) > decisions log > handoff (perishable). [SOURCE: fable-mode-profile.md:284-291; fable-mode.md:24] **Net-new** as an explicit dialectic; partially echoes our Logic-Sync / amendment protocol — but sharper: it names *undocumented* deviation, not deviation itself, as the fault.

- **F10 — Two-register voice + the lexicon + averted-disaster framing.** [register/measurement] Two registers switched *by what you are doing*: clipped, colon-terminated while working ("GREEN. Committing:"); dense and complete at turn boundaries, every clause load-bearing. Lead with the verdict, then the receipts (file:line, hash, measured-vs-target). **Convey importance through averted disaster, not adjectives** ("the shell would have died on the skeleton screen") — never "thorough/robust/comprehensive." No celebration, apology theater, or flattery. Signature **lexicon**: a test *pins* (not "covers"); *verify against*; *trap/tripwire*; *drift*; *owed*; *deliberately*; *fail closed*; *load-bearing* vs defense-in-depth; *the seam*; *byte-identical*; *fixpoint*. [SOURCE: fable-mode-profile.md:420-461; fable-mode.md:27-33] **Net-new + the spec's named Tier-B target** ("lexicon + register-split").

- **F11 — Multi-agent house rules (LEAF discipline + typed status + two-stage review + personal backstop).** [mechanism] Implementer subagents are scoped to **disjoint paths**, **banned from git and from editing `.md` docs**, and return a **typed status enum** (`DONE | DONE_WITH_CONCERNS | NEEDS_CONTEXT | BLOCKED`, plus `files`, `tests`, `concerns`). Each track gets a **two-stage review** — spec-compliance (to the letter) THEN code-quality (judge craft, don't relitigate the contract) — each with its own fix loop. A **"verify personally" backstop**: when a fix loop hits its round cap the track returns `DONE_VERIFY_PERSONALLY`. The orchestrator commits; it re-runs every gate itself and never commits on a subagent's word. [SOURCE: fable-mode-profile.md:351-360] **Net-new + strongly maps** to our @orchestrate/@code LEAF convention, the agent-io-contract, and the "finding is a hypothesis" rule.

- **F12 — Reproduce-before-fix + "a second break behind the first" + suspect-yourself-first + throwaway instruments.** [method] Don't fix a bug you can't first make fail (true-assertion RED). Assume a fixed bug masks another and **generalize the fix so the next instance cannot hide** ("so a third drift cannot hide behind fixtures"). Suspect your *own* assumptions before the system ("before blaming production, verify the selector"); build throwaway instruments to *locate* a fault rather than fire another guess, discard wrong hypotheses out loud, and clean the instruments up afterward. [SOURCE: fable-mode-profile.md:159-192; fable-mode.md:15] **Partially-new** sharpening of Fable5.md's "reproduce a diagnosis before you call it the cause" — folded, not stand-alone.

- **F13 — Measurement integrity (a number you can't trust is a bug, even green).** [measurement] Never report a number bare — pair it with its target, bold the measured value, re-run benchmarks in production. An inaccurate gate is a gate that lies: fix a gate's undercount *even at ~28KB against a 60KB budget* (huge headroom). [SOURCE: fable-mode-profile.md:316-322] **Net-new + Tier-C relevant** (a metric that lies is worse than no metric — directly governs the leak_test port in G3).

- **F14 — Ration live/destructive actions; harnesses over production; cleanup as a privacy ethic.** [doctrine] Ration deliberate live failures against shared systems ("at most one deliberate failed login per session… never TDD-iterate auth against production"); build loopback harnesses so you needn't touch production; clean up and **verify the deletion** (data minimization, not tidiness). [SOURCE: fable-mode-profile.md:328-331, 366-372; fable-mode.md:25] **Partially-new** beyond Fable5.md's rollback rule; **lower direct relevance** (we have no shared production box).

- **F15 — Worst-first triage in named severity buckets, scored by downstream consequence to a cold reader.** [ritual] Order findings by blast radius with literal named tiers ("Would actively misdirect" > "Real omissions" > "Paperwork-level (low)"); the severity metric is damage-to-a-successor, not difficulty/novelty; the #1 item is the most damning even when it's your own ("One claim I added yesterday was flat wrong"). [SOURCE: fable-mode-profile.md:58-67; fable-mode.md:20] **Partially-new** sharpening; maps to our P0/P1/P2 review severity (deep-review).

**Cross-cutting observation (the root conviction / doctrine spine):** every technique descends from one belief — *the expensive failures live exactly where confirmation is cheapest to skip* (the gap between green and reality, between a doc and the truth) — so spend lavishly to close those gaps [SOURCE: fable-mode-profile.md:21-31]. This is the spine the Tier-A recommendation should make legible in a single line, not a paragraph.

**Guardrail the command itself ships (load-bearing for our adoption):** "Emulate the strengths, not the costs… **Scale rigor to blast radius.** This mode is not a license to spawn agent fleets or run multi-hour workflows for a one-liner" [SOURCE: fable-mode.md:11]. The profile's §11 names the real costs: verbosity at turn boundaries, a 1,323-line monotonically-growing decision log, process outweighing a small change, autonomy outrunning operator tolerance, em-dash non-compliance, operational slips the gates don't catch [SOURCE: fable-mode-profile.md:491-507]. **Any adoption recommendation must inherit this guardrail or it imports the cost with the strength** — and our framework already has the matching primitive (CLAUDE.md §1 "Match effort to blast-radius").
