## Rationale

SKIPPED — Requires issuing a /rerank request via cocoindex consumer with a forced short timeout (adapter timeout_s set to 1s) and an unusually long document list to trigger timeout. The CLI does not expose a way to override the HttpSidecarRerankerAdapter timeout_s parameter. This scenario requires either modifying the cocoindex code to add a CLI flag for timeout, or creating a custom Python test harness that directly instantiates the adapter with custom timeout. The current Qwen model is fast enough that normal document lists don't timeout at the default 30s.
