# v1.0.0.1 - make-interfaces-feel-better surface craft

**Released:** 2026-06-27

This release folds the genuinely net-new surface micro-craft from a study of the external `make-interfaces-feel-better` corpus into the design family, and fixes a hub documentation bug. The corpus is prescriptive frontend polish; this release adopts only what `sk-design` did not already encode, lands each rule in `foundations` with a matching `audit` detector, closes the one real `motion` gap, and rules out the rest (no global review format, no wholesale numeric defaults, no per-mode logic in the hub). All edits are additive guidance; nothing changes runtime behavior.

> Source research: `.opencode/specs/skilled-agent-orchestration/154-sk-design-parent/022-mifb-design-research/research/research.md`
> Build phase: `.opencode/specs/skilled-agent-orchestration/154-sk-design-parent/023-mifb-design-adoption` (Level 3)

## Foundations (rules)

The static visual-system mode is the rule home for the new surface craft.

#### Concentric border radius
Nested rounded surfaces now have an explicit rule: the outer radius should equal the inner radius plus the inset between them, and layers more than ~24px apart are treated independently. Mismatched nested radii were the most common reason a careful UI still felt slightly off, and the family had no formula for it.

#### Image-edge outline as an optical exception
Images get a 1px inset outline using pure black `rgba(0,0,0,0.1)` (light) or pure white `rgba(255,255,255,0.1)` (dark). This is a deliberate exception to the family's normal rule of tinting neutrals toward the brand hue — a tinted image outline picks up the surrounding surface colour and reads as dirt on the image edge.

#### Shadow-as-border and dark-mode depth
A transparent shadow ring may replace a decorative depth border on raised surfaces, but must never coexist with a solid border plus a wide shadow on one element (that pairing is the existing ghost-card tell). Dark-mode elevation comes from lightness steps, with a low-opacity white ring used only as an optical or state separator.

#### Typography precision
Root-only macOS font smoothing, `text-wrap` line-count caveats (`balance` for short headings, `pretty` for short-to-medium text, neither for long), and a dynamic-number rule for `tabular-nums` (counters, timers, prices, updating tables) to stop layout shift.

## Audit (detectors)

Each foundations rule gained a matching detector so violations surface as findings.

#### New production anti-pattern detectors
Same-radius nested surfaces, image outlines that use tinted/branded neutrals or layout-affecting borders, the border-plus-wide-shadow ghost-card pairing (cross-linked to the existing fingerprint tell, with the shadow-ring replacement allowed), and hit-area collisions (the 44x44 floor is kept; expanded hit areas must not overlap neighbours).

#### transition: all static-risk detector
The accessibility-and-performance audit now flags `transition: all` (and Tailwind's bare `transition` shorthand) next to the existing broad/permanent `will-change` guidance, asking for named transition properties.

## Motion

The motion mode already covered interruptible transitions, `AnimatePresence initial={false}`, press scale `0.96`, and zero-bounce springs, so this release adds only the missing pieces: a no-dependency contextual icon-swap CSS fallback (both icons in the DOM, cross-fading scale `0.25 -> 1`, opacity `0 -> 1`, blur `4px -> 0`), a `static` escape hatch for press feedback, semantic split/stagger enters, and small fixed-translate exits — all without overriding existing stagger caps or exit-duration rules.

## Interface, md-generator, shared, and the hub

#### Interface preflight reminders
The interface mode gained optical-alignment examples and preflight reminders for alignment, image-edge outlines, concentric radii, and hit-area collision — checklist reminders, not a new rule family.

#### md-generator measured capture
The extraction pipeline now preserves measured image outlines, shadows, nested radii, root font smoothing, and hit-target affordances when present, without authoring new taste defaults.

#### Shared vocabulary
`image-edge outline` and `shadow ring` are defined in the shared token vocabulary so modes share consistent language; the mechanics stay in foundations and audit.

#### Hub documentation fix
The hub previously cited a `references/` shared base that does not exist; the real directory is `shared/`. The layout tree, the backend paragraph, and the references list now point at `shared/`. Per-mode `references/` paths were not touched.
