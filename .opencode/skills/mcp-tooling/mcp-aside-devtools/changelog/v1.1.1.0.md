## [**1.1.1.0**] - 2026-07-16

Live Code Mode discovery ran on 2026-07-16 (direct stdio MCP probe of CodeMode-MCP: `initialize`, then `tools/call` on `list_tools`, `search_tools`, `tool_info`, with `UTCP_CONFIG_FILE=.utcp_config.json`). The dated fixture `references/discovery-fixture-2026-07-16.json` is the packet's new ground truth, and this release flips every discovery-pending claim to the observed facts.

#### Discovery flips (callable naming)

- Need: The packet predicted `aside.aside_repl` as THE callable name and marked it UNCONFIRMED/UNKNOWN across SKILL.md, README, install-guide, references, the asset snapshot, the feature catalog, the playbook, and the doctor.
- Change: All those claims now record both observed facts precisely: discovery returns the registry name **`aside.aside.repl`** (dot-separated `{manual}.{server}.{tool}` — the pre-discovery registry prediction lacked the dot), while the TypeScript call surface inside `call_tool_chain` is **`aside.aside_repl(args)`** (the fixture's `Access as:` line, matching mcp-code-mode's `{manual_name}.{manual_name}_{tool_name}` convention). The one-`repl`-tool inventory is re-confirmed live: `aside.aside.repl` is the only `aside.*` registry entry.
- Why it matters: The two name forms differ by design (dotted for discovery queries, underscore for TS calls); recording both prevents the #1 Code Mode error class — calling with the discovery name or searching with the call name.

&nbsp;

#### Helper-surface refresh

- Need: `references/mcp-wiring.md` and `references/aside-cli-reference.md` carried a version-pinned "advertised helpers" description with the full surface flagged unconfirmed.
- Change: Both now carry the full helper surface from the fixture (`listBrowserTabs`/`attachBrowserTab`/`attachActiveBrowserTab`/`getTabByTargetId`/`openTab`/`closeTab`, `snapshot`, `display`, `page`, `tabs`, screenshots/PDF, `fs`/`pwd`/`path`/`Buffer`/`sleep`/`fetch`) plus the fixture-documented rules (console.log-only returns; neutral-session tab attachment order). Bound-page output shapes stay probe-required — discovery lists the surface, it does not exercise calls.
- Why it matters: The REPL contract consumers code against now matches what the live server describes, not an older pinned snapshot.

&nbsp;

#### What stays open

- Authenticated/live invocation items remain SKIP-valid operator work: an actual `call_tool_chain` smoke invocation, browser-profile binding (procedure still UNKNOWN), bound-page output shapes, permission-mode inheritance, and the multi-client isolation test behind the single-vs-dual-manual question. Their preconditions are updated: callable-name confirmation is now satisfied by fixture.
