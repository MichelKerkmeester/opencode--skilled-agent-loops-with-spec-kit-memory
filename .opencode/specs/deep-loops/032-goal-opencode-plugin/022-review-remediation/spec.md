## 2. PROBLEM & PURPOSE

### Problem Statement
The dual-model review confirmed one real runtime defect and a broad documentation-drift cluster. (1) `sweepOrphanedActiveStates` (mk-goal.js:~1231) makes its staleness decision - stat, readFile, `updatedAtMs` comparison - OUTSIDE the per-session mutation queue, then calls `archiveGoalStateFile` (:~1205) which enqueues a rename WITHOUT re-validating; a goal written between the sweep's read and the queued rename is archived while active (silent active-goal loss, recoverable only from archive). (2) Eight child phases (009-speckit-command-goal-prompt-offer, 012, 015, 016, 017, 018, 019, 021) carry spec.md Status rows of Planned/Draft while their implementation-summary.md says Complete - the work verifiably shipped (fresh full-suite run 2026-07-04: 103/103 pass) but the spec rows and parent phase-map were never flipped, and both reviewer models mis-inferred the fix direction from this drift. (3) The audit dossier (`../scratch/2026-07-03-four-reviewer-audit-findings.md`) still presents findings F4, F5, DOC-2, and e-2.2 as open although each is verifiably mitigated in current code. (4) The parent spec.md narrative claims a "6-file test suite" (actual: 8 goal test files) and a "16-seam" test contract (actual: 17 `__test` seams). (5) Both feature catalogs omit `speckit-goal-offer-contract.test.cjs` and describe `mk-goal-state.test.cjs` inconsistently with each other.

### Purpose
The one confirmed runtime bug is fixed with regression proof, and every packet document tells the same story as the shipped code: statuses match reality, the dossier no longer resurrects fixed findings, narrative counts are accurate, and the catalogs list the full test suite.

## 6. RISKS & DEPENDENCIES

| Type | Item | Impact | Mitigation |
|------|------|--------|------------|
| Risk | Re-validation inside the queue could suppress legitimate explicit archives (session.deleted, clear) | Medium - stale state files would accumulate | Scope the re-validation to sweep-initiated archives only (option or dedicated enqueued op); existing explicit-archive tests must stay green unmodified |
| Risk | The interleaving test could be flaky if it relies on timing instead of promise ordering | Medium - flaky suite | Drive the interleave deterministically: hold the session's mutation queue with a deferred mutator, trigger the sweep's read, resolve the mutator to write a fresh goal, then let the sweep's queued archive run |
| Risk | Flipping spec Status rows without checking each impl-summary could overclaim an unshipped phase | Low - 020 is the only ambiguous row | Verify each phase's implementation-summary content (not just the row) before flipping; report any phase left unflipped with the reason |
| Dependency | Generated metadata (graph-metadata.json, description.json) refresh | Low | Orchestrator runs backfill-graph-metadata.js after doc edits land; not in dispatch write scope |
