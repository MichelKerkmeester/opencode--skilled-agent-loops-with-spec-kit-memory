## Findings

### Q4 — Save/index path (CONFIRMED, file:line)
1. **The save path already splits write from enrichment, and a deferred mode already exists.** The save handler set lives in `handlers/save/` (`create-record.ts`, `embedding-pipeline.ts`, `response-builder.ts`, `types.ts`). `OperationStatus` is `'ran' | 'skipped' | 'failed' | 'deferred' | 'partial'` [SOURCE: handlers/save/types.ts:138], and `create-record.ts` logs `"Using deferred indexing"` while `embedding-pipeline.ts` logs `"Async embedding mode - deferring embedding"` behind an `asyncEmbedding` flag ("embedding generation is deferred (non-blocking)") [SOURCE: handlers/save/create-record.ts:297, handlers/save/embedding-pipeline.ts:152, handlers/save/types.ts:304]. So the *fast episode-write + deferred enrichment* shape aionforge describes is **already partially implemented** for embeddings/BM25, not absent.
2. **`memory_index_scan` already supports a background job and reports pending vectors.** Scan args carry `background?: boolean` ("run the scan as a background job and return a jobId immediately") [SOURCE: handlers/memory-index.ts:293-294]. The scan result tracks a `deferred` count and returns `status: results.deferred > 0 ? 'complete_with_pending_vectors' : 'complete'` with `pendingVectors` [SOURCE: handlers/memory-index.ts:153, 1062-1064, 1376-1377]. This is the *cursor/work-queue* analogue at coarse grain (file-level), not episode/fact grain.
3. **Content-addressing already exists at the record/scan level via `content_hash` (sha256), but IDs are NOT content-addressed.** `memory_index.content_hash` is a sha256 over content [SOURCE: handlers/memory-index.ts:281-284 (`createHash('sha256')`), handlers/memory-crud-types.ts:73]. Scan idempotency uses a sha256 `scanKey` over normalized options [SOURCE: handlers/memory-index.ts:272-285]. But the primary key is the sequential `memory_index.id` (autoincrement; rows reference `nextMemoryId` [SOURCE: handlers/pe-gating.ts:407]). aionforge instead makes the *id itself* a content hash (`fact_id(namespace, subject, predicate, object, episode_id, rule_version)`) so re-extraction is a no-op [SOURCE: external/aionforge-memory-development/docs/consolidation.md:84-90]. **INFERENCE:** internal IDs are sequential, not content-addressed — content-addressing is present as a *dedup signal* (`content_hash`) but not as *identity*. Closing the gap means deriving derived-artifact IDs (e.g. causal edges, future "facts/notes") from content, not the episode row id.
4. **Idempotency-receipts already exist and are aionforge-shaped — but default OFF.** `lib/storage/idempotency-receipts.ts` defines receipts keyed by `{operation, contentHash, requestFingerprintHash, payloadHash}` over `memory_save`/`memory_update`, with explicit `miss | replay | conflict` lookup states (replay returns the stored response; conflict flags a same-key different-payload collision) [SOURCE: lib/storage/idempotency-receipts.ts:9-29, 87-90]. Client idempotency tokens are stripped before hashing so the receipt is content-derived, not token-derived [SOURCE: lib/storage/idempotency-receipts.ts:43-79]. Enablement is gated: `isMemoryIdempotencyEnabled()` reads `SPECKIT_MEMORY_IDEMPOTENCY` and is **off unless explicitly set** [SOURCE: lib/storage/idempotency-receipts.ts:54-57]. Expired receipts are pruned on the server lifecycle [SOURCE: context-server.ts:51, 1831]. So "idempotency receipts fall out of the async split" is already true *in code* — the work is promotion + wiring the replay/conflict path into the deferred-save flow, not invention.

### Q3 — Causal edge model + tombstones + bi-temporal (CONFIRMED, file:line)
5. **The edge relation vocabulary already includes supersedes/contradicts/supports.** `RELATION_TYPES = {CAUSED, ENABLED, SUPERSEDES, CONTRADICTS, DERIVED_FROM, SUPPORTS}` with traversal weights `supersedes:1.5` (strongest, "new info replaces old"), `contradicts:0.8` (dampened) [SOURCE: lib/storage/causal-edges.ts:21-43]. aionforge's two key currentness edges (SUPERSEDED_BY, CONTRADICTS) thus already have internal counterparts — `supersedes` and `contradicts` — though internal edges are memory↔memory, not fact↔fact (no fact layer yet).
6. **Bi-temporal currentness is already partially modeled as edge-presence, not a node flag — and it is flag-gated OFF.** `lib/graph/temporal-edges.ts` adds `valid_at` / `invalid_at` columns to `causal_edges` (`ensureTemporalColumns`, fail-open) and exposes `invalidateEdge()` that sets `invalid_at` to now [SOURCE: lib/graph/temporal-edges.ts:35-80]. `lib/graph/contradiction-detection.ts` queries only *live* edges (`WHERE invalid_at IS NULL OR invalid_at = ''`) [SOURCE: lib/graph/contradiction-detection.ts:85-90], auto-invalidates on `supersedes` ("'supersedes' replaces any existing edge on the same pair") and on conflicting pairs `supports↔contradicts`, `caused↔contradicts`, `enabled↔contradicts` [SOURCE: lib/graph/contradiction-detection.ts:38-42, 99-110]. The whole path is gated by `isTemporalEdgesEnabled()` and returns early when OFF [SOURCE: lib/graph/contradiction-detection.ts:75-77]. This is *exactly* aionforge's "currentness = edge presence; superseded edges retired with a closed window, not deleted" — at the causal-edge layer rather than a fact-ABOUT-edge layer [SOURCE: external/aionforge-memory-development/docs/bi-temporal-model.md:35-39, 89-113].
7. **Two distinct retirement mechanisms coexist — and they are not yet unified.** (a) *Temporal invalidation* (soft-close via `invalid_at`, retains the row, drives `Current`-style reads by edge presence) [SOURCE: lib/graph/temporal-edges.ts:64-80]. (b) *Tombstone sweep* (`causal_edge_tombstones` with `lifecycle_generation`, `restore_metadata`, identity index on `(source_id,target_id,relation,lifecycle_generation DESC)`) which is a hard-delete-with-restore-receipt for active deletion [SOURCE: lib/causal/sweep.ts:68-100, 175-188]. aionforge keeps a *single* non-destructive discipline (close the window; never delete; `History` mode still reads it). **INFERENCE:** the internal system has BOTH a non-destructive temporal close AND a destructive-but-restorable tombstone sweep; a bi-temporal currentness model would lean on (a) for "not current" and reserve (b) for genuine off-state forgetting — they answer different questions and should not be collapsed.
8. **What is missing for full bi-temporal parity (CONFIRMED gaps):** internal `causal_edges` has the *transaction-time-ish* pair (`valid_at`/`invalid_at`) but **not aionforge's four-timestamp split** of event-time (`valid_from`/`valid_to`) vs transaction-time (`ingested_at`/`expired_at`) [SOURCE: external/aionforge-memory-development/docs/bi-temporal-model.md:14-20 vs lib/graph/temporal-edges.ts:35-54]. There is **no `TemporalMode` (Current/AsOf/AsKnownAt/History) on the recall path** and **no maintained `current_support_facts` provider** [SOURCE: external/.../bi-temporal-model.md:46-60, 116-145; INFERENCE: negative — no such enum/provider surfaced in causal-boost.ts or contradiction-detection.ts]. So "as-of / as-known-at" temporal queries are not yet expressible internally even with the columns present.

## Candidate Proposals (NOT implementations)

### Q4 candidates
- **C4-A (S): Promote idempotency-receipts to default-on for `memory_save`/`memory_update`** behind a validated rollout, wiring the `replay`/`conflict` result into the deferred-save flow so a retried save returns the stored response instead of re-indexing [evidence: idempotency-receipts.ts:9-29, already pruned at context-server.ts:1831]. Effort **S** — module exists, off; cost is wiring + tests + flag flip.
- **C4-B (M): Add content-addressed identity for *derived* artifacts** (causal edges first, future facts/notes later) — derive a stable `derived_id = sha256(canonical-triple + source + rule_version)` mirroring aionforge `fact_id`, so async re-derivation is an idempotent no-op [evidence: consolidation.md:84-90; internal sha256 helpers already at memory-index.ts:281]. Effort **M** — leaves the sequential `memory_index.id` as the episode/row id, adds a content id only on derived rows.
- **C4-C (M/L): Generalize the existing `background` scan + `deferred` status into an explicit episode→consolidation cursor** (a coarse internal analogue of aionforge's `ConsolidationCursor` + per-episode `raw|in_progress|consolidated|failed` state) so enrichment (embeddings/BM25/causal extraction) runs on a background tick with resume-not-reprocess [evidence: memory-index.ts:293-294 `background`, 1376-1377 `complete_with_pending_vectors`; consolidation.md:16-42]. Effort **M/L** — the fast/deferred seam exists; the missing piece is a durable cursor + per-item state machine.

### Q3 candidates
- **C3-A (S): Flip `SPECKIT_TEMPORAL_EDGES` on by default** so contradiction-detection's edge-presence currentness (`invalid_at` close + conflict auto-invalidation) becomes the live retirement path, replacing flag-implicit staleness [evidence: contradiction-detection.ts:75-77, 99-110]. Effort **S** — code complete, gated OFF.
- **C3-B (M): Split the single `valid_at`/`invalid_at` pair into aionforge's four-timestamp window** (`valid_from`/`valid_to` event-time + `ingested_at`/`expired_at` transaction-time) on `causal_edges`, with a `windows_ordered` fail-closed write guard [evidence: bi-temporal-model.md:14-44 vs temporal-edges.ts:35-54]. Effort **M** — additive columns + write-path validation + backfill.
- **C3-C (M/L): Add a `TemporalMode` (Current/AsOf/AsKnownAt/History) to recall + a maintained `current-support` provider** that excludes edges with a live outgoing `supersedes`/`contradicts`, so currentness is computed by edge presence at query time rather than by post-hoc filtering [evidence: bi-temporal-model.md:46-60, 116-145]. Effort **M/L** — new query-path enum + provider; depends on C3-B for AsOf/AsKnownAt.
- **C3-D (S): Keep tombstone-sweep and temporal-close as separate concerns, documented** — temporal `invalid_at` = "not current" (recall-excluded, History-readable); `causal_edge_tombstones` = off-state forgetting (restorable hard delete). Effort **S** — primarily a contract/decision note; prevents collapsing two mechanisms that answer different questions [evidence: sweep.ts:68-100 vs temporal-edges.ts:64-80].

## Questions Answered
- **Q4 (ANSWERED):** Yes — the save path *can* split into fast episode-write + deferred consolidation, and it **already partially does** (deferred indexing + async embedding + background scan + `complete_with_pending_vectors`). Content-addressed idempotent IDs are **partially present** (`content_hash` sha256 as a dedup signal; sequential row ids as identity) and **idempotency-receipts already exist** as an aionforge-shaped module that is flag-gated OFF (`SPECKIT_MEMORY_IDEMPOTENCY`). Remaining work is promotion + a durable consolidation cursor + content-addressed identity on derived artifacts (C4-A/B/C). Save path located: `handlers/save/` + `handlers/memory-index.ts` + `lib/storage/idempotency-receipts.ts`.
- **Q3 (ANSWERED):** Edge-based bi-temporal currentness integrates *cleanly* because the substrate already does it at the causal-edge layer: `supersedes`/`contradicts` relations, `invalid_at` edge-close (currentness = edge presence, not a node flag), and conflict auto-invalidation all exist in `lib/graph/temporal-edges.ts` + `lib/graph/contradiction-detection.ts` — but are gated OFF (`isTemporalEdgesEnabled()`). Integration with off-state tombstones is a *separation-of-concerns* decision (C3-D): temporal-close = "not current" (non-destructive, History-readable), tombstone-sweep = off-state forgetting (restorable). Full parity needs the four-timestamp window (C3-B) and a `TemporalMode` + current-support provider (C3-C).
