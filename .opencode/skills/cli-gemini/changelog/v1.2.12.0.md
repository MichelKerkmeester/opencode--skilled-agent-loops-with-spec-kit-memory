This release adds the missing sk-doc section dividers to cli-gemini's reference and asset docs. Several files ran their numbered H2 sections together with no `---` separator between them, which drifts from the sk-doc reference and asset templates. The separators are in now, so the docs scan cleanly and match the rest of the skill set. No dispatch behavior changed.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/130-prompt-knowledge-layering/014-doc-divider-sweep-and-sk-prompt-cleanup/` (Level 1)

## What Changed

#### Documentation

- **Section dividers added** -- `references/cli_reference.md`, `references/integration_patterns.md`, `assets/prompt_quality_card.md` and `assets/prompt_templates.md` were missing the standalone `---` thematic break that the sk-doc templates require before each numbered H2 section. We inserted the missing blank-padded dividers before every non-first H2. The edits add only blank and `---` lines, so a content-skeleton diff against the prior version is byte-identical on every non-blank, non-divider line, verified. An adversarial per-skill audit confirmed every non-first H2 now has its immediate divider.
