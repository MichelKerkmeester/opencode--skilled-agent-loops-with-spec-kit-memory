### 3.1 CF-BM-4 / CF-BM-1: name the fix-site explicitly (frozen-file safety, load-bearing)
`score-skill-benchmark.cjs:1765-1773` is inside the **frozen** trio. The synthesis's own fix text ("the outer switch inspects... BEFORE deciding BLOCKED/CONDITIONAL/DEGRADED") is defensible in intent but ambiguous in wording — a spec author implementing it literally against the cited evidence line could add a branch to the frozen file, which directly violates `014/spec.md` REQ-001 (byte-for-byte SHA-256 preservation) and its four repeated "editing the frozen files is out of scope" statements. **Fix:** state explicitly that the new sub-verdict/degraded classification and the `BLOCKED-BY-COMPILED-DRIFT` exit-3 branch must live in `run-skill-benchmark.cjs` (confirmed non-frozen, and already has the exact precedent pattern at lines 300-310 mapping `report.verdict` values to exit codes) or in the new `compiled-routing-parity.cjs`, and that `score-skill-benchmark.cjs:1765-1773` is cited only to show the shape of the existing (insufficient) enum — never as an edit target.

### 3.2 Findings-count error
Frontmatter/provenance claims "143 raw findings deduplicate to **48** consolidated findings." Actual count: `grep -oE 'CF-[A-Z]+-[0-9]+' synthesis-v1.md | sort -u | wc -l` = **44** distinct CF-IDs across the 7 workstream tables, plus 3 standalone items in §3 "Unnamed gaps" that carry no CF-ID (catalog census undercount, 009 Phase Map drift, 015 JSONL schema gap) = **47** total distinct consolidated findings identifiable in the document, not 48. ### 3.3 Section 3 item 13 (009 Phase Map) mischaracterizes its own source finding
Synthesis text: *"009-non-hub-rollout Phase Map lists 1 of 4 children (**005-mcp-code-mode missing entirely**)."* I confirmed via `ls` that `009-non-hub-rollout/` contains exactly 4 children on disk: `001-sk-git, 002-system-code-graph, 003-system-skill-advisor, 004-system-spec-kit` — **no `005-mcp-code-mode` directory exists at all.** The raw source finding (`F-8-6`, read directly from `findings-registry.json`) says something meaningfully different and more precise: *"...the directory contains 4 children with full packet docs... and `mcp-code-mode` has no entry at all — 3 children undocumented, 1 candidate missing."* The synthesis's compression (a) invents a "005-" numbering never asserted by the source, and (b) collapses two distinct problems (3-of-4 **existing, built** children absent from the Phase Map table; a wholly separate, not-yet-onboarded 5th candidate with zero directory presence) into one parenthetical that reads as if a numbered-but-undocumented folder exists. **Fix:** restate per F-8-6's original wording; the higher-severity part (3 completed children invisible in the Phase Map) is currently underweighted.
