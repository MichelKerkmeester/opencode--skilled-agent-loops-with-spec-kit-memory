### Q3 Conflict Decisions

1. **Shadow-as-border vs ghost-card slop.** Legitimate: a tokenized transparent shadow ring replacing a decorative depth border, usually no solid border on the same element, and scoped to raised controls, cards, containers, overlays, or hover/focus lift. Slop: a `1px solid` border plus a wide shadow blur on the same card/button, especially repeated across grids. This resolves the conflict by using shadow-as-border as a replacement material, not an excuse to stack every edge treatment. [SOURCE: .opencode/skills/sk-design/design-audit/references/ai_fingerprint_tells.md:38] [SOURCE: .opencode/skills/sk-design/design-audit/references/ai_fingerprint_tells.md:42] [SOURCE: external/surfaces.md:125]
2. **Image-outline pure rgba vs tinted neutral tokens.** The image-outline rule should be an explicit optical exception. Existing foundations correctly tints neutral scales toward brand hue, but image edges are not semantic borders; tinted outlines can dirty image edges. Add this under foundations color as an exception, not as a new token scale. [SOURCE: .opencode/skills/sk-design/design-foundations/assets/token_starter.md:38] [SOURCE: external/surfaces.md:184] [SOURCE: external/surfaces.md:186]
3. **Global Review Output Format.** Rule out. The external corpus has a review table at `SKILL.md:100`, but `audit` already owns findings-first output and severity schema. Importing a second global format would conflict with mode-owned contracts and the hub's no-per-mode-logic rule. [SOURCE: external/SKILL.md:100] [SOURCE: .opencode/skills/sk-design/design-audit/references/audit_contract.md:16] [SOURCE: .opencode/skills/sk-design/SKILL.md:15]
4. **Wholesale numeric defaults.** Rule out. Keep existing sk-design values where already owned (`0.96` press scale, `initial={false}`, no `transition: all`), but do not turn corpus numbers into universal constants. Examples: 40px must not replace the existing 44x44 target; 100ms group stagger must not override current per-item stagger caps; blur is bounded and verified, not a default flourish. [SOURCE: .opencode/skills/sk-design/design-audit/references/accessibility_performance.md:42] [SOURCE: .opencode/skills/sk-design/design-motion/references/animate_presence_patterns.md:91] [SOURCE: .opencode/skills/sk-design/design-motion/assets/motion_performance_failure_card.md:42]
5. **Shared reference path.** The prompt's `.opencode/skills/sk-design/references/` path is stale in this checkout. The hub describes shared vocabulary, but the mode packets point to `../shared/register.md`, `../shared/anti_slop_principles.md`, `../shared/design_token_vocabulary.md`, and `../shared/cognitive_laws.md`. Shared edits should therefore target `shared/`, not a nonexistent `references/` directory. - **Q2 answered for adoptable techniques.** Each technique now has a precise home, anchor, coverage status, and minimal future edit.
