### FINDINGS

1. Council’s dark window is Phase 2 plus Phase 3: one CLI round may “dispatch or simulate distinct seats,” then run cross-seat critique and convergence before persistence. The reference only defines final seat contents, not per-seat writes, so a compliant executor can hold all simulated seats and critique internally until the final persist phase. `.opencode/skills/deep-loop-workflows/deep-ai-council/references/integration/loop_protocol.md:22-27`, `.opencode/skills/deep-loop-workflows/deep-ai-council/references/integration/loop_protocol.md:57-65`, `.opencode/skills/deep-loop-workflows/deep-ai-council/references/integration/loop_protocol.md:98-120`

2. Council auto mode collapses session orchestration into one command, with validation only after `round_completed`. The contract lists writes for the orchestrator, but does not require `round_started`, `seat_started`, `seat_completed`, or `critique_started` records before the round completes. `.opencode/commands/deep/assets/deep_ai-council_auto.yaml:119-131`

3. Council convergence is lightweight in the reference, but auto mode adds adjudicator-verdict stability across rounds. That can invite an executor to run an internal evaluator loop after seats and critique, especially in absorption probes, before any observable round-complete write. `.opencode/commands/deep/assets/deep_ai-council_auto.yaml:10-11`, `.opencode/commands/deep/assets/deep_ai-council_auto.yaml:102-103`, `.opencode/skills/deep-loop-workflows/deep-ai-council/references/convergence/convergence_signals.md:54-58`

4. Deep-context’s primary stall window is `step_parallel_sweep`: all seats launch, then the host barrier-joins before collection, merge, and iteration writes. If one GPT CLI seat starts output then stalls before returning structured JSON, the host can wait with no state append until the watchdog kills it. `.opencode/commands/deep/assets/deep_context_auto.yaml:408-416`, `.opencode/commands/deep/assets/deep_context_auto.yaml:496-504`, `.opencode/commands/deep/assets/deep_context_auto.yaml:536-546`

5. Deep-context does mandate artifacts, but most liveness artifacts are after completion of a seat or full sweep, not during long reasoning. The protocol says seats return only structured findings JSON and the host writes returned findings; that favors silent internal work for GPT, while Claude likely stays alive by executor habit rather than because the protocol forces progress. `.opencode/commands/deep/assets/deep_context_auto.yaml:455-461`, `.opencode/commands/deep/assets/deep_context_auto.yaml:462-470`, `.opencode/commands/deep/assets/deep_context_auto.yaml:818-820`
