This release graduates the reciprocal-rank-fusion spine out of the dark and closes the self-recommendation hole the graduation exposed. The dark-flag-graduation benchmark phase 007 measured the RRF spine and the conflict-rerank seam on a widened 42-prompt set, and the verdict was GRADUATE the RRF core paired with the conflict-rerank seam and CUT the separate self-recommendation guard. The follow-up work then documented and locked the implicit penalty that did the guard's job, and a later opus deep review found the penalty matched only the exact id by string so the `skill-advisor` alias still self-recommended to the top. The penalty value is unchanged at -0.25 and the full scorer suite stayed green at 119 of 119.

## What Changed

#### RRF fusion graduated with its regression net

- **Benchmark phase 007 (advisor-rrf-fusion)** measured the reciprocal-rank-fusion spine on a widened 42-prompt set. BEFORE: the RRF fusion spine and the conflict-rerank seam were dark (default off). AFTER: RRF top-1 went 37/42 to 38/42 with zero regressions, 0.976 agreement and deterministic output, so the RRF core graduates.
- **The conflict-rerank seam ships with the core** because the seam is RRF's regression net. It repairs a regression plain RRF introduces and corrects 1 of 5 conflict cases, so the two graduate together rather than the core landing alone.
- **The separate self-recommendation guard was CUT** as behaviorally redundant. It moved 0 of 4 top-1 because the generic explainer floor and an un-flagged audit penalty already did its job.

#### Self-recommendation penalty documented, locked, then the alias closed

- **The penalty was the sole undocumented defense.** BEFORE: cutting the explicit self-recommendation guard left an implicit `auditRecsAdvisorPenalty` as the only thing stopping the advisor recommending itself on audit and explainer prompts, and that penalty was undocumented, so a future refactor could silently remove it. AFTER: the penalty has a durable WHY comment and a regression contract test, validated two ways by zeroing the penalty and by reverting to exact-id-only.
- **The alias gap, found by opus deep review.** BEFORE: the penalty matched only the exact id `system-skill-advisor` by string, so the `skill-advisor` alias still self-recommended to the top in the default guard-off config. AFTER: the predicate uses the canonical self-recommendation id set, so BOTH `system-skill-advisor` and the `skill-advisor` alias are demoted.
- **Test reconciliation.** A pre-existing test had codified the alias bug by asserting the alias should rank top. It was reconciled to the corrected behavior, and the full scorer suite stayed green at 119 of 119.

#### Benchmark posture

- **Phase 007 verdict:** GRADUATE the RRF core with the conflict-rerank seam, CUT the self-recommendation guard.
- **Default-off was byte-identical** across the 42 prompts, and no production default was flipped by the benchmark itself.

## Upgrade

No migration required. The RRF core and conflict-rerank seam graduate as the new defaults from the phase 007 benchmark, and default-off behavior was byte-identical across the 42-prompt set so no flag flip changes existing routing. The self-recommendation penalty value is unchanged at -0.25, so adopters see the same demotion magnitude for both the canonical id and the alias. Adopt via fresh session so the daemon respawns with the graduated defaults. Verify with an `advisor_recommend` for an audit or explainer phrase (expect neither `system-skill-advisor` nor the `skill-advisor` alias at top-1) and re-run the scorer suite (expect 119 of 119).
