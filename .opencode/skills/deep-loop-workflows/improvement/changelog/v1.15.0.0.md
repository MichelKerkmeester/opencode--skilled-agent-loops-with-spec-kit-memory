# v1.15.0.0 - Lane D Kit 1.2.0 + Standards Alignment

The non-dev-ai-system onboarding kit can now reproduce the pilot's real derivation model (shared-global docs as relative symlinks, skill copies with sk-doc stripping), a scaffolder bug that made every rendered Python file crash at import was found and fixed by the first live execution of a rendered template, and the lane's code and doc surfaces follow the sk-code and sk-doc standards.

---

## 1. KIT 1.2.0 - SYMLINK + TRANSFORM DERIVE (NEW)

- `packaging_config.schema.json`: `derived_copies[].targets[]` gains three optional fields with behavior-preserving defaults - `mode: copy|symlink`, `link_target` (relative target, required for symlinks, worktree-safe by construction) and `transform: none|skill_strip`. Existing configs stay valid and render byte-identically.
- `templates/derive.py.template`: symlink branches in `derive()` (create relative link) and `check()` (`NOT A SYMLINK` / `BROKEN LINK` / `WRONG TARGET` via basename-of-realpath, worktree-tolerant), plus `_strip_for_skill` (drop TOC section and decorative emojis, keep the do/don't markers) proven regex-equal to the pilot via AST comparison.
- `templates/gauntlet.py.template`: A2/A8 now skip symlinked docs when picking a file to dirty - appending writes THROUGH the link into the shared globals and `git checkout` of a symlink path cannot restore it. Check count stays 10.
- `init_packaging.py` validates the new fields (symlink requires `link_target`, rejects `is_skill_ref`/`transform` combinations); `packaging_config.example.json` mirrors the live pilot (6 symlinked globals, 6 `skill_strip` system docs); operator guide carries the kit 1.2.0 changelog note.

## 2. SCAFFOLDER FIX - PYTHON LITERAL RENDERING

**Problem:** 16 placeholder tokens were rendered with `json.dumps`, so configs containing booleans put JSON `true`/`false` into rendered Python source. Every rendered file compiled (`true` is a valid identifier to the parser) but crashed with `NameError` at import - the latent bug survived all prior `py_compile` smoke checks and surfaced on the first-ever live execution of a rendered `derive.py`.
**Fix:** all 16 Python-context tokens render through a new `py_lit()` (repr) helper. Verified at runtime: rendered `gates.py` and `derive.py` now import cleanly, including configs with `in_rubric: false`.

## 3. PILOT PARITY PROVEN AND ADOPTED

The rendered `derive.py` was executed against the live Copywriter pilot and validated all 24 derived surfaces (12 project mirrors, 6 transformed skill copies, 6 symlinks) - byte parity, not just compile parity. The rendered file was then adopted as the pilot's live `_gates/derive.py`, closing the instance/kit gap for that file. Audit context: an instance-vs-kit review confirmed everything else stays packaging-owned by design (lock, journal, resume, family check), and nothing moves into deep-loop-runtime.

## 4. CODE-STANDARDS ALIGNMENT (KIT 1.1.1)

All nine `assets/non_dev_ai_system/templates/` aligned with the sk-code :opencode Python and shell standards: single-line imports, type hints on every function, Google-style docstrings, and WHY-comments on their own line starting with a capital (21 trailing/lowercase comments fixed). Token inventories stayed byte-identical per template, and `run.sh.template` deliberately keeps `set -uo pipefail` without `-e` so the dispatch status report still runs on failure.

## 7. SAME-DAY REVIEW REMEDIATION + FLEET HARDENING

A maximum-effort review of this release surfaced 8 P1s and a ranked optimization list; all were remediated the same day (two parallel GPT-5.5 agents, two Fable 5 agents and the operator session):

- **Template bugs that only fire for the second packaging**: gauntlet A3's same-family grader is now constructed from `{{PROPOSER_FAMILY}}` instead of a hardcoded deepseek literal, and `check_convergence` honors the runtime `--max-iters` instead of the module constant. Both fixes mirrored into the live Copywriter and Barter deals instances.
- **`loop.py` is now self-locating**: the packaging root derives from the file's own location with `CW_ROOT` as explicit override - worktree runs can no longer silently dispatch against the real packaging (template + both instances).
- **Loop efficiency (Fable)**: the held-out baseline is measured once per session and cached (only when measurable), and the first accepted promotion ends the session (`promotedStop`, canonical `converged`) - at most one kept worktree per run, no burned dispatches re-sampling unchanged HEAD.
- **Kill-switch taxonomy corrected everywhere**: hard halts (frozen-surface drift, derivation drift, candidate held-out HVR violation, grader family, lock) vs promote_reject-and-continue (floor breach, held-out regression) - loop_contract, operator_guide, catalog, grader_calibration and the template docstring all now match the code.
- **Doc-code mismatches closed**: stale "Packaging-Benchmark-Refine" frontmatter, phantom `run-packaging-refine.cjs` filename, fixture-lint guardrail claims that did not exist, the PR-001 playbook scenario (worktree subdir + mandatory CW_ROOT), the duplicated LOOP_SKIP_PROBE knob, the iteration-keyed grade-record claim, schema editing debris.
- **Scaffolder hardening**: unresolved `{{TOKENS}}` are now fatal at render, derived-copies validation requires src_relpath/relpath and value-checks `transform`, `self_score_regex` is guarded against quote/newline breakage, and the emoji-strip regex no longer eats the Unicode Arrows block (it had corrupted `->` separators in transformed copies).
- **Shared-script dedup (Fable)**: one `parse-args.cjs` now backs loop-host, the Lane D adapter and fixture-lint (verified byte-identical behavior); `extract-deliverable.cjs` gained the same ANSI stripping as its Python counterparts; the packaging-config schema encodes the symlink conditionals (`link_target` required, no `is_skill_ref`, no non-none `transform`).
- **Lane D test coverage**: 4 new vitest cases (mode validity, fail-closed, plan shape, ENV_FORWARD map) and a documented battery runner - full battery green (loop-host 25/25; the single failure is the pre-existing Lane C e2e fixture issue).
