## [**1.0.10.0**] - 2026-07-16

This update brings the packet to full inventory parity with its sibling mcp-tooling modes. It adds the feature catalog, the registered-state snapshot of the Code Mode manuals, and the mcp-servers install pointers — all derived from the packet's own existing documentation, with no new tool facts invented.

#### Feature catalog

- Need: The skill had no `feature_catalog/`, so the CLI + MCP capability surface was only discoverable by reading SKILL.md, the references, the examples, and the playbook end to end.
- Change: Added `feature_catalog/feature_catalog.md` plus 29 per-feature files across 7 domains (cli_bdg_lifecycle, protocol_discovery, dom_and_screenshot, console_and_network, mcp_parallel_instances, automation_and_performance, recovery_and_troubleshooting), mirroring the mcp-click-up catalog structure and frontmatter. The domain taxonomy mirrors the packet's own manual-testing-playbook categories.
- Why it matters: Agents and operators get a single inventory that names each capability, its command, and where the execution truth lives.

#### Code Mode manuals snapshot

- Need: The dual `chrome_devtools_1` / `chrome_devtools_2` registration existed only in the repo `.utcp_config.json`, with no packet-local record to verify against.
- Change: Added `assets/utcp_chrome_devtools_manuals.md`, a byte-true snapshot of both manual entries (extracted and verified programmatically with `jq`), with the dual-manual-for-parallelism rationale and a verify-don't-re-add provenance contract.
- Why it matters: Drift between the packet's documentation and the live registration becomes detectable, and nobody re-adds an already-registered manual.

#### mcp-servers install pointers

- Need: Sibling packets (mcp-click-up, mcp-figma) carry `mcp-servers/` pointer folders; this packet had none.
- Change: Added `mcp-servers/bdg-cli/README.md` (install pointer per INSTALL_GUIDE.md — nothing vendored) and `mcp-servers/chrome-devtools-mcp/README.md` (the Code Mode server behind the manuals).
- Why it matters: The packet now matches the hub-wide embedded-server layout, so install and verification questions have one predictable home.
