This release fixes the long-standing problem where AI agents working on Webflow sites pulled JavaScript rules into CSS tasks (and vice versa), then produced code with the wrong comment style, naming, and banner formatting. The root cause was a single monolithic standards folder mixing three languages in five files. This release replaces that folder with separate per-language references that mirror the OpenCode side of the skill, restructures the routing layer to match the proven Barter sk-code pattern, and adds five copy-paste templates so authors can start from production-quality scaffolding instead of inventing one. An empirical probe with GPT 5.5 confirms the fix works: generated CSS plus JavaScript scored 12 out of 12 on the JavaScript checklist and 6 out of 6 on the CSS checklist under the new structure.

> Spec folder: `specs/skilled-agent-orchestration/099-webflow-per-language-style-guides` (Level 3, complete, strict-validate PASSED)

## Architecture

The Webflow reference tree was rebuilt from the ground up so that JavaScript, CSS, and HTML each have their own home. The routing layer that decides which references to load was simultaneously consolidated to match how the Barter sk-code fork already works. Both changes are structural and breaking for any external consumer that hardcoded paths under the old layout.

#### Per-language reference tree

The previous layout put all coding rules into five files under `references/webflow/standards/` (3,831 lines that mixed JavaScript naming with CSS animation rules with HTML data attributes). When a model loaded the skill for a CSS-only task, it pulled in all the JavaScript rules too, which led to wrong choices like applying snake_case to CSS class names or applying CSS animation property restrictions to JavaScript callbacks. The fix splits the tree into per-language folders that mirror the OpenCode side of the skill: `webflow/javascript/` carries `style_guide.md`, `quality_standards.md`, `quick_reference.md`. `webflow/css/` carries the same trio plus a fourth file `patterns.md` (the 1,263-line Webflow CSS deep-dive that was previously misfiled under `implementation/`). `webflow/html/` carries `style_guide.md` and `quality_standards.md`. A new `webflow/shared/` folder holds the three rules that genuinely apply to all languages (file naming, comment principles, file-header banner format) plus the cross-language enforcement workflow and dev-tools patterns. The legacy `webflow/standards/` folder is physically deleted because git history is the authoritative archaeology surface for the old layout.

&nbsp;

#### Routing layer mirrors the Barter sk-code pattern

The previous routing layer scattered detection, intent scoring, resource loading, and phase lifecycle across four files inside a `references/router/` subfolder. The Barter sk-code fork (the production sibling that ships in the Barter monorepo) uses a flatter pattern with three files at the references root: `stack_detection.md`, `smart_routing.md`, `phase_detection.md`. This release adopts that pattern. `router/code_surface_detection.md` is now `references/stack_detection.md`. `router/phase_lifecycle.md` is now `references/phase_detection.md`. The intent classification and resource loading content from `router/intent_classification.md` and `router/resource_loading.md` is consolidated into a single new `references/smart_routing.md` that contains the authoritative routing pseudocode in one place. SKILL.md section 2 SMART ROUTING gains explicit ANCHOR markers and a new Phase Detection visual that matches the Barter format. All 24 cross-references across the skill that pointed at the old `router/` paths are updated.

&nbsp;

#### Comment style grounded in production code

The previous JavaScript and CSS commenting sections invented rules that did not match production practice. The "max 5 comments per 10 lines" rule was wrong (real Webflow JavaScript averages 1 comment per 5 to 10 lines). The JavaScript section-header closing format `──── */` (with a space before the closing) was wrong (production uses `────*/` glued together). The function-preamble pattern, group-introductory comments, silent-catch shorthand `catch (_) {}`, and underscore-prefix for internal state were all missing entirely. This release rewrites the JavaScript style guide section 5 with nine subsections grounded in actual code from `anobel.com/src/3_staging/`. The CSS style guide gains a new section 7 covering file-header banners (correct format with space before closing), section-header sub-numbering (`1.1` for fine-grained sub-states), multi-line section descriptions for non-obvious sections, and CSS-specific comment density (sparser than JavaScript: file header always, section headers always, sub-section headers sometimes, above complex selector groups when warranted). The HTML style guide gains a new section 7 covering Webflow Site Settings custom code panels (Head and Footer): file-level markers, three-line section headers with five-space indent (HTML uses extra padding because `<!--` is wider), parenthetical-context titles, Title-cased inline labels above each script or link tag, end-markers for multi-line script blocks, and the convention that sub-sections use letter suffixes (`4b.`) rather than decimal sub-numbering.

&nbsp;

#### CSS quality standards expanded with seven deep-dive sections

The CSS quality standards file was 327 lines compared to the JavaScript equivalent at 1,166 lines. The asymmetry left several common CSS pain points uncovered. This release expands the CSS quality standards to 763 lines with seven new sections. Fluid typography deep-dive shows the per-breakpoint variable-override pattern that scales type from mobile through ultra-wide displays without the brittleness of `clamp()`. Autofill override patterns cover the `:-webkit-autofill` quartet (default plus hover plus focus plus active) with the `box-shadow inset` trick and `5000s transition` that paint over Chrome's yellow autofill background. Color-mix state interpolation shows how to derive hover and active tints from a single brand color token using `color-mix(in srgb, var(--color-brand) 90%, white)`. Content-visibility optimization shows the `auto` value paired with `contain-intrinsic-size` for skipping render work on below-fold sections. The `:focus-visible` versus `:focus` discipline section covers the migration path from the legacy `body.using-keyboard` JavaScript-driven pattern to the modern browser-native heuristic. The `:has()` parent-state patterns section shows how to eliminate JavaScript class toggles for form-field validation, card layouts, and section visibility. The print stylesheet patterns section covers the `@media print` reset (hide nav and modals, force white background, show full URLs after links) and modern `@page` rules.

## New Features

#### Five copy-paste templates

Authors no longer need to invent component scaffolding from scratch. The new `assets/webflow/templates/` directory contains five production-style annotated templates. `component_template.js` is an 8.9 KB JavaScript template implementing every convention in the new style guide (file header, numbered sections, IIFE wrapper, INIT_FLAG guard, snake_case identifiers, function preambles, cleanup hook, optional public API). `component_template.css` is a 7.1 KB CSS template (file header, numbered sections, BEM naming, custom properties with semantic prefixes, attribute selectors with the case-insensitivity flag, GPU-only animation properties, prefers-reduced-motion override, sub-numbered breakpoint sections). `embed_template.html` is a 3.9 KB Webflow custom embed scaffolding template (data-component root, data-action and data-target for JavaScript event delegation, ARIA initial state matching the visual state, JavaScript handoff notes at the bottom). `head_footer_code_template.html` is a 9.4 KB template for the Webflow Site Settings custom code panels covering all eight standard Head sections (analytics, security meta tags, connection hints, preloads, speculative prefetch, JavaScript detection, module scripts, schema markup) and the two standard Footer sections (third-party libraries, custom scripts). `form_scaffold_template.html` is an 8.9 KB form template with `data-form-field` and `data-form-input` validation hooks, ARIA attributes, custom select bridge pattern, and a complete JavaScript handoff notes block at the bottom.

&nbsp;

#### Webflow staging vs production deployment workflow

A new file at `references/webflow/deployment/webflow_staging_production.md` (260 lines) covers the staging-to-production promotion workflow that was previously missing from the deployment folder. It documents Webflow's built-in `.webflow.io` staging subdomain (and the `noindex` headers gotcha that gives false-negative SEO scores on staging), the CSS export workflow for source-controlled mirror projects like A. Nobel, the externally-hosted JavaScript bundle workflow with version-pin discipline (the `?v=1.4.0` query string serves three purposes: cache-busting, promotion gating, rollback hook), schema markup and structured-data sync considerations, CDN cache invalidation behavior on R2 versus jsDelivr, and a full pre-promote checklist.

&nbsp;

#### Webflow HTML quality standards

A new file at `references/webflow/html/quality_standards.md` (354 lines) gives HTML the same quality-standards companion that JavaScript and CSS already had. The previous gap meant data-attribute hygiene rules, ARIA state-sync enforcement, custom embed validation, and head/footer panel structure validation lived nowhere or were scattered across other docs. The new file consolidates all of these into one enforcement reference.

## Bug Fixes

A multi-AI council audit (three lenses: maintainability, discoverability, symmetry) over the post-refactor tree surfaced 14 alignment findings. All 14 are resolved in this release.

#### Restored a quick-reference file lost during refactor

A python regex pass during the refactor accidentally deleted `webflow/javascript/quick_reference.md` from disk. The file was untracked at the time of the recovery so `git checkout HEAD --` did not restore it. Four other files were already cross-referencing the missing file. This release restores `webflow/javascript/quick_reference.md` (294 lines covering pre-commit checklist, code snippets like `waitForElement` and `waitForLibrary`, eleven one-liners for MutationObserver, IntersectionObserver, Lenis scrollTo, Motion.dev animate and inView, custom event dispatch, optional chaining, and form validation classes).

&nbsp;

#### Fixed stale references the bulk sweep missed

The Phase 3 bulk sed pass missed six references in `webflow/shared/enforcement.md` because the sweep regex did not match the bare-filename references like `[code_quality_standards.md](code_quality_standards.md)` (no path prefix). This release replaces all six with the correct per-language tree paths and rewrites the surrounding "Related Documents" block as a five-link tree map.

&nbsp;

#### Fixed broken relative paths in moved files

When `implementation/css_patterns.md` moved to `css/patterns.md`, three internal links continued pointing at the old `./` (same directory) path that no longer resolved (`./animation_workflows.md`, `./webflow_patterns.md` are in `../implementation/`, not the new `css/` location). Three more broken paths in `shared/dev_workflow.md` section 9 pointed at sibling files that actually live in `../implementation/` and `../debugging/`. This release fixes all six paths with the correct relative prefixes.

&nbsp;

#### Added inbound references to orphan deep-dive

The 1,263-line `css/patterns.md` had zero inbound references after the move (orphan: a new contributor would never discover it through the reference tree). This release adds inbound links from `css/style_guide.md`, `css/quality_standards.md`, `implementation/webflow_patterns.md`, and `implementation/animation_workflows.md`.

&nbsp;

#### Added bidirectional cross-references between performance docs

The umbrella performance checklist at `implementation/performance_patterns.md` and the five topical deep-dives under `webflow/performance/` (Core Web Vitals remediation, resource loading, third-party scripts, interaction-gated loading, Webflow platform constraints) had zero cross-references between them despite massive topical overlap. This release adds bidirectional links so readers can navigate from any topical deep-dive to the umbrella and from the umbrella to any deep-dive in one hop.

&nbsp;

#### Fixed display-text mismatch in three workflow files

The display text in three sibling workflow files showed `[shared_patterns.md]` while the actual link target was `../shared/dev_workflow.md` (the file was renamed during refactor but the display text was not updated). This release matches the display text to the actual filename in all three files.

&nbsp;

#### Promoted CSS to first-class status in the implementation layer

Eight of the nine implementation reference files used to reference only `../javascript/quality_standards.md`, leaving CSS as an unintentional second-class citizen. This release adds parallel `../css/quality_standards.md` references to five implementation files where CSS overlap exists (animation, webflow patterns, form upload, focus management, Swiper).
