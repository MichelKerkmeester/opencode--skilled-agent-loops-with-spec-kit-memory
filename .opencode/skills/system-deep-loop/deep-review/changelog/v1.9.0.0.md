This release re-baselines the `deep-review` skill against the current sk-doc templates and Human Voice Rules before the broader 116-deep-skill-evolution arc begins. The work ran in three phases inside packet `131/000/003-deep-review`: a Level-3 spec packet was authored end-to-end, a surgical audit produced 23 schema-validated findings, and every actionable finding was either resolved or formally closed with rationale (zero deferrals shipped). The README rewrite that closes the release calibrates tone to roughly 70 percent of the root `Public/README.md` voice and mirrors the `system-spec-kit/README.md` 9-section pattern so operators can read it like every other skill README in the workspace.

## Documentation Surface

The README, SKILL.md, and four reference files now align with the current sk-doc templates and pass HVR cleanly. No behavioral code shipped in this release. The reducer (`scripts/reduce-state.cjs`) was bug-scan only per the surgical-edit policy.

#### README rewrite to current tone and structure

The previous README ran 452 lines in a mixed technical voice that pre-dated the recent shift toward marketing-leaning HVR copy. The rewrite covers every unique feature from the catalog (21 entries across loop lifecycle, state management, review dimensions, and severity system), keeps every cross-system link named with an explicit target path, and scores 100 of 100 against the banned-word and punctuation rules. The structural pattern matches the `system-spec-kit/README.md` 9-section layout so a reader who has seen one skill README can navigate this one without re-learning the conventions.

&nbsp;

#### SKILL.md restructure under surgical-edit policy

The runtime SKILL.md kept its Smart Router intact and received targeted moves that pull implementation-detail subsections out of the activation section. The Executor Selection Contract and the Code-Graph Readiness TrustState surface (which both document how the skill works, not when to invoke it) moved from section 1 to section 3. The two REFERENCES sections that had accumulated over time merged into one. A packet-citation parenthetical in a section header was removed in favor of content-only naming, and a deprecated planning placeholder was dropped entirely. The version frontmatter now matches the changelog stream at 1.9.0.0 (previously stuck at 1.4.0.0 despite changelog entries reaching 1.8.0.0).

&nbsp;

#### References cleanup

The four reference files (`convergence.md`, `loop_protocol.md`, `state_format.md`, `quick_reference.md`) gained the `### When to Use` subsection that the sk-doc reference template prescribes, scrubbed 33 em-dashes that violated HVR, removed a meta-commentary line about earlier doc drafts, and pulled the convergence threshold semantics block out of the SKILL.md preamble into its rightful home in `references/convergence.md`. The Smart Router section in SKILL.md was preserved exactly as-shipped per the preservation default.

&nbsp;

#### Assets cleanup

Two markdown assets and one prompt template scrubbed 19 em-dashes between them. The asset dashboard intro was condensed from three sentences to two per the asset template. The `prompt_pack_iteration.md.tmpl` was left structurally unchanged after verifying that `prompt-pack.ts` consumes it as a raw template via `readFileSync`. Adding YAML frontmatter, condensing the intro, or inserting horizontal rules would inject metadata text into the rendered agent prompt, so the template-conformance rules under `skill_asset_template.md` do not apply to that file.

&nbsp;

#### Feature catalog index repair

The root `feature_catalog/feature_catalog.md` was missing one of the six loop-lifecycle entries, leaving `06-resource-map-emission.md` orphaned on disk but invisible from the root catalog. The rewrite adds the Resource Map Emission entry in the correct position and links it to the snippet file. Total entries now match disk reality at 21 (1 root + 20 per-feature snippets).

&nbsp;

#### Manual testing playbook index repair and DRV filename collisions

The root playbook claimed 32 scenarios across 6 categories but actually carried 45 files across 8 categories. The two stress-test categories (command-flow stress tests CP-052 through CP-057 and the review-depth v2 rollout DRV-058 through DRV-063) had no body sections and were not listed in the index. The rewrite adds section 15 and section 16 with per-scenario summaries, updates the table of contents and the section 14 index, and corrects the overview count. Filename collisions in the convergence-and-recovery directory (where files 015 and 021 through 023 collided with files in sibling directories under the same numbers but mapping to different DRV IDs) were renamed to match their DRV IDs (030, 032 through 034). Fourteen cross-references in other files that pointed to the old filenames were patched in the same commit.

&nbsp;

#### Cross-system path migration to deep-loop-runtime

When the 118-FULL_ISOLATE_NO_MCP arc moved deep-loop code out of `system-spec-kit/mcp_server/` into the peer `deep-loop-runtime/` skill, deep-review's docs did not get the memo. Twelve current-doc references across the SKILL, references, and playbook tests migrated to their new homes under `deep-loop-runtime/lib/deep-loop/`, `deep-loop-runtime/lib/coverage-graph/`, and `deep-loop-runtime/tests/integration/`. The DRV-015 playbook scenario, which previously pointed at an archived test, now verifies the current `LoopType='review'` contract in `coverage-graph-db.ts`. Two stale `commands/speckit/deep-review.md` paths in stress tests were updated to the current `commands/deep/review.md`. A `sample-040-corpus.json` reference that suffered an extension typo was fixed to `.jsonl`. Seven historical changelog references (in v1.0.0.0 through v1.3.1.0) were preserved as-is to keep the chronological record accurate for the paths that existed at the time those changelogs were authored.
