This release lands the code-graph portion of the 028 dark-flag graduation arc. Two flags graduated on benchmark evidence (the staleness-repair reverse-dependency degree cap and the bitemporal edge writer) and the closed-vocabulary edge governance was cut. The bitemporal writer is the headline: a pre-graduation validation found it built but unwired, the first wiring was then caught broken by a 10-iteration opus deep review with three P0 defects, and the corrected wiring landed an as-of query surface that finally gives the preserved edge history a production reader. Everything except the degree cap lives behind `SPECKIT_CODE_GRAPH_EDGE_BITEMPORAL_READS` (default OFF, byte-identical when off). The degree cap reads only inside the force-parse branch.

## New Features

#### The staleness-repair degree cap defaults to 15

BEFORE: the staleness-repair reverse-dependency degree cap defaulted to uncapped (0), so a rename of a high-fan-in dependency reparsed every importer. AFTER: a degree-sweep cost benchmark drove the real scan handler across importer counts 5 to 25 and found rebind cost is linear with no knee (about 0.3 to 0.8ms per importer over a 17ms scan floor), so the default is set to 15 on that evidence. That fully repairs the common-to-moderate refactor at negligible cost while bounding the unmeasured 100-plus-importer tail. The hot-hub cost is recorded: above the cap a high-fan-in rename leaves importer edges stale until each is next edited. This cap reads only inside the force-parse branch, so it is not gated by the bitemporal flag.

&nbsp;

#### The bitemporal edge writer, wired and then caught

BEFORE: a pre-graduation validation (deep review plus research) found the bitemporal close-and-insert writer was built but NOT wired into the reindex path, so it was demoted to refine. The follow-up wired it into `replaceEdges`. AFTER: a 10-iteration opus deep review caught that the first wiring was broken with three P0 defects, because the cli test pass was green only because the existing tests never exercised the real failure modes.

The three P0 defects:

1. The live edge readers `queryEdgesFrom` and `queryEdgesTo` had no `invalid_at` filter, so a flag-on reindex returned both the closed and the open edge and corrupted every live read.
2. The production reindex prune `pruneDanglingEdges` hard-deleted dangling edges regardless of the flag.
3. `replaceNodes` hard-deleted edges before the close path could run, so nothing was left to close (`asOfEdgesFrom` returned empty).

The fix: `replaceNodes` and `pruneDanglingEdges` now close-not-delete under the flag, the live readers filter `invalid_at IS NULL`, a next-generation stamp fixes an off-by-one and a zero-width-lifetime, and the integration test was rewritten to drive the real `code_graph_scan` twice under production ordering. A second opus re-review confirmed PASS.

&nbsp;

#### The as-of query surface gives the history a reader

BEFORE: even once the writer worked, `asOfEdgesFrom` had no production caller, so the preserved history was dead. AFTER: `code_graph_query` takes an optional `asOf` generation parameter that routes edge reads through `asOfEdgesFrom` and a new `asOfEdgesTo` inbound mirror, skipping the dangling-edge exclusion because a historical edge legitimately points at a now-deleted node. An as-of query at a past generation returns the old target while the default query returns the new.

Two residual as-of-window-integrity fixes landed alongside it. The ensure-ready auto-index path now bumps the generation so windows are non-zero-width, and `recordSupersedesLineage` stamps `valid_at` so lineage edges are as-of readable.

&nbsp;

#### Closed-vocabulary edge governance cut

The same graduation phase that promoted the degree cap and the bitemporal writer cut the closed-vocabulary edge governance, removing that constraint from the edge surface.

## Backward Compatibility

Fully backward-compatible with the flag off. `SPECKIT_CODE_GRAPH_EDGE_BITEMPORAL_READS` defaults OFF and is byte-identical when off, so the writer, the live-reader filtering, the close-not-delete paths, and the as-of query surface are inert for operators who never set it. The degree cap is the one always-on change: it reads only inside the force-parse branch and trades an unbounded reparse for a benchmark-bounded one at 15 importers.

---

## Upgrade

No migration required. The degree cap applies automatically with a default of 15. To preserve and read edge history, set `SPECKIT_CODE_GRAPH_EDGE_BITEMPORAL_READS` and pass an `asOf` generation to `code_graph_query` for historical reads. The flag and the degree-cap default are documented in system-spec-kit's `mcp-server/ENV-REFERENCE.md`.
