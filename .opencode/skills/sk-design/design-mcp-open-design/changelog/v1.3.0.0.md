# v1.3.0.0 - sk-design-interface is mandatory for all design work

**Released:** 2026-06-16

The coupling to `sk-design-interface` is upgraded from a conditional best-effort ("apply it when a read or run feeds a design decision") to an absolute hard precondition. Any UI/design work through Open Design — every generation/`start_run`, and every read that feeds a design decision (grounding in a system, reusing its tokens/components) — MUST load `sk-design-interface` and run its ground → token-system → critique first, and shape the brief and discovery-form answers with that judgment. An interface can never be produced or shaped from Open Design without it. Pure transport (wiring the MCP server, bare project listing that feeds no design decision) stays exempt. No change to the `od` CLI contract, the tool surface, the gating taxonomy, or the daemon model.

## What Changed

### SKILL.md — the coupling is now a hard block at every layer

- A **MANDATORY PAIRING** banner is added before Section 1, stating the hard precondition and the transport-vs-taste framing, with the pure-transport exemption.
- Section 2 (SMART ROUTING) gains a phase-detection **HARD GATE** (STEP 2) before any RUN or design-feeding READ, the two design-work resource rows are marked **⛔ MANDATORY**, and the router pseudocode adds a `design_gate` precondition that blocks a design step composed without `sk-design-interface`.
- Section 3 (Run Direction) adds a **MANDATORY before turn 1** step: load `sk-design-interface` and shape the `start_run` brief and every `od ui respond` answer with it.
- Section 4 (RULES) rewrites ALWAYS #5 as a hard precondition and adds NEVER #6: never produce or shape UI from Open Design without `sk-design-interface`.
- Section 6 (SUCCESS CRITERIA) adds an Always gate: a design step is not complete without `sk-design-interface` evidence.
- Sections 7 and 8 reword the relationship to a **MANDATORY partner for all design work**.
- Frontmatter `version` 1.2.0.0 → 1.3.0.0 and the `description` now states the mandatory pairing.

### README — the mandate is reflected

- A MANDATORY callout is added under the lead, and the Design Grounding, Related Skills, and FAQ sections are reworded from "applied whenever ... feeds a design decision" to a hard precondition with the pure-transport exemption.

## Notes

- The asymmetry is intentional and unchanged: `sk-design-interface` does NOT require Open Design (Open Design is an optional grounding source there), but Open Design design-work requires `sk-design-interface`. This release does not edit `sk-design-interface`.
- Behaviour for pure transport (WIRE / bare inventory) is unchanged — those operations make no design decision and are explicitly exempt.
- Historical changelog entries are preserved.
