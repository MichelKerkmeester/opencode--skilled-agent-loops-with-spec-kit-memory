This release narrows the design-generation prompt pattern to a single tool. The `mcp-magicpath` skill is removed, so `references/design-generation-patterns.md` no longer documents MagicPath canvas authoring and the `DESIGN_GEN` router no longer keys on "magicpath". The Open Design `start_run` usecase, the grounded anti-default brief, the String Seed of Thought variation technique, and the discovery-form pre-answer are unchanged.

---

## What Changed

#### Changed

- **`references/design-generation-patterns.md`** -- drops the MagicPath canvas-authoring usecase. The reference now covers the `mcp-open-design` `start_run` usecase only: the grounded anti-default brief, the seed-of-thought variation technique, pre-answering the multi-turn discovery form, and the `sk-code` handoff. The MagicPath-parallel front-loading note is removed, and the design-system-ground row drops the `get-theme` mechanism while keeping `od tools design-systems read`.
- **Smart router `DESIGN_GEN` intent** -- `SKILL.md` removes the "magicpath" keyword from the `INTENT_MODEL` `DESIGN_GEN` entry. The remaining signals (open design, start_run, design generation, generate ui, canvas, design brief, variations) still route design-generation requests. The design-generation use case, the resource list, and the §9 references re-point to open-design only.
