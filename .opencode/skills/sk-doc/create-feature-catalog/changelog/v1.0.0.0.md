# v1.0.0.0 - Initial Release

Initial release of `create-feature-catalog`, one of the ten workflow packets in the `sk-doc` parent hub's workflow-only architecture. The packet authors canonical feature-inventory packages: a root `feature-catalog/feature-catalog.md` catalog organized into numbered category folders, with one source-anchored per-feature file per root entry, so a system's shipped behavior has a single reviewable current-state reference.

## What's Included

- `SKILL.md` — the packet's inline workflow contract: when to create a catalog versus keep a README summary, the canonical package shape (`feature_catalog.md` + `NN--category-name/` folders + per-feature files), the 14-step creation workflow, root-catalog and per-feature-file content requirements, the catalog/playbook boundary, and the ALWAYS/NEVER/ESCALATE rules.
- `references/feature_catalog_creation.md` — the deep-dive creation reference for overflow guidance beyond the inline workflow.
- `assets/feature-catalog/feature-catalog-template.md` — the scaffold for the root `feature-catalog/feature-catalog.md`.
- `assets/feature-catalog/feature-catalog-snippet-template.md` — the scaffold for each per-feature reference file.

This packet has no packet-local `scripts/` or `graph-metadata.json`; validation and advisor identity are consumed from the shared `sk-doc` backbone.
