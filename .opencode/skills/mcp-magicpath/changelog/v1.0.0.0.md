# v1.0.0.0 - Initial Release

**Released:** 2026-06-13

Initial release of the `mcp-magicpath` skill. It vendors the MagicPath agent skill into the framework and gives an agent a clean contract for the `magicpath-ai` CLI: find, preview, inspect, and install MagicPath UI components into an app, or author and edit components on the canvas. MagicPath has no MCP server, so the CLI is the only interface and the `mcp-` prefix is the framework's external-tool family bucket.

---

## What's New

### Core Skill

- `SKILL.md` - lean house-template runtime instructions under the 5k-word cap
  - Section 1: WHEN TO USE (activation triggers, install vs author use cases, anti-patterns)
  - Section 2: SMART ROUTING (direction detection, phase flow, resource loading levels, guarded router pseudocode)
  - Section 3: HOW IT WORKS (first-step auth, install direction, author direction, repo import, embedded browser)
  - Section 4: RULES (ALWAYS / NEVER / ESCALATE IF)
  - Sections 5-8: references, success criteria, integration points, related resources

### References

- `references/magicpath_operations.md` - full operational playbook relocated verbatim from the upstream guide (discover, understand, install, integrate, themes, canvas authoring, Design Defaults)
- `references/cli_reference.md` - complete `magicpath-ai` command reference with flags, JSON shapes, and the `code` subcommands
- `references/working_with_repositories.md` - recreate UI from a local path or Git repo onto the canvas
- `references/working_with_embedded_browsers.md` - keep a MagicPath project open as a persistent canvas in hosts with an embedded browser
- All four conform to the sk-doc reference template (frontmatter, OVERVIEW, numbered sections)

### Installation

- `scripts/install.sh` - global installer (`npm install -g magicpath-ai`)
  - Checks Node 16+ and npm, verifies the version, prints login steps, idempotent on re-run
  - `--check-only` and `--force` flags
- `mcp-servers/magicpath-cli/` - in-skill CLI vendor (`package.json` + `setup.sh`), mirroring the `mcp-click-up/mcp-servers/` pattern
- `INSTALL_GUIDE.md` - phased install guide with validation checkpoints
  - AI-first prompt, three install paths, browser authentication, end-to-end verification, troubleshooting

### Discovery and Routing

- `graph-metadata.json` - schema-2 advisor node with intent signals, an `enhances` edge to `sk-code`, and a `related_to` link to `mcp-chrome-devtools`
- Registered in the skills catalog under External Tool Surfaces
- Advisor routes a MagicPath prompt to `mcp-magicpath` as the top match, passing Gate 2

## Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Skill name | `mcp-magicpath` | Keeps the framework family-prefix convention; operator chose the prefix over upstream parity |
| Upstream body | Relocated to `references/magicpath_operations.md` | Upstream SKILL.md was 7.2k words with no house sections; relocation preserves it while SKILL.md stays conformant |
| Reference naming | snake_case | Matches the sk-doc convention used by `mcp-click-up` |
| CLI availability | Global installer, in-skill vendor, and npx | Covers global use, vendored use, and zero-install use |
| Transport | CLI only (no MCP server) | MagicPath ships no MCP server; the `mcp-` prefix is organizational |

## Known Limitations

- Authentication is required before any operation (`magicpath-ai login`). There is no offline mode.
- The `mcp-` prefix is organizational, not literal. MagicPath has no MCP server.
- `add` is React and TypeScript only. For other languages, use `inspect` and translate the source by hand.
