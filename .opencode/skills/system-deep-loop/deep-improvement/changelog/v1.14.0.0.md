# v1.14.0.0 - Lane D Rename + Onboarding Kit

Lane D is now **non-dev-ai-system** (mode `non-dev-ai-system-refine`, command `/deep:ai-system-improvement`), and it ships the machinery to onboard a new AI-system packaging without copy-editing a sibling.

---

## 1. RENAMES

- `assets/{agent_improvement, model_benchmark, skill_benchmark}` and `references/{agent_improvement, model_benchmark, skill_benchmark, non_dev_ai_system}` use snake_case. All live path references across skills, commands and agents were rewritten. Spec-packet history was left untouched.
- Lane D surfaces renamed end to end: `scripts/non-dev-ai-system/run-non-dev-ai-system.cjs`, catalog `06--non-dev-ai-system`, playbook `11--non-dev-ai-system`. The old mode id warns and falls back per the documented unknown-mode contract.

## 2. ONBOARDING KIT (NEW)

- `assets/non_dev_ai_system/packaging_config.schema.json` + `packaging_config.example.json`: every packaging-specific value (dimensions, floors, maxes, frozen-surface anchors, technique-doc map, fixtures, models, lexicon) in one config.
- `assets/non_dev_ai_system/templates/`: nine parameterized templates derived verbatim from the pilot implementations, guardrail fixes included.
- `scripts/non-dev-ai-system/init_packaging.py`: stdlib-only scaffolder that validates the config, renders `_loop/`, `_gates/` and `benchmark/` into a packaging root and prints the conformance checklist. Motivated by a real hand-port defect that silently disabled floor enforcement for two dimensions.

## 3. LANE REFERENCES (NEW)

`references/non_dev_ai_system/`: `loop_contract.md` (the formal `_loop/loop.py` contract), `guardrails_teachings.md` (the twelve pilot teachings and where each guardrail lives), `fixture_authoring.md` and `grader_calibration.md`, plus the operator guide's new scaffolder onboarding section.

## 4. VALIDATION

Battery 243/243. Lane D dry-run E2E green through the renamed mode. Rendered templates compile and match the pilot constants.
