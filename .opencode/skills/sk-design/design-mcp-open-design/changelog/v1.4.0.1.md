# v1.4.0.1 - Verb surface table: add missing od CLI commands

**Released:** 2026-06-21

The `od` CLI verb surface table now covers eight commands discovered from live usage that were missing or under-documented. All new entries were verified against `--help` output, and the gating policy in `tool_surface.md` was updated with their read-only/mutating classifications.

---

## What Changed

### Verb surface table (od_cli_reference.md)

- Added `od daemon start` (mutating, starts daemon headless or with web UI)
- Added `od project create` (mutating, with `--skill`, `--design-system`, `--json` flags)
- Added `od files list` / `od files read` (read-only); also added `od files write` (mutating)
- Added `od skills list` (read-only)
- Added `od design-systems list` (read-only)
- Added `od daemon status` (read-only; `od status` is a subcommand, not standalone)
- Promoted `od doctor` from `[UNVERIFIED]` to confirmed read-only
- Documented inline form-answer pattern: discovery form answers can be submitted as a follow-up `--message` on the same `--conversation` in `od run start`

### Gating policy (tool_surface.md)

- Added `od daemon start` and `od project create` to the "Surface but GATE" list
- Added `od files write` to the "Surface but GATE" list
- Added `od skills list`, `od design-systems list`, `od files list`, `od files read`, `od doctor`, and `od daemon status` to the "Surface freely" read-only list
- Removed the `[UNVERIFIED]` annotation on `od doctor`

---

## Notes

- All eight new verb entries were confirmed against live `node "$OD_BIN" <verb> --help` output on macOS with Open Design desktop app installed
- `od status` was corrected to `od daemon status` during verification (the standalone `od status` does not exist)
- The mandatory `sk-design-interface` pairing, the multi-turn generation flow, and the socket/discovery model are unchanged
