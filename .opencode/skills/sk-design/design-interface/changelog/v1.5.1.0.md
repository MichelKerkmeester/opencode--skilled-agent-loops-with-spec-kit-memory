# v1.5.1.0 - Aesthetics reconciled as grounding cues, tool contract tightened

**Released:** 2026-06-25

A deep-review remediation pass. The headline fix reconciles `references/aesthetics/` with the skill's own no-preset contract: the four files are no longer framed as a pick-a-vibe preset chooser, they are illustrative grounding cues for naming a realized default to critique against. The skill's tool-access contract is also brought in line with what a judgment-only skill actually uses and with the Code Mode routing it documents.

---

## What Changed

### Aesthetics reframed from preset chooser to grounding cues

`references/aesthetics/` previously presented brutalist/minimalist/soft/apple-bento as selectable presets ("pick a design preset", "preset catalog", "use one preset as the dominant aesthetic per direction"), which contradicted the skill's hard rule in `real_ui_loop.md` Section 8 (no style presets, no pick-a-vibe menu, no named aesthetic dials).

- The corpus is kept, not deleted. Each file is recast as an illustrative cue for the realized look it names, to be cited once at the critique step as the default to deviate from, subordinate to grounding and the anti-default critique.
- Chooser framing removed: the index is no longer a "preset catalog", the "pick a design preset" / "design preset catalog" / "visual direction presets" trigger phrases are gone, and the per-file "use one preset as the dominant aesthetic" language is replaced with critique-against framing.
- `real_ui_loop.md` Section 8 gained a one-line reconciliation clarifying the cues are reference material for grounding, never a chooser or pick-a-vibe axis.
- The cues are wired into the SKILL.md Section 2 routing table (ON_DEMAND, critique-against only), Section 5 references, and `graph-metadata.json` `key_files`, so they are no longer orphaned, but stay clearly subordinate.

### Tool-access contract aligned with documented routing

- `allowed-tools` narrowed from `[Read, Write, Edit, Bash, Grep, Glob]` to `[Read, Grep, Glob, Task]`, matching the sibling design-judgment skills and the skill's own "owns the look, not the build" boundary. Implementation is delegated to `sk-code` and screenshots to `mcp-chrome-devtools`.
- The optional Mobbin/Refero real-world reference lookups are Code Mode (UTCP) manuals, not in-skill tools. SKILL.md Section 2, ALWAYS rule 7, and README now state the skill co-loads `mcp-code-mode` and routes the lookup through it, falling back to the generic process when Code Mode is unavailable.

### Doc hygiene

- SMART ROUTING detection heuristic reworded to test for a pinned value rather than a mere axis mention, so "pick a palette" is correctly read as free.
- README Section 8 skill-structure verification command now uses `--check` (validation only) instead of defaulting to building a zip.
- README Section 9 related-documents table now lists `ux_quality_reference.md` and `design_inventory.md`.
- `feature_catalog.md` documents the intentionally reserved `06--` category gap (kept aligned with the playbook topic numbering, where `06` is licensing-and-provenance).
- `graph-metadata.json` `last_updated_at` and `causal_summary` refreshed; `key_files` now includes the aesthetics index.
- SKILL.md and README package version bumped to 1.5.1.0.

## Notes

- The anti-default guardrails are unchanged in substance: one reference only, read live, never copied, never a chooser, grounding stays upstream and non-negotiable, and a pinned brief always wins.
- The vendored `design_principles.md` Apache-2.0 content is untouched.
