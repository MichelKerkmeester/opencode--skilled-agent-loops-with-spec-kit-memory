# Deep Review — sk-design-md-generator (phases 002-005)

**Scope:** the session's phase 002-005 work (extraction fixes, doc/prompt fixes, validator semantic checks, doc-as-view formatters + build-write-prompt, docs alignment, anobel regeneration, sk-code alignment).
**Method:** 10 review dimensions, each finding adversarially verified (round 2). 52 raw findings → 47 confirmed, 5 refuted/downgraded.

## Verdict

PASS-WITH-FIXES

The phase 002-005 work is fundamentally sound — zero P0 defects, no critical correctness or security failures, and several of the core phase-002/004 mechanisms (the coverage-election pre-gate, classifyShadow de-duplication, the placeholder a11y shape, PRESENT/ABSENT data semantics, and the 8px font guard direction) were verified as correct as designed. The blocking concerns are concentrated in nine P1 findings that cluster into two themes: spec non-compliance in the validator gate (the dual-score split is computed but isPass() ignores claimsScore, defeating REQ-004-2, so a doc can pass on hex-fidelity alone despite multiple fabrication warnings) and a half-wired doc-as-view seam (three deterministic renderers for §4/§5/§9 exist but the write prompt never calls them, leaving the exact AI-hand-assembled table hallucination surface that Phase A was built to remove). Equally load-bearing is the test-coverage gap: every phase-002 classifier fix (classifyVariant's 8 branches, the coverage-election pre-gate, Badge/Card heuristics, interaction-default behavior) ships with zero regression tests, and several existing "scoring" tests assert arithmetic on hand-built literals without ever invoking the real code — so the named fixes could silently regress undetected. The 38 P2 findings are dominated by order-dependent CLI flag overrides, unvalidated/unguarded input paths that crash or fabricate tokens on malformed data ('0px' fonts, empty-{} a11y election, ENOENT stack traces), latent type-drift from pervasive `as unknown as` casts, and a consistent family of documentation claims (validate.md, feature_catalog.md, interaction-capture.md, SKILL.md) that invert or misattribute how section-coverage gating actually works. None of these block correctness today, but the P1 spec-compliance and zero-coverage items must be remediated before this can be claimed done; the verdict is PASS-WITH-FIXES rather than CHANGES-REQUIRED because the pipeline is correct end-to-end and every fix is scoped and well-localized.
