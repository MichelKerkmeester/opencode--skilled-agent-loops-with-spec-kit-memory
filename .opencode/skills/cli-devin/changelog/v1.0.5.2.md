## Summary

v1.0.5.0 established **RCAF as the SWE-1.6 default framework** in SKILL.md §3 (Default Invocation callout), SKILL.md §4 ALWAYS #12, `assets/prompt_templates.md` §2, and `assets/prompt_quality_card.md` §3. This patch aligns those examples with the RCAF default so a caller who copies-and-pastes the iter skeletons gets the empirically-best framework immediately, without having to reach the SKILL.md §4 #12 paragraph that overrides them.

## Upgrade

**No invocation-shape changes.** Iter prompts that explicitly chose `Framework: STAR` or `Framework: BUILD` continue to dispatch normally — the framework tag is still free-form per SKILL.md §4 #12. Iter prompts that copy the template default now get RCAF, matching the §4 #12 contract that already applied to every SWE-1.6 dispatch in this skill.

The dispatcher YAML branches (`spec_kit_deep-research_auto.yaml`, `spec_kit_deep-review_auto.yaml`, `spec_kit_deep-review_confirm.yaml` `if_cli_devin:` blocks) carry no framework hardcode — the framework tag lives in the prompt body, which the dispatcher reads from the cli-devin iter template at run time. So this asset/reference patch propagates to dispatched iters with no YAML changes needed.

## Why this lives in changelog/ and not SKILL.md

Per the project convention recorded in user memory (`feedback_skill_docs_no_phase_references.md`): the skill body (SKILL.md, assets, references) states present-tense desired logic only — no phase, test, or spec citations. The desired logic ("RCAF is the SWE-1.6 default") is already in SKILL.md §3/§4 #12 and the two prompt-quality assets. This patch makes the iter-template examples match that desired logic. The historical reason — "RCAF beat STAR by 33% and BUILD by 35% in the 003 eval-loop, then survived the 113/005 file-extraction unlock and the v3 mutation confirmation" — stays exclusively in `changelog/`.
