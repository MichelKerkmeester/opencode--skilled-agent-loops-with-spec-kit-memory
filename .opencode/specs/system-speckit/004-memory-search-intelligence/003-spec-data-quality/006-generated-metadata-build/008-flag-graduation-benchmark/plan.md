### Technical Context

| Aspect | Value |

| **Framework** | The phase 029 benchmark harness and the phase 025 false-confirm driver |

| **Testing** | One vitest proving single-flag isolation, the dual safety gate and the kept-off default |

### Overview
This phase delivers the Stage 4 graduation benchmark. It builds a driver that, for every default-OFF flag from this program, toggles that one flag in isolation against the migrated live corpus and the same query set, measures a before-and-after on the phase 029 benchmark harness for retrieval and scoring and on the phase 025 false-confirm driver for the safety metric, and decides graduation on the measured delta. A flag that clears both bars graduates to default-ON, a flag that does not stays off with the reason recorded. The verdicts land in `benchmark-status.md` and `keep-off-flag-roadmap.md`, per the 028 earn-or-delete discipline. No new measurement machinery is built, the driver composes the two existing harnesses.

## 3. ARCHITECTURE

### Pattern
A per-flag before-and-after over a fixed corpus and query set, composed from two existing harnesses. One driver iterates the flag set, toggles one flag at a time, runs the harness and the false-confirm driver for the baseline and the flag-on state, computes the delta, and applies the dual-gate graduation rule. No new ranking or scoring logic.

### Key Components
- **Flag set iterator**: the canonical list of default-OFF program flags, each toggled in isolation so its delta is attributable.
- **Before-and-after runner**: runs the phase 029 benchmark harness and the phase 025 false-confirm driver for the baseline and the flag-on state over the same corpus and query set.
- **Dual graduation gate**: graduates a flag only when the measured delta clears the bar on both the retrieval-or-scoring metric and the false-confirm safety metric, otherwise keeps it off.
- **Verdict writer**: writes the per-flag before-and-after and decision to `benchmark-status.md` and the kept-off flags with reasons and revisit conditions to `keep-off-flag-roadmap.md`.
- **Default flipper**: flips each graduated flag's default to ON in the capability-flag set and leaves the kept-off flags default-OFF.

### Data Flow
The driver checks the corpus is on the new format, then for each flag it runs the harness and the false-confirm driver with the flag off to get the baseline, runs them again with only that flag on to get the candidate, and computes the delta on both metrics. The dual gate decides graduation, the verdict writer records the numbers and decision, and the default flipper applies the graduation to the capability-flag set. A neutral delta inside the noise band is recorded as a keep-off reason.
