## P1 - Required

### P1-1 - mem-eval - Flag eval hardcodes `forceAllChannels: true`, bypassing default query routing

`scripts/evals/run-retrieval-flag-eval.mjs:355` (gpt-5.5)

`search()` sets `forceAllChannels: true`. Production routing uses `routeQuery()` unless `forceAllChannels` is set (`hybrid-search.ts:1394-1396`), so simple and default production queries get measured with graph, degree, and summary-capable channels forcibly active. This is acute for `SPECKIT_CARDINALITY_PENALTY`, which only affects the degree lane, so the measured off vs on delta does not reflect the default production path. A flag can appear baseline-neutral in the harness while changing real default behavior.

Verify reason (claude-confirmed, real, P1): `forceAllChannels:true` sets `activeChannels` to ALL channels instead of `routeResult.channels`. The complexity router is default-ON (`query-classifier.ts:61-64`, `raw!=='false'`), so production default simple and moderate queries route to `[vector,fts]` / `[vector,fts,bm25]` and drop the degree lane (`query-router.ts:413`, `getChannelSubset` 114-116). But `cardinalityPenalty` only damps the degree channel (`graph-search-fn.ts:551-567`), so the per-flag off vs on delta this driver feeds into the criterion-4 flip decision (`benchmark-status.md`) is measured on a non-representative all-channels path. The defect is 028's own code (single `feat(028)` commit), in scope, real.
