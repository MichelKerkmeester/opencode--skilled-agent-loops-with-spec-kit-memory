## [**1.2.0.0**] - 2026-07-14

This release replaces the flat `wt/{NNNN}-{name}` worktree/branch namespace with an owner-first grammar: every managed task branch now starts with the owning skill (or `skilled` for cross-cutting/system/release work), so a Git-UI branch tree groups by owner instead of sitting in one flat pile. A clone-wide locked allocator (`worktree-naming.sh`) replaces hand-computed counters, the launch-wrapper lane and its reaper gain several hardening fixes, and naming-grammar enforcement moves to a migration-tolerant pre-push hook.

---

## What Changed

#### Owner-First Branch + Worktree Grammar

- **New grammar** -- Task branches are now `{OWNER}/{NNNN}-{slug}` (e.g. `sk-git/0001-add-oauth`) with a matching `.worktrees/{NNNN}-{OWNER}-{slug}` directory. `{OWNER}` is a canonical skill id (the `name:` frontmatter of an `.opencode/skills/**/SKILL.md`) or the literal `skilled` for cross-cutting/system/release work; `{NNNN}` stays a 4-digit zero-padded counter, now clone-wide rather than hand-computed. `main`, `skilled/vA.B.C.D` release branches, and the launch wrapper's `work/{runtime}/{slug}` lane are unchanged and sit outside the task-branch grammar.
- **Legacy `wt/{NNNN}-{name}` branches** -- Permitted-but-non-conformant; migrate by renaming the branch (and its worktree directory) into the owner-first form, never by rewriting history.

#### Naming Allocator (`worktree-naming.sh`)

- **New sourceable script + CLI** -- `.opencode/skills/sk-git/scripts/worktree-naming.sh` exposes `is_valid_owner` / `is_valid_slug` / `is_valid_nnnn` / `is_valid_branch` / `is_wrapper_branch` / `is_valid_pair` validators, `scan_max_number` / `next_number` / `allocate_number` for the counter, and `create_named_worktree` / `create_detached_worktree` for one-step creation. CLI subcommands: `skill-ids`, `next`, `allocate`, `scan-max`, `validate-owner`, `validate-slug`, `validate-branch`, `validate-pair`, `create`, `create-detached`.
- **Locked, clone-wide counter** -- Allocation holds a lock in the shared common git dir and seeds its max from the stored high-water mark, every registered worktree basename, and all local + remote refs, so a partial scan can never reissue a live number. Owner ids are discovered live from every `name:` frontmatter under `.opencode/skills/**/SKILL.md`.
- Covered by a hermetic test suite exercising the grammar validators, number scanning, locked concurrent allocation, and named + detached worktree creation.

#### Wrapper + Reaper Hardening

- **`worktree-session.sh`** -- Validates the runtime token (safe charset + `command -v`) before it ever reaches `exec`, and writes an active-session marker (the launching pid, which survives `exec`) to `<common-git-dir>/worktree-sessions/<runtime>-<slug>.pid` — kept outside the worktree so it can never dirty the tree; a write failure is logged but non-fatal.
- **`worktree-reaper.sh`** -- The live integration target is now the primary checkout's actual `HEAD` (never a possibly-stale local `main`). Auto-reap is restricted to the launch-wrapper lane (`work/{runtime}/{slug}` pairs) and fires only when the tree is clean, the branch is merged into that live tip, and the session marker proves the recorded pid is dead. Owner-first task worktrees, detached worktrees, and any wrapper worktree with a missing/unreadable marker or a live pid are report-only — absence of proof is never proof of absence. Cleanup order is worktree-remove before branch-delete, and neither call ever passes `--force`.

#### Enforcement

- **Migration-tolerant pre-push hook** -- Naming-grammar enforcement gates only the creation of brand-new remote branches; existing non-conformant branches are never rewritten or blocked, and `skilled/v*` release branches are never blocked.

#### Router and Rules (`SKILL.md`)

- **ALWAYS #4 rewritten** to the owner-first grammar, pointing at the allocator script instead of a hand-computed counter.
- **New ALWAYS #17** -- cleanup ordering (worktree before branch), the reaper's wrapper-only + proven-inactive scope, and the pre-push enforcement contract.
- **Success Criteria** branch-naming line updated to `{OWNER}/{NNNN}-{slug}`.
- **Keyword/Owns comment lines and the `ON_DEMAND_KEYWORDS` router list** extended with `owner-first branch`, `skill-scoped worktree`, `worktree naming allocator`, `skilled branch` — existing entries (including the legacy `wt-branch` / `wt/ branch` terms) are kept, not replaced.

#### Reference Doc (`worktree_workflows.md`)

- Steps 1-4, the Branch Strategy Guide, Decision Matrix, and all three Example Workflows rewritten from hand-computed `wt/{NNNN}-{name}` shell snippets to `worktree-naming.sh create` / `create-detached` invocations.
- **Accuracy fix**: detached-experiment worktrees are now documented as still numbered (`.worktrees/{NNNN}-detached-{slug}`) even though they carry no branch/owner — the prior doc incorrectly claimed no number was assigned.
- **Accuracy fix**: the fast-merge cleanup examples now remove the worktree before deleting its branch, matching the new ALWAYS #17 (git refuses to delete a branch still checked out by a worktree).
- The two-lane note (owner-first vs. launch-wrapper) and a new legacy-migration note are both kept in the Naming Convention callout.

#### Graph Metadata

- `graph-metadata.json` vocabulary arrays (`domains`, `intent_signals`, `derived.trigger_phrases`, `derived.key_topics`, `derived.intent_signals`) extended with owner-first naming terms; no existing entries removed.
