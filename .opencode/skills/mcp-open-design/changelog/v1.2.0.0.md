# v1.2.0.0 - Retire the mcp-magicpath sibling reference

**Released:** 2026-06-14

The `mcp-magicpath` skill is removed now that this skill drives the installed Open Design app from the terminal and covers the local design-generation usecase. This release drops the now-dead `mcp-magicpath` sibling reference so the related-skill set and graph edges point only at live skills. There is no change to the Open Design transport, the tool surface, or the gating policy.

---

## What Changed

### Sibling reference dropped

- SKILL.md removes the `mcp-magicpath` related-skill bullet and the magicpath mention in the Section 8 related-skills prose.
- README drops the `mcp-magicpath` row from the Related Skills table.
- `graph-metadata.json` removes the `mcp-magicpath` siblings edge and removes it from `manual.related_to`. The `mcp-figma` sibling edge and the `sk-interface-design` design-judgment dependency are unchanged.

## Notes

- This is a reference-hygiene release tied to the `mcp-magicpath` deprecation. The Open Design generation flow, the multi-turn `start_run` plus discovery-form behavior corrected in v1.1.0.0, and every verb and gate are unchanged.
- Historical changelog entries are preserved. The shared `claude_design_parity.md` protocol now names this skill as the single generation transport.
