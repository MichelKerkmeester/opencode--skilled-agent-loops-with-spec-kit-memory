# v1.0.0.0 — animation surface established

First release of the `animation` surface packet. The Motion.dev evidence that previously lived under `code-implement/references/motion_dev/` + `assets/motion_dev/` is now consolidated under one read-only surface packet on the sk-code hub's surface axis.

## New

- `SKILL.md` — the surface contract: the cross-stack overlay model, the reference map, the non-negotiable motion standards (principles before API, compositor-only animation, right-tool-per-case via the decision matrix, respect reduced motion), and the copy-ready snippet tier.
- `README.md` — packet at-a-glance and layout.
- `references/` — animation principles, quick start, the animate/timeline/scroll/gesture API, integration patterns, decision matrix, and performance/pitfalls.
- `assets/` — install card, playbook entries, and drop-in `snippets/`.

## Notes

This packet is `packetKind: surface`, `backendKind: evidence-base`, read-only (`toolSurface` ⊆ {Read, Bash, Grep, Glob}), and advisor-invisible (`routingClass: metadata`). Unlike webflow/opencode, animation is a **cross-stack overlay**: the hub bundles it on top of whichever base surface is detected whenever an animation intent fires. It never routes as a primary and mutates nothing.
