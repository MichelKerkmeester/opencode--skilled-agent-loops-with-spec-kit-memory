# v1.4.0.0 — Skill Rename: deep-agent-improvement → deep-agent-improvement

Date: 2026-05-06

## 1. OVERVIEW

This changelog records the active skill rename from `deep-agent-improvement` to `deep-agent-improvement` and the migration boundary for paths, metadata, and runtime references.

## 2. SUMMARY

This release renames the skill from `deep-agent-improvement` to `deep-agent-improvement` so the skill folder, skill identity, metadata, and active documentation use the new deep-* naming convention while preserving the existing agent and command surfaces.

## 3. WHAT CHANGED

- Skill folder: `.opencode/skill/deep-agent-improvement/` was replaced by `.opencode/skill/deep-agent-improvement/`.
- Skill identity: `skill_id`, skill frontmatter, trigger phrases, key files, and derived entities now use `deep-agent-improvement`.
- Advisor scoring tables: skill advisor routing and scoring references are expected to target the new skill name.
- Command surfaces in 4 runtimes: OpenCode/Copilot, Claude, Codex, and Gemini command surfaces keep `/improve:agent` while routing references move to the new skill path.
- Agent skill-matrix lines in 4 runtimes: OpenCode/Copilot, Claude, Codex, and Gemini agent matrices continue to expose `@improve-agent` while pointing at `deep-agent-improvement`.
- Root docs: top-level references are expected to describe the new skill name and path.
- Install guides: installation and packaging instructions are expected to install `deep-agent-improvement`.

## 4. MIGRATION

The old path `.opencode/skill/deep-agent-improvement/` no longer resolves. Use `.opencode/skill/deep-agent-improvement/` for skill files, references, assets, scripts, feature catalog entries, and manual testing playbooks. The agent name `@improve-agent` and command `/improve:agent` are unchanged.

## 5. PRECEDENT

This rename follows the sister rename in `specs/skilled-agent-orchestration/070-sk-deep-rename/`, which moved `sk-deep-review` and `sk-deep-research` to `deep-review` and `deep-research`.
