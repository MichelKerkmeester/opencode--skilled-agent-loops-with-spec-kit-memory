Documented component-choice routing resilience and the packet's intentional use of flat resources instead of keyed runtime discovery.

## What Changed

### Component-choice routing
Documented the agent-vs-skill-vs-command decision path so an ambiguous request resolves to a clear owner instead of defaulting to an agent. The distinguishing signals, lighter-alternative cues, and the decision rule live in `references/agent-vs-skill-vs-command.md`, loaded when a request could plausibly be a skill or command or mixes responsibilities.

### Flat resource layout
Recorded the packet's deliberate choice of flat `references/` and `assets/` folders instead of keyed `references/<key>/` subdirectories. The resource set is small enough that the flat route map in `references/README.md` is clearer than keyed runtime discovery, so the packet adds no per-key resource subfolders.
