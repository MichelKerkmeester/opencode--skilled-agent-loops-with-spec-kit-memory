# v1.17.0.0 - Repo-Wide Agent-Mirror-Sync Gate

This release folds the deep-improvement slice of the packet 146 ponytail-based refinement into the agent-mirror surface. Until now, the check that keeps every agent's three runtime copies in agreement only ran inside a deep-improvement promotion. An edit to an agent landed by any other path could silently desync the `.opencode`, `.claude`, and `.codex` mirrors and nothing would catch it until production routing behaved differently per runtime. This release promotes that check to a repo-wide gate, sharpens the normalizer so it stops crying drift over legitimate per-runtime wording, and clears the real and phantom drift the sharper check then surfaced.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/146-sk-code-ponytail-based-refinement` (Level 2)

---

## New Features

The three changes below promote an existing library into a commit-time and CI gate, make that gate accurate enough to run on every change, and fix the one real divergence it found.

#### Repo-wide agent-mirror-sync gate

Every agent is authored once under `.opencode/agents/<name>.md` and mirrored to `.claude/agents/<name>.md` and `.codex/agents/<name>.toml` so each runtime ships the same behavior. Editing one copy without the others makes the runtimes silently disagree. A new `scripts/check-agent-mirror-sync.cjs` wraps the existing `verifyMirrorSync` library and turns it into a gate: given the changed agent files (or `--all`), it fails when any agent's mirrors drift from its `.opencode` canonical body — including a deleted mirror, or a deleted canonical whose runtime copies still linger. It is wired changed-files-scoped over added, modified, and deleted agent paths into `.opencode/hooks/pre-commit` and into a new `.github/workflows/agent-mirror-sync.yml` that fails closed if the checker is ever removed, so the divergence is caught at commit and pull-request time instead of in production. This promotes the mirror check from a deep-improvement promotion-only gate to an every-commit and every-PR gate.

&nbsp;

#### Mirror normalizer accuracy fix

A gate that runs on every commit has to be precise, and the old normalizer was both too loose and too narrow. Each runtime mirror legitimately describes itself — a clause along the lines of "this runtime's mirror; the canonical source lives in .opencode" — and refers to its own runtime agent-file path (for example `.codex/agents/<name>.toml`), neither of which the canonical file carries. The old `normalizeRuntimeSpecificText` in `scripts/lib/mirror-sync-verify.cjs` counted the self-description clause as body drift and false-flagged healthy mirrors, while a blanket rule that collapsed every bare `.md`/`.toml` extension token was loose enough to mask a real format-extension mismatch. The normalizer now strips the self-description clause and normalizes only the runtime agent-file paths — so a mirror's reference to its own `.codex/agents/<name>.toml` no longer reads as drift — and the blanket bare-extension collapse is gone. The fix is deliberately conservative: it removes only the known self-description and agent-path shapes, and real body drift, including a genuine format-extension mismatch, is still caught.

&nbsp;

#### Pre-existing drift fixed

Running the now-accurate gate over all agents surfaced one genuine divergence and cleared one false positive. The `context` codex mirror had a dropped Tool-Inventory row, which is restored to match the canonical body. The `orchestrate` agent was a false positive that the normalizer fix clears on its own. With both resolved, all twelve agents report in sync.

## Files Changed

- `.opencode/skills/deep-improvement/scripts/check-agent-mirror-sync.cjs` — new changed-files-scoped (or `--all`) gate wrapping `verifyMirrorSync`.
- `.opencode/skills/deep-improvement/scripts/lib/mirror-sync-verify.cjs` — `normalizeRuntimeSpecificText` now strips per-runtime self-description and normalizes runtime agent-file paths, dropping the blanket bare-extension collapse that was masking real format-extension drift.
- `.opencode/hooks/pre-commit` — runs the gate against changed agent files on commit.
- `.github/workflows/agent-mirror-sync.yml` — new CI workflow running the gate on pull requests.
- `.codex/agents/context.toml` — restored the dropped Tool-Inventory row to match the canonical agent body.
- `.opencode/skills/deep-improvement/README.md` — surfaced the repo-wide mirror-sync gate in the Integration Scan section.

## Upgrade

No migration required. The gate is additive — it runs against changed agent files at commit time and against the full set in CI, and it only fails when a mirror genuinely diverges from its `.opencode` canonical. Run `node .opencode/skills/deep-improvement/scripts/check-agent-mirror-sync.cjs --all` to verify all mirrors locally.
