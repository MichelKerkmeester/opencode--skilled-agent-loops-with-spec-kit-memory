## What Changed

#### Architecture

- **Canonical card relocated** -- `cli_prompt_quality_card.md` lived in `sk-prompt`, which is meant to stay a generic forkable framework engine, yet the card is full of CLI-dispatch specifics. We moved it in via a git rename to `assets/cli-prompt-quality-card.md` here, so the card now sits with the model-craft knowledge it serves. The card resolves from every consumer and its own links resolve, verified.
- **Card links repointed** -- the relocated card's internal links to the generic framework definitions would have broken after the move, so they were repointed to `../../sk-prompt/references/{patterns_evaluation,depth_framework}.md` (those definitions stay in sk-prompt), keeping the card's references intact.
- **Consumer pointers updated** -- the hub `SKILL.md` §5 reference now points at the local `./assets/cli-prompt-quality-card.md`, and the five `cli-*` SKILL.md files plus their cards plus `cli-devin/references/context-budget.md` were repointed from `../sk-prompt/assets/...` to `../sk-prompt-models/assets/...` (roughly fifteen references). No active skill file references the old `sk-prompt/assets/` card path anymore, verified.
