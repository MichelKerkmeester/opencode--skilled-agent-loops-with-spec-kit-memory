This is the first shipped release of `deep-loop-runtime`. The earlier `v0.1.0` state was a phase-001 scaffold. Version `1.0.0` marks the runtime as populated and verified: shared deep-loop libraries, coverage-graph ownership, direct script entry points, SQLite storage, and runtime tests now live together in the peer skill instead of inside `system-spec-kit/mcp_server/`.

> Spec folder: `.opencode/specs/skilled-agent-orchestration/118-deep-loop-full-isolation-no-mcp/008-verification-changelog-closeout/` (Level 2)

---

## New Features

#### Runtime libraries

- **Deep-loop support code** -- Ten shared TypeScript modules now live under `.opencode/skills/deep-loop-runtime/lib/deep-loop/`: executor config, executor audit, prompt-pack rendering, post-dispatch validation, atomic state, JSONL repair, loop locking, permissions, Bayesian scoring, and fallback routing.

#### Coverage graph ownership

- **Graph schema and queries** -- Three coverage-graph modules now live under `.opencode/skills/deep-loop-runtime/lib/coverage-graph/`: `coverage-graph-db.ts`, `coverage-graph-query.ts`, and `coverage-graph-signals.ts`. The SQLite database is owned by `.opencode/skills/deep-loop-runtime/storage/deep-loop-graph.sqlite`.

#### Direct scripts

- **No MCP graph tools** -- Four CommonJS entry points replace the removed MCP tool surface: `scripts/status.cjs`, `scripts/query.cjs`, `scripts/upsert.cjs`, and `scripts/convergence.cjs`. Workflow YAMLs and collateral call these scripts directly with `--spec-folder`, `--loop-type`, and `--session-id`.

#### Runtime tests

- **Split by responsibility** -- Runtime-focused unit, integration, and lifecycle tests moved into `.opencode/skills/deep-loop-runtime/tests/`. The system-spec-kit MCP server Vitest config discovers them, while MCP-specific tests for deleted deep-loop tools were removed.

## Upgrade

Consumers should call the four `.cjs` scripts directly rather than attempting to use `mcp__mk_spec_memory__deep_loop_graph_*` tools. The migration source was `.opencode/skills/system-spec-kit/mcp_server/`; after 118, the runtime owner is `.opencode/skills/deep-loop-runtime/`.
