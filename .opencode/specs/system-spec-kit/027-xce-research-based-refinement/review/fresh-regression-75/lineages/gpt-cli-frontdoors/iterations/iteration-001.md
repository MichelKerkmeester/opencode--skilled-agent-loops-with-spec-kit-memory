P1 security: the spec-memory CLI trusts an environment/default IPC socket directory under `/tmp` before authenticating the socket perimeter. The wrapper defaults unset `SPECKIT_IPC_SOCKET_DIR` to `/tmp/mk-spec-memory` and only creates/checks path length; the CLI-side `ensureSocketEnvironment()` follows the directory with `statSync()` and only rejects group/world-writable modes, with no `lstat()` symlink rejection and no uid check. The bridge probe then treats any JSON-RPC initialize reply as an alive daemon, so a local attacker can pre-create `/tmp/mk-spec-memory` as a symlink to an attacker-controlled executable directory containing a fake `daemon-ipc.sock`; warm-only and normal CLI calls will connect to the fake daemon instead of spawning the real one.

No shell-argument injection found on the reviewed path: process launches use argv arrays (`spawnSync(process.execPath, [cliDist, ...argv])`, `spawn(process.execPath, [launcherPath])`), and JSON-RPC frames are serialized with `JSON.stringify`.
