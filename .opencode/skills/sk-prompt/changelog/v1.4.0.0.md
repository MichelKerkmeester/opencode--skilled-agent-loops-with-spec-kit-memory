# sk-prompt v1.4.0.0 — model-profile recommended_frameworks field + prompt-composition precedence

**Released**: 2026-06-02
**Status**: Feature addition to `assets/model-profiles.json`; documentation update to `references/model-profiles.md` and `assets/cli_prompt_quality_card.md`.

---

## Summary

Three coordinated changes land together to make per-model framework guidance first-class and eliminate duplication with the sk-prompt-models hub:

1. **`recommended_frameworks` field added to all 8 active model profiles.** The new additive field lists the ranked frameworks each model responds to best, surfacing guidance that was previously scattered across prose.
2. **`references/model-profiles.md` rebuilt.** The stale model count (was 10, now correct) is fixed; the new `recommended_frameworks` field schema is documented with an example; the Architecture-A entry is split so data rows and explanatory prose no longer mix.
3. **Prompt-Composition Precedence section added to `assets/cli_prompt_quality_card.md`.** A 3-tier hierarchy (framework selection → density → per-model override) makes composition order explicit. The per-model-override note is repointed to sk-prompt-models as the canonical hub for model-specific profiles. The Mirror Sync section is refreshed to a duplication-guard description so readers know where each concern lives.
