# v1.0.3.0 — Three-Way Isolation Finalize

**Release date**: 2026-05-15
**Source directive**: user — "all 3 system skills need to be completely isolated; users can delete system-spec-kit and expect a functioning system-code-graph and skill-advisor"
**Remediation packets**: 040 (active arc) + 040-followon (deferred items)

## Headline

After v1.0.2.0, system-code-graph still had 1 production `.ts` cross-skill import (handlers/query.ts:14) + 87 stale source-level `.js` artifacts + 29 stale `.d.ts` artifacts + 4 test files importing from system-spec-kit. v1.0.3.0 closes every remaining cross-skill production-`.ts` dependency at the import level and proves it via an actual delete-system-spec-kit smoke test.

**Verdict from the smoke test** (script + log committed at `018-three-way-isolation-finalize/smoke-test/`):
```
code-graph tsc exit BEFORE removing system-spec-kit:  2 (baseline)
code-graph tsc exit AFTER removing system-spec-kit:   2 (UNCHANGED — full isolation)
code-graph tsc exit AFTER restoring system-spec-kit:  2 (back to baseline)
```

Zero new failures when system-spec-kit is gone. Delete-system-spec-kit works as advertised.

## What changed

#### Last production cross-skill .ts import closed

- Need: handlers/query.ts:14 imported `attachGraphEdgeEnrichment`, `attachStructuralTrustFields`, `EdgeEvidenceClass`, `HotFileBreadcrumb` from `system-spec-kit/mcp_server/lib/context/shared-payload.ts` (the only remaining production .ts cross-skill leak).
- Change: the 4 symbols were already duplicated into `lib/shared/shared-payload.ts` during v1.0.2.0's Path-1 refactor — only the import line in query.ts needed to switch. Also removed the `@spec-kit/shared` dependency declaration from package.json (was unused after the duplication).
- Why it matters: production .ts cross-skill imports go from 1 → 0. Combined with no `@spec-kit/*` workspace-alias imports, code-graph's TypeScript source is fully decoupled from system-spec-kit.

&nbsp;

#### 87 stale source-level .js + 29 stale .d.ts deleted

- Need: legacy compiled artifacts sat alongside .ts source files (29 .js + 29 .js.map + 29 .d.ts.map + 29 .d.ts). They contained outdated cross-skill imports that surfaced in audit grep but weren't used at runtime (the launcher boots `mcp_server/dist/index.js`, not source-level siblings).
- Change: physically deleted 116 files (-5505 + -757 lines).
- Why it matters: audit grep is now clean across `mcp_server/**/*.{js,d.ts}`. Future tsc compiles to `dist/` (outDir setting), so source-level siblings won't regenerate.

&nbsp;

#### 7 test/stress files isolated

- Need: 7 test files (`tests/code-graph-indexer`, `tests/graph-payload-validator`, `tests/startup-brief`, `stress_test/code-graph/{walker-dos-caps, w10-degraded-readiness-integration}`) imported cross-skill helpers; 2 files (`tests/crash-recovery`, `tests/p0-a-cross-runtime-tempdir-poisoning`) tested system-spec-kit internals from the wrong skill.
- Change: 5 files refactored to `vi.mock` + local `__fixtures__/` stubs; 2 files deleted (Option C — they belonged in system-spec-kit/tests, not here).
- Why it matters: full test-suite isolation. Discovery: 527 code-graph tests post-refactor. Zero cross-skill imports across the entire test surface.

&nbsp;

#### CI guardrail extended

- Need: the existing reverse-direction audit only checked code-graph → spec-kit relative-path imports. Workspace-alias imports (`@spec-kit/*`) weren't caught.
- Change: `.github/workflows/isolation-check.yml` extended with 3 new blocking audit steps — skill-advisor reverse-direction, skill-advisor + code-graph `@spec-kit/*` workspace-alias checks. The skill-advisor `@spec-kit/*` audit excludes `@spec-kit/shared/embeddings/*` as the single documented exception (visible via the embeddings symlink in skill-advisor's lib/shared/).
- Why it matters: 6 blocking audit steps now prevent any future cross-skill import from sneaking in. Failure mode: GitHub Actions error with the exact import line.

## Known limitations

- **Source-level `.d.ts` files were retained where they declare types consumed externally** — none in code-graph after this release (all 29 deleted; audit found 0 consumers), but if a future export needs a `.d.ts` shim it should land in `dist/` via tsc, not at source level.
- **`tsconfig.json` extends from `system-spec-kit/tsconfig.json`** — at the build-config level, code-graph still references system-spec-kit. Deleting system-spec-kit doesn't break TypeScript SOURCE compilation (smoke test verified), but it does break `tsc --build` if you rely on the inherited config. Inlining the base config is a future cleanup if needed.
- **Source-level `.js` artifacts won't regenerate** under current `tsc` config (outDir: `./dist`), but if a future contributor changes the build to emit at source level, the cleanup needs to be re-done. The CI guardrails would catch the resulting cross-skill imports either way.
